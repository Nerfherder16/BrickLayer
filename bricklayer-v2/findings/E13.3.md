Any "optimization" based on this signal is random. ## Investigation

### 1. Compatibility check: `improve_agent.py` vs research-analyst

`improve_agent.py` runs this internal loop:
1. **Eval**: runs `eval_agent.py` on held-out records → gets a baseline score
2. **Optimize**: calls Claude to generate improved instructions from high/low examples
3. **Re-eval**: runs `eval_agent.py` again → compares scores
4. **Decide**: keep new instructions if score improved, else revert

`eval_agent.py` is a **tool-free eval** harness. It passes each question to the research-analyst agent WITHOUT any MCP tools available. The research-analyst agent requires tools to answer questions — it must read files, search code, query Masonry, etc.

**Result**: tool-free research-analyst cannot produce meaningful verdicts. It outputs either:
- "I need to check the file but cannot without tools" → prose → score=0.20 (partial credit)
- Guess at a verdict with no evidence → score=0.20-0.30

**Dry-run confirmation** (run during this session):
```
improve_agent.py research-analyst --dry-run --eval-size 5
baseline_score = 0.20 (1/5 passed)
```

This 0.20 baseline is noise. With 5 records, random chance alone produces 0-2 passes. Any optimization cycle comparing 0.20 vs 0.25 is not measuring real agent capability — it's measuring prompt formatting luck.

**Conclusion: `improve_agent.py research-analyst --loops 2` was not run.** Running it would corrupt the agent's instructions with noise-optimized prompts that perform well on tool-free eval but degrade live performance.

## Root Cause: `improve_agent.py` Architecture Incompatibility

The fundamental issue: `improve_agent.py` was designed for agents that can be evaluated without tools. research-analyst is inherently tool-dependent. The eval backend mismatch makes the optimization loop meaningless.

```
improve_agent.py
    └── eval_agent.py  ← tool-free, scores research-analyst at 0.20 (noise)

What's needed:
improve_agent.py
    └── eval_agent_live.py  ← tools enabled, scores research-analyst at 0.61-0.84
```

**Required change to enable real optimization:**

`masonry/scripts/improve_agent.py` needs a `--live-eval` flag (or auto-detection) that substitutes `eval_agent_live.py` calls for `eval_agent.py` calls when the agent is tool-dependent.

```python
# improve_agent.py change (not implemented — diagnosis only)
def _run_eval(agent: str, eval_size: int, use_live: bool = False) -> float:

            ["python", "masonry/scripts/eval_agent_live.py", agent, "--eval-size", str(eval_size)],

    else:
        result = subprocess.run(

            ...
```
