## Bugs Fixed

### Bug 1: `score_ops_agents._score_karen()` â€” wrong input context

**Before**: `input.files_modified` captured the doc files karen WROTE (CHANGELOG.md, ROADMAP.md), not the source files that triggered the update. The model correctly inferred "docs already written â†’ skip" from this misleading signal.

**After**: Fixed to use the PARENT commit's non-doc source files:
```python
parent = all_commits[idx + 1]  # git log is newest-first
source_files = [f for f in parent.get("files", []) if Path(f).name not in _KAREN_FILES]
"input": {"commit_subject": parent_subject[:200], "files_modified": source_files}
```
Also added filter: skip records where parent is itself a bot commit.

**New training distribution**: 362 records with source files (was 252 CHANGELOG-only), 0 doc-only, 0 bot commit subjects.

---

### Bug 2: `build_karen_metric._derive_expected()` â€” wrong labels for bot commits

**Before**: "chore: update CHANGELOG for abc123" commits had `doc_files_written=1` â†’ metric derived expected="updated" (wrong â€” 237/321 records, 73% mislabeled).

**After**: Added `_RE_BOT_COMMIT_SUBJECT` check first:
```python
_RE_BOT_COMMIT_SUBJECT = re.compile(
    r"^chore:\s+update\s+CHANGELOG\s+for\s+[0-9a-f]{7,}", re.IGNORECASE
)
commit_subject = str(getattr(example, "commit_subject", "") or "")
if _RE_BOT_COMMIT_SUBJECT.search(commit_subject):
    return 0.0, "skipped"
```
Bot commits now correctly labeled "skipped" regardless of doc file count.

### Bug 3: karen.md has Unicode arrows â†’ cp1252 encoding failure on Windows

**Before**: `â†’` (U+2192) characters in karen.md "Reasoning approach" section (positions 3277-3430) cannot be encoded in Windows cp1252. When `eval_agent.py` passed `--system-prompt` to `claude.cmd` subprocess, the argument was silently truncated at the first arrow. The model never received the `feat/fix â†’ action='updated'` rule.

**After**: Replaced all `â†’` arrows with ASCII `->` in all karen.md files. Also fixed `eval_agent.py` to pass the user message via `stdin` (using `--print` + `input=user_msg`) instead of as a `-p` argument, preventing double-quote argument quoting issues on Windows.
