# Finding: Q1.1 — Minimal campaign.py change for mode dispatch

**Question**: What is the minimal change to `bl/campaign.py` to support mode dispatch — reading `mode:` from each question and loading the corresponding `modes/{mode}.md` as loop context?
**Agent**: quantitative-analyst
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: Medium
**Mode**: diagnose
**Target**: `bl/campaign.py`, `bl/runners/agent.py`

## Summary

Three targeted additions to `campaign.py` implement full mode dispatch: (1) a `_load_mode_context()` helper that reads `modes/{mode}.md`, (2) injection of that context into the question dict before `run_question()`, and (3) reading the `operational_mode` field from the question (distinct from the runner `mode` field). No changes to the runner registry, runner protocol, or findings pipeline are required.

## Evidence

### Current architecture

`campaign.py:run_and_record()` at line 28–84 calls `run_question(question)` where `question["mode"]` is the **runner type** (agent, http, subprocess, correctness). The runner dispatch in `runners/__init__.py:run_question()` routes on `question["mode"]`. The agent runner in `runners/agent.py` builds its prompt from `question["hypothesis"]`, `question["test"]`, `question["verdict_threshold"]`, and the agent file content.

### What needs to change

**Step 1 — `_load_mode_context()`** (new helper in `campaign.py`):
```python
def _load_mode_context(operational_mode: str) -> str:
    """Load modes/{operational_mode}.md as loop context. Returns empty string if not found."""
    modes_dir = cfg.project_root / "modes"
    mode_file = modes_dir / f"{operational_mode}.md"
    if mode_file.exists():
        return mode_file.read_text(encoding="utf-8")
    return ""
```

**Step 2 — inject into `run_and_record()`** (one line added at line 35, before `run_question`):
```python
def run_and_record(question: dict) -> dict:
    qid = question["id"]
    # NEW: attach mode context to question for agent runner to include
    op_mode = question.get("operational_mode", "")
    if op_mode:
        question = dict(question)  # don't mutate the original
        question["mode_context"] = _load_mode_context(op_mode)
    result = run_question(question)
    ...
```

**Step 3 — agent runner reads `mode_context`** (in `runners/agent.py`, the prompt builder needs one addition):
The agent runner already builds a prompt string. Add `mode_context` as a preamble:
```python
mode_ctx = question.get("mode_context", "")
if mode_ctx:
    prompt = f"## Mode Program\n\n{mode_ctx}\n\n---\n\n{prompt}"
```

### What does NOT need to change

- Runner registry: all existing runners (agent, http, subprocess, correctness) remain unchanged
- Runner protocol: `(question: dict) -> dict` contract unchanged
- findings.py: write_finding() doesn't care about operational_mode
- questions.py: parse_questions() just needs to read the `operational_mode` field (see Q1.2)
- results.tsv: unchanged format
- The `mode:` field in questions continues to mean runner type — no rename needed in the runner layer

### Proof of minimality

The 3 changes above touch ~15 lines total across 2 files. The mode dispatch is purely additive — existing campaigns without `operational_mode` fields continue to work identically (the `_load_mode_context` returns empty string and no injection happens).
