# Finding: E13.8 — Baseline eval scores for peer-reviewer, agent-auditor, retrospective

**Question**: Three candidate-tier agents — peer-reviewer, agent-auditor, and retrospective — have `last_score: null` in agent_registry.yml and no records in scored_all.jsonl. Generate 5 training records each (15 total) using the live record generation approach validated in E12.1. What baseline eval scores do these agents achieve on first run?
**Agent**: research-analyst
**Verdict**: BLOCKED
**Severity**: Medium
**Mode**: evolve
**Target**: `masonry/agent_registry.yml`, `masonry/training_data/scored_all.jsonl`, `masonry/scripts/eval_agent_live.py`

## Summary

Cannot generate baseline eval scores for peer-reviewer, agent-auditor, or retrospective — none of the three agents have instruction files (`.md`) in either the project agents directory or the global `~/.claude/agents/` directory. `eval_agent_live.py` invokes agents via `claude -p {instructions}` where instructions are read from the agent `.md` file. Without instruction files, no eval records can be generated. The agents exist in `masonry/agent_registry.yml` as `tier: candidate` entries but are shell registrations only — they cannot be invoked.

### Root cause analysis

The three agents appear to have been added to the registry (likely via `masonry-agent-onboard.js` hook during a campaign) referencing agent descriptions in CLAUDE.md or system context, but the corresponding `.md` instruction files were never written. The `masonry-agent-onboard.js` hook creates registry entries from `.md` file content — if the hook ran on a different agent file that mentioned these three agents, or if they were manually added, the entries exist without corresponding instruction files.

Agents listed as available in CLAUDE.md (global rules) include peer-reviewer, agent-auditor, and retrospective with brief descriptions:
- **peer-reviewer**: Re-runs tests from completed findings, verifies fix code, appends peer review sections
- **agent-auditor**: Audits active fleet, scores against finding history, identifies underperformers, writes AUDIT_REPORT.md
- **retrospective**: Post-campaign quality analyst — scores process efficiency, audits content integrity, identifies agent tooling gaps

These descriptions are sufficient to create stub `.md` instruction files. However, writing those files is out of scope for this finding — it requires a separate implementation task.

## Verdict Threshold

BLOCKED: Cannot generate eval records without instruction files. The three agents are registry-only entries with no executable instructions. Creating the `.md` files is a prerequisite.

## Mitigation Recommendation

Priority order:
1. **Create instruction files** for all three agents. Minimum viable `.md` files can be derived from the CLAUDE.md descriptions plus standard BrickLayer agent frontmatter. Estimated: 30 minutes per agent, 90 minutes total.
2. **Re-run E13.8** after instruction files are created — `python masonry/scripts/eval_agent_live.py --agent peer-reviewer --n 5` etc.
3. **Expected baselines**: Given that these are new agents without optimized prompts, expect 0.40–0.60 initial scores. Scores below 0.50 warrant immediate optimization.

## Open Follow-up Questions

The broader fleet gap (E13.9) identified 9 agents lacking eval baselines — peer-reviewer, agent-auditor, and retrospective represent 3 of those 9 with the additional blocking condition of missing instruction files. Resolving E13.8 requires implementation work (agent file authoring) before research work (eval generation).
