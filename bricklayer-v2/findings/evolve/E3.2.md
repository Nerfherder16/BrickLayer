## Root Cause Analysis

### 1. Heterogeneous training records

The 61 quantitative-analyst records (45 from scored_all.jsonl + 16 from wave13) span multiple
question types from the Recall project campaign:
- Research findings (correct use case for quantitative-analyst)
- "What should the next wave test?" planning questions → model returns QuestionPayload
- "Route this to the right agent" inputs → model returns RoutingDecision
- "Verify this assumption" questions → model returns inspection plan

The eval instruction `"Respond ONLY with a valid JSON object matching the expected output schema"`
is insufficient — it doesn't tell the model WHAT schema to use.

### 2. Missing eval schema instruction

Unlike karen eval (which has `_KAREN_JSON_INSTRUCTION` with explicit field list), the research
signature uses only the generic:
```
"Respond ONLY with a valid JSON object matching the expected output schema."
```

For agents with diverse output formats, this is ambiguous.

## Improvement Path

**Short-term (eval harness fix)**:
Add a `_RESEARCH_JSON_INSTRUCTION` constant to `eval_agent.py` with explicit fields:
```python
_RESEARCH_JSON_INSTRUCTION = (
    "You are being evaluated on a research task. "
    "Based ONLY on the provided question_text, produce a research finding. "
    "Respond ONLY with a valid JSON object with exactly these keys: "
    '"verdict" (one of: "HEALTHY", "WARNING", "FAILURE", "INCONCLUSIVE"), '
    '"summary" (1-2 sentence summary), '
    '"evidence" (detailed evidence string >300 chars), '
    '"confidence" (decimal 0.0-1.0).'
)
```

**Long-term (training data quality)**:
Filter quantitative-analyst records to only include research finding outputs. Remove records where
`expected.verdict` is absent or where the input is a planning/routing question.
