## Root Cause: Tool-Free Eval vs Agentic Agent

The research-analyst in production uses tools (file reads, web search) to verify claims.
The eval harness runs it with `--no-session-persistence` and no tool calls — only its
training prompt and the question text.

For "Is X sound?" questions requiring code inspection:
- **Agent response**: INCONCLUSIVE ("No implementation details found in provided context")
- **Expected verdict**: HEALTHY (based on my prior knowledge of the system state)
- **Result**: 0.00 score every time

Agent output for E9.3-rec-2 ("Is masonry-guard.js sound?"):
```
"No implementation details, design documentation, or code inspection of masonry-guard.js
found in provided context [...] To evaluate the 3-strike design soundness requires: (1)
actual source code showing fingerprint generation logic..."
```

This is CORRECT behavior. Without tool access, INCONCLUSIVE is the honest answer.

For taxonomy questions (E9.3-rec-1), the agent identifies real design tensions:
```
"IMPROVEMENT shifts to a delta-based verdict ('did quality change?'), a fundamentally
different question type requiring relative comparison rather than absolute threshold
assessment."
```

Agent's WARNING verdict is MORE CORRECT than my expected HEALTHY — the IMPROVEMENT vs
state-based verdict design tension IS a real concern.
