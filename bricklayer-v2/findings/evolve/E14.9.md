## Failure Analysis

### 1. E12.1-live-15 (persistent)
HEALTHY→WARNING. The cosmetic timeout message discrepancy (print says '>120s', actual 180s) still fires as WARNING across all instruction versions. This is a hard failure that persists through E13.3 and E14.1 changes. Requires explicit calibration example naming this pattern.

### 2. Wrong verdicts on INCONCLUSIVE records (E8.2, E9.4)
4 INCONCLUSIVE records predicted as WARNING or FAILURE:
- E8.2-rec-6: INCONCLUSIVE→FAILURE
- E8.2-rec-2: INCONCLUSIVE→WARNING
- E9.4-rec-1: INCONCLUSIVE→WARNING

The agent over-fires concrete verdicts when it should recognize unresolvable uncertainty. The INCONCLUSIVE rule (Rule 1 / calibration anchor) is not firing on these records.

### 3. Wrong verdicts on WARNING records (E8.2, E9.1)
3 WARNING records predicted as HEALTHY:
- E9.1-rec-2: WARNING→HEALTHY
- E9.3-rec-3: WARNING→HEALTHY
- E8.2-rec-5: WARNING→HEALTHY

The agent under-fires WARNING on older record formats. These records may describe gaps more abstractly (less code-inspection framing).

### 4. Timeouts (E9.4, E9.4b, E7.2)
4 records timed out after >120 seconds. These appear to be computationally intensive questions (simulation runs, complex code analysis). The eval harness's 120s timeout may need to be extended for certain question types, or these records should be excluded from standard eval runs.
