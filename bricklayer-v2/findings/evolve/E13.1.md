## Root Cause Analysis

### 1. Relabeling was net-neutral for records 5 and 16

E12.1-live-5 and E12.1-live-16 predicted WARNING during the gold label generation run, which led to the hypothesis that relabeling FAILURE→WARNING would make them pass. But in E13.1, both records predict FAILURE again. The agent oscillates stochastically between FAILURE and WARNING across runs. Neither label produces a consistently passing record.

**Conclusion**: Both records are inherently stochastic at the FAILURE/WARNING boundary and should be REMOVED and REPLACED.

### 2. 4 empty responses — stochastic run-to-run variance

Records 8, 11, 17, 19 produced empty or near-empty responses this run but scored 0.95-0.97 in E12.1. This reflects variance in the live eval subprocess — tools-enabled agent sometimes fails to produce JSON output for complex questions.

### 3. True mean score is approximately 0.75

E12.1 Run 2: 0.84 (17/20). E13.1: 0.69 (13/20). Mean: 0.77. The E12.1 0.84 was a favorable run of a distribution with mean ~0.75. The live eval score has inherent ±0.10 variance per run.
