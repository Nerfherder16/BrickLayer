## Root Cause Analysis

### Why INCONCLUSIVE records are miscalibrated
Tool-free agents cannot read files or run searches. Questions asking "does X guard exist?" or "is Y implemented?" produce INCONCLUSIVE by default — the agent can reason but not verify. Tool-enabled agents immediately verify by reading the actual files and return definitive verdicts (WARNING or FAILURE).

**Affected records:**
- E8.2-rec-2: Question about whether a specific security check is implemented
- E8.2-rec-6: Question about a missing guard condition
- E9.4-rec-1: Question about routing layer coverage (confirmed from E11.1 pilot data)

### Why E8.2-rec-7 (HEALTHY → WARNING) is miscalibrated
Tool-free agent rated the design as HEALTHY based on surface-level reasoning. Tool-enabled agent read the actual implementation and identified a design gap that warranted WARNING. The tool-free gold label over-optimistically rated something it could not fully examine.

### E8.2-rec-5 Prose Issue
A separate stochastic issue — the agent sometimes produces prose instead of JSON for this record. Not a calibration gap; a formatting consistency issue. Max score: 0.40 (partial credit).

## Key Observations

### 1. INCONCLUSIVE is a structural artifact of tool-free eval
INCONCLUSIVE verdicts in tool-free training data are not genuine uncertainty — they are forced by the tool-free constraint. Every INCONCLUSIVE record where the question asks about codebase state is a miscalibration candidate. The old 18 records have at least 3 confirmed INCONCLUSIVE → (WARNING/FAILURE) gaps.

### 2. Scale of miscalibration in old 18 records
From the 18 pre-E12.1 research-analyst records:
- INCONCLUSIVE records tested: 3 (E8.2-rec-2, E8.2-rec-6, E9.4-rec-1) → all 3 re-classify
- Additional untested: E8.2-rec-8 (INCONCLUSIVE — meta question about live eval viability; now answered by E11.1/E12.1, likely HEALTHY)
- HEALTHY record: E8.2-rec-7 → escalates to WARNING

Estimated total miscalibrated in old 18: **4-5 records** (22-28% of old dataset)

### 3. E12.1 new records are better calibrated
E12.1 live-calibrated records use tool-enabled gold labels → no structural INCONCLUSIVE artifacts. Score 17/20 = 0.85. The new records are 3-4x better calibrated than the old records.

### 4. Full re-labeling impact estimate
If all 4 identified miscalibrated records are corrected:
- Old 18 records: estimated ~12/18 pass rate (vs current ~8/18 estimate)
- Combined 38 records: estimated ~29/38 pass rate = **0.76**
- Combined with E12.1 relabel of E12.1-live-5 and E12.1-live-16: ~31/38 = **0.82**
