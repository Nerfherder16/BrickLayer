### Loop 1 Optimized Instructions (kept)

Key changes injected into `## DSPy Optimized Instructions`:
- **Rule 1**: Mandatory file-read-before-verdict — INCONCLUSIVE only when tools fail, never from choosing not to read
- **Rule 2**: Evidence format with numbered `[file:line]` citations ≥300 chars with quantitative content
- **Rule 3**: Summary formula — quantitative fact + mechanism + verdict in ≤200 chars
- **Rule 4**: Verdict calibration — explicit HEALTHY/WARNING/FAILURE/INCONCLUSIVE decision tree
- **Rule 5**: Confidence targeting — default 0.75, ±0.10-0.15 band, never 1.0 or <0.50
- **Rule 6**: Root cause chain — mechanism → gap → impact for FAILURE/WARNING
- **Rule 7**: Never leave verdict as "?" or summary as empty

## Key Observations

### 1. Optimization produced measurably better instructions (+0.07 live eval)
The tool-free eval signal was noisy (0.35→0.45→0.25 across runs) but the kept instructions genuinely improved live eval from 0.84 to 0.91. The 7-rule DSPy optimization section focuses on the exact failure modes observed in the corpus: INCONCLUSIVE predictions on readable files, weak evidence, verdict calibration gaps.

### 2. Tool-free eval variance is too high for reliable loop comparison
Variance of ±0.10 at eval-size=20 means:
- Loop 1: 0.35→0.45 (+0.10) — genuine improvement confirmed by live eval
- Loop 2: 0.25→0.15 (-0.10) — likely stochastic noise, not real regression (reverted correctly)
- The revert-on-regression design prevented harm but also may have blocked a genuine second improvement

### 3. One new failure: E12.1-live-15 (HEALTHY→WARNING)
Post-optimization, the agent over-applies the WARNING/gap-finding rules to a HEALTHY record. The injected Rule 6 (root cause chain for WARNING) may be causing the agent to look harder for gaps even when none exist. This is a known risk when strengthening warning-detection instructions.

### 4. E13.10 prediction confirmed (partial)
E13.10 predicted "loop 1 is expected to improve, loop 2 is borderline" — confirmed. The actual convergence behavior matched the prediction: +0.10 improvement in loop 1, regression in loop 2 reversed by revert.
