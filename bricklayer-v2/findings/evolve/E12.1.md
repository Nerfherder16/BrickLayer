## Key Observations

### 1. Live calibration approach validated
Expected verdicts from tool-enabled agents produce high consistency (85% match rate) on re-runs. The approach scales: generate questions → collect gold labels via live eval → use those as expected verdicts. Tool-free eval on same records would score ~0.45 (structural mismatch).

### 2. FAILURE verdict is the calibration gap
Both failures are FAILURE→WARNING mismatches. The gold label said FAILURE (original live agent was definitive: "this IS a security risk / this IS missing"), but re-runs produce WARNING (potential risk). FAILURE vs WARNING is the most stochastic boundary in the eval — depends on how urgently the agent frames the finding.

**Implication**: Changing E12.1-live-5 and E12.1-live-16 expected verdicts to WARNING would raise score to 19/20 = 0.95.

### 3. One prose response (E12.1-live-14)
Record 14 (synthesis.md accuracy check) produced prose in Run 2 but HEALTHY in Run 1. This is the same stochastic prose pattern seen in earlier waves. The 2-stage eval gives it 0.40 (partial pass, below 0.50 threshold).

### 4. Verdict distribution of calibrated dataset
- HEALTHY×11, WARNING×5, FAILURE×4, INCONCLUSIVE×1 (20 total)
- HEALTHY-heavy (55%) but that's correct — the codebase is largely sound, and tool-enabled agents can verify this, unlike tool-free agents
