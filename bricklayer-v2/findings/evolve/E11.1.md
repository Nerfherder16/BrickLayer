## Key Observations

### 1. WARNING records with clear self-evident evidence score excellently (0.97-0.98)
The agent reads the actual implementation, finds the described flaw, and produces specific
file-referenced evidence. Score matches or exceeds the best tool-free scores.

### 2. INCONCLUSIVE records become definitive with tool access — a feature, not a bug
E9.4-rec-1 expected INCONCLUSIVE ("no way to know if question selection is sound without
reviewing the mechanism"). With tools, the agent READ the mechanism and produced a definitive
WARNING. This is MORE CORRECT — the agent should not be INCONCLUSIVE when it can verify.

Implication: INCONCLUSIVE expected verdicts in training data are calibrated for tool-free
agents. For live eval, these records should be re-labeled as HEALTHY/WARNING based on what
a tool-enabled agent actually finds.

### 3. Severity escalation is evidence of deeper accuracy
E7.2-pilot-1 expected WARNING but agent predicted FAILURE with tool evidence. This may be
the more accurate verdict — the agent's access to actual code revealed a more serious issue.

### 4. Timeout variability
1 record timed out at 120s (extensive optimization.py inspection). With 180s timeout, all
records completed. Some records will require 3-5 minutes of tool use — realistic for
production research questions.

## Calibration Gap: Why Live Eval Doesn't Immediately Score Higher

The training data expected verdicts were produced by tool-free agents. Live eval agents:
- Find MORE evidence → escalate severity (WARNING→FAILURE)
- Find ENOUGH evidence → no longer INCONCLUSIVE (INCONCLUSIVE→WARNING)
- Find CONTRADICTING evidence → sometimes more severe than expected

Result: live eval scores on existing training data are not necessarily better than
tool-free — they're DIFFERENT and arguably more accurate.

**To reach 0.85+ with live eval requires new training data**:
1. Generate 20+ new questions with clear expected verdicts (facts about the current codebase)
2. Run live eval to establish what the tool-enabled agent produces
3. Use THOSE responses as the gold labels for live eval training data
4. Score against live-calibrated expected verdicts
