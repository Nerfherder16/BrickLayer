## Key Observations

### 1. Target ≥0.60 reached
synthesizer-bl2 scores 0.62 on live-calibrated data, breaking the 0.55 tool-free ceiling (E11.2). The improvement comes from calibrated gold labels: questions whose answers are verifiable by reading actual files produce consistent verdicts on re-run.

### 2. PROSE records are miscalibrated gold labels — not a real failure
The 4 PROSE records failed not because synthesizer-bl2 got the wrong answer — they failed because the gold label was "PROSE" while re-runs produced proper verdicts (WARNING, FAILURE, WARNING, empty).

The generation run produced PROSE on these questions because synthesizer-bl2's JSON compliance rate is lower than research-analyst's. The gold label should be the verdict from a JSON-producing run, not a prose-producing run.

**After re-labeling PROSE records to their re-run verdicts:**
- E12.3-synth-3: PROSE → WARNING (re-run: WARNING) → WARNING→WARNING = PASS (est. 0.97)
- E12.3-synth-4: PROSE → FAILURE (re-run: FAILURE) → FAILURE→FAILURE = PASS (est. 0.96)
- E12.3-synth-6: PROSE → WARNING (re-run: WARNING) → WARNING→WARNING = PASS (est. 0.97)
- E12.3-synth-9: stochastic (re-run: empty) → remains PARTIAL (~0.40)

**Estimated post-relabel score: 9/10 = 0.90**

### 3. synthesizer-bl2 has 40% prose rate in generation vs 5% for research-analyst
The agent produces prose more often when asked to do cross-file comparison or subjective quality assessment. This is a prompt compliance gap. Questions requiring multi-file synthesis are more likely to trigger prose. The E12.3 record set can be improved by replacing prose-prone questions with cleaner verification questions (similar to E13.2 plan for research-analyst).

### 4. JSON-producing questions score excellently (5/6 = 0.83)
When synthesizer-bl2 produces JSON (6 out of 10 records), the verdict is correct and well-calibrated. The problem is not verdict quality — it's output format compliance.
