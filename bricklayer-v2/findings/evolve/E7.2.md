## Key Discovery: Question Type Determines JSON Compliance

The eval harness immediately returns 0.00 when the model produces non-JSON output
(`eval_agent.py` lines 131-135: `except (json.JSONDecodeError, ValueError): return 0.0, {}`).

Two question types behave very differently:

### Code-inspection questions → Non-JSON (0.00 stochastic)
Questions requiring the agent to read specific files and verify implementation details:
- "Does eval_agent.py have test coverage for metrics.py?" → 0.00 both runs
- "Is optimize_with_claude.py correctly scoped?" → 0.98 run1, 0.00 run2

These trigger the agentic research-analyst behavior: the model reads files, writes up findings in
markdown prose, and sometimes forgets to output JSON at the end.

### Knowledge/reasoning questions → Clean JSON (0.97 consistent)
Questions that can be answered from existing knowledge, patterns, and reasoning:
- "Could 2-stage eval improve research-analyst score?" → 0.97 both runs

These produce reliable JSON because the model can answer from in-context reasoning without needing
to "do research" first.

## Implication for Full 25-Record Plan (E6.3)

The E6.3 plan assumed all question types would perform equally. The pilot reveals a critical split:

**Use reasoning questions** for the remaining 15 records:
- "Would approach X significantly improve metric Y?" → PROMISING
- "Is the current [design/spec/architecture] sound for [goal]?" → HEALTHY or WARNING
- "Does [system] have any [class of vulnerability/gap] based on its design?" → WARNING or FAILURE
- "If [scenario] occurred, what would break first?" → FAILURE or INCONCLUSIVE

**Avoid pure code-inspection questions** (until the JSON compliance issue is fixed):
- "Does file X have coverage for function Y?" → triggers agentic behavior
- "Is the implementation of X at line Y correct?" → triggers tool use

Alternatively: implement the 2-stage eval (E7.2-pilot-5's PROMISING finding) where prose responses
score on evidence quality only. This would turn 0.00 scores on code-inspection questions into
0.2-0.4 range scores, raising the overall baseline.
