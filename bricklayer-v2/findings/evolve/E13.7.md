**Question**: Audit the deterministic routing layer keyword coverage in `masonry/src/routing/router.py`: list all registered slash-command patterns and mode-keyword patterns. Are there any commonly-used BrickLayer commands missing from the deterministic layer that require LLM fallback unnecessarily?
**Agent**: research-analyst
**Verdict**: WARNING
**Severity**: Medium
**Mode**: evolve
**Target**: `masonry/src/routing/deterministic.py`

## Summary

The deterministic layer has 36 patterns covering nearly all BrickLayer workflows. However, 4 coverage gaps cause unnecessary LLM fallback for common use cases: `improve_agent.py` invocations, mode-guidance queries, "what is broken" phrasing variants, and architect/design queries that don't use the exact registered vocabulary. Adding 4 targeted pattern extensions would raise deterministic coverage from 75% to ~90%.

### Coverage gaps (cause unnecessary LLM fallback)

**Gap 1: eval/improve-agent workflow** ← MISSING PATTERN
- No pattern covers: `run eval on X`, `improve_agent.py X`, `eval score`, `optimize X agent`
- These are common operations in the BL dev workflow, occur dozens of times per wave
- Fix: Add `_EVAL_PATTERN → evolve-optimizer`:
  ```python
  _EVAL_PATTERN = re.compile(
      r"\b(run\s+eval\b|improve.agent|eval\s+score|optimize\s+\w+\s+agent|"
      r"eval_agent\.py|baseline\s+eval|improve\s+agent\s+prompt)\b",
      re.IGNORECASE,
  )
  ```

**Gap 2: "what is broken" phrasing** ← PARTIAL COVERAGE
- `_DIAGNOSE_PATTERN` requires multi-word phrases like "why is it broken"
- "what is broken", "what's broken", "it's broken" miss entirely
- Fix: Add `\bwhat\s+(is|'s)\s+broken\b`, `\bit.s\s+broken\b` to `_DIAGNOSE_PATTERN`

**Gap 3: architect phrasing** ← PARTIAL COVERAGE
- `_ARCHITECT_PATTERN` requires "architecture decision", "system design" — narrow vocabulary
- "design a X", "architect a X", "how should I design" miss
- Fix: Add `\barchitect\s+a\b`, `\bhow\s+should\s+I\s+design\b`, `\bdesign\s+a\b` to `_ARCHITECT_PATTERN`

**Gap 4: mode guidance** ← MISSING PATTERN
- No pattern covers: "what mode should I use", "which mode for X", "what BL mode"
- Fix: Add `_MODE_GUIDANCE_PATTERN → mortar`:
  ```python
  _MODE_GUIDANCE_PATTERN = re.compile(
      r"\b(what\s+mode|which\s+mode|what\s+bricklayer\s+mode|"
      r"what\s+stage\s+is|what\s+phase)\b",
      re.IGNORECASE,
  )
  ```

**Gap 5: campaign status** ← PARTIAL COVERAGE
- `_CAMPAIGN_PATTERN` doesn't match bare "campaign status", "show campaign", "project status"
- Fix: Add `\bcampaign\s+status\b`, `\bshow.*(campaign\|project)\b` to `_CAMPAIGN_PATTERN`
