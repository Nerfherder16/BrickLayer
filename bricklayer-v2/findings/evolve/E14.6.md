### Agent 3: karen (static, tool-free)

**Status**: BLOCKED — format incompatibility.

karen's 379 training records have the structure:
```json
{"verdict": "?", "action": "changelog", "commit_hash": "abc123", ...}
```

`eval_agent.py` expects `FindingPayload` format:
```json
{"verdict": "HEALTHY|WARNING|FAILURE|INCONCLUSIVE", "summary": "...", "evidence": "...", ...}
```

The `verdict='?'` field indicates these are **commit-classifier records** (karen categorizes commits as changelog-worthy or skip), not agent finding records. The `eval_agent.py` scorer calls `_score_karen()` which expects `predicted.action` — a different field entirely from FindingPayload.verdict.

**Root cause**: karen operates on a fundamentally different task (commit classification → action dispatch) than research agents (question investigation → verdict). The shared `eval_agent.py` entry point cannot accommodate both task types without per-agent scorer routing.

**E2.3 historical note**: Karen's last measured score (E2.3) was 1.00 using a custom `score_ops_agents.py` pipeline, not `eval_agent.py`. That score is specific to the commit-classification task. The 379 records in `scored_all.jsonl` are correctly formatted for that pipeline, not for `eval_agent.py`.
