# Finding E5.1 — Add PROMISING to _RESEARCH_JSON_INSTRUCTION

**ID**: E5.1
**Mode**: evolve
**Verdict**: IMPROVEMENT
**Date**: 2026-03-24

---

## Summary

Added `PROMISING` to the allowed verdict set in `_RESEARCH_JSON_INSTRUCTION` in
`eval_agent.py`. quantitative-analyst eval score rose from **0.70 → 0.90**
(7/10 → 18/20). Stable across two independent runs. Karen confirmed 1.00 (20/20)
with `--signature karen` — prior low score (13/20) was caused by using the wrong
signature (research instead of karen). Secondary finding: `--signature karen` must
be passed explicitly; eval_agent.py does not auto-detect the agent signature.

---

## Delta

- Metric: quantitative-analyst eval score
- Baseline: 0.70 (7/10 passed) — E4.1 measurement
- After: **0.90 (18/20 passed, also 9/10 on 10-example run)** — exceeds 0.85 target
- Improvement: +29%
- Karen verification: 1.00 (20/20) with `--signature karen` — unchanged from E2.3

---

## Root Cause of Prior Score Cap

42% of quantitative-analyst training records (26/61) have `PROMISING` expected
verdict. The previous `_RESEARCH_JSON_INSTRUCTION` only allowed:
`HEALTHY/WARNING/FAILURE/INCONCLUSIVE`. This prevented the model from producing
the correct PROMISING verdict on 26 records — all those records had verdict_match=0
(0 of 0.4 possible) and scored at most 0.60 (evidence 0.4 + confidence ~0.2).
Records at the boundary of "good enough evidence" could fall below 0.50.

With PROMISING now permitted, the model can correctly classify frontier/research
findings that don't clearly fit the 4-class schema.

---

## Change Made

**Before**:
```python
'"verdict" (one of: "HEALTHY", "WARNING", "FAILURE", "INCONCLUSIVE"), '
```

**After**:
```python
'"verdict" (one of: "HEALTHY", "WARNING", "FAILURE", "INCONCLUSIVE", "PROMISING"), '
```

## Secondary Finding — Wrong Signature Bug

During karen regression check, running `eval_agent.py karen` (default
`--signature research`) applied `build_metric()` (research metric using
verdict/evidence/confidence) instead of `build_karen_metric()` (karen metric
using action/quality_score/changelog). This produced scores of 0.36-0.65 that
appeared to be a regression but were an evaluation error.

**Impact**: All prior karen regression checks in Waves 3-4 that used default
`--signature research` are invalid. The actual karen score (with correct
`--signature karen`) remains 1.00 (20/20).

**Required fix**: Auto-detect signature from agent name, or document the flag
prominently. When `agent == "karen"`, default signature should be "karen".
