**Question**: Establish a routing accuracy baseline for the Masonry four-layer router: create `masonry/scripts/score_routing.py` that runs 20 routing queries through `masonry/src/routing/router.py` and scores deterministic vs semantic vs LLM layer hit rates. What is the current deterministic layer coverage (target ≥60%)?
**Agent**: research-analyst
**Verdict**: HEALTHY
**Severity**: Info
**Mode**: evolve
**Target**: `masonry/src/routing/deterministic.py`, `masonry/scripts/score_routing.py`

## Summary

`score_routing.py` already exists (routes from routing_log.jsonl, not query testing). Deterministic layer coverage on 20 representative queries is **75%** (15/20), exceeding the ≥60% target. The 5 misses all fall into known semantic-layer territory: architect patterns, broken-system phrasing variants, eval/masonry-ops commands, campaign guidance, and status queries.

### Registered deterministic patterns

30 keyword patterns + 6 slash commands. All primary workflows covered:
- Slash: `/plan`, `/build`, `/fix`, `/verify`, `/bl-run`, `/masonry-run`
- Keywords: git, ui/figma, karen, diagnose, security, kiln, solana, refactor, architect, campaign, research, competitive, regulatory, quantitative, benchmark, bug-catcher, fix-implementer, code-reviewer, cascade, evolve, health, compliance-auditor, agent-auditor, skill-forge, prompt-engineer, devops, spreadsheet, self-host, hypothesis

### Miss analysis

| Query | Gap | Fix |
|-------|-----|-----|
| "architect a caching layer" | _ARCHITECT_PATTERN requires "system design", "architecture decision" etc — "caching layer" alone doesn't match | Add `\bcaching\s+layer\b`, `\bdesign\s+a\b`, `\barchitect\b` to pattern |
| "what is broken and why?" | _DIAGNOSE_PATTERN requires "why is it broken" (3-word phrase) — "broken and why" doesn't match | Add `\bwhat is broken\b` |
| "run eval on research-analyst" | No eval/improve-agent pattern in deterministic layer | Add `\brun eval\b`, `\bimprove.agent\b`, `\beval.score\b` → evolve-optimizer or research-analyst |
| "what mode should I use?" | No mode-guidance pattern | Add `\bwhat mode\b`, `\bwhich mode\b` → mortar |
| "show me campaign status" | _CAMPAIGN_PATTERN matches `masonry.run` but not bare "campaign status" | Add `\bcampaign\s+status\b`, `\bshow.*(campaign\|project)\b` |

## Verdict Threshold

Deterministic layer at 75% exceeds the 60% target. Three patterns missing from `_DIAGNOSE_PATTERN`, `_ARCHITECT_PATTERN`, and `_CAMPAIGN_PATTERN` would raise coverage to ~90%.

## Mitigation Recommendation

Add 5 pattern extensions to `masonry/src/routing/deterministic.py`:
1. `_ARCHITECT_PATTERN`: add `\bcaching\b`, `\bdesign\s+a\b`
2. `_DIAGNOSE_PATTERN`: add `\bwhat\s+is\s+broken\b`
3. New `_EVAL_PATTERN` → evolve-optimizer: `\brun\s+eval\b`, `\bimprove.agent\b`
4. `_CAMPAIGN_PATTERN`: add `\bcampaign\s+status\b`
5. Add mortar routing for `\bwhat\s+mode\b` queries
