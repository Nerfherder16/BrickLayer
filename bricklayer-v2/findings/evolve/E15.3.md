## Summary

Flagged 5 eval-incompatible records in scored_all.jsonl with `eval_compatible: false`. Updated eval_agent_live.py to skip them by default with a `--include-incompatible` override flag. Full-corpus eval will now report 31 records instead of 36, eliminating 4-5 guaranteed timeout failures that were masking the real score.

## Evidence

### Records flagged

| Record | Reason | Expected verdict |

| E9.4-rec-1 | Malformed input (uses `question_text` key, not `question`) + timeout | INCONCLUSIVE |
| E9.4-rec-2 | Malformed input (uses `question_text` key, not `question`) | WARNING |
| E9.4b-rec-1 | Malformed input (uses `question_text` key, not `question`) | FAILURE |
| E9.4b-rec-2 | Malformed input (uses `question_text` key, not `question`) | WARNING |
| E7.2-pilot-5 | Timeout (>120s) during full-corpus live eval (E14.9) | WARNING |

### Root cause for E9.4/E9.4b records

The E9.4 and E9.4b records use a different input schema — `question_text` instead of `question`. The eval harness generates questions using the standard `question` key, so when these records are run through live eval, the agent receives an empty question field. This causes the agent to either time out (trying to infer the question) or produce empty output. These records cannot be meaningfully evaluated without a schema migration.

### Root cause for E7.2-pilot-5

The question ("Could the research-analyst eval score be significantly improved by a 2-stage eval approach...") requires the agent to reason about hypothetical eval redesigns. This is computationally intensive and the 120s eval harness timeout is insufficient.

### Changes applied

**scored_all.jsonl**: Added `eval_compatible: false` and `eval_skip_reason: 'timeout-or-malformed-input'` to all 5 records.

**eval_agent_live.py `_load_agent_records()`**: Added `include_incompatible: bool = False` parameter. Records with `eval_compatible == False` are skipped by default with a log message. Pass `--include-incompatible` to include them.

**eval_agent_live.py `run_live_eval()`**: Added `include_incompatible: bool = False` parameter, passed through to `_load_agent_records()`.

**eval_agent_live.py `main()`**: Added `--include-incompatible` CLI flag.
