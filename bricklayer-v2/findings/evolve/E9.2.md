---

## Root Cause (Prior Defect)

The original `build_metric()` awarded evidence quality and confidence points regardless
of whether the verdict was correct. This allowed calibration inversion:

```
wrong verdict + good evidence + conf=0.75:
  0.0 (verdict_match) + 0.4 (evidence) + 0.2 (calibration) = 0.60 → PASS
```

A prediction with the WRONG verdict could pass the 0.50 threshold if its evidence
was sufficiently detailed. This made the eval metric unsound as a quality signal —
an agent could score well by writing verbose wrong answers.

---

## Fix

**`masonry/src/metrics.py`** — `build_metric()` internal metric function:

```python
# Added prerequisite gate after verdict_match check:
if not verdict_matched:
    return min(score, 0.2)
```

If the verdict doesn't match, the total is capped at 0.2 (below the 0.50 pass threshold)
regardless of evidence quality or confidence calibration.

Score behavior after fix:
- Wrong verdict + good evidence: 0.00 (was 0.60)
- Correct verdict + good evidence + conf=0.75: 1.00 (unchanged)
- Correct verdict + poor evidence + conf=0.75: 0.80 (unchanged)
