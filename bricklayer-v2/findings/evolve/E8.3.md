## Why 0.90 Target Wasn't Reached

The synthesizer-bl2 exhibits the same scoring structure as research-analyst:
- Prose responses (when Q6.5 triggers non-JSON): score = 0.40 via 2-stage eval
- Clean JSON with correct verdict: score = 0.80-0.97
- Clean JSON with wrong verdict but good evidence: score = 0.60

With 10 records, ~3-4 are right at the 0.40-0.50 boundary, causing ±30% swings.

The specific problem records:
- **Q6.5** (WARNING: mypy errors 0→0): stochastic JSON compliance — produces prose in 1/2 runs
- **New records**: diverse campaign contexts but most scoring 0.45-0.58 (not reliably above threshold)

**Path to 0.90**: Either replace Q6.5 with a more stable question, or add 5-10 more records
that consistently produce correct JSON with correct verdicts (score 0.80+). The current borderline
pattern means even 20 records won't reach 0.90 unless the borderline records are replaced.

---

## Recommendation

The 2-stage eval fix (E8.1) helped by raising the floor, but both research-analyst and
synthesizer-bl2 need a **training data quality audit**: identify which records produce
stochastic wrong-verdict predictions and replace them with cleaner questions.

Specifically for synthesizer-bl2: Q6.5 (WARNING: mypy errors 0→0) is the structural weak
point — this question is ambiguous (0 changes = HEALTHY or WARNING?) and should be replaced
with a question that has an unambiguously correct verdict.
