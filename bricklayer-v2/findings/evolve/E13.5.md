E12.3 score of 0.62 remains the synthesizer-bl2 ceiling.

## Root Cause Analysis

### 1. Re-labeling raised the bar without optimization

PROSE gold labels scored at 0.40 if the agent also produced prose (partial credit). Re-labeling to WARNING/FAILURE now scores at 0.00 if the agent produces prose or the wrong verdict. Records synth-3, synth-4, synth-6 now require the agent to produce a structured JSON verdict — but since optimization failed, the agent was never retrained on these harder targets. Net effect: harder eval with same agent = lower score.

### 2. Optimization blocked by tool-call in claude -p subprocess

`optimize_with_claude.py` calls `claude -p --no-session-persistence` with a text prompt asking for a JSON response. The claude subprocess decided to use the Write tool to write directly to `synthesizer-bl2.md` rather than returning JSON in stdout. Without `--dangerously-skip-permissions`, the masonry-approver.js hook intercepted this and emitted an approval-flow message to stdout, which broke JSON parsing.

**Fix required in `optimize_with_claude.py`**: The subprocess call needs either:
- `--dangerously-skip-permissions` (and the prompt must be rewritten to NOT write files), OR
- `--output-format text` if that suppresses tool use, OR
- An explicit system constraint in the prompt: "Do not use file tools. Output ONLY a JSON object in your response text."

### 3. Additional stochastic regressions

Records synth-1 (HEALTHY → predicted FAILURE) and synth-8 (HEALTHY → predicted WARNING) both regressed from their E12.3 correct predictions. These are synthesizer-bl2's inherent stochastic variance — same questions produce different verdicts across runs.

## Comparison to Hypothesis

> "After re-labeling the 4 PROSE records and running `improve_agent.py synthesizer-bl2 --loops 2`, does live eval score exceed the E12.3 baseline of 0.62?"

**NOT CONFIRMED**: Score is 0.41 vs baseline 0.62. The optimization loop did not run (blocked by claude -p tool-call issue). Re-labeling alone made the eval harder, producing a net regression.

---

## Path Forward

**Fix optimize_with_claude.py** (E14.x scope): Modify the subprocess call to prevent `claude -p` from using Write tools. Simplest fix: add a system instruction at the start of the prompt: "You must respond with ONLY a JSON object in your text output. Do not use any file tools. Do not write to any files."

**Re-run optimization after fix**: Once the subprocess returns valid JSON, run `improve_agent.py synthesizer-bl2 --loops 3` against the 20-record dataset (9 E12.3-synth + 11 prior records) to see if instruction optimization can raise the score above 0.62.

**Accept current ceiling**: If optimization continues to fail, E12.3's 0.62 score on the calibrated 9-record eval is the current synthesizer-bl2 live ceiling. The pre-PROSE records (synth-1, 5, 7, 8, 10) achieve 3/5 correct (0.60) without optimization — this is the natural agent floor.
