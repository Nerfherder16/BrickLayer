# E1.2: Should `modes/validate.md` explicitly document FAILURE routing?

**Status**: IMPROVEMENT
**Date**: 2026-03-24
**Mode**: evolve
**Agent**: evolve-optimizer

## Hypothesis

Adding explicit FAILURE routing guidance to `modes/validate.md` should prevent agents from
misrouting validate FAILURE findings — an agent reading validate.md alone would otherwise not
know whether to hand off to Research (design flaw) or Diagnose (behavior mismatch), even though
this distinction is correctly captured in `program.md`'s cross-mode handoff table.

## Before State

- `modes/validate.md` had no guidance on FAILURE routing
- FAILURE findings ended at `validation-report.md` with only "exact location in design doc"
- The two-path split (new system → Research; deployed system → Diagnose) was only in `program.md`
- An agent reading only validate.md had no routing signal; a Go/No-Go recommendation without
  a clear next step creates a decision gap between session end and the next campaign start

## Change Applied

Added "FAILURE routing" section to `modes/validate.md` (before Session end):
- Routing table: new system → Research | deployed system → Diagnose
- Explicit instruction to write routing decision in `validation-report.md` for each FAILURE
- Fallback: if unclear, use SUBJECTIVE and ask human
- Updated `validation-report.md` format spec to include routing decisions

## Delta

- **Metric**: Completeness of validate.md as a standalone loop program
- **Baseline**: validate.md session end had no routing guidance; agents required program.md
  cross-mode table knowledge to correctly route failures
- **After**: validate.md contains the full two-path routing decision tree; can be understood
  in isolation. `validation-report.md` now includes routing decisions per FAILURE.
- **Improvement**: +1 missing action path documented; validate.md is now self-contained
- **Regression check**: program.md cross-mode table unchanged. validate.md verdict set
  unchanged (HEALTHY | WARNING | FAILURE | SUBJECTIVE). No existing behavior paths removed.
- **Method**: Read before/after state of modes/validate.md

## What Would Change This Verdict

Would become HEALTHY (already sufficient) if: agents were guaranteed to always read program.md
before validate.md. They are not — the session start protocol in program.md says "read the mode
program", not "also read program.md for cross-mode handling". The local routing table is needed.
