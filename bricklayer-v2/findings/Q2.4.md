# Finding: Q2.4 — Predict mode methodology soundness

**Question**: Does the Predict mode program (`modes/predict.md`) have a sound methodology? Can the IMMINENT/PROBABLE/POSSIBLE/UNLIKELY verdict set be objectively assigned, or is it inherently SUBJECTIVE?
**Verdict**: WARNING
**Severity**: Medium
**Mode**: validate
**Target**: `bricklayer-v2/modes/predict.md`

## Summary

Predict mode's causal chain methodology is sound — the format forces explicit mechanism and timeline statements, which is the right structure for failure cascade analysis. However, the IMMINENT/PROBABLE/POSSIBLE/UNLIKELY verdict set is partially subjective: IMMINENT ("within 1 wave / 1 sprint") and PROBABLE ("within 5 waves / 1 month") have objective criteria, but POSSIBLE and UNLIKELY are assigned by agent judgment without measurable criteria. For projects with quantitative growth rates (like Recall), the verdicts can be made objective. For qualitative projects, they cannot.

## Evidence

### Causal chain format — sound

The required format forces three things that prevent lazy prediction:
1. **Trigger**: named finding ID + current state (not vague "there's a bug")
2. **Cascade mechanism**: explicit "↓ [mechanism]" — must name the failure pathway
3. **Impact**: user-visible or system-critical consequence (not just "bad thing happens")
4. **Timeline**: concrete IMMINENT/days/weeks estimate

This is the right structure. It prevents "if X is bad, Y will also be bad" circular predictions.

### IMMINENT/PROBABLE objective criteria — sound

The time-based criteria are objective:
- `IMMINENT` — within 1 wave / 1 sprint if not addressed
- `PROBABLE` — within 5 waves / 1 month

These are testable. For Recall's floor-clamped count:
- Q29.6 confirmed floor-clamped at 941, growing +268 in one wave cycle
- Rate: ~268/wave; threshold for ALERT: 1,000
- Remaining capacity: 59 units; 59/268 = 0.22 waves → IMMINENT by formula ✓

### POSSIBLE/UNLIKELY objective criteria — MISSING

The program defines:
- `POSSIBLE` — could occur given current trajectory; not certain
- `UNLIKELY` — trajectory doesn't support this cascade; explain why

These are not time-bounded. "Could occur" is judgment, not calculation. Two agents analyzing the same trajectory could assign POSSIBLE vs UNLIKELY based on risk tolerance, not evidence.

**Root cause of the gap**: POSSIBLE and UNLIKELY are appropriate for qualitative causal chains (e.g., "if the vendor ecosystem shrinks, the ADBP network effect reverses") where no growth rate is available. But the verdicts lack decision criteria.

**Suggested fix**: Add to predict.md:
```
### Verdict decision criteria

For POSSIBLE vs UNLIKELY:
- If the trajectory has a measurable rate: compute time-to-threshold.
  <30d → IMMINENT; 30d-6mo → PROBABLE; 6mo-2yr → POSSIBLE; >2yr → UNLIKELY
- If the trajectory has no measurable rate (qualitative chain):
  Assign POSSIBLE if the mechanism has 3+ documented instances in the finding graph.
  Assign UNLIKELY if the mechanism requires a NEW failure (not yet observed) to trigger.
  Any qualitative assignment must state: "POSSIBLE — 3 documented instances; no rate available"
  or "UNLIKELY — requires unobserved prerequisite [X]"
```

### Interaction pair questions — sound but expensive

"One question per major open failure + one question per failure interaction pair" — for a project with N open failures, this is O(N²) questions. At N=10 (Recall has 12 open FAILUREs), this is 10 + 45 = 55 questions per Predict session. That's a full campaign wave. The program should add:

```
Prioritize interaction pairs by: (a) both failures are IMMINENT/PROBABLE, and
(b) both failures share a common resource (same service, same data store).
Skip interaction pairs where one failure is UNLIKELY.
```

### Retrospective validation

Predict mode has no feedback loop — there's no mechanism to check whether IMMINENT predictions actually came true within 1 sprint. Adding a retrospective check at Predict Wave 2+ would improve accuracy:

```
### Retrospective check (Wave 2+)
For each IMMINENT/PROBABLE verdict from the prior Predict wave:
- Did the cascade occur? YES → CONFIRMED; NO → FALSE POSITIVE (update the causal chain)
```

## Verdict Rationale

WARNING: The causal chain format is sound and correctly prevents vague predictions. IMMINENT/PROBABLE have objective time-based criteria. POSSIBLE/UNLIKELY lack decision criteria and will be assigned subjectively without the suggested additions. The O(N²) interaction pair scaling is a practical problem for projects with many open failures.
