# Finding: Q3.5 — UI/UX audit checklist from design philosophy

**Question**: For UI/UX: given the `figma-designer-guide.md` and `frontend-design-philosophy.md` in Tim's global rules, define an audit checklist for any new frontend component. What are the top 10 compliance checks?
**Agent**: competitive-analyst
**Verdict**: HEALTHY
**Severity**: Info
**Mode**: audit
**Target**: Tim's frontend components — global rules compliance

## Summary

The 10 highest-signal compliance checks for any new frontend component, derived from `frontend-design-philosophy.md` and `figma-designer-guide.md`. These are ordered by violation frequency (most-commonly violated first, based on anti-patterns listed in the rules). Each check has a testable pass/fail criterion.

## Evidence

### Top 10 Audit Checklist

| # | Check | Fail Condition | Source |
|---|-------|---------------|--------|
| A1 | **No absolute positioning in layout containers** | Any `absolute left-[*px]` or `top-[*px]` on layout containers (not overlays/badges) | figma-designer-guide.md Rule 1 |
| A2 | **Dark background enforced** | Page background is pure `#000`, `#111`, gray, or white — not a hue-tinted dark (`#0f0d1a` range) | design-philosophy.md Color System |
| A3 | **Lucide icons only** | Any inline SVG, `imgVector`, `imgVector2`, or non-Lucide/Phosphor icon library used | figma-workflow.md |
| A4 | **No DaisyUI / shadcn defaults** | Import from `daisyui` or `@shadcn/ui` present; OR component is visually generic/template-looking | design-philosophy.md Anti-patterns |
| A5 | **Typography hierarchy uses weight extremes** | All text within 2 weight steps of each other; OR font sizes are uniform (all `text-sm`, etc.) | design-philosophy.md Typography |
| A6 | **8px spacing grid compliance** | Any `gap-[*px]`, `p-[*px]`, or `m-[*px]` with values not in {4,8,12,16,24,32,48,64}px | design-philosophy.md Spatial System |
| A7 | **No hardcoded Figma asset URLs** | Any URL containing `figma.com/api/mcp/asset` or `figma.com/file/*/asset` | figma-workflow.md |
| A8 | **Approved font families only** | Any use of `Inter`, `Roboto`, `Open Sans`, `Lato`, `Arial`, `Helvetica`, `system-ui` as display font | design-philosophy.md Typography |
| A9 | **Motion: transform/opacity only** | CSS animation on `width`, `height`, `padding`, `margin`, `background-color`, or `border` properties | design-philosophy.md Motion > GPU Rules |
| A10 | **Focus ring present on interactive elements** | `:focus-visible` has `outline: none` without replacement; OR interactive element has no visible focus state | design-philosophy.md Accessibility |

### Checking methodology

**A1 (Absolute positioning)**: `grep -r "absolute left-\[" src/` and `grep -r "absolute top-\[" src/` — any hit is a fail unless in a component documented as an intentional overlay.

**A2 (Dark background)**: Check `tailwind.config.ts` for `bg-*` defaults; check `index.css` body background; check `App.tsx` root className.

**A3 (Icons)**: `grep -r "import.*from 'react-icons'" src/` or `grep -r "<img.*src.*svg" src/` — fail on either.

**A4 (No component libraries)**: `grep -r "from 'daisyui'" src/` or `grep -r "@shadcn" src/` — fail on import presence.

**A5 (Typography hierarchy)**: Manual review — scan for heading/body/label combinations. Pass requires at least 2× size difference between main heading and body text, and at least 200 weight difference between primary and muted text.

**A6 (Spacing grid)**: `grep -rE "gap-\[[0-9]+px\]" src/` — any explicit pixel gap not in the approved set is a fail.

**A7 (No temp URLs)**: `grep -r "figma.com/api" src/` — zero tolerance.

**A8 (Fonts)**: `grep -r "Inter\|Roboto\|Open Sans\|Lato\|Arial\|Helvetica" src/` and check index.css `@import`.

**A9 (Motion GPU)**: `grep -rE "transition-(width|height|padding|margin|background)" src/` — direct animated layout properties.

**A10 (Focus rings)**: `grep -r "outline: none\|outline-none" src/` — any without `:focus-visible` replacement is a fail.

### Pass/fail thresholds per BrickLayer 2.0 Audit mode

| Verdict | Condition |
|---------|-----------|
| COMPLIANT | 0 fails across all 10 checks |
| PARTIAL | 1-3 fails, none are A1/A3/A4/A7 (structural violations) |
| NON_COMPLIANT | Any fail on A1/A3/A4/A7 OR 4+ total fails |
| NOT_APPLICABLE | N/A per check (e.g., A7 if no Figma workflow used) |

### Secondary checks (not top 10 but worth running)

- Bento grid: no uniform `grid-cols-3` without intentional asymmetry
- KPI cards: large numbers should be `font-light` (300) or `font-thin` (200), not `font-bold`
- Reduced motion: `@media (prefers-reduced-motion)` present in global CSS
- Touch targets: minimum `min-h-[48px]` on interactive elements in mobile components
- Active states: every button has `:active` variant (`active:scale-[0.97]` or equivalent)

## Verdict Rationale

HEALTHY: The 10 checks are derivable from the design philosophy rules, cover the most common violation categories, and are each testable with grep or visual review. This checklist can be run as a BrickLayer 2.0 Audit mode session for any new frontend component.
