# Finding: V7.1 — `masonry_drift_check` Requires `agent_db_path` with No Default, Blocking Kiln Integration

**Question ID**: V7.1
**Status**: DONE
**Verdict**: FAILURE
**Agent**: design-reviewer
**Severity**: Low
**Confidence**: 0.95

## Summary

`_tool_masonry_drift_check()` in `mcp_server/server.py` makes `agent_db_path` a required argument with no default. If not provided, it returns `{"error": "agent_db_path is required", "reports": []}`. The actual `agent_db.json` is at `_REPO_ROOT / "agent_db.json"` — a deterministic, computable path. By contrast, `registry_path` in the same function has a sensible default of `str(_REPO_ROOT / "masonry" / "agent_registry.yml")`. This inconsistency means Kiln (which calls the MCP tool without knowing filesystem paths) receives an error response on every invocation.

## Evidence

### `_tool_masonry_drift_check()` — argument handling (lines 374–388)

```python
def _tool_masonry_drift_check(args: dict) -> dict:
    agent_db_path_str = args.get("agent_db_path", "")          # NO DEFAULT
    registry_path_str = args.get(
        "registry_path", str(_REPO_ROOT / "masonry" / "agent_registry.yml")  # HAS DEFAULT
    )

    if not agent_db_path_str:
        return {"error": "agent_db_path is required", "reports": []}  # ← FIRES FOR KILN
```

### Downstream impact

| Caller | Behavior without fix |
|---|---|
| Kiln DRIFT CHECK button | Receives `{"error": "agent_db_path is required", "reports": []}` |

Note: Even if fixed, D7.1 establishes that the tool would return `{"reports": [], "count": 0}` due to empty `verdicts` in `agent_db.json`. Both bugs compound.

## Fix Specification

Change `agent_db_path` from required to optional with a default:

```python
def _tool_masonry_drift_check(args: dict) -> dict:
    agent_db_path_str = args.get(
        "agent_db_path",
        str(_REPO_ROOT / "agent_db.json"),          # ← add default
    )
    registry_path_str = args.get(
        "registry_path", str(_REPO_ROOT / "masonry" / "agent_registry.yml")
    )

    # Remove the "required" error check — path always has a default now
    agent_db_path = Path(agent_db_path_str)
```

And update the tool schema to remove `agent_db_path` from `"required"`.

## Verdict

**FAILURE** — `agent_db_path` has no default despite being a deterministic path from `_REPO_ROOT`. Kiln and other non-path-aware callers receive an error response on every invocation. Fix: add `str(_REPO_ROOT / "agent_db.json")` as the default and remove from `"required"` list. Severity is Low because D7.1 already establishes that drift detection returns no reports regardless — the path bug compounds an already non-functional feature.
