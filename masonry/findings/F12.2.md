## Change Applied

**File**: `masonry/scripts/sync_verdicts_to_agent_db.py`

```python
# When scope is narrowed (--questions-md specified), clear old verdicts/confidences
# for ALL agents before writing so stale data from prior broader scans is purged.
if questions_md_path is not None:
    for entry in agent_db.values():
        if isinstance(entry, dict):
            entry["verdicts"] = []
            entry["confidences"] = []
```

Applied in `sync_verdicts()` after `_valid_agent_name` filter, before the write loop. The full scope-clear + write sequence is:
1. Clear all agents' verdicts/confidences (when `questions_md_path` is not None)
2. Write only the agents found in the current scoped scan
3. Result: agent_db reflects exactly and only the current scope

## Design Choice

Option A (zero-all before write) was chosen over Option B (selective delete) because:
- Simpler — one clear loop, then one write loop
- Deterministic — same result regardless of prior state
- Correct for the masonry use case — the masonry scope IS the authoritative view
- Backward compatible — when `--questions-md` is NOT specified (full scan), no clearing occurs
