# Finding: D2.3 — DSPy Drift Detector Has Two Correctness Defects: baseline=0.0 False Negative and UNCALIBRATED Misclassification

**Question ID**: D2.3
**Status**: DONE
**Verdict**: FAILURE
**Agent**: diagnose-analyst
**Severity**: Low
**Confidence**: 1.0

## Summary

Two correctness defects confirmed in the DSPy pipeline:

1. **`baseline_score == 0.0` false negative** (`drift_detector.py` lines 81-84): When an agent has a baseline score of 0.0 (new/uncalibrated agent), `drift_pct` is hardcoded to 0.0 regardless of `current_score`. An agent with baseline 0.0 that consistently produces FAILURE verdicts (`current_score = 0.0`) reports `alert_level = "ok"` and "performing within acceptable range." The agent will never be flagged for re-optimization through the drift pathway.

2. **"UNCALIBRATED" verdict scored as 0.0** (`drift_detector.py` lines 22-42): "UNCALIBRATED" is absent from `_OK_VERDICTS` and `_PARTIAL_VERDICTS` and therefore scores 0.0 (treated as failure). Research-analyst findings with verdict "UNCALIBRATED" (meaning insufficient calibration data — a data gap, not a quality failure) would penalize research-analyst in drift scoring. Current impact is low because findings in this campaign's format lack inline agent attribution (`"agent"` field), causing `score_example()` to exclude them from training data entirely.

## Evidence

### Defect 1: baseline_score == 0.0 False Negative

**Code** (`drift_detector.py` lines 81-84):
```python
if baseline_score == 0.0:
    drift_pct = 0.0
else:
    drift_pct = (baseline_score - current_score) / baseline_score * 100
```

**Scenario: New agent with all-FAILURE verdicts**
```
agent_name:       "diagnose-analyst"
baseline_score:   0.0   (agent has not been optimized → agent_db score = 0.0)
recent_verdicts:  ["FAILURE", "FAILURE", "FAILURE", "FAILURE"]
current_score:    0.0   (all verdicts score 0.0)

drift_pct:        0.0   (forced by baseline_score == 0.0 branch)
alert_level:      "ok"  (0.0 < 10 → "ok")
recommendation:   "diagnose-analyst is performing within acceptable range (drift 0.0%)..."
```

The drift detector reports an agent with four consecutive FAILURE verdicts as "performing within acceptable range."

**Why this matters**: New agents start with `score = 0.0` in agent_db. The intention of the guard is to avoid division-by-zero. But the guard overcorrects — it reports the situation as "no drift" rather than "baseline unknown, cannot assess." For an agent with zero baseline AND zero current performance, the correct recommendation is "insufficient baseline data — run optimization before drift assessment."

**Correct behavior**: When `baseline_score == 0.0`:
- If `recent_verdicts` is empty: current behavior (skip → stable) is correct
- If `recent_verdicts` is non-empty: should report `alert_level = "warning"` with a "baseline not established" message, regardless of `current_score`

### Defect 2: UNCALIBRATED Misclassified as Failure

**Code** (`drift_detector.py` lines 22-32 + `_score_verdict` lines 35-42):
```python
_OK_VERDICTS = frozenset({
    "HEALTHY", "FIXED", "COMPLIANT", "CALIBRATED", "CONFIRMED", "OK",
    "IMPROVEMENT", "PROMISING",
})
_PARTIAL_VERDICTS = frozenset({
    "WARNING", "PARTIAL", "PROBABLE", "POSSIBLE",
})
# Everything else = failure (0.0)

def _score_verdict(verdict: str) -> float:
    v = verdict.strip().upper()
    if v in _OK_VERDICTS: return 1.0
    if v in _PARTIAL_VERDICTS: return 0.5
    return 0.0  # ← "UNCALIBRATED" falls here
```

"UNCALIBRATED" is not in either set → scores 0.0.

This is semantically incorrect. "UNCALIBRATED" means "the parameter cannot be assessed without empirical measurement" — it reflects a research data gap, not a quality failure by research-analyst. Scoring it 0.0 in drift would penalize research-analyst for accurately reporting calibration limitations.

**Current impact assessment**: Low, because the training extractor's `score_example()`:
```python
def score_example(finding: dict[str, Any], agent_db: dict[str, Any]) -> float:
    agent_name = finding.get("agent")  # ← research findings don't have "agent" field
    if not agent_name:
        return 0.0  # excluded from training
```

Findings written by this campaign don't include `**Agent**:` fields in the body (only questions.md has the agent). So `score_example` returns 0.0 for all current findings → all excluded from training data. The UNCALIBRATED misclassification has no effect on current training.

**Future risk**: If a future schema version adds `**Agent**: research-analyst` to finding bodies, UNCALIBRATED findings would enter training data and score 0.0, penalizing research-analyst unfairly.

### Verdict Taxonomy Gap (Broader Finding)

Beyond UNCALIBRATED, the following verdicts appear in the Wave 1/2 findings and are also absent from `_OK_VERDICTS` / `_PARTIAL_VERDICTS`:

| Verdict | Appears in | Correct interpretation |
|---------|-----------|----------------------|
| `UNCALIBRATED` | R1.1, R1.2 | Data gap, not failure → should be ~0.5 |
| `DIAGNOSIS_COMPLETE` | F2.1, F2.2 | Success (fix spec produced) → should be 1.0 |
| `FAILURE` | D1.x, R1.x, V1.x | Confirmed failure → 0.0 ✓ (correct) |
| `HEALTHY` | V1.1, D2.1 | Confirmed safe → 1.0 ✓ (correct) |

`DIAGNOSIS_COMPLETE` scores 0.0 under the current taxonomy, which would penalize diagnose-analyst and fix implementers for producing successful fix specifications.

### agent_db Integration Gap

`run_drift_check` reads `agent_db.json`:
```python
for agent_name, entry in agent_db.items():
    verdicts = entry.get("verdicts", [])
    if not verdicts:
        continue  # skip agents without verdict history
    baseline_score = entry.get("score", 0.0)
```

The `verdicts` list is populated from agent_db, not directly from finding files. The drift detector doesn't call `training_extractor` — it reads pre-aggregated verdicts from agent_db. The training extractor writes training data for DSPy optimization; the drift detector uses agent_db scores. These are two separate data flows. Defect 2's impact is therefore limited to the (not-yet-implemented) path where training extractor data flows into agent_db verdicts.

## Recommended Fixes

**Defect 1 — baseline_score == 0.0**:
```python
if baseline_score == 0.0:
    if recent_verdicts:
        # New agent with data: flag as needing baseline calibration
        drift_pct = 0.0
        alert_level = "warning"
        recommendation = (
            f"{agent_name} has no established baseline. "
            f"Current score: {current_score:.2f}. Run optimization to set baseline."
        )
        return DriftReport(...)
    else:
        drift_pct = 0.0  # no data + no baseline → assume stable
```

**Defect 2 — Verdict taxonomy**:
```python
_OK_VERDICTS = frozenset({
    "HEALTHY", "FIXED", "COMPLIANT", "CALIBRATED", "CONFIRMED", "OK",
    "IMPROVEMENT", "PROMISING", "DIAGNOSIS_COMPLETE",  # add
})
_PARTIAL_VERDICTS = frozenset({
    "WARNING", "PARTIAL", "PROBABLE", "POSSIBLE", "UNCALIBRATED",  # add
})
```

This promotes UNCALIBRATED to 0.5 (data gap, not failure) and DIAGNOSIS_COMPLETE to 1.0 (successful output).
