**Status**: DONE

**Agent**: research-analyst

## Summary

`sync_verdicts_to_agent_db.py` is a standalone script with no integration into the wave-end workflow. Neither `synthesizer-bl2.md`, `trowel.md`, nor any hook references it. For drift detection to stay current, the script must be run manually after each wave synthesis. Without automated invocation, `agent_db.json["verdicts"]` will be stale from the moment new findings are written.

(no results)

Only the script itself and campaign findings/questions reference `sync_verdicts`.

### `synthesizer-bl2.md` — no verdict sync

`synthesizer-bl2.md` reads `agent_db.json` to display the **Agent Fleet** score table, but never invokes `sync_verdicts_to_agent_db.py`. The synthesizer updates `CHANGELOG.md`, `ARCHITECTURE.md`, and `ROADMAP.md` at wave end — verdict syncing is not in scope.

### `trowel.md` — tracks verdicts in masonry-state.json, not agent_db.json

Trowel tracks verdict counts in `masonry-state.json` (`verdicts: { HEALTHY: N, WARNING: N, FAILURE: N }`) — campaign-level aggregate counts. It records per-question agent performance via `record_run()` but this writes to `agent_db.json` as individual run scores, not the `verdicts` list field that `sync_verdicts_to_agent_db.py` populates.

### `karen.md` — no reference

Karen handles docs, changelogs, and folder audits. No verdict sync integration.

## Integration Plan

The best integration point is **`synthesizer-bl2.md` wave-end step**, since synthesis already:
1. Reads all findings from the wave
2. Updates `agent_db.json` scores (Agent Fleet table)
3. Commits docs changes

A one-line addition after step 2:
```bash
python -m masonry.scripts.sync_verdicts_to_agent_db || echo "[SYNTHESIS] verdict sync failed (non-blocking)"
```

This should be non-blocking (the `|| echo` fallback ensures synthesis doesn't fail if the script errors). The script already handles the case where `agent_db.json` doesn't exist gracefully.

**Alternative**: Add to `karen.md` as a post-synthesis cleanup step invoked by Trowel at wave end alongside CHANGELOG/ROADMAP updates.

## Verdict

**WARNING** — `sync_verdicts_to_agent_db.py` requires manual invocation. Drift detection data will become stale after each wave unless the user remembers to run the script. Integration into `synthesizer-bl2.md` as a non-blocking post-synthesis step is the minimal fix.
