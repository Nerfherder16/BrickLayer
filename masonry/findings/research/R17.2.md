# Finding: R17.2 — Vigil Shows HEALTHY from masonry/ Dir; False Thorn Gone, but Signal Quality Low Due to Missing `**Agent**:` Field in BL2.0 Finding Template

**Severity**: Medium
**Confidence**: 0.90
**Agent**: research-analyst

## Summary

Running `python scripts/run_vigil.py` from `masonry/` dir produces `HEALTHY` with `1 roses, 0 buds, 0 thorns. Roses: unknown`. The false Thorn for diagnose-analyst (R14.1, triggered by BL1.x findings at BL root dir) is no longer present — running from masonry/ dir avoids those BL1.x findings entirely. However, the HEALTHY verdict is a low-quality signal: **109/127 findings (86%) are classified as "unknown" agent** because the standard BL2.0 finding template does not include a `**Agent**:` metadata field. Only the 7 findings written in this session (F16.1 through F17.2) use `**Agent**:` — they have only 1-3 findings per agent, below the `MIN_FINDINGS_FOR_METRICS=5` threshold, so they're excluded from classification. The `scored_all.jsonl` augmentation path also fails silently (path resolves to `masonry/masonry/training_data/scored_all.jsonl` when CWD is masonry/).

### scored_all.jsonl Augmentation Failure

`run_vigil.py` line 53:
```python
jsonl_path = base_dir / "masonry" / "training_data" / "scored_all.jsonl"
```

When `base_dir = masonry/` (self-research run), path resolves to:
```
masonry/masonry/training_data/scored_all.jsonl  ← does not exist
```

`load_scored_all()` returns `{}` silently. The 243 training records in `masonry/training_data/scored_all.jsonl` are never consulted for augmentation. Even if the path were correct, the augmentation only applies to agents **not already in** the findings-dir metrics — since all training record agents (quantitative-analyst, karen, etc.) are not in the findings-dir metrics (their findings show as "unknown"), they would have been eligible candidates, but the path failure prevents this.

**Corrected path would be** (for self-research mode):
```python
# Option A: smart detection
if (base_dir / "training_data").exists():
    jsonl_path = base_dir / "training_data" / "scored_all.jsonl"
else:
    jsonl_path = base_dir / "masonry" / "training_data" / "scored_all.jsonl"
```

3. Fix `load_scored_all` path for self-research mode (minor, augmentation path is secondary)
