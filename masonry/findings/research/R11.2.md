## Summary

The F10.2 `cmd.exe /c` fix in `runBackground()` correctly handles both absolute ruff paths with spaces and `npx` .cmd shim resolution without regression. When `spawn("cmd", ["/c", absolutePath, ...args], {shell: false})` is called, Node.js quotes space-containing arguments when building the `CreateProcess` command line — cmd.exe receives the path correctly quoted. The synchronous `run()` calls (for `ruff check`) use template strings with explicit quotes (`"${ruff}" check "${winPath}"`) and are unaffected by the `runBackground()` change.

## Analysis

### Code path for `findRuff()` returning an absolute path

```javascript
// masonry-lint-check.js line 184
runBackground(ruff, ["format", winPath], projectRoot);
```

On Windows with `ruff = "C:\\Users\\trg16\\AppData\\Local\\Python\\pythoncore-3.14-64\\Scripts\\ruff.exe"`:

```javascript
// runBackground() after F10.2
const finalCmd = "cmd";
const finalArgs = ["/c", "C:\\Users\\...\\ruff.exe", "format", "C:\\path with spaces\\file.py"];
spawn("cmd", finalArgs, { shell: false, ... });
```

When `shell: false`, Node.js's Windows `CreateProcess` binding builds the command-line by quoting any argument that contains spaces. The effective command line becomes:

```
cmd.exe /c "C:\Users\...\ruff.exe" format "C:\path with spaces\file.py"
```

cmd.exe with `/c "executable" arg1 arg2` correctly invokes the quoted executable. **No regression for absolute paths with spaces.**

### Before F10.2 (`shell: true`) — why it broke

```javascript
// Old: spawn(ruff, ["format", winPath], { shell: true })
// Node + shell:true joins without quoting:
// cmd.exe /c C:\path with spaces\ruff.exe format C:\path with spaces\file.py
// cmd.exe splits on spaces → "C:\path" is not found → silent failure
```

### `npx` resolution after F10.2

```javascript
// masonry-lint-check.js line 196
runBackground("npx", ["prettier", "--write", filePath], projectRoot);
// → spawn("cmd", ["/c", "npx", "prettier", "--write", filePath], {shell: false})
```

cmd.exe receives `/c npx prettier --write filePath`. cmd.exe resolves `npx` by searching the system `PATH` for `npx.cmd` (the .cmd shim). This is the same mechanism as before — cmd.exe's PATH resolution works identically whether invoked with `shell: true` or `shell: false`, because in both cases cmd.exe is the process executing the lookup.

**Result: npx .cmd shim resolution is unaffected. ✓**

### Synchronous `run()` calls are untouched

```javascript
// masonry-lint-check.js line 187
const check = run(`"${ruff}" check "${winPath}"`, cwdOpts);
```

`run()` uses `execSync` with a shell string. Explicit `"${ruff}"` quoting handles spaces in the absolute path. This code path was not changed by F10.2. **No regression. ✓**
