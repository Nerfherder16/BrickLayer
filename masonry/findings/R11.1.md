### New Issue: Sync Is Update-Only, Not Replace-All

The fix needed for the stale verdict problem is one of:
1. **Option A** (simplest): When `--questions-md` is specified, zero out verdicts for ALL agents before writing the masonry-scoped results. This ensures agent_db reflects only the specified scope.
2. **Option B**: Add a `--clear-scope` flag that explicitly deletes `verdicts` for any agent not found in the current sync.
3. **Option C**: Store verdicts per-project in agent_db (e.g., `verdicts: {masonry: [...], adbp: [...]}`) and have drift check use the project-scoped list.

Option A is the most straightforward and matches the intended use case.
