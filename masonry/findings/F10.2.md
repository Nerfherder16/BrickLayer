# Finding: F10.2 — Fix `runBackground()` Windows Path-With-Spaces Bug (R6.1)

**Question ID**: F10.2
**Status**: DONE
**Verdict**: FIX_APPLIED
**Agent**: fix-implementer
**Severity**: Low
**Confidence**: 0.98

## Summary

`runBackground()` in `masonry-lint-check.js` now uses the F4.2 pattern: `spawn("cmd", ["/c", cmd, ...args], { shell: false })` on Windows instead of `spawn(cmd, args, { shell: true })`. This eliminates the path-with-spaces breakage (R6.1 FAILURE): Node.js no longer joins args into a space-separated string without quoting, and `.cmd` shims (npx) are still resolved via `cmd.exe`.

## Root Cause (from R6.1)

`shell: true` on Windows causes Node.js to call `cmd.exe /c <cmd> <arg1> <arg2>` with args joined by spaces. File paths with spaces (e.g., `C:\Users\trg16\My Documents\file.py`) were split at the space — `ruff format C:\Users\trg16\My` with `Documents\file.py` as an unrecognized argument. All three background formatters (ruff, prettier, eslint) failed silently because `stdio: "ignore"` discards all output.

## Change Applied

**File**: `masonry/src/hooks/masonry-lint-check.js` lines 22-34

```javascript
// Before
function runBackground(cmd, args, cwd) {
  try {
    const proc = spawn(cmd, args, {
      detached: true,
      stdio: "ignore",
      cwd,
      shell: process.platform === "win32",
    });
    proc.unref();
  } catch {
    // ignore spawn errors — formatting is best-effort
  }
}

// After
function runBackground(cmd, args, cwd) {
  try {
    // On Windows, wrap in cmd.exe to resolve .cmd shims (npx, etc.) without
    // shell: true — shell: true joins args without quoting, breaking paths
    // with spaces (R6.1 / F10.2).
    const finalCmd = process.platform === "win32" ? "cmd" : cmd;
    const finalArgs = process.platform === "win32" ? ["/c", cmd, ...args] : args;
    const proc = spawn(finalCmd, finalArgs, {
      detached: true,
      stdio: "ignore",
      cwd,
      shell: false,
    });
    proc.unref();
  } catch {
    // ignore spawn errors — formatting is best-effort
  }
}
```

**Why this works**: With `shell: false` and an explicit `["cmd", "/c", "npx", "--write", "C:\\path with spaces\\file.js"]`, Node.js passes each argument as a separate element to the Win32 `CreateProcess` API. cmd.exe receives `/c npx --write "C:\path with spaces\file.js"` with proper argument boundaries preserved. The `.cmd` extension resolution still works via cmd.exe.

**Matches F4.2 pattern**: `llm_router.py` uses the same approach for spawning claude.exe: `["cmd", "/c", "claude", ...]` with `shell: false`.

## Verdict

**FIX_APPLIED** — R6.1's FAILURE is resolved. Background formatters on Windows will now correctly process files in paths with spaces. The fix is identical in structure to the F4.2 approach already proven in `llm_router.py`.
