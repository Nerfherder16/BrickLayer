## Summary

`route_llm()` now validates the LLM-returned `target_agent` against the registry before constructing a `RoutingDecision`. Hallucinated agent names (e.g., "general-assistant", "code-helper", "Claude") are rejected with a stderr log and `None` return, identical to a timeout or parse failure.

## Change Applied

### `src/routing/llm_router.py`

**Before:**
```python
target = parsed.get("target_agent")
if not target:
    print("[llm_router] No target_agent in LLM response.", file=sys.stderr)
    return None

reason = parsed.get("reason", "LLM routing")[:100]

return RoutingDecision(target_agent=target, ...)
```

**After:**
```python
target = parsed.get("target_agent")
if not target:
    print("[llm_router] No target_agent in LLM response.", file=sys.stderr)
    return None

# Validate against registry — reject hallucinated agent names
registry_names = {a.name for a in registry}
if target not in registry_names:
    print(f"[llm_router] LLM returned unknown agent '{target}' not in registry.", file=sys.stderr)
    return None

reason = parsed.get("reason", "LLM routing")[:100]

return RoutingDecision(target_agent=target, ...)
```

## Why This Fix Is Correct

The `registry` parameter is the live registry passed by the router — the same list used for semantic routing. The membership check is O(1) via set construction (`registry_names = {a.name for a in registry}`). No hardcoded agent names.

Edge cases:
- **Empty registry**: `registry = []` → `registry_names = set()` → all names fail → `route_llm()` returns `None`. Correct: the early-exit check for empty registry in `router.py` should prevent `route_llm()` from being called with empty registry, but this is defensive.
- **Valid name**: in set → passes through to `RoutingDecision`. No behavior change.
- **Invalid name**: not in set → `None` returned, falls to Layer 4. Correct.

## Security Improvement

Without this check, a malformed or adversarially-prompted LLM response could route to an arbitrary string as `target_agent`. Mortar's agent dispatch layer would then attempt to spawn or communicate with a non-existent agent, producing an error that could confuse the user or disrupt the session. The check closes this vector.

## Verdict

**FIX_APPLIED** — Registry membership validation in place. All hallucinated agent names will now be rejected at Layer 3 with a diagnostic log message.
