### E2: `_record_id()` helper added to eval_agent.py

```python
def _record_id(record: dict) -> str:
    """Stable identifier for a record — uses question_id if present, else SHA-256 hash of content."""
    if record.get("question_id"):
        return str(record["question_id"])
    content = json.dumps(record, sort_keys=True)
    return hashlib.sha256(content.encode()).hexdigest()[:16]
```

Helper added to `masonry/scripts/eval_agent.py`. `held_out_ids` captured from the held-out slice and returned in the result dict: `result["held_out_ids"] = held_out_ids`.

## Analysis

The convergence trap (E5 in P-w40.1) required eval and optimization to use disjoint record sets. The `_record_id()` approach handles both cases — records with `question_id` fields use that stable identifier; records without use a content hash. The `excluded_ids` parameter flows cleanly through the four-function call chain without disrupting optional callers (callers that don't pass `excluded_ids` default to `None`, which excludes nothing).

The MIN_TIER_RECORDS guard prevents a degenerate case where exclusion removes all records from one tier, causing `_tier_examples()` to return empty tiers that would produce nonsensical optimization prompts.
