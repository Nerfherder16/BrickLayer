## Summary

`_load_registry()` in `router.py` now logs both failed path attempts before returning `[]`. Operators can now distinguish "missing file at primary path" from "missing file at both paths" and diagnose wrong-CWD errors without reading source code.

## Changes Applied

### `src/routing/router.py` — `_load_registry()`

**Before:**
```python
def _load_registry(project_dir: Path) -> list[AgentRegistryEntry]:
    registry_path = project_dir / "masonry" / "agent_registry.yml"
    if not registry_path.exists():
        # Also try relative to CWD
        fallback = Path("masonry") / "agent_registry.yml"
        if fallback.exists():
            return load_registry(fallback)
        return []  # ← silent empty return
    return load_registry(registry_path)
```

**After:**
```python
def _load_registry(project_dir: Path) -> list[AgentRegistryEntry]:
    registry_path = project_dir / "masonry" / "agent_registry.yml"
    if not registry_path.exists():
        print(f"[ROUTER] Registry not found at {registry_path}", file=sys.stderr)
        # Also try relative to CWD
        fallback = Path("masonry") / "agent_registry.yml"
        if fallback.exists():
            return load_registry(fallback)
        print(f"[ROUTER] Registry not found at {fallback} (CWD fallback) — returning empty registry", file=sys.stderr)
        return []
    return load_registry(registry_path)
```

## Logging Coverage Post-Fix

| Scenario | Logs emitted |
|---|---|
| Primary path found | (no log — success path) |
| Primary path missing, CWD fallback found | `[ROUTER] Registry not found at {primary}` |
| Both paths missing (empty return) | `[ROUTER] Registry not found at {primary}` + `[ROUTER] Registry not found at {fallback} (CWD fallback) — returning empty registry` |

Combined with `route()`'s existing log `[ROUTER] Layer fallback resolved: user (registry empty, confidence 0.0)`, a full trace now looks like:

```
[ROUTER] Registry not found at C:\Users\trg16\Dev\project\masonry\agent_registry.yml
[ROUTER] Registry not found at masonry\agent_registry.yml (CWD fallback) — returning empty registry
[ROUTER] Layer fallback resolved: user (registry empty, confidence 0.0)
```

This clearly identifies that the registry YAML is missing, with both attempted paths shown, and explains why all requests are falling to Layer 4.

## Verdict

**FIX_APPLIED** — Path-level diagnostic logs added. V1.4 carry-forward resolved. D1.3 root cause (silent empty return) addressed.
