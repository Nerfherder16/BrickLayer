## Summary

Added a `backend` parameter to `configure_dspy()`, `optimize_agent()`, and `optimize_all()` in
`masonry/src/dspy_pipeline/optimizer.py`. When `backend="ollama"` is passed, `configure_dspy`
constructs an `ollama_chat/<model>` LM pointing at `http://192.168.50.62:11434` instead of
calling the Anthropic API. The Anthropic path remains the default (`backend="anthropic"`).
The `--backend` CLI flag was also added to `masonry/scripts/run_optimization.py`, and the
MCP server handler in `masonry/mcp_server/server.py` was updated to read and forward a
`backend` argument from tool args.

## Evidence

### Fix Applied

**File**: `masonry/src/dspy_pipeline/optimizer.py`

`configure_dspy` — lines 73–90 (before → after):

Before:
```python
def configure_dspy(model: str = "claude-sonnet-4-6") -> None:
    """Configure DSPy with an Anthropic LM (requires ANTHROPIC_API_KEY)."""
    lm = dspy.LM(f"anthropic/{model}")
    dspy.configure(lm=lm)
```

After:
```python
def configure_dspy(model: str = "claude-sonnet-4-6", backend: str = "anthropic") -> None:
    """Configure DSPy with the specified LM backend."""
    if backend == "ollama":
        lm = dspy.LM(
            f"ollama_chat/{model}",
            api_base="http://192.168.50.62:11434",
            max_tokens=4096,
        )
    else:
        lm = dspy.LM(f"anthropic/{model}")
    dspy.configure(lm=lm)
```

`optimize_agent` signature — added `backend: str = "anthropic"` parameter (line 101).

`optimize_all` signature — added `backend: str = "anthropic"` parameter; passes it through
to the `optimize_agent` call (line 201+ area).

**File**: `masonry/scripts/run_optimization.py`

- `run()` now accepts `backend: str = "anthropic"` and passes it to `configure_dspy(backend=backend)`
  and `optimize_agent(..., backend=backend)`.
- `_main()` argparse now includes `--backend {anthropic,ollama}` flag.

**File**: `masonry/mcp_server/server.py`

- `_tool_masonry_optimize_agent` reads `backend = args.get("backend", "anthropic")` and passes
  it to both `configure_dspy(model=model, backend=backend)` and
  `optimize_agent(..., backend=backend)`.
