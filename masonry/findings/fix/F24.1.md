## Summary

`masonry/scripts/run_optimization.py` previously had no CLI surface to control MIPROv2 trial count or validation set size, forcing source file edits for every experiment run. The fix adds `--num-trials` and `--valset-size` argparse flags and threads them end-to-end through `run()` → `optimize_agent()` → `optimizer.compile()`.

## Fix Applied

### `masonry/scripts/run_optimization.py`

1. Extended `run()` signature to accept `num_trials: int = 10` and `valset_size: int = 100` parameters.
2. Added two new `argparse` arguments to `_main()`:
   - `--num-trials` (type=int, default=10, help="Number of Bayesian optimization trials (default: 10)")
   - `--valset-size` (type=int, default=100, help="Validation set size for trials (default: 100)")
3. Forwarded both values from `args` into the `run()` call in `_main()`.
4. Forwarded both values from `run()` into the `optimize_agent()` call.
5. Added `[init]` log lines to echo the effective values at startup.

### `masonry/src/dspy_pipeline/optimizer.py`

1. Extended `optimize_agent()` signature to accept `num_trials: int = 10` and `valset_size: int = 100`.
2. Passed `num_trials=num_trials` and `valset_size=valset_size` directly into `optimizer.compile()`.

### Before (optimizer.compile call)
```python
optimized = optimizer.compile(
    module,
    trainset=trainset,
    max_bootstrapped_demos=3,
    max_labeled_demos=3,
)
```

### After (optimizer.compile call)
```python
optimized = optimizer.compile(
    module,
    trainset=trainset,
    max_bootstrapped_demos=3,
    max_labeled_demos=3,
    num_trials=num_trials,
    valset_size=valset_size,
)
```
