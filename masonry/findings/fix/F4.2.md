## Summary

The Windows-specific injection risk in `llm_router.py` is resolved. The `shlex.quote(full_prompt)` + `shell=True` pattern has been replaced with list-form `["cmd", "/c", "claude", "--model", ..., "-p", full_prompt]` for Windows, and `["claude", ..., "-p", full_prompt]` for all other platforms. `shell=True` is no longer used.

## Changes Applied

### `src/routing/llm_router.py` — Windows subprocess rewrite

**Before:**
```python
_is_windows = platform.system() == "Windows"
if _is_windows:
    import shlex
    cmd_str = f"claude --model {_LLM_MODEL} --print -p {shlex.quote(full_prompt)}"
else:
    cmd_str = None

result = subprocess.run(
    cmd_str if _is_windows else ["claude", "--model", _LLM_MODEL, "--print", "-p", full_prompt],
    capture_output=True, text=True, timeout=_LLM_TIMEOUT, shell=_is_windows,
)
```

**After:**
```python
if platform.system() == "Windows":
    cmd = ["cmd", "/c", "claude", "--model", _LLM_MODEL, "--print", "-p", full_prompt]
else:
    cmd = ["claude", "--model", _LLM_MODEL, "--print", "-p", full_prompt]

result = subprocess.run(cmd, capture_output=True, text=True, timeout=_LLM_TIMEOUT)
```

## Why `["cmd", "/c", "claude", ...]`

On Windows, the Claude Code CLI installs as `claude.cmd` — a `.cmd` file which is not a native executable. `subprocess.run(["claude", ...], shell=False)` cannot find `.cmd` files. The options are:

1. `shell=True` with a string argument — **injection risk** via `shlex.quote` (POSIX escaping only)
2. `["cmd", "/c", "claude", ...]` with `shell=False` — **chosen**: cmd.exe resolves `.cmd` files; Python's `subprocess.list2cmdline()` applies Windows-appropriate double-quote escaping on each element before passing to CreateProcess; no POSIX escaping
3. `["claude.cmd", ...]` explicitly — fragile (assumes exact extension)

Option 2 is the correct Windows approach: cmd.exe handles `.cmd` extension resolution while Python controls argument quoting via `list2cmdline`, eliminating the `shlex.quote` POSIX mismatch.

## Security Impact

| Vector | Before | After |
|--------|--------|-------|
| Prompt with `&` (cmd AND) | Shell command injection risk | Escaped by list2cmdline |
| Prompt with `|` (pipe) | Shell injection risk | Escaped by list2cmdline |
| Prompt with `>` / `<` (redirect) | Shell injection risk | Escaped by list2cmdline |
| Prompt with `"` (double quote) | Escaping ambiguous | Escaped by list2cmdline |
| Prompt with `'` (single quote) | `shlex.quote` wraps in `'...'` — unrecognized by cmd.exe | Irrelevant (list2cmdline uses double quotes) |

Confirmed: no other `shell=True` in `llm_router.py`. The `import shlex` was inline (inside the `if _is_windows:` block) and is fully removed.
