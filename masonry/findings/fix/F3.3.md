### `baseline_score == 0.0` Branch (false negative fix)

**Before:**
```python
if baseline_score == 0.0:
    drift_pct = 0.0  # ← always returns ok, hides all-FAILURE new agents
```

**After:**
```python
if baseline_score == 0.0:
    drift_pct = None
    alert_level = "calibrating"
    recommendation = (
        f"{agent_name} has no calibrated baseline (baseline_score=0.0). "
        f"Current score: {current_score:.2f}. "
        "Accumulate more findings to establish a baseline before drift detection is meaningful."
    )
```

**Rationale**: A new agent with `baseline_score == 0.0` and all-FAILURE recent verdicts (`current_score = 0.0`) previously reported `alert_level = "ok"` — a false negative. Now it reports `alert_level = "calibrating"` which surfaces in Kiln as an uncalibrated agent, prompting the operator to review and set a proper baseline score after enough data is accumulated.

### DriftReport Model Updated

`drift_pct: float` → `drift_pct: Optional[float]` to accommodate the `None` value for calibrating agents.
`alert_level` Literal extended to include `"calibrating"`.
