## Root Cause

The `([^\n]+)` capture group consumed all non-newline characters following `**Agent**:`. Two failure modes:

1. **Spurious match before metadata field** (R17.2): The regex found `` `**Agent**:` `` inside the H1 title, which appears before the real metadata block. The backtick-quoted phrase is a literal occurrence of the pattern, causing the wrong line to be matched first.

2. **Body-text match instead of metadata field** (R14.1): No `**Agent**:` metadata field exists. The pattern matched `**Agent**:` inside a paragraph, then captured the rest of the line — including backtick spans and additional sentences.

The `([\w-]+)` fix constrains the capture to word characters and hyphens only, which matches agent slugs exactly and stops at any non-slug character (backtick, space, punctuation), eliminating both corruption modes.

## Fix Applied

Changed line 49 of `masonry/scripts/score_findings.py`:
- `([^\n]+)` → `([\w-]+)`

This is a targeted single-character-group change. No other code was modified.
