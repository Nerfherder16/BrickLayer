## Why This Fix Is Correct

`build_dataset()` shapes ALL examples (regardless of agent) to the `ResearchAgentSig` field set:
- Inputs: `question_text`, `project_context`, `constraints`
- Outputs: `verdict`, `severity`, `evidence`, `mitigation`, `confidence`

`DiagnoseAgentSig` expects inputs `symptoms`, `affected_files`, `prior_attempts` and outputs `root_cause`, `fix_strategy`, `affected_scope`, `confidence` — none of which are populated by `build_dataset()`.

The `DiagnoseAgentSig` specialization makes semantic sense (diagnose agents should reason about symptoms, not questions), but it requires `build_dataset()` to produce a separate example format for diagnose agents. That is future work. For now, using `ResearchAgentSig` for all agents is correct and complete given the current data pipeline.
