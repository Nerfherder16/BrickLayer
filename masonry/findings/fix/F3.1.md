## Summary

The fix specified in F2.1 has been implemented. The plugin-level `hooks/hooks.json` `hooks` object has been emptied — all 10 hook registrations removed. The global `~/.claude/settings.json` remains the sole source of hook registrations and provides a strict superset of coverage including additional hooks (masonry-approver, masonry-lint-check, masonry-design-token-enforcer, masonry-context-monitor, masonry-tdd-enforcer, masonry-session-summary) that were never present in the plugin file.

## Change Applied

**Before** (`hooks/hooks.json`):
```json
{
  "description": "Masonry research-loop hooks",
  "hooks": {
    "SessionStart": [...masonry-session-start.js...],
    "UserPromptSubmit": [...masonry-register.js...],
    "PreToolUse": [...masonry-context-safety.js (ExitPlanMode)...],
    "PostToolUse": [...masonry-observe.js (no matcher)..., ...masonry-guard.js (no matcher)...],
    "PostToolUseFailure": [...masonry-tool-failure.js...],
    "SubagentStart": [...masonry-subagent-tracker.js...],
    "Stop": [...masonry-stop-guard.js..., ...masonry-build-guard.js..., ...masonry-ui-compose-guard.js...]
  }
}
```

**After** (`hooks/hooks.json`):
```json
{
  "description": "Masonry research-loop hooks",
  "hooks": {}
}
```

| masonry-build-guard | Stop | ✓ removed | ✓ present |

**Additional note on PostToolUse matcher improvement**: The plugin registered masonry-observe and masonry-guard with NO matcher (fires on all PostToolUse events, including Bash, Read, Glob, etc.). The global settings.json registration has `matcher: "Write|Edit"`. Removing the plugin registrations means these hooks now fire only on Write and Edit events, which is the intended behavior. This is a functional improvement — observe and guard no longer receive spurious notifications for non-file-modifying tool calls.
