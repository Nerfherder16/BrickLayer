## Summary

`build_dataset()` in `training_extractor.py` now injects `project-brief.md` content (up to 2000 chars) as `project_context` for each training example. A new `_load_project_brief()` helper walks up from each finding's `source_file` path to find the `findings/` ancestor, then reads `project-brief.md` from the parent directory. A per-finding cache avoids repeated disk reads. The `extract_finding()` function was also hardened to use `errors="replace"` after a UnicodeDecodeError was encountered in one finding file.

End-to-end validation confirms the full training pipeline now produces 89 examples across 5 agents with `project_context len: 2000` for all attributed findings.

## Changes Applied

### 1. `_PROJECT_BRIEF_MAX_CHARS` constant

```python
_PROJECT_BRIEF_MAX_CHARS = 2000
```

### 2. `_load_project_brief()` helper

```python
def _load_project_brief(source_file: str) -> str:
    p = Path(source_file).resolve()
    for ancestor in p.parents:
        if ancestor.name == "findings":
            brief_path = ancestor.parent / "project-brief.md"
            try:
                text = brief_path.read_text(encoding="utf-8")
                return text[:_PROJECT_BRIEF_MAX_CHARS]
            except (OSError, FileNotFoundError):
                return ""
    return ""
```

Walks up the path until a `findings/` directory is found, then reads `project-brief.md` from the project root. Returns empty string gracefully if not found. For wave subdirs (`findings/wave1/R1.1.md`), the walk finds `findings/wave1/` first, skips it (name is "wave1"), then finds `findings/` next — correct.

Wait: the walk checks `ancestor.name == "findings"`. For `findings/wave1/R1.1.md`, the parents are `[findings/wave1/, findings/, masonry/, ...]`. It finds `findings/` and reads `findings/../project-brief.md` = `masonry/project-brief.md`. ✓

### 3. `build_dataset()` cache + injection

```python
_brief_cache: dict[str, str] = {}

for finding in findings:
    ...
    source_file = finding.get("source_file", "")
    if source_file not in _brief_cache:
        _brief_cache[source_file] = _load_project_brief(source_file)
    project_context = _brief_cache[source_file]

    example = {
        "question_text": ...,
        "project_context": project_context,   # ← was ""
        ...
    }
```

### 4. `extract_finding()` encoding hardening

```python
text = finding_path.read_text(encoding="utf-8", errors="replace")
```

A UnicodeDecodeError (byte 0x97 = Windows em-dash) was encountered during live testing. `errors="replace"` substitutes unmappable bytes with the Unicode replacement character — preserves all semantic content.
