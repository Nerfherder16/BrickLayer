## Summary

The `--backend anthropic` path in `run_optimization.py` routes through DSPy → LiteLLM →
Anthropic API and requires `ANTHROPIC_API_KEY` (or `ANTHROPIC_AUTH_TOKEN`) to be set as an
explicit environment variable. Neither is set in this shell environment. The Claude Code
process does not inject any passthrough credentials that LiteLLM can discover. The MIPROv2
run for research-analyst cannot proceed until auth is provided.

Prerequisites F29.1 and F29.2 are both confirmed DONE: research-analyst training data is
available (56 records, median question_text 500 chars) and `scored_all.jsonl` exists at
`masonry/training_data/scored_all.jsonl` (602 total records). The only remaining blocker is
authentication.

### No native auth path

There is no Claude Code credential passthrough to LiteLLM. The Claude Code session token
used by the Claude Code CLI is not exposed as `ANTHROPIC_API_KEY` and LiteLLM has no
mechanism to discover it. The `anthropic` backend requires an explicit API key.
