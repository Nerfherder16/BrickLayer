## Changes Applied

### New function: `_build_qid_to_agent_map(questions_md_path)`

```python
def _build_qid_to_agent_map(questions_md_path: Path) -> dict[str, str]:
    """Extract question_id → agent_name mapping from a questions.md file.

    Handles both ``**Agent**:`` (Wave 1) and ``**Method**:`` (Wave 2+) fields.
    """
    try:
        text = questions_md_path.read_text(encoding="utf-8")
    except (OSError, FileNotFoundError):
        return {}

    result: dict[str, str] = {}
    blocks = re.split(r"\n---\n", text)
    for block in blocks:
        qid_m = re.search(r"###\s+(\w+\d+\.\d+):", block)
        agent_m = re.search(r"\*\*(?:Agent|Method)\*\*:\s*(\S+)", block)
        if qid_m and agent_m:
            result[qid_m.group(1)] = agent_m.group(1)
    return result
```

Splits `questions.md` on `---` section dividers, extracts `question_id` from `### XX.Y:` headers and agent name from `**Agent**:` or `**Method**:` fields. Returns 42 entries for the current `questions.md` (36 DONE questions + 6 Wave 4 PENDING questions).

### Modified: `extract_training_data(projects_dir, questions_md_path=None)`

Added `questions_md_path: Path | None = None` parameter. For each `findings/` directory found during scanning:
1. Checks for `questions.md` at `findings_dir.parent` (auto-discovery) or uses explicit override
2. Builds `qid_map` via `_build_qid_to_agent_map()`
3. After each `extract_finding()` call, populates `finding["agent"] = qid_map.get(finding["question_id"])`

### Modified: `build_dataset(projects_dir, agent_db_path, questions_md_path=None)`

Added `questions_md_path` parameter and passes it through to `extract_training_data()`.

## Root Cause Resolved

R3.2 identified two design gaps:
1. **Finding template gap**: BL 2.0 finding format has no `**Agent**:` field — fixed by looking up attribution from `questions.md` at extract time
2. **Extractor gap**: `extract_finding()` never populated "agent" — fixed by `_build_qid_to_agent_map()` + `extract_training_data()` wiring

Option A (questions.md lookup) was chosen over Option B (template change) as it retroactively fixes all 35 existing findings without requiring a finding template migration.
