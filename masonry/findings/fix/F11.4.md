# Finding: F11.4 — Fix `best_score = 0.0` in `optimize_agent()`: Score Lives on Returned Module, Not Optimizer

**Question ID**: F11.4
**Status**: DONE
**Verdict**: FIX_APPLIED
**Agent**: fix-implementer
**Severity**: Low
**Confidence**: 0.96

## Summary

`optimize_agent()` now reads the optimization score from `optimized.score` (the compiled program) instead of `optimizer.best_score` (which does not exist on `MIPROv2`). After `MIPROv2.compile()`, the returned program has its `.score` attribute set to the best trial score (`best_program.score = best_score` at line 665 of `mipro_optimizer_v2.py`). This attribute is available whenever `track_stats=True` (the default). The old `hasattr(optimizer, "best_score")` check always returned False, keeping `result["score"]` at 0.0.

## Root Cause (from R10.3)

`dspy` 3.1.3 `MIPROv2` stores `best_score` as a local variable inside `compile()`. It is NOT exposed as an instance attribute on the optimizer. The compiled program returned by `compile()` receives the score instead (`best_program.score = best_score`, guarded by `self.track_stats` which defaults to `True`).

```python
# mipro_optimizer_v2.py line 663-665
if best_program is not None and self.track_stats:
    best_program.trial_logs = trial_logs
    best_program.score = best_score  # ← THIS is where the score lives
```

## Change Applied

**File**: `masonry/src/dspy_pipeline/optimizer.py`

```python
# Before (always produces best_score = 0.0)
best_score = 0.0
try:
    if hasattr(optimizer, "best_score"):
        best_score = float(optimizer.best_score)
except Exception:
    pass

# After (reads from returned optimized program)
best_score = 0.0
try:
    if hasattr(optimized, "score"):
        best_score = float(optimized.score)
except Exception:
    pass
```

## Verification

Cannot run a full MIPROv2 compile without an active Anthropic API key (as noted in R10.3). However, the fix is correct by source inspection:
1. `MIPROv2` constructor (L63): `track_stats: bool = True` — always True by default
2. `compile()` (L663-665): `if best_program is not None and self.track_stats: best_program.score = best_score`
3. `compile()` return (L680): `return best_program`
4. Therefore: `optimized = optimizer.compile(...)` → `optimized.score` = best optimization score

The `try/except float()` conversion handles the edge case where compilation fails and `optimized` falls back to the unoptimized module (which would have no `.score` attribute), defaulting to 0.0 safely.

## Verdict

**FIX_APPLIED** — `optimize_agent()` now reads the score from `optimized.score` (the compiled program). When a real optimization run completes, `result["score"]` will be the actual best trial score (0.0–1.0 range) instead of a hardcoded 0.0. Kiln will display meaningful optimization quality scores.
