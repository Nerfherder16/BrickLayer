**Before**: Used `max()` to find best agent, no margin check.

**After**: Uses `sorted()` to rank all agents, applies margin check:

```python
# Compute similarities and sort descending
scored = sorted(
    [(agent, _cosine_similarity(request_emb, emb)) for agent, emb in agent_embs],
    key=lambda x: x[1], reverse=True,
)
best_agent, best_score = scored[0]

if best_score < threshold:
    return None

# Margin check: fall through to Layer 3 on thin-margin ties
if len(scored) >= 2:
    second_score = scored[1][1]
    if best_score - second_score < margin_threshold:
        return None

return RoutingDecision(
    target_agent=best_agent.name,
    layer="semantic",
    confidence=best_score,
    reason=f"Semantic match: {best_score:.2f} (margin: {best_score - scored[1][1]:.3f})"[:100]
    if len(scored) >= 2 else f"Semantic match: {best_score:.2f}"[:100],
)
```

Edge cases handled:
- **Single-agent registry**: No margin to compute — routes directly if above threshold
- **Thin margin**: Returns `None` (falls through to Layer 3 for LLM disambiguation)
- **Reason string**: Now includes margin for observability (`"Semantic match: 0.65 (margin: 0.072)"`)
