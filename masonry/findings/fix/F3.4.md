## Summary

The `fallback_reason: Optional[Literal[...]] = None` field has been added to `RoutingDecision` in `src/schemas/payloads.py`, and the Layer 4 return path in `src/routing/router.py` has been updated to populate it. This enables callers to distinguish `"ambiguous"` (genuine multi-agent ambiguity) from `"registry_empty"` (infrastructure failure where no agents were loaded).

## Changes Applied

### `src/schemas/payloads.py` — RoutingDecision field added

```python
class RoutingDecision(BaseModel):
    target_agent: str
    layer: Literal["deterministic", "semantic", "llm", "fallback"]
    confidence: float = Field(ge=0.0, le=1.0)
    reason: str = Field(max_length=100)
    fallback_agents: list[str] = Field(default_factory=list)
    fallback_reason: Optional[Literal[
        "ambiguous", "ollama_timeout", "llm_timeout", "registry_empty", "multi_failure"
    ]] = None
```

**Non-breaking**: The field has `= None` default. All existing callers that construct `RoutingDecision` without specifying `fallback_reason` continue to work. `extra="forbid"` is satisfied because the field is declared.

### `src/routing/router.py` — `_make_fallback()` and populated return sites

Replaced the module-level `_FALLBACK` constant with `_make_fallback(reason)` helper:

```python
def _make_fallback(fallback_reason: str = "ambiguous") -> RoutingDecision:
    return RoutingDecision(
        target_agent="user",
        layer="fallback",
        confidence=0.0,
        reason="Ambiguous request -- asking user for clarification",
        fallback_reason=fallback_reason,
    )
```

Two Layer 4 return sites:
1. **Empty registry** → `_make_fallback("registry_empty")` — fast path before attempting any layers
2. **L1-L3 exhausted** → `_make_fallback("ambiguous")` — all layers ran but none matched
