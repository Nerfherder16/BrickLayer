## Summary

The `re.IGNORECASE` flag has been added to `_MODE_FIELD_RE` in `src/routing/deterministic.py`, and the extracted mode value is now normalized to lowercase before registry lookup. No additional case-sensitivity issues were found in the deterministic layer.

## Changes Applied

### `_MODE_FIELD_RE` ÔÇö added `re.IGNORECASE`

```python
# Before:
_MODE_FIELD_RE = re.compile(r"\*\*Mode\*\*:\s*(\w+)")

# After:
_MODE_FIELD_RE = re.compile(r"\*\*Mode\*\*:\s*(\w+)", re.IGNORECASE)
```

Now correctly matches all three variants:
- `**Mode**: diagnose` (original ÔÇö was working)
- `**mode**: diagnose` (lowercase M ÔÇö was failing)
- `**MODE**: diagnose` (all caps ÔÇö was failing)

### `mode_value` normalization ÔÇö added `.lower()`

```python
# Before:

# After:
mode_value = m.group(1).strip().lower()
```

The registry stores mode strings in lowercase (`"diagnose"`, `"research"`, `"fix"`, etc.). Without normalization, `**Mode**: Diagnose` would extract `"Diagnose"` (capital D), which would not match registry entries. Normalizing to lowercase ensures consistent comparison regardless of the case used in the mode field.
