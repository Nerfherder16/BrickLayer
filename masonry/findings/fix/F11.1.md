# Finding: F11.1 — Fix `AgentRegistryEntry` `extra="forbid"` → `extra="ignore"`; All 46 Agents Now Load

**Question ID**: F11.1
**Status**: DONE
**Verdict**: FIX_APPLIED
**Agent**: fix-implementer
**Severity**: High
**Confidence**: 0.99

## Summary

`AgentRegistryEntry` in `masonry/src/schemas/payloads.py` now uses `model_config = ConfigDict(extra="ignore")` instead of `extra="forbid"`. All 46 agents in `agent_registry.yml` load without validation errors. Previously, 14 agents (including `fix-implementer`, `code-reviewer`, `git-nerd`, etc.) were silently skipped by `registry_loader` due to extra fields added by `masonry-agent-onboard.js` during auto-onboarding.

## Root Cause (from R10.2)

`masonry-agent-onboard.js` adds extra fields to `agent_registry.yml` entries:
- `dspy_status: not_optimized`
- `drift_status: ok`
- `last_score: null`
- `runs_since_optimization: 0`
- `registrySource: frontmatter`

`model_config = ConfigDict(extra="forbid")` caused `AgentRegistryEntry` to raise 5 validation errors per offending agent. `registry_loader` caught the error and printed `[registry_loader] Skipping invalid agent 'fix-implementer': ...` to stderr, silently returning only 30/44 agents.

## Change Applied

**File**: `masonry/src/schemas/payloads.py` — `AgentRegistryEntry`

```python
# Before
model_config = ConfigDict(extra="forbid")

# After
model_config = ConfigDict(extra="ignore")
```

Note: `extra="forbid"` is still correct for other payload models (`QuestionPayload`, `FindingPayload`, `RoutingDecision`, `DiagnosePayload`, `DiagnosisPayload`) where strict field validation is a safety property at agent handoff boundaries. Only `AgentRegistryEntry` needs `extra="ignore"` because it's sourced from a YAML file that is extended by hooks with monitoring fields.

## Validation

```python
registry = load_registry(Path('masonry/agent_registry.yml'))
len(registry)  # 46 — was 30 before the fix
```

No `[registry_loader] Skipping invalid agent` stderr output. All 46 agents including `fix-implementer` (25 verdicts), `code-reviewer`, `git-nerd`, `synthesizer-bl2`, `trowel` are now loaded and eligible for drift checking.

## Verdict

**FIX_APPLIED** — Registry validation now correctly loads all 46 agents. The extra fields from auto-onboarding are silently ignored without schema violations. `masonry_drift_check` can now check all agents that have verdict history.
