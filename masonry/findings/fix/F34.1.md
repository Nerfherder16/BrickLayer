## Summary

`load_training_data_from_scored_all()` in `masonry/scripts/run_optimization.py` hardcoded `"project_context": ""` for every training example. This meant all 57 research-analyst records produced empty `project_context` fields, starving DSPy of the system context needed to optimize prompt quality.

The fix adds an optional `base_dir: Path | None = None` parameter. When `base_dir` is provided, a new `_read_project_brief(base_dir)` helper reads `project-brief.md` directly from that directory (up to 2000 chars) and sets it as `project_context` for all examples in that call. The fallback to `""` is preserved when `base_dir` is None. The call site in `run()` was updated to pass `base_dir=base_dir`.
