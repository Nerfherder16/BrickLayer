## Summary

Three co-occurring defects in `masonry/scripts/score_routing.py` caused the routing scorer to produce only 0 training records against 34 log events. The flat dict collision caused all agents sharing a parent session to overwrite each other (last-write-wins on findings). The agent alias mismatch caused legacy short names (`fix`, `research`, `diagnose`) to be unrecognized. The missing `general-purpose` category caused those start events to score 0 and be dropped. All three were fixed surgically. After the fix, 12 training records are produced from 34 events with correct per-agent attribution.

## Evidence

### Before Fix

- Events scanned: 34
- Sessions matched: 0
- Training records written: 0
- All Wave 18-19 dispatches produced no output due to flat dict collision on session `315da739`, alias mismatch, and missing category.

### Fix Applied

**Defect 1 — Flat dict collision (masonry/scripts/score_routing.py)**

Added `from collections import defaultdict` import.

Changed `findings_by_session: dict[str, dict[str, Any]] = {}` (line ~130) to a compound-key `defaultdict`:

```python
# Before
findings_by_session: dict[str, dict[str, Any]] = {}
for ev in events:
    if ev.get("event") == "finding":
        sid = ev.get("session_id", "")
        if sid:
            findings_by_session[sid] = ev  # last-write-wins collision

# After
findings_by_key: defaultdict[tuple[str, str], list[dict[str, Any]]] = defaultdict(list)
for ev in events:
    if ev.get("event") == "finding":
        sid = ev.get("session_id", "")
        agent_norm = _normalize_agent(ev.get("agent", ""))
        if sid and agent_norm:
            findings_by_key[(sid, agent_norm)].append(ev)  # compound key, no collision
```

Downstream lookup changed from `findings_by_session.get(session_id)` to `findings_by_key.get((session_id, agent), [])[-1]`.

**Defect 2 — Agent alias normalization (masonry/scripts/score_routing.py)**

Added `_AGENT_ALIASES` map and `_normalize_agent()` helper after line 53:

```python
_AGENT_ALIASES: dict[str, str] = {
    "fix": "fix-implementer",
    "research": "research-analyst",
    "diagnose": "diagnose-analyst",
}

def _normalize_agent(agent: str) -> str:
    """Resolve legacy shortened agent names to canonical full names."""
    return _AGENT_ALIASES.get(agent, agent)
```

Applied normalization to both the start-event agent and the finding-event agent at all lookup sites. Also updated `_score_session()` to accept an optional `normalized_agent` parameter so the AGENT_CATEGORIES check uses the canonical name.

**Defect 3 — Missing `general-purpose` category**

In `masonry/src/scoring/rubrics.py` (line 56, after `"trowel": "routing"`):
```python
# general category (catch-all for multi-purpose subagents)
```
