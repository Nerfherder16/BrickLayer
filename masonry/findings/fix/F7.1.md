## Summary

The `_tool_masonry_route()` error fallback in `mcp_server/server.py` has been corrected. The previous fallback `{"error": str(exc), "target_agent": "user", "layer": 4, "confidence": 0.0}` had three schema mismatches against `RoutingDecision` (integer `layer`, missing required `reason`, extra `error` field). The fix replaces integer `4` with string `"fallback"`, adds the required `reason` field, and includes `fallback_agents` and `fallback_reason` to match `decision.model_dump()` shape.

## Changes Applied

### `mcp_server/server.py` — `_tool_masonry_route()` error path

**Before:**
```python
except Exception as exc:
    return {"error": str(exc), "target_agent": "user", "layer": 4, "confidence": 0.0}
```

**After:**
```python
except Exception as exc:
    return {
        "error": str(exc),
        "target_agent": "user",
        "layer": "fallback",
        "confidence": 0.0,
        "reason": f"Router import failed: {str(exc)[:80]}",
        "fallback_agents": [],
        "fallback_reason": "multi_failure",
    }
```

## Why This Fix Is Correct

The corrected fallback now matches the shape of `decision.model_dump()` (the success path) on all schema-required fields:

| Field | Before | After | Schema | Compatible? |
|---|---|---|---|---|
| `target_agent` | `"user"` | `"user"` | `str` | ✓ |
| `layer` | `4` (int) | `"fallback"` (str) | `Literal[...]` | ✓ fixed |
| `confidence` | `0.0` | `0.0` | `float` | ✓ |
| `reason` | absent | `"Router import failed: ..."` | `str` (required) | ✓ fixed |
| `fallback_agents` | absent | `[]` | `list[str]` | ✓ fixed |
| `fallback_reason` | absent | `"multi_failure"` | `Optional[Literal[...]]` | ✓ |
| `error` | `str(exc)` | `str(exc)` | extra (diagnostic) | ~ (preserved for diagnostics) |

The `error` key is retained as a diagnostic field. It would still fail `RoutingDecision(**result)` deserialization due to `extra="forbid"`, but since MCP callers receive the raw JSON dict (not a Pydantic model), this is acceptable for error surfacing. If strict schema compliance is needed in future, move the error text into `reason`.

The `reason` field is truncated to 80 characters (`str(exc)[:80]`) to stay within the `max_length=100` constraint on `RoutingDecision.reason`.

## Verdict

**FIX_APPLIED** — All three V6.1 schema mismatches corrected: `layer` is now `"fallback"` (str), `reason` is present and bounded to 80 chars, `fallback_reason` is `"multi_failure"`. The error and success paths now return structurally compatible dicts.
