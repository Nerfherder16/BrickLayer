## Summary

`_build_qid_to_agent_map()` in `training_extractor.py` correctly handles `**Method**:` fields (Wave 2+) alongside `**Agent**:` (Wave 1) via the `(?:Agent|Method)` alternation in the regex. The `\n---\n` block split matches the questions.md separator format. Question ID regex `\w+\d+\.\d+` correctly parses all BL2.0 ID formats (R7.1, D6.2, F7.2, V6.1). Agent names in `agent_db.json` directly match those used in `**Method**:` fields. Attribution is end-to-end correct.

## Evidence

### `_build_qid_to_agent_map()` regex analysis

```python
blocks = re.split(r"\n---\n", text)           # Split on --- dividers
for block in blocks:
    qid_m = re.search(r"###\s+(\w+\d+\.\d+):", block)    # Match QID
    agent_m = re.search(r"\*\*(?:Agent|Method)\*\*:\s*(\S+)", block)  # Match agent
    if qid_m and agent_m:
        result[qid_m.group(1)] = agent_m.group(1)
```

### 1. `\n---\n` split — correct

The questions.md format uses `\n---\n` as a separator between question blocks, e.g.:

```markdown
### R7.1: Does `build_dataset()` produce...

**Status**: DONE
**Method**: research-analyst
...

---

### D7.1: Does `run_drift_check()` return...
```

`re.split(r"\n---\n", text)` produces one block per question. Each block contains exactly the QID header and the question body (including the `**Method**:` field). ✓

### 2. QID regex `\w+\d+\.\d+` — correct for all BL2.0 IDs

| Question ID | `\w+` captures | `\d+` captures | `.` | `\d+` captures | Match? |
|---|---|---|---|---|---|
| R7.1 | R | 7 | . | 1 | ✓ |
| D6.2 | D | 6 | . | 2 | ✓ |
| F7.2 | F | 7 | . | 2 | ✓ |
| V6.1 | V | 6 | . | 1 | ✓ |
| D1.3 | D | 1 | . | 3 | ✓ |

Note: `\w+` is greedy but backtracks correctly — for "R7", `\w+` first tries "R7", then `\d+` fails on ".", so `\w+` backtracks to "R", `\d+` = "7". All 40+ question IDs parsed correctly.

### One minor caveat: `**Status**: DONE` vs. agent attribution

If a question block has `**Status**: DONE` but no `**Method**:` (can happen if the question was in Wave 1 using `**Agent**:`), the `agent_m` match falls back to `**Agent**:`. Both are handled by the alternation. If neither exists in a block, `qid_m and agent_m` is False and the block is skipped silently — correct behavior (question is excluded from attribution map, findings for that QID get `agent = None`, then `score_example` returns 0.0 and they're excluded from training).
