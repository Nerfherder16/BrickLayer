## Summary

The "Agents: 1" column in `score_all_agents.py` summary table for `score_findings` does NOT mean 1 agent contributed findings — it means only 1 agent has ≥10 records (`agents_with_10_plus` = 1). Five agents actually contribute: quantitative-analyst=36, synthesizer-bl2=9, competitive-analyst=6, regulatory-researcher=5, research-analyst=5 (total=61). `score_findings` scans 675 findings across 7 projects (recall-arch-frontier=275, recall=219, adbp=99, bricklayer-meta=27, bl-audit=23, bricklayer-v2=20, root=12), but 614 score below the 60-point threshold. Masonry findings are confirmed excluded (0 included). The 61→57 reduction in scored_all.jsonl after dedup is caused by 4 cross-project duplicate question_id collisions.

### Why "Agents: 1" in summary table

`score_all_agents.py` populates the `agents_covered` field for score_findings using only `agents_with_10_plus`:

```python
# score_all_agents.py:234-235
"agents_covered": list(sf_summary.get("agents_with_10_plus", {}).keys()),
"agents_10plus": len(sf_summary.get("agents_with_10_plus", {})),
```

The printed "Agents" column = `len(agents_covered)` = 1 (only quantitative-analyst, with 36 records, meets the ≥10 threshold). Other scorers (code, ops) use their actual `agents_covered` lists.

**This is a summary table inconsistency**: `score_findings` → `agents_with_10_plus` count; `score_code_agents` / `score_ops_agents` → `total agents covered`. The columns represent different things per row.

### 61 → 57 dedup reduction in scored_all.jsonl

```
scored_findings.jsonl:  quantitative-analyst=36, synthesizer-bl2=9, ...
scored_all.jsonl:       quantitative-analyst=35, synthesizer-bl2=6, ...
Difference: -1 quantitative-analyst, -3 synthesizer-bl2 = -4 total
```

The dedup key is `f"qid:{question_id}:{agent}"`. Cross-project ID collisions: if two projects both have question_id `1.1` attributed to quantitative-analyst (e.g., adbp and recall both start their wave at question 1.1), the last-write-wins rule eliminates one. Similarly, synthesizer-bl2 synthesis findings use question IDs like `synthesis_wave1` which appear in multiple projects.

### Masonry exclusion confirmed

`masonry` appears in `child.name not in ("findings", "masonry", ".git")` exclusion list. Verified: `len([p for p in paths if 'masonry' in p.parts]) = 0`.

## Verdict

Five agents actually contribute to scored_findings.jsonl. No masonry findings are accidentally included.
