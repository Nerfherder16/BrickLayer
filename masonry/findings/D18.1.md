## Summary

The `_AGENT_PATTERN` regex in `run_vigil.py` is correct and case-insensitive — it matches `**Agent**:` wherever it appears. The root cause is that 109 of 130 findings (84%) were written without the `**Agent**:` field because the BL2.0 finding template never enforced it as a required header field. The field was added sporadically starting with some earlier waves (D2.3, D9.1, R3.2, R7.2, F4.1) and became more consistent from Wave 16 onward, but was never retroactively backfilled for the 109 pre-consistent findings. A bulk backfill script is warranted and low-risk.

## Evidence

### Regex Analysis

`_AGENT_PATTERN = re.compile(r"\*\*agent\*\*\s*:\s*(.+)", re.IGNORECASE)`

- The `re.IGNORECASE` flag means `**Agent**:`, `**AGENT**:`, and `**agent**:` all match
- Pattern correctly handles optional whitespace between `:` and the value
- The regex is not the problem

### Field Count (empirical, run against live findings/)

```
With **Agent**: field:  21  (16.2%)
Without **Agent**: field: 109  (83.8%)
```

This aligns precisely with R17.2's 109/127 observation (R17.2 saw 127 because synthesis files

### Distribution of Findings WITH Agent Field

Not a clean Wave 16+ cutover — the field was inconsistently introduced earlier:
- Sporadic early inclusion: D2.3, D9.1, R3.2, R7.2, F4.1, R13.3, R14.1
- Consistent from Wave 16+: D16.1, F16.1, R16.1, R16.2, F17.1, F17.2, R17.1, R17.2, R18.1

This confirms the template was never mandatory — individual agents sometimes included it, sometimes didn't, until it became part of the standard format in the campaign prompt around Wave 16.

### Backfill Risk Assessment

A backfill script would:
1. Open each `.md` finding that lacks `**Agent**:` field
2. Infer agent from the question ID prefix (D → diagnose-analyst, R → research-analyst,
   V → research-analyst/validate, F → fix-implementer) or leave as `unknown`
3. Insert `**Agent**: {inferred}` after the `**Confidence**:` line

Risk factors:
- Files are append-only artifacts — no running process writes to them
- The field is purely metadata for vigil scoring — no simulation or code depends on it
- Regex insertion after `**Confidence**:` line is well-defined (all 109 missing files have that field)
- A dry-run mode showing proposed changes before commit eliminates risk entirely

**Risk verdict**: LOW. Backfill is safe with a dry-run flag. The existing `backfill_agent_fields.py`
in `scripts/` is likely already staged for this purpose.
