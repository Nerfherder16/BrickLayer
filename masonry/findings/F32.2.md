## Summary

Added `api_key: str | None = None` to `configure_dspy()` in `masonry/src/dspy_pipeline/optimizer.py`. When non-None, the key is forwarded to `dspy.LM()` via `**lm_kwargs`, covering both the Anthropic and Ollama backends. Added matching `--api-key` argument to the `run_optimization.py` CLI argparse and threaded the value through `run()` → `configure_dspy()`. When `api_key` is not supplied, behavior is unchanged — LiteLLM falls back to the `ANTHROPIC_API_KEY` environment variable as before.

```python

# Anthropic branch
```
