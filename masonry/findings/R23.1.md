## Evidence

### Root Cause: Metric Arity Bug

`build_metric()` in `optimizer.py` returned a 2-argument closure:
```python
def metric(example: Any, prediction: Any) -> float:
```

DSPy's MIPROv2 bootstrapper (`dspy/teleprompt/bootstrap.py`) calls the metric as:
```python
metric(example, prediction, trace)
```
The `trace` argument is execution metadata used to determine valid bootstrapping candidates.
Every bootstrap attempt failed with `TypeError: metric() takes 2 positional arguments but 3 were given`.

### Fix Applied (`optimizer.py` line 34)

```python
# Before:
def metric(example: Any, prediction: Any) -> float:

# After:
def metric(example: Any, prediction: Any, trace: Any = None) -> float:
```
