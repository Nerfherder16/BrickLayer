# Finding: D6.2 — `masonry_optimize_agent` MCP Tool Is Missing from `mcp_server/server.py`

**Question ID**: D6.2
**Status**: DONE
**Verdict**: FAILURE
**Agent**: diagnose-analyst
**Severity**: High
**Confidence**: 1.0

## Summary

The `masonry_optimize_agent` MCP tool referenced in `project-brief.md`, `ARCHITECTURE.md`, and CLAUDE.md does not exist in `mcp_server/server.py`. The TOOLS dict (lines 434–662) contains no optimization-trigger handler. The only optimization-related tool is `masonry_optimization_status`, which is read-only (reads existing JSON files from `optimized_prompts/` — cannot trigger a new MIPROv2 run). DSPy optimization is therefore unreachable via MCP, blocking the Phase 16 DSPy training roadmap and the Kiln "OPTIMIZE" button integration.

### `masonry_optimization_status` is read-only

```python
def _tool_masonry_optimization_status(args: dict) -> dict:
    """Return DSPy optimization scores for all agents in the optimized_prompts directory."""
    optimized_dir = Path(args.get("optimized_dir", ...))
    agents: list[dict] = []
    if not optimized_dir.is_dir():
        return {"agents": [], "count": 0}
    for json_file in sorted(optimized_dir.glob("*.json")):
        data = json.loads(json_file.read_text(...))
        agents.append({"agent": data["agent"], "score": data.get("score"), ...})
    return {"agents": agents, "count": len(agents)}
```

This only reads existing JSON files. It cannot trigger `optimizer.py` or `optimize_all()`.

### Required implementation

A correct `masonry_optimize_agent` tool would need to:
1. Accept `agent_name` (required), `projects_dir`, `agent_db_path`, `output_dir` (optional)
2. Call `configure_dspy()` — set the Anthropic LM before any optimization
3. Call `build_dataset(projects_dir, agent_db_path)` to get training examples for the agent
4. Call `optimize_agent(agent_name, ResearchAgentSig, dataset, output_dir)` (or `optimize_all()` for fleet-wide)
5. Return the result dict (agent, score, optimized_at)

The `configure_dspy()` pre-condition is critical: without it, the first DSPy forward pass fails with "No LM is loaded."

## Verdict

**FAILURE** — `masonry_optimize_agent` is a missing implementation. The MCP server was built with `masonry_optimization_status` (read-only) but not the optimization trigger. All references in project-brief, ARCHITECTURE, and CLAUDE.md describe a tool that does not exist in `server.py`.

**Priority**: P1 — the Kiln OPTIMIZE button and the entire Phase 16 DSPy training pipeline depend on this tool.

## Fix Specification

Add to `mcp_server/server.py`:

```python
def _tool_masonry_optimize_agent(args: dict) -> dict:
    """Run MIPROv2 optimization for a specific agent."""
    agent_name = args.get("agent_name", "")
    if not agent_name:
        return {"error": "agent_name is required"}

    projects_dir = Path(args.get("projects_dir", str(_REPO_ROOT)))
    agent_db_path = Path(args.get("agent_db_path", str(_REPO_ROOT / "agent_db.json")))
    output_dir = Path(args.get("output_dir", str(_REPO_ROOT / "masonry" / "optimized_prompts")))

    try:
        from masonry.src.dspy_pipeline.optimizer import configure_dspy, optimize_agent
        from masonry.src.dspy_pipeline.signatures import ResearchAgentSig
        from masonry.src.dspy_pipeline.training_extractor import build_dataset

        configure_dspy()  # must be called before any DSPy forward pass
        datasets = build_dataset(projects_dir, agent_db_path)
        agent_dataset = datasets.get(agent_name, [])
        if not agent_dataset:
            return {"error": f"No training data for agent '{agent_name}'", "examples": 0}

        result = optimize_agent(agent_name, ResearchAgentSig, agent_dataset, output_dir)
        return result
    except Exception as exc:
        return {"error": str(exc)}
```

And register it in TOOLS dict with `masonry_optimize_agent` key.
