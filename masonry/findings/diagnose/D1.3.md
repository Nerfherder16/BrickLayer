# Finding: D1.3 — `_load_registry` Silently Returns Empty List on CWD Mismatch

**Question ID**: D1.3
**Status**: DONE
**Verdict**: FAILURE
**Agent**: diagnose-analyst
**Severity**: High
**Confidence**: 1.0

## Summary

`_load_registry` in `router.py` silently returns `[]` when both path resolution attempts fail. No warning is logged. This disables Layer 1 Rule 5 (Mode field), all of Layer 2 (semantic), and degrades Layer 3 (LLM router sees empty agent list). Any caller that invokes `route()` from a CWD that is not the repository root will receive degraded routing with no indication of the failure.

## Evidence

### The Exact Code Path (router.py lines 26-35)

```python
def _load_registry(project_dir: Path) -> list[AgentRegistryEntry]:
    """Load registry from the standard location inside a project dir."""
    registry_path = project_dir / "masonry" / "agent_registry.yml"
    if not registry_path.exists():
        # Also try relative to CWD
        fallback = Path("masonry") / "agent_registry.yml"
        if fallback.exists():
            return load_registry(fallback)
        return []                           # ← SILENT RETURN
    return load_registry(registry_path)
```

When both `{project_dir}/masonry/agent_registry.yml` and `./masonry/agent_registry.yml` (relative to CWD) are absent, the function returns `[]` with zero logging.

### Downstream Consequences

**Layer 1 (deterministic)** — partially affected:
- Rules 1-4 (slash commands, autopilot state, campaign state, UI state) work without the registry.
- Rule 5 (Mode field matching) calls `get_agents_for_mode(registry, mode_value)` which returns `[]` for an empty registry → no match → falls through. A `**Mode**: diagnose` field in a request that should route to `diagnose-analyst` would instead fall to Layer 2.

**Layer 2 (semantic)** — completely disabled:
```python
def route_semantic(...) -> RoutingDecision | None:
    if not registry:
        return None   # ← immediate exit for empty registry
```
Layer 2 is completely skipped when registry is empty.

**Layer 3 (LLM router)** — degraded:
```python
agent_list = "\n".join(
    f"- {a.name}: {a.description or ', '.join(a.capabilities)}"
    for a in registry   # ← empty list → empty agent_list string
)
```
The prompt sent to Claude Haiku contains `"Available agents:\n"` with no agents listed. The LLM is asked to pick from an empty set — likely produces a hallucinated agent name or `None`.

### When This Fails in Practice

The MCP server is invoked with a `project_dir` argument. If this argument is incorrect (e.g., points to a subdirectory, or is the BrickLayer repo root rather than the masonry subdirectory), or if it is not provided and CWD is a subdirectory, both paths fail.

Example invocation that fails silently:
- CWD = `C:/Users/trg16/Dev/Bricklayer2.0/` (repo root, not masonry subdirectory)
- `project_dir` = same
- `registry_path` = `C:/Users/trg16/Dev/Bricklayer2.0/masonry/agent_registry.yml` → exists ✓ (this case works)

But:
- CWD = `C:/Users/trg16/Dev/some-other-project/`
- `project_dir` = same (wrong project)
- Both paths fail → `[]` silently

### No Logging at the Failure Point

`_load_registry` has no `print()` or logging call in the failure branches. The first observable signal is degraded routing behavior. `load_registry()` itself logs only on YAML parse errors, not on file-not-found.

## Mitigation

Add a stderr warning when returning empty:

```python
def _load_registry(project_dir: Path) -> list[AgentRegistryEntry]:
    registry_path = project_dir / "masonry" / "agent_registry.yml"
    if not registry_path.exists():
        fallback = Path("masonry") / "agent_registry.yml"
        if fallback.exists():
            return load_registry(fallback)
        print(
            f"[router] WARNING: agent_registry.yml not found at {registry_path} "
            f"or {fallback}. Routing will be degraded (no semantic/mode matching).",
            file=sys.stderr,
        )
        return []
    return load_registry(registry_path)
```

This converts a silent failure into a visible warning without changing behavior.
