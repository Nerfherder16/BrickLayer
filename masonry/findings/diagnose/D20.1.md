# Finding: D20.1 — score_routing._match_events() flat dict causes last-write-wins collision

**Question ID**: D20.1
**Status**: DONE
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: Medium
**Confidence**: 0.97
**Agent**: diagnose-analyst

## Summary

`_match_events()` in `score_routing.py` line 130 builds `findings_by_session` as a flat `dict[str, dict]` keyed only by `session_id`. All 17 finding events from session `315da739` overwrite each other, leaving only the last one (R19.2). This collapses 17 distinct findings to 1, causing finding-producing starts to either (a) get credited with the wrong finding or (b) lose their `downstream_success` 30pts. Additionally, `general-purpose` is absent from `AGENT_CATEGORIES` in both `rubrics.py` and the fallback dict, causing 2 start events to score 0 and be dropped entirely.

A simple compound `(session_id, agent_name)` key is necessary but NOT sufficient alone -- finding events use inconsistent agent names (`fix` vs `fix-implementer`, `research` vs `research-analyst`, `diagnose` vs `diagnose-analyst`). An alias normalization map is also required.

## Evidence

```python
# Line 130-135
findings_by_session: dict[str, dict[str, Any]] = {}
```

## Root Cause

Three co-occurring defects in `score_routing.py::_match_events()`:

1. **Line 130**: Flat `dict[str, dict]` keyed by `session_id` alone -- last-write-wins when multiple agents share a parent session.
2. **No agent name normalization**: Finding events use inconsistent agent names (`fix` vs `fix-implementer`) that don't match start event agent names.
3. **`general-purpose` missing from `AGENT_CATEGORIES`** in both `rubrics.py` (line 17-57) and the fallback dict in `score_routing.py` (line 31-43).

## Fix Specification

### Fix 1: Compound key with normalization in `_match_events()`

- **File**: `masonry/scripts/score_routing.py`
- **Lines**: 124-135
- **Change**: Replace flat `findings_by_session` dict with a compound-keyed `defaultdict(list)` using `(session_id, normalized_agent)` as key. Add an `_AGENT_ALIASES` normalization map. When looking up findings at line 157, use `findings_by_compound.get((session_id, agent), [])` and select the first non-INCONCLUSIVE finding.

Exact diff for lines 124-135:

```python
# Add before _match_events:
_AGENT_ALIASES: dict[str, str] = {
    "fix": "fix-implementer",
    "research": "research-analyst",
    "diagnose": "diagnose-analyst",
}

def _normalize_agent(name: str) -> str:
    return _AGENT_ALIASES.get(name, name)
```

Replace lines 129-135:
```python
    # OLD:
    # findings_by_session: dict[str, dict[str, Any]] = {}
    # for ev in events:
    #     if ev.get("event") == "finding":
    #         sid = ev.get("session_id", "")
    #         if sid:
    #             findings_by_session[sid] = ev

    # NEW: compound key (session_id, normalized_agent) -> list of findings
    from collections import defaultdict
    findings_by_compound: dict[tuple[str, str], list[dict[str, Any]]] = defaultdict(list)
    for ev in events:
        if ev.get("event") == "finding":
            sid = ev.get("session_id", "")
            agent = _normalize_agent(ev.get("agent", ""))
            if sid and agent:
                findings_by_compound[(sid, agent)].append(ev)
```
