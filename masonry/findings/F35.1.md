## Root Cause

`extract_finding_fields()` hardcoded the section name `"Evidence"` when calling `_extract_section()`. Waves 11-13 findings used `## Analysis`, `## Verification Results`, and `## Code Trace` as their primary evidence containers, following an earlier finding template that did not use `## Evidence`. This caused `evidence = ""` for those findings, making `_score_evidence_quality()` return 0 of 40 possible points, pushing affected findings below the 60-point `TRAINING_THRESHOLD` and excluding them from DSPy training data.

The change is purely additive: findings with `## Evidence` sections are unaffected (first match wins and breaks the loop). Findings with alternate section names now receive evidence quality points.
