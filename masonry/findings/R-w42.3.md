# Finding: R-w42.3 — Metric Blind Spots: 2/3 Active Components Are Format Proxies

**Question**: Metric blind spots (P-w40.1 E2) — does the eval metric measure reasoning quality or only format proxies?
**Agent**: research-analyst
**Verdict**: WARNING
**Severity**: Medium

## Evidence

### E1: build_metric() component classification

`masonry/src/metrics.py` — three scoring components:

| Component | Weight | Type | What it measures |
|---|---|---|---|
| `verdict_match` | 0.40 | Weak correctness proxy | Exact string match against training label |
| `evidence_quality` | 0.40 | **Format proxy** | `len(evidence) > 300` AND `re.search(r"\d+\.?\d*", evidence)` OR threshold keywords |
| `confidence_calibration` | 0.20 | **Format proxy** | `0.2 * (1 - abs(pred_conf - 0.75))` — rewards proximity to 0.75 unconditionally |

### E2: Prerequisite gate closes the worst prior exploit

Lines 36-39: `if not verdict_matched: return min(score, 0.2)`

Wrong-verdict predictions cap at 0.2 — below the 0.5 pass threshold (eval_agent.py line 327). The "calibration inversion" exploit (wrong verdict + good evidence = 0.60 false pass) documented in prior wave E8.2 is closed by this gate. An agent cannot pass eval with a wrong verdict regardless of evidence quality.

### E3: evidence_quality is a pure format proxy

Passes if: `len(evidence) > 300` AND (`has_numbers` OR `has_threshold_language`).
Threshold keywords: `threshold`, `baseline`, `%`, `ms`, `pts`, `seconds`.

A 350-character paragraph of plausible-sounding fiction containing "85%" earns the full 0.40. A correct finding with precise line references that happens to be 290 characters earns 0.0. The metric cannot distinguish fabricated evidence from accurate evidence.

### E4: confidence_calibration rewards hard-coded 0.75

`0.2 * (1 - abs(pred_conf - 0.75))` — an agent that outputs `confidence: 0.75` on every response earns 0.20 per example unconditionally. No calibration measurement occurs.

### E5: Residual blind spot — label-correct + fabricated evidence

The prerequisite gate prevents wrong-verdict predictions from passing. But it does not prevent a finding that:
- Correctly matches the training label verdict (e.g., WARNING)
- Contains fabricated evidence (cites non-existent line numbers, invents statistics)
- Is well-formatted (>300 chars, contains numbers)

Such a finding scores up to 1.0 — identical to a substantively correct finding. The metric cannot distinguish them.

### E6: Training data cross-check (5 research-analyst records, scored_all.jsonl lines 473-479)

All 5 records score 60-65. All 5 are substantively correct findings. Score variation reflects evidence_quality boundary noise, not reasoning quality. The training data contains no label-correct + fabricated-evidence case — the residual blind spot is unexercised in current training data.

### E7: Prior exploit training record predates the gate

`scored_all.jsonl` line 477 (E8.2-rec-5, score=65): notes the calibration inversion exploit as "currently unaddressed." The prerequisite gate in current `metrics.py` (lines 36-39) addresses it. The training record predates the fix — the gate was added after E8.2. This confirms the gate is a later improvement not reflected in the training label rationale.

## Analysis

The metric provides one effective correctness check (via the prerequisite gate) and two format proxies. The gate is necessary — without it, the calibration inversion exploit re-opens. But it is insufficient: it validates that the verdict label matches, not that the reasoning behind that verdict is sound.

An optimization loop running 5-10 cycles will exert pressure toward: correct-verdict label (mandatory for gate), long evidence with numbers (0.40 weight), confidence ≈ 0.75 (0.20 weight). The loop exerts zero pressure toward whether the evidence is actually accurate. This is the E2 blind spot from P-w40.1.

The practical risk is bounded by the training data quality: if labeled verdicts in `scored_all.jsonl` are correct, a wrong-verdict fabricated-evidence finding fails the gate. The blind spot only manifests if the optimizer generates instructions that produce correct-verdict-label responses with meaningless evidence — which requires the optimizer to distinguish between verdict selection and evidence quality in a way that current training data does not incentivize.

## Verdict Reasoning

WARNING: The prerequisite gate (lines 36-39) closes the calibration inversion exploit (prior wave E8.2). But 2/3 active scoring components are format proxies — evidence length+numbers and unconditional 0.75 reward. No component validates that cited evidence is accurate. The metric can detect wrong-verdict predictions (gate) but cannot detect label-correct + fabricated-evidence findings. This is the P-w40.1 E2 residual blind spot — bounded by training label quality but not closed by the metric itself.
