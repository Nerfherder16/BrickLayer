# Finding: F17.1 — `extractMarkdownField` Regex Fixed: `[\w]+` → `[\w-]+` Enables Hyphenated Agent Name Extraction

**Question ID**: F17.1
**Status**: DONE
**Verdict**: FIX_APPLIED
**Severity**: High
**Confidence**: 0.96
**Agent**: fix-implementer

## Summary

`extractMarkdownField()` in `masonry-observe.js` used `[\w]+` which stops at hyphens, causing all hyphenated agent names to be truncated at the first hyphen (e.g., `fix-implementer` → `"fix"`, `research-analyst` → `"research"`). Since most Masonry specialist agents use hyphenated names and none of the truncated forms are in `AGENT_CATEGORIES`, `correct_agent_dispatched` always scored 0pts for specialist agents → total ≤ 30 → below `min_training_score(65)` → 0 routing training records even when events pair correctly. Fix: changed `[\w]+` to `[\w-]+` on line 30 of masonry-observe.js. Verified: all five tested hyphenated agent names now extract correctly.

## Evidence

### The bug

`masonry-observe.js` line 30 (before fix):
```javascript
function extractMarkdownField(content, label) {
  const re = new RegExp(`\\*{0,2}${label}\\*{0,2}:\\s*([\\w]+)`, 'i');
  //                                                   ^^^^^ [\w] = [A-Za-z0-9_]
  //                                                         stops at hyphens!
  const m = content.match(re);
  return m ? m[1].trim() : null;
}
```

`[\w]` matches word characters: letters, digits, underscore. Does NOT match `-`. So for `**Agent**: research-analyst`:
- Before fix: captures `"research"` (stops at `-`)
- After fix: captures `"research-analyst"` (continues through `-`)

### Fix applied

```javascript
// Before:
const re = new RegExp(`\\*{0,2}${label}\\*{0,2}:\\s*([\\w]+)`, 'i');

// After:
const re = new RegExp(`\\*{0,2}${label}\\*{0,2}:\\s*([\\w-]+)`, 'i');
```

Single character added: `-` inside the character class `[\w-]+`.
