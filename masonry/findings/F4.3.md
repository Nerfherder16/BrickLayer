# Finding: F4.3 — `llm_router.py` Fixed: Platform-Conditional `_LLM_TIMEOUT` (20s Windows / 10s Linux)

## Summary

`_LLM_TIMEOUT` is now platform-conditional: 20 seconds on Windows, 10 seconds on Linux/macOS. The previous hardcoded 8-second timeout caused Layer 3 to fail on approximately 30% of cold-start calls on this Windows machine (R1.3, R2.2), effectively making Layer 3 dead and routing ~65-70% of requests to Layer 4 fallback.

## Change Applied

### `src/routing/llm_router.py`

**Before:**
```python
_LLM_TIMEOUT = 8  # seconds — Layer 4 fallback is acceptable, don't wait long
```

**After:**
```python
# Windows cold-starts for cmd.exe + claude subprocess average 4-6s (vs ~2s on Linux).
# 20s gives sufficient headroom without exceeding Claude Code's hook timeout threshold.
_LLM_TIMEOUT = 20 if platform.system() == "Windows" else 10
```

`platform` was already imported in `llm_router.py` (used for the subprocess path conditional added in F4.2), so no additional import is required.

## Rationale

From R2.2 (Layer 3 latency benchmark on Windows):

| Call type | Median latency | With 8s timeout |
|-----------|---------------|-----------------|
| Cold start (first call) | ~6.2s | ~30% timeout rate |
| Warm (subsequent) | ~2.1s | Never times out |

With F4.2 eliminating the cmd.exe→shell wrapper overhead (estimated 1-2s reduction), projected cold-start latency drops to ~4-5s. The 20s timeout provides:
- **Sufficient headroom**: 20s >> 4-5s projected cold start
- **Below hook threshold**: Claude Code's async hook timeout is 30s; 20s stays safely under
- **Non-Windows unchanged**: 10s on Linux/macOS (was 8s, slight increase for safety margin)

## Dependency Satisfied

This fix was listed as requiring F4.2 first (dependency note in synthesis_wave3.md). F4.2 (list-form subprocess) was applied in the same session immediately prior. The `platform` import used here was already present from the F4.2 change.

## Verdict

**FIX_APPLIED** — Layer 3 timeout failures on Windows should now be eliminated. With 20s headroom, even cold-start Layer 3 calls should complete within the timeout window, enabling Layer 3 to handle the ~20-25% of requests that fall through Layer 2 with thin margins.
