## Hypothesis Under Test

`writeback_optimized_instructions()` in `masonry/scripts/run_optimization.py` is claimed to:
1. Gracefully skip missing `.md` paths (no crash, just silent continue)
2. Replace — not duplicate — the `## DSPy Optimized Instructions` block when instruction text changes between runs (true idempotency)
3. Handle an empty or `None` instructions field without writing a blank block

## Evidence

### Edge Case 1 — Missing .md path (graceful skip)

**Source**: `run_optimization.py` lines 175–185, static code analysis.

```python
for md_path in unique_candidates:
    if not md_path.exists():
        continue          # <-- guard
    content = md_path.read_text(encoding="utf-8")
    ...
```

The function explicitly checks `md_path.exists()` at the top of the loop body and calls `continue` if the file is absent. No exception is raised. The caller (`run()`) also wraps the entire writeback call in a `try/except Exception` (line 368–380), logging a `[warn]` and proceeding on any unexpected failure. Two independent layers of protection.

**Verdict on EC1**: PASS — graceful skip confirmed by code.

### Edge Case 2 — Changed instruction text idempotency

**Source**: `run_optimization.py` lines 170–184, verified by live Python execution.

The replacement regex:

```python
section_pattern = re.compile(
    rf"{re.escape(_DSPY_SECTION_HEADER)}.*?{re.escape(_DSPY_SECTION_END)}\n?",
    re.DOTALL,
)
```

Uses `re.DOTALL` so `.` matches newlines, and `.*?` (non-greedy) so it matches the shortest possible span between the open and close sentinels. On second run with changed instructions:

```python
if section_pattern.search(content):
    new_content = section_pattern.sub(section_text.lstrip("\n"), content)
```

`sub()` replaces the matched span wholesale with the new `section_text`. The old instructions are discarded entirely. Live execution confirmed exactly 1 `## DSPy Optimized Instructions` header after the replacement, regardless of how much the text changed:

```
Number of DSPy section headers after second write: 1
IDEMPOTENCY: PASS
```

**Verdict on EC2**: PASS — no double-write bug. Regex replacement is idempotent.

### Edge Case 3 — Empty or None instructions field

**Source**: `run_optimization.py` lines 355–382.

The caller guards the writeback with:

```python
if instructions_text:
    ...writeback_optimized_instructions(...)
else:
    print(f"[writeback] No instructions field in optimized JSON — skipping writeback.")
```

`None` and `""` both evaluate falsy in Python. Neither triggers `writeback_optimized_instructions()`. The function itself is never reached, so no empty block can be written. Confirmed by Python evaluation:

```
None -> writeback called? NO (skipped)
Empty string -> writeback called? NO (skipped)
```

**Verdict on EC3**: PASS — empty/None instructions result in clean skip with informational log line.

## Minor Observations (Non-Blocking)

1. **One-level sub-project scan uses `base_dir.iterdir()`** — this silently ignores OS permission errors on any child directory. Not a bug but worth noting if deeply nested structures are added.
2. **Deduplication uses resolved path only for existing files** (`c.resolve() if c.exists() else c`). Two non-existent candidate paths with different logical paths but same target would not deduplicate — harmless since both would be skipped by the `exists()` check, but the deduplication logic is technically imprecise for non-existent paths.
3. **`projects/bl2`** quantitative-analyst.md lacks the DSPy block. This is expected behavior (not a bug) — the file was not reached during the F31.1 optimization run because `projects/bl2` is two levels deep from the BL root, outside the one-level scan.

## What Would Change This Verdict

- If a future change removed the `if not md_path.exists(): continue` guard, EC1 would become a crash-on-missing-file failure.
- If the regex were changed from non-greedy `.*?` to greedy `.*`, and an agent file contained two DSPy blocks (e.g., from manual editing), the greedy match would consume everything between the first open and last close sentinel — potentially over-replacing content. Current non-greedy pattern is correct.
- If `instructions_text` guard in `run()` were removed, an empty string would propagate to `writeback_optimized_instructions()` and write a blank section block to all agent files.
