## Summary

The preagent tracker (PreToolUse:Agent) and subagent tracker (SubagentStart) used CWD-relative paths to write and read slot files. Because the two hooks can fire in different working directory contexts — the preagent hook uses `process.cwd()` while the subagent hook uses `input.cwd || process.cwd()` — the slot file could be written to a different directory than the one the consumer looked in. The fix moves both hooks to use `os.homedir()` as the base for the shared slot directory, making the path unconditionally stable regardless of CWD at event time.

---

## Evidence

### Root Cause

**masonry-preagent-tracker.js (before fix, lines 52-63):**
```
const cwd = process.cwd();
const masonryDir = path.basename(cwd) === "masonry" && fs.existsSync(cwd)
  ? cwd
  : path.join(cwd, "masonry");
const pendingDir = path.join(masonryDir, ".masonry", "pending_agent_prompts");
```
This always includes the `masonry/` directory segment in the path. When `process.cwd()` is the repo root (e.g. `C:/Users/trg16/Dev/Bricklayer2.0`), the written path is:
`C:/Users/trg16/Dev/Bricklayer2.0/masonry/.masonry/pending_agent_prompts/spec-writer_latest.json`

**masonry-subagent-tracker.js (before fix, line 196):**
```
const pendingDir = path.join(cwd, '.masonry', 'pending_agent_prompts');
```
This uses `input.cwd || process.cwd()` for `cwd` with no masonry-dir resolution. When the SubagentStart hook fires with `cwd` set to the repo root, the read path is:
`C:/Users/trg16/Dev/Bricklayer2.0/.masonry/pending_agent_prompts/spec-writer_latest.json`

The slot was written to `…/masonry/.masonry/…` but read from `…/.masonry/…` — a different directory. The slot file remained unconsumed after the SubagentStart event, confirming the miss.

### Fix Applied

Both hooks now use:
```javascript
const pendingDir = path.join(os.homedir(), ".masonry", "pending_agent_prompts");
// → C:/Users/trg16/.masonry/pending_agent_prompts/
```

The preagent hook gained `const os = require("os");` at the top and the entire masonryDir resolution block was removed (it was only needed to construct the old pendingDir). The subagent hook already imported `os` and already used `os.homedir()` for its `~/.masonry/state/` path — the fix aligns the slot path to the same pattern.
