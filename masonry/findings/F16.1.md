# Finding: F16.1 — CWD Guard Fixed in Both Routing Hooks; routing_log.jsonl Now Writeable in Self-Research Sessions

**Question ID**: F16.1
**Status**: DONE
**Verdict**: FIX_APPLIED
**Severity**: High
**Confidence**: 0.95
**Agent**: fix-implementer

## Summary

Both `masonry-observe.js` and `masonry-subagent-tracker.js` had a CWD guard bug that silently dropped all routing_log.jsonl writes when Claude Code ran from the masonry/ directory. The fix resolves the `masonryDir` and `routingLogPath` using CWD-agnostic logic: check if `path.basename(cwd) === 'masonry'` first, and if so use `cwd` directly rather than `path.join(cwd, 'masonry')`. The fix was verified correct via logic trace across all three CWD scenarios: masonry-as-CWD (self-research), BL-root, and project subdir (subprocess suppression preserved).

## Root Cause (from R15.1)

Both hooks computed:
```javascript
const masonryDir = path.join(cwd, 'masonry');
if (fs.existsSync(masonryDir)) { ... }
```

When CWD = `C:\Users\trg16\Dev\Bricklayer2.0\masonry`:
- `masonryDir = C:\Users\trg16\Dev\Bricklayer2.0\masonry\masonry` (doesn't exist)
- Guard fails → routing_log.jsonl write silently dropped

Additionally, `routingLogPath` was pre-computed as `path.join(cwd, 'masonry', 'routing_log.jsonl')` before `masonryDir` was resolved, so even if the guard had been corrected, the `appendFileSync` would still write to the wrong path.

## Fix Applied

### masonry-observe.js (line 294 → inside guard)

**Before:**
```javascript
const routingLogPath = path.join(cwd, 'masonry', 'routing_log.jsonl');
// ...
try {
  const masonryDir = path.join(cwd, 'masonry');
  if (fs.existsSync(masonryDir)) {
    fs.appendFileSync(routingLogPath, findingEntry + '\n', 'utf8');
  }
} catch (_err) { /* non-fatal */ }
```

**After:**
```javascript
// routingLogPath pre-computation removed
// ...
try {
  // Resolve masonry/ dir — cwd might be the masonry dir itself (self-research sessions)
  const masonryDir = path.basename(cwd) === 'masonry' && fs.existsSync(cwd)
    ? cwd
    : path.join(cwd, 'masonry');
  if (fs.existsSync(masonryDir)) {
    const routingLogPath = path.join(masonryDir, 'routing_log.jsonl');
    fs.appendFileSync(routingLogPath, findingEntry + '\n', 'utf8');
  }
} catch (_err) { /* non-fatal */ }
```

### masonry-subagent-tracker.js (line 93 → inside guard)

Same pattern: removed pre-computed `routingLogPath`, moved computation to inside the guard using `masonryDir`.

## Verification: CWD Resolution Logic

|-----|-------------------|------------|----------------|--------------|
