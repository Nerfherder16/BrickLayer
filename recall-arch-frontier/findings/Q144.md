### Persistence across restarts
`PATCH /collections/{name}` writes to Qdrant's persistent collection config on disk (`/qdrant/storage`). The `qdrant_data` Docker volume preserves it across container restarts. No action needed to persist the change.

The existing `_ensure_optimizer_config()` call only sets `optimizer_config.indexing_threshold=50000`. It does not touch `hnsw_config`. The ef=128 change will survive application restarts because the app never calls `update_collection` with `hnsw_config` — it only calls it with `optimizer_config`. Safe to apply.
