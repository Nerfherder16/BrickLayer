## Complete Implementation

### Unit 1: `src/core/simhash.py` (new file, ~45 lines)

```python
"""
SimHash implementation for near-duplicate detection.

Computes a 64-bit locality-sensitive hash from token unigrams and bigrams.
Used as Layer 1 dedup before the Ollama embedding call.
"""

import hashlib
import re


def compute_simhash(text: str) -> tuple[int, list[int]]:
    """
    Compute a 64-bit SimHash and 8 band integers for LSH bucket search.

    Returns:
        (full_64bit_int, [b0, b1, ..., b7])
        where bN = bits 8N..8N+7 of the full hash (unsigned, 0-255).

    Algorithm:
        1. Tokenize to unigrams and bigrams (lowercased, stopwords kept for
           sensitivity; punctuation stripped).
        2. For each token, hash via SHA-256 and interpret first 8 bytes as
           a 64-bit integer (big-endian).
        3. For each bit position 0..63, accumulate +1 if bit is set, -1 if not,
           weighted by 1.0 (uniform weights — unigrams and bigrams equally).
        4. Final bit i = 1 if accumulator[i] > 0 else 0.
        5. Convert to unsigned 64-bit integer; split into 8 x uint8 bands.
    """
    normalized = re.sub(r"[^\w\s]", " ", text.lower())
    tokens = normalized.split()

    # Unigrams + bigrams
    features: list[str] = tokens[:]
    for i in range(len(tokens) - 1):
        features.append(tokens[i] + "_" + tokens[i + 1])

    if not features:
        return 0, [0] * 8

    # 64-element accumulator
    v = [0.0] * 64

    for token in features:
        digest = hashlib.sha256(token.encode()).digest()
        # First 8 bytes → 64-bit int
        token_hash = int.from_bytes(digest[:8], byteorder="big")
        for bit in range(64):
            if (token_hash >> bit) & 1:
                v[bit] += 1.0
            else:
                v[bit] -= 1.0

    # Build final 64-bit fingerprint (unsigned)
    fingerprint = 0
    for bit in range(64):
        if v[bit] > 0:
            fingerprint |= 1 << bit

    # Mask to 64-bit unsigned (Python ints are arbitrary precision)
    fingerprint = fingerprint & 0xFFFFFFFFFFFFFFFF

    # Split into 8 x uint8 bands: b0 = bits 0-7, b1 = bits 8-15, ...
    bands = [(fingerprint >> (8 * i)) & 0xFF for i in range(8)]

    return fingerprint, bands


def hamming_distance(a: int, b: int) -> int:
    """
    Compute Hamming distance between two 64-bit unsigned integers.

    Uses Python 3.10+ int.bit_count() (popcount).
    XOR isolates differing bits; bit_count() counts them.
    """
    return (a ^ b).bit_count()
```

**Overflow guard**: The `& 0xFFFFFFFFFFFFFFFF` mask on line `fingerprint = fingerprint & 0xFFFFFFFFFFFFFFFF` ensures the value fits in a uint64 before storage. Python's arbitrary-precision integers never overflow, but the mask prevents Qdrant from rejecting a value that exceeds int64 range when stored.

**Note on int64 vs uint64**: Qdrant's integer payload field is signed int64 (range -2^63 to 2^63-1). Values with bit 63 set will appear negative when stored but compare identically by value when filtered via `MatchValue`. Store as Python int directly — no cast needed; Python will pass the correct value. Alternatively, store as signed by interpreting with `struct.unpack(">q", ...)` — but unsigned storage via the mask is simpler and avoids confusion.

### Unit 4: `src/storage/qdrant.py` — `find_by_simhash_bands()` new method (~35 lines)

Add this method adjacent to `find_by_content_hash()` (after line 478):

```python
    async def find_by_simhash_bands(
        self,
        bands: list[int],
        user_id: int | None = None,
    ) -> list[tuple[str, int]]:
        """
        Find candidate near-duplicates by SimHash band matching (LSH bucket search).

        At least one band must match (OR logic via `should` filter).
        Returns list of (memory_id, simhash_value) for Hamming distance check.

        The `should` filter in Qdrant means: at least one condition must be true.
        With min_should = 1 (Qdrant default), any single band match returns a candidate.
        Callers must then apply hamming_distance() to confirm true near-duplicates.
        """
        band_keys = [
            "simhash_b0", "simhash_b1", "simhash_b2",
            "simhash_b3", "simhash_b4", "simhash_b5",
            "simhash_b6", "simhash_b7",
        ]
        should_conditions = [
            FieldCondition(
                key=band_keys[i],
                match=MatchValue(value=bands[i]),
            )
            for i in range(8)
        ]

        # must: user scope; should: OR across band columns
        must_conditions = self._user_conditions(user_id) + [
            IsNullCondition(is_null=PayloadField(key="superseded_by")),
        ]

        scroll_filter = Filter(
            must=must_conditions if must_conditions else None,
            should=should_conditions,
        )

        results, _ = await self.client.scroll(
            collection_name=self.collection,
            scroll_filter=scroll_filter,
            limit=50,           # band collision pool — Hamming check narrows this
            with_payload=["simhash"],   # only fetch the field we need
            with_vectors=False,
        )

        return [
            (str(point.id), (point.payload or {}).get("simhash", 0))
            for point in results
        ]
```

**Filter semantics**: In Qdrant's filter model, `should` is an OR list with an implicit `min_should=1`. A point matches if it satisfies all `must` conditions AND at least one `should` condition. This is exactly the band LSH semantics: any bucket collision is a candidate.

**`with_payload=["simhash"]`**: Qdrant supports selective payload projection as a list of field names. This avoids fetching `content`, `tags`, and other large fields for the ~50 candidates; only the 64-bit fingerprint is returned for Hamming comparison.

**limit=50**: Band collisions are rare for non-duplicates (probability ~1/256 per band by birthday paradox at corpus size 22k). 50 is a safe upper bound that won't cause memory pressure. If Q160 calibration reveals band collision rates are higher, this can be reduced.

## Backfill Migration

Existing 22,423 points have no simhash payload fields. Qdrant returns `None` for missing fields on the `with_payload=["simhash"]` projection, so the band filter will never match legacy points until they are backfilled.

**Migration script** (`scripts/backfill_simhash.py`):

```python
"""
One-time backfill: compute SimHash for all existing Qdrant points and patch payload.

Run:
    python scripts/backfill_simhash.py

Cost estimate at 22,423 points:
    - scroll (batch 100): ~225 HTTP calls @ ~5ms each = ~1.1s total
    - compute_simhash(): ~0.1ms per point = ~2.2s total
    - set_payload() (batch 100): ~225 HTTP calls @ ~5ms each = ~1.1s total
    Total wall-clock: ~5s (Qdrant local). Safe to run live; set_payload is non-blocking.
"""
import asyncio
from qdrant_client import AsyncQdrantClient
from src.core.simhash import compute_simhash
from src.core import get_settings

async def backfill():
    settings = get_settings()
    client = AsyncQdrantClient(url=settings.qdrant_url)
    collection = settings.qdrant_collection
    offset = None
    patched = 0

    while True:
        points, next_offset = await client.scroll(
            collection_name=collection,
            scroll_filter=None,
            limit=100,
            offset=offset,
            with_payload=["content"],
            with_vectors=False,
        )
        if not points:
            break

        for point in points:
            content = (point.payload or {}).get("content", "")
            fingerprint, bands = compute_simhash(content)
            await client.set_payload(
                collection_name=collection,
                payload={
                    "simhash": fingerprint,
                    "simhash_b0": bands[0], "simhash_b1": bands[1],
                    "simhash_b2": bands[2], "simhash_b3": bands[3],
                    "simhash_b4": bands[4], "simhash_b5": bands[5],
                    "simhash_b6": bands[6], "simhash_b7": bands[7],
                },
                points=[point.id],
            )
            patched += 1

        if next_offset is None:
            break
        offset = next_offset

    print(f"Backfilled {patched} points.")

asyncio.run(backfill())
```

**Safe to run live**: `set_payload` in Qdrant is a merge operation, not a replace. Existing payload fields (content, tags, etc.) are untouched. The migration can be interrupted and resumed (already-patched points will just be re-patched idempotently with the same values).

## Key Findings

1. **Exact insertion point confirmed**: L1 SimHash block inserts at `memory.py` line 192 (between the content_hash early-return at line 191 and the embedding call at line 194). This is the only location that achieves the Ollama short-circuit goal.

2. **Qdrant scroll OR filter pattern**: `Filter(must=[...user_scope, IsNullCondition(superseded_by)], should=[FieldCondition(band_key, MatchValue(band_value)) x 8])`. The `should` list is the existing pattern used in time-machine queries (qdrant.py:334-336, 359-362).

3. **`with_payload=["simhash"]`**: Selective field projection reduces payload transfer from ~800 bytes/point to ~20 bytes/point for the candidate pool, critical when limit=50 returns a full batch.

4. **`_dedup_skip_types` guard must be replicated at L1**: The L2 block already exempts `episodic` and `session_summary` types. L1 must apply the same guard, or session summaries will be incorrectly collapsed by SimHash even though their text is nearly identical across sessions by design.

5. **No simhash function exists in the codebase**: Confirmed Q175. `src/core/simhash.py` is a net-new file with zero imports from existing Recall modules except `hashlib` and `re` (stdlib only), making it fully isolated and independently testable.

6. **Hamming threshold 8/64**: Corresponds to ~87.5% bit agreement. At 22,423 points, the expected false positive rate per store call is roughly `22423 × (8/64) × (1/256)^1` ≈ 10 false candidates per band match, each requiring a Hamming check (O(1)). The actual confirmed false positives (Hamming ≤ 8 on genuinely distinct content) require Q160 calibration to quantify.

7. **Band collision pool limit=50 is safe**: With 8 bands of 8 bits each, birthday collision probability for a random non-duplicate at corpus size 22k is ~`22423/256 ≈ 87.6` points per band. Returning 50 and Hamming-filtering is correct behavior; the Hamming check discards all non-duplicates in the pool.

8. **Simhash fingerprint propagation via `memory.metadata`**: Avoids modifying the `Memory` dataclass or `store()` signature (minimum diff). The `_simhash` and `_simhash_bands` underscore-prefixed keys follow the existing `_provenance` convention already in `memory.metadata` (memory.py:144-148).

## Ideas

- **idea**: [batch_set_payload_optimization] Instead of one `set_payload` call per point in the backfill migration, batch into groups of 100 using `points=[id1, id2, ...]` with the same payload dict. This reduces HTTP round-trips from 22,423 to ~225. Only works when all points in the batch have identical simhash values — which they won't. Use `asyncio.gather()` with 10 concurrent `set_payload` coroutines instead to get 10x throughput without Qdrant connection saturation.

- **idea**: [simhash_band_width_tuning] The 8×8 band split (8 bands, each 8 bits wide) is a tunable parameter. Changing to 4×16 (4 bands, 16-bit buckets) reduces false positive collision rate by 256x per band at the cost of lower recall for high-hamming-distance duplicates. The split should be a config param `simhash_band_count` / `simhash_band_width` pending Q160 calibration results.

- **idea**: [l1_metrics_instrumentation] Add structlog metrics at the L1 call site: `logger.info("simhash_l1_hit", candidate_id=_candidate_id, hamming=distance, bands_matched=N)` and `logger.debug("simhash_l1_miss", candidate_count=len(_sh_candidates))`. This enables the Q160 calibration script to use production logs rather than synthetic data.
