## Research Question

What is "forgetting" in physical terms? Design a forgetting mechanism using only physical requirements.

**Forbidden terms**: delete, remove, erase, wipe, purge, drop, overwrite, clear, discard, forget, decay, expire, age-out, TTL, clean, prune.

---

## Physical Definition of Forgetting

A memory M is **functionally forgotten** when: the system's output distribution P(output | M present) = P(output | M absent).

The memory must have zero influence on all future outputs. This is the only physical requirement. The question is whether the stored text encoding must be destroyed to achieve it, or whether there are weaker physical states that satisfy the same condition.

---

## Four Physical Mechanisms for Zero Future Influence

### Mechanism 1: Retrieval Weight Reduction to Zero

If a memory's retrieval score is permanently set to zero — it never appears in the K=10 results — it has zero influence on all future outputs via the retrieval path.

Physical state: an entry in a "weight override" table mapping `memory_id → retrieval_weight_multiplier = 0.0`.

Properties:
- Reversible: the multiplier can be changed back to 1.0 (the memory "returns")
- Storage-efficient: one float per suppressed memory
- Does not destroy the encoded information
- Effective as long as retrieval is the only path to influence

Limitation: if spreading activation (Q235) can propagate activation through the zero-weight memory, the memory still influences output indirectly. The zero-weight multiplier must be applied at the activation-propagation step too.

### Mechanism 2: Activation Floor at Zero (Spreading Activation Suppression)

In Q235's CO_RETRIEVED graph, a memory's activation level drives its influence. If the activation floor is set to zero AND incoming activation from neighbors is blocked (the node is "masked" in the activation graph), the memory contributes nothing to spreading activation.

Physical state: a boolean `activation_masked` flag per memory_id in LMDB.

This is stricter than Mechanism 1: it prevents indirect influence via spreading activation, not just direct retrieval.

### Mechanism 3: Entity-Predicate Index Suppression

The Q236 entity-predicate conflict index determines which memory wins for any given entity-predicate pair. If a memory's entity-predicate entries are all shadowed by a contradicting memory with a higher timestamp (Q249 LWW rule), the memory is effectively forgotten for the purpose of contradiction detection — it never produces a supersession event.

This is forgetting at the semantic level: the memory's facts are "overwritten" in the index without the memory text being touched.

Physical state: new memory with same entity-predicate + higher timestamp → old memory is shadow-suppressed by the index.

### Mechanism 4: Zero Embedding Similarity (Query-Space Isolation)

If a memory's embedding vector is modified to be maximally dissimilar from all possible query embeddings (e.g., set all components to zero, or to a "null" embedding with zero cosine similarity to any real text), it will never appear in ANN search results.

Physical state: zeroed or null-vector stored in HNSW for the memory's node.

This achieves retrieval isolation at the vector level without touching the text content.

## The Physical Architecture of Forgetting

Full functional forgetting (zero influence via all paths) requires all four simultaneously:

```
functional_forgetting(memory_id):
  1. LMDB: set retrieval_weight[memory_id] = 0.0
  2. LMDB: set activation_masked[memory_id] = true
  3. entity_predicate_index: apply timestamp=MAX to shadow all M's facts
  4. HNSW: set embedding[memory_id] = null_vector
```

This achieves functional forgetting with zero information destruction — the original text is intact in LMDB. The memory is "isolated" from all output paths.

**The reversible forgetting operation**: set all four. The memory text is preserved. If needed (legal hold, audit, recovery), the isolation can be lifted.

**The irreversible equivalent** (traditional deletion): also required in some cases (GDPR right-to-erasure). But for the homelab case, reversible isolation is strictly better — it preserves the append-only log guarantee (Q240) while achieving functional forgetting.

## The Q240 Connection: Permanence and Forgetting Are Orthogonal

Q240 established that physical permanence = append-only log. Nothing is destroyed. The WAL grows monotonically.

Q250 establishes that functional forgetting = isolation from output paths. The WAL entry persists; its influence is zeroed.

**These are orthogonal**:
- Physical permanence: the text is in the WAL (true for all memories, always)
- Functional forgetting: the text has zero influence on output (tunable per memory)

This resolves the apparent contradiction between "memories are permanent" (Q240) and "the user wants to forget something." Both can be true simultaneously: the fact is recorded permanently AND has zero influence on future outputs.

## The Forgetting Half-Life: Asymmetric Recovery

One additional property: **asymmetric recovery time**.

If functional forgetting is implemented via isolation multipliers, recovering the memory (un-isolating) is instant — flip the flag, the memory is immediately retrievable. But for biological and computational memory systems, forgetting is typically asymmetric: hard to forget, easy to remember. The Recall 2.0 isolation mechanism makes forgetting instant but recovery equally instant, which may not be desirable for "soft" forgetting (gradual relevance reduction).

**Soft forgetting mechanism**: apply a retrieval weight multiplier that decreases over time:
```
retrieval_weight[memory_id](t) = decay_factor ^ (t - suppression_start_time)
```
Where `decay_factor < 1.0`. After enough time, the weight approaches zero — functional forgetting without an explicit "set to zero" step. This is the continuous-time equivalent of spreading activation suppression.

## Verdict

**BREAKTHROUGH.** Physical forgetting = functional forgetting = zero influence on all output paths. Four physical mechanisms achieve this without information destruction: retrieval weight zero, activation masking, entity-predicate index shadow, and null embedding. All four are reversible — the memory text is preserved in the WAL (Q240 permanence holds). This resolves the Q240-Q250 apparent contradiction: permanence and forgetting are orthogonal properties of the same system. The text is permanent; its influence is tunable. For GDPR/erasure cases, the irreversible path (text removal) remains available but is not the only option. Soft forgetting via decay multiplier provides a continuous-time alternative that avoids the sharp transition of hard isolation.
