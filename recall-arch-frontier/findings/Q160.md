## SimHash Approach: Token Bigram from Q146

Per Q146's finding, the token bigram SimHash approach is:
1. Tokenize content: lowercase, extract `[a-zA-Z0-9]+`, remove stopwords, min length 2
2. Generate bigrams from token list
3. Weight each bigram by hash → 64 bit accumulators
4. Sign the accumulator → 64-bit fingerprint
5. Hamming distance = `bin(fp_a XOR fp_b).count('1')`

The threshold per Q146 is **k=8** for 80% detection at cosine >= 0.95. The question asks for k<=3 pairs but Q146 determined k=8 is the operative threshold. The script reports both to compare with previous estimates.

```python
#!/usr/bin/env python3
"""
simhash_calibration.py — Measure syntactic near-duplicate rate in Tim's Recall corpus.

Reads 1,000 memories from Qdrant, computes 64-bit token-bigram SimHash for each,
samples 1,000 random pairs, and reports Hamming distance distribution.

Usage (on CasaOS where Qdrant port 6333 is accessible):
    pip install qdrant-client
    python simhash_calibration.py

Usage (from external host via Recall API):
    python simhash_calibration.py --use-api

Outputs:
    - D_near at k<=3 (question baseline)
    - D_near at k<=8 (Q146 operative threshold)
    - Hamming distance histogram
    - Implementation decision
"""

import argparse
import hashlib
import random
import re
import struct
import sys
from collections import Counter

# ---------------------------------------------------------------------------
# SimHash implementation (token bigram, 64-bit)
# ---------------------------------------------------------------------------

_STOPWORDS = frozenset({
    "a", "an", "the", "is", "it", "in", "on", "at", "to", "for", "of",
    "and", "or", "but", "with", "from", "by", "as", "be", "was", "are",
    "were", "been", "have", "has", "had", "do", "did", "my", "i", "me",
    "we", "you", "he", "she", "they", "this", "that", "what", "how",
    "when", "where", "which", "who", "not", "no",
})

def _tokenize(text: str) -> list[str]:
    tokens = re.findall(r"[a-zA-Z0-9]+", text.lower())
    return [t for t in tokens if t not in _STOPWORDS and len(t) >= 2]

def _token_hash64(token: str) -> int:
    """64-bit hash of a token string."""
    digest = hashlib.md5(token.encode()).digest()
    return struct.unpack("<Q", digest[:8])[0]

def simhash64(text: str) -> int:
    """
    Compute 64-bit SimHash of text using token bigrams.
    Uses unit weights (all tokens weighted equally).
    Returns a 64-bit integer fingerprint.
    """
    tokens = _tokenize(text)
    if not tokens:
        return 0

    # Generate bigrams; fall back to unigrams if fewer than 2 tokens
    if len(tokens) >= 2:
        features = [f"{tokens[i]}_{tokens[i+1]}" for i in range(len(tokens) - 1)]
    else:
        features = tokens

    # Accumulate bit votes
    v = [0] * 64
    for feature in features:
        h = _token_hash64(feature)
        for i in range(64):
            v[i] += 1 if (h >> i) & 1 else -1

    # Sign to fingerprint
    fingerprint = 0
    for i in range(64):
        if v[i] > 0:
            fingerprint |= (1 << i)
    return fingerprint

def hamming64(a: int, b: int) -> int:
    """Hamming distance between two 64-bit integers."""
    return bin(a ^ b).count("1")

# ---------------------------------------------------------------------------
# Data fetching
# ---------------------------------------------------------------------------

def fetch_via_qdrant_client(host: str, port: int, collection: str, n: int) -> list[str]:
    """
    Fetch n random memories directly from Qdrant using qdrant-client.
    Works when running on CasaOS or when port 6333 is accessible.
    """
    try:
        from qdrant_client import QdrantClient
        from qdrant_client.models import ScrollRequest
    except ImportError:
        print("ERROR: qdrant-client not installed. Run: pip install qdrant-client")
        sys.exit(1)

    client = QdrantClient(host=host, port=port)

    print(f"Connecting to Qdrant at {host}:{port}, collection '{collection}'")

    # Get collection info to know total count
    info = client.get_collection(collection)
    total = info.points_count
    print(f"Collection '{collection}': {total} points")

    if total == 0:
        print("ERROR: Collection is empty")
        sys.exit(1)

    # Scroll through all memories (or enough to sample n)
    # For efficiency, stop after collecting n*5 to get a diverse random sample
    target_pool = min(n * 5, total)
    batch_size = 250
    contents = []
    offset = None

    print(f"Scrolling up to {target_pool} memories for sampling pool...")
    while len(contents) < target_pool:
        points, next_offset = client.scroll(
            collection_name=collection,
            limit=batch_size,
            offset=offset,
            with_payload=["content"],
            with_vectors=False,
        )

        for point in points:
            if point.payload and "content" in point.payload:
                text = point.payload["content"]
                if isinstance(text, str) and len(text) >= 10:
                    contents.append(text)

        if next_offset is None or len(contents) >= target_pool:
            break
        offset = next_offset

    print(f"Collected {len(contents)} memories with content field")

    # Random sample n from pool
    if len(contents) < n:
        print(f"WARNING: Only {len(contents)} memories available, using all")
        return contents

    return random.sample(contents, n)


def fetch_via_recall_api(api_base: str, n: int) -> list[str]:
    """
    Fetch memories via Recall's REST API at port 8200.
    Uses the /api/memories/browse or /api/search endpoint with pagination.
    Falls back to /api/search with a broad query.
    """
    try:
        import urllib.request
        import json
    except ImportError:
        pass  # stdlib, always available

    import urllib.request
    import json

    contents = []
    page = 0
    page_size = 100

    print(f"Fetching memories via Recall API at {api_base}")

    # Try browse-style scroll endpoint first
    while len(contents) < n * 3:
        url = f"{api_base}/api/memories?limit={page_size}&offset={page * page_size}"
        try:
            with urllib.request.urlopen(url, timeout=10) as resp:
                data = json.loads(resp.read())

            # Handle different response shapes
            memories = data if isinstance(data, list) else data.get("memories", data.get("results", []))

            if not memories:
                break

            for m in memories:
                # Content might be in 'content', 'text', or nested
                text = m.get("content") or m.get("text", "")
                if isinstance(text, str) and len(text) >= 10:
                    contents.append(text)

            page += 1
            if len(memories) < page_size:
                break  # Last page

        except Exception as e:
            print(f"API browse failed (page {page}): {e}")
            break

    if len(contents) < 100:
        print(f"WARNING: Only fetched {len(contents)} via API browse, results may be limited")

    if len(contents) < n:
        return contents
    return random.sample(contents, n)


# ---------------------------------------------------------------------------
# Analysis
# ---------------------------------------------------------------------------

def analyze(contents: list[str], n_pairs: int, k_values: list[int]) -> None:
    """
    Compute SimHash for all memories, sample n_pairs random pairs,
    report Hamming distance distribution and D_near at each k.
    """
    n_memories = len(contents)
    print(f"\nComputing SimHash for {n_memories} memories...")

    fingerprints = []
    empty_count = 0
    for text in contents:
        fp = simhash64(text)
        if fp == 0:
            empty_count += 1
        fingerprints.append(fp)

    if empty_count > 0:
        print(f"  WARNING: {empty_count} memories produced empty SimHash (too short or all stopwords)")

    # Sample n_pairs random pairs (without replacement of pair combinations)
    print(f"Sampling {n_pairs} random pairs from {n_memories} memories...")
    indices = list(range(n_memories))
    pair_indices = set()
    sampled_pairs = []

    max_possible = n_memories * (n_memories - 1) // 2
    actual_pairs = min(n_pairs, max_possible)

    if actual_pairs < n_pairs:
        print(f"  NOTE: Only {max_possible} unique pairs possible with {n_memories} memories")

    while len(sampled_pairs) < actual_pairs:
        i = random.randint(0, n_memories - 1)
        j = random.randint(0, n_memories - 1)
        if i == j:
            continue
        key = (min(i, j), max(i, j))
        if key not in pair_indices:
            pair_indices.add(key)
            sampled_pairs.append((i, j))

    # Compute Hamming distances
    print(f"Computing {len(sampled_pairs)} Hamming distances...")
    distances = []
    for i, j in sampled_pairs:
        d = hamming64(fingerprints[i], fingerprints[j])
        distances.append(d)

    total = len(distances)

    # Report D_near at each k
    print("\n" + "=" * 60)
    print("SIMHASH NEAR-DUPLICATE CALIBRATION RESULTS")
    print("=" * 60)
    print(f"Corpus sample: {n_memories} memories")
    print(f"Pairs sampled: {total}")
    print()

    for k in k_values:
        near_dup_count = sum(1 for d in distances if d <= k)
        d_near = near_dup_count / total
        print(f"D_near (k<={k:2d}): {d_near:.4%}  ({near_dup_count}/{total} pairs)")

    print()
    print("Hamming distance histogram:")
    dist_counter = Counter(distances)
    print(f"{'Ham':>5}  {'Count':>7}  {'Frac':>8}")
    for d in sorted(dist_counter.keys()):
        frac = dist_counter[d] / total
        bar = "#" * int(frac * 200)
        print(f"{d:>5}  {dist_counter[d]:>7}  {frac:>8.4%}  {bar}")

    print()
    print(f"Min Hamming: {min(distances)}")
    print(f"Max Hamming: {max(distances)}")
    print(f"Mean Hamming: {sum(distances)/len(distances):.2f}")

    # Decision logic
    print()
    print("=" * 60)
    print("IMPLEMENTATION DECISION")
    print("=" * 60)
    d_near_k8 = sum(1 for d in distances if d <= 8) / total
    d_near_k3 = sum(1 for d in distances if d <= 3) / total

    print(f"D_near at k<=3: {d_near_k3:.4%}")
    print(f"D_near at k<=8: {d_near_k8:.4%}")
    print()

    if d_near_k8 < 0.001:
        print("DECISION: D_near < 0.1% — SimHash is NOT worth implementing.")
        print("  Layer 0 SHA-256 (already live) is sufficient for this corpus.")
        print("  Near-duplicate pairs at cosine >= 0.95 are rarer than break-even threshold.")
    elif d_near_k8 < 0.005:
        print("DECISION: D_near 0.1-0.5% — BORDERLINE.")
        print("  SimHash is net-positive only if write latency measurements confirm")
        print("  that Redis HSET lookup (<1ms) does not dominate savings (~200ms embedding).")
        print("  Recommended: run latency benchmark before implementing.")
    else:
        print("DECISION: D_near > 0.5% — SimHash IS net-positive. IMPLEMENT.")
        print("  Near-duplicate rate exceeds Redis HSET break-even (0.49% at S=203ms).")
        print("  Use k=8 threshold (Q146), Redis HSET storage (Q152).")
        print("  Implementation: token bigram SimHash, 64-bit fingerprint, Redis HSET per bucket.")

    print("=" * 60)

    # Also print as TSV for logging
    print("\nTSV output (for results.tsv):")
    print("question\td_near_k3\td_near_k8\tmean_hamming\tn_memories\tn_pairs")
    mean_h = sum(distances) / len(distances)
    print(f"Q160\t{d_near_k3:.6f}\t{d_near_k8:.6f}\t{mean_h:.2f}\t{n_memories}\t{total}")


# ---------------------------------------------------------------------------
# Main
# ---------------------------------------------------------------------------

def main():
    parser = argparse.ArgumentParser(description="SimHash near-duplicate calibration")
    parser.add_argument("--use-api", action="store_true",
                        help="Use Recall REST API instead of direct Qdrant client")
    parser.add_argument("--api-base", default="http://192.168.50.19:8200",
                        help="Recall API base URL (default: http://192.168.50.19:8200)")
    parser.add_argument("--qdrant-host", default="localhost",
                        help="Qdrant host (default: localhost, use on CasaOS)")
    parser.add_argument("--qdrant-port", type=int, default=6333,
                        help="Qdrant port (default: 6333)")
    parser.add_argument("--collection", default="memories",
                        help="Qdrant collection name (default: memories)")
    parser.add_argument("--n-memories", type=int, default=1000,
                        help="Number of memories to sample (default: 1000)")
    parser.add_argument("--n-pairs", type=int, default=1000,
                        help="Number of random pairs to evaluate (default: 1000)")
    parser.add_argument("--seed", type=int, default=42,
                        help="Random seed for reproducibility (default: 42)")
    args = parser.parse_args()

    random.seed(args.seed)

    if args.use_api:
        contents = fetch_via_recall_api(args.api_base, args.n_memories)
    else:
        contents = fetch_via_qdrant_client(
            args.qdrant_host, args.qdrant_port, args.collection, args.n_memories
        )

    if len(contents) < 10:
        print(f"ERROR: Fetched only {len(contents)} memories — not enough for calibration")
        sys.exit(1)

    analyze(
        contents=contents,
        n_pairs=args.n_pairs,
        k_values=[2, 3, 5, 8, 10, 12, 16],
    )


if __name__ == "__main__":
    main()
```

## Decision Thresholds Explained

| D_near (k<=8) | Meaning | Action |
|---|---|---|
| < 0.1% | Fewer than 1 in 1,000 pairs are near-duplicates | **Skip SimHash** — SHA-256 is sufficient |
| 0.1% – 0.5% | Borderline — marginal net benefit | **Benchmark first** — only implement if Redis HSET <1ms confirmed |
| > 0.5% | Exceeds Redis break-even (0.49% at S=203ms per Q152) | **Implement SimHash** — net-positive |
| > 2% | Clearly worthwhile | **High priority** — ship SimHash + Redis |

**Note on k=3 vs k=8**: The question asks about k<=3 (Hamming distance 3). Q146 established that k=8 is the operative threshold for 80% detection at cosine >= 0.95. The script reports both. For the break-even analysis in Q152, the relevant metric is **what fraction of writes are near-duplicates that SimHash would catch** — and that depends on which k is used. D_near at k=3 is likely 10-100x lower than at k=8, meaning the break-even threshold is harder to exceed at k=3.

## Theoretical D_near Estimate

Given Tim's usage pattern (memories from different sessions about homelab, AI tools, coding):

### Factors pushing D_near UP
- Hook re-fires: `recall-session-summary.js` fires on Stop — if two sessions summarize the same ongoing work ("Tim is working on Recall SimHash"), the summaries will share most bigrams (cosine > 0.95)
- `observe-edit.js` fires on every PostToolUse — repeated edits to the same file produce similar "Tim edited X.py to add Y" memories
- Stable facts repeated across sessions: CasaOS IP, Ollama endpoint, model names — these appear verbatim in multiple memories

### Factors pushing D_near DOWN
- Tim's memories span diverse domains (homelab/infra, Recall itself, FamilyHub, Media Server, coding)
- SHA-256 dedup (Layer 0, already live) catches byte-identical pairs before they enter the corpus
- 22,381 memories × diverse sessions = most pairs are truly unrelated

### Estimate

The theoretical D_near at k=8 is most likely in the **0.2% – 1.5%** range:

| Scenario | D_near (k<=8) | Justification |
|---|---|---|
| Conservative (mostly unique memories) | 0.1% – 0.3% | SHA-256 catches exact dups; diverse topics; few repeated facts |
| Central estimate | 0.3% – 0.8% | Hook re-fires produce similar summaries; stable facts repeated |
| Upper bound (heavy hook-driven corpus) | 0.8% – 2.0% | Many sessions overlap on same ongoing projects |

**Most likely: 0.3% – 0.8%**, which straddles the break-even line for Redis HSET (0.49%).

At k=3 (the question's stated threshold), expected D_near is much lower — likely **0.01% – 0.05%** (mostly byte-identical memories that survived SHA-256 due to minor whitespace differences before normalization was added). At k=3, SimHash is almost certainly **below break-even** for a single-user corpus where Layer 0 SHA-256 is already live.

This means:
- **k=3 question is the wrong question** (confirmed by Q146) — even if D_near(k=3) = 0.05%, it's below break-even and SimHash at k=3 is net-negative
- **k=8 is the actionable threshold** — D_near(k=8) ~0.5% sits right at the Redis break-even line, making the empirical measurement critical
