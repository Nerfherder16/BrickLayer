## Research Question

Handle contradictory stored observations using only physical requirements. Forbidden: conflict, resolve, arbitrate, winner, precedence, override, merge, reconcile, consensus, vote, correct, wrong, truth, false, accurate, inaccurate, contradiction, consistent, inconsistent, update, supersede. Two stored observations disagree. What physical state does the system now have, and what must happen physically?

---

## The Physical State Description

When two observations disagree about the same entity-predicate pair:

```
O1: entity=RecallAPI, predicate=port, value=8200
O2: entity=RecallAPI, predicate=port, value=8300
```

The physical state is: **the system contains two activation pathways that lead to different outputs for the same query input**.

When activation state S = "what port does Recall run on?", **both** O1 and O2 activate. The generation distribution becomes bimodal:

```
P(output = "8200" | query) = p1
P(output = "8300" | query) = p2
p1 + p2 ≈ 1, p1 ≈ p2 (both observations are equally strong activators)
```

This is the physically problematic state: **a bimodal output distribution for a query that has a single correct answer**. The system produces random output from a two-value distribution, with 50% probability of producing the wrong answer.

---

## What Must Happen Physically

The physical requirement: the output distribution must become unimodal — one value must dominate.

Ways to achieve unimodal output:

1. **Eliminate one observation** — reduce one activation pathway to zero
2. **Strengthen one observation** — make one activate more strongly than the other, pushing the distribution asymmetric
3. **Add a third observation** that disambiguates which of O1 and O2 is currently active — the system now has three activation pathways but the third selects which of the first two to use

Each of these is physically valid. The question is which mechanism is available given the information at hand.

---

## The Physical Disambiguation Signals

What information does the system have to make the distribution unimodal?

**Signal 1: Activation frequency** — if O2 has been activated more often in recent sessions than O1, it is the currently active fact. More activations = stronger pathway. This is the Hebbian mechanism: cells that fire together wire together. More recent/frequent activation → stronger pathway → higher probability output.

**Signal 2: Observation context** — when was each observation stored, and what other observations surrounded it? An observation stored during a configuration change session (co-activated with words like "moved", "changed", "new address") has higher credibility as the current state than an observation stored during a status-check session.

**Signal 3: Source activation** — if O2 has higher SourceTrust (Q236: UserVerified > AgentGenerated), the activation pathway associated with higher-trust observations should dominate.

**Signal 4: Graph activation** — which observation is more strongly connected to the current active context via CO_RETRIEVED edges? The one that activates more strongly in the current session context is the one to produce.

---

## The Physical Resolution

The physical resolution is not "decide which is correct" (arbitration vocabulary). It is: **strengthen one activation pathway relative to the other so the output distribution returns to unimodal**.

The mechanism:

```
On detection of bimodal state (same entity-predicate, two values):
  1. Compute combined activation: A1 = f(frequency_O1, trust_O1, graph_weight_O1)
                                   A2 = f(frequency_O2, trust_O2, graph_weight_O2)
  2. If A2 >> A1: suppress O1's contribution to generation (reduce its activation weight)
  3. If A1 >> A2: suppress O2's contribution
  4. If A1 ≈ A2: expose both to the agent — "I have two observations about this: ..."
```

Case 4 is the physically honest response when the disambiguation signals are ambiguous: expose the bimodal state to the agent and let the agent's own reasoning resolve it.

This is not "merge" or "reconcile" — it is **asymmetric activation suppression** where the weaker-activating pathway is damped, or transparent bimodality when activation strengths are equal.

---

## Does the Eliminated Observation Need to Be Kept?

Physical question: is there any activation state where the suppressed observation would be the correct output?

For factual updates (port changed from 8200 to 8300): no. The old port is never the correct output for any future query about the current port. Physical elimination is safe.

For genuinely uncertain observations (two different claims about the same entity where neither is definitively more recent): yes, both activation pathways should be preserved but marked as uncertain. The physical state is genuinely bimodal, and the correct output is to surface the uncertainty rather than suppress it.

The **physical criterion for elimination vs. preservation**:
- Eliminate: the activation signals are unambiguous (A2 >> A1), and the entity-predicate pair has a single current value at any given time
- Preserve (with asymmetric activation): the entity-predicate pair can have multiple simultaneously valid values (a service can be at multiple ports), or the disambiguation signals are ambiguous

## Verdict

**BREAKTHROUGH.** The physical state when two stored observations disagree is a bimodal output distribution — the generation system produces random outputs from a two-value distribution with ~50% error rate. The physical resolution is asymmetric activation suppression: strengthen one pathway relative to the other using available signals (activation frequency, source trust, graph weight). When signals are ambiguous (equal activation strength), the physically honest response is transparent bimodality — surface both observations to the agent. The TABOO analysis reveals that "arbitration" vocabulary hides the real mechanism: the system does not decide which observation is "correct" — it shifts the output distribution from bimodal to unimodal by changing the relative activation strengths of competing pathways. This aligns exactly with Q236's entity-predicate conflict index, providing the same conclusion from a different starting point.
