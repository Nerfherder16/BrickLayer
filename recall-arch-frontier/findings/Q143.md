### Architecture Clarification: co_retrieval_gravity Is Not a Table

The `co_retrieval_gravity` metric visible in the dashboard and API health force profile is computed at query time from Neo4j, not from a PostgreSQL table:

```python
# src/core/health.py lines 118-128
rels = await self.neo4j.get_relationships_for_memory(memory_id)
co_retrieval = sum(
    r.get("strength", 0.5)
    for r in rels
    if r.get("rel_type", "").upper() == "RELATED_TO"
)
co_retrieval = min(co_retrieval / 5.0, 1.0)
```

This is the live computation path. There is no edge accumulation table anywhere in the deployed system.

## Qualifying Ideas

- **co-retrievals-single-migration-deploy-path**: (N=0.82, E=0.92, F=0.98) — The concrete deployment gap is precisely bounded. Q136 produced the exact diff needed: add `_MIGRATE_CO_RETRIEVALS_OUTBOX` after `_MIGRATE_INJECTION_LOG_V5` in `postgres_store.py`, add `await conn.execute(_MIGRATE_CO_RETRIEVALS_OUTBOX)` to `connect()`, and restart the service. Because the constant uses `CREATE TABLE IF NOT EXISTS`, the migration is idempotent and safe to apply to the running production database without downtime or data loss risk. The entire deployment is a two-line source change plus a service restart — the full corrected code already exists in Q136's findings file.

- **neo4j-as-interim-co-retrieval-store**: (N=0.73, E=0.65, F=0.88) — The current system computes `co_retrieval_gravity` from Neo4j `RELATED_TO` edge strength, not from accumulated retrieval co-occurrence. This means co-retrieval reinforcement (each retrieve call strengthening edges between K co-retrieved memories) is currently unimplemented — Neo4j edges are written by other pathways (graph bootstrap, contradiction backfill, cognitive distillation), not by retrieval events. The absence of the co_retrievals table means that 6+ months of retrieval co-occurrence data that could have been strengthening memory graph edges has instead been discarded silently. The deployment gap is not merely a missing table — it is a missing reinforcement signal source feeding both the PostgreSQL edge store and the Neo4j graph.
