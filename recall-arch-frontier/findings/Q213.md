## Evidence

### OS Tier Architecture: Direct Mapping

The two-tier agent memory design from Q211 is structurally identical to OS virtual memory:

| OS Layer | Agent Memory Layer | Physical Properties |
|---|---|---|
| Physical RAM (hot pages) | Working memory tier | Fast random access, bounded capacity, volatile (RAM only) |
| Disk storage (cold pages) | Long-term memory (LMDB) | Durable, high capacity, higher access latency |
| Page table (address translation) | HNSW index | Maps from semantic key to physical storage location |
| Page fault handler | Cache miss handler | On miss: load from slow store into fast tier |
| Page eviction algorithm | Working memory discard | Select which entries to remove from hot tier |
| Page promotion (fault → RAM) | Observation write | Move item from "not loaded" to "in working memory" |
| Page demotion (eviction → disk) | Promotion to long-term | Move item from working memory to LMDB+HNSW (expensive write) |

**Key structural insight**: OS page eviction (RAM → disk) maps to **working memory discard** (discard, not promote). OS page write-back (dirty page → disk) maps to **working memory promotion** (agent decided this is worth keeping → embed and write to LMDB). These are different operations — the agent cannot afford to write all working memory to disk (embedding cost), just as OS cannot afford unlimited disk writes.

### Denning's Working Set Model (1968) Applied to Agent Memory

**Working Set Model**: W(t, τ) = the set of pages referenced in the interval (t - τ, t).

- **τ** (window parameter) = the time horizon defining "currently active" pages
- Optimal τ: captures all pages the process will need, without including cold pages
- Pages not in W(t, τ) at time t → candidates for eviction

**Mapping to agent working memory**:

| OS Concept | Agent Analog |
|---|---|
| τ (time window) | Task duration |
| W(t, τ) | Set of observations accessed during the current task |
| Page referenced in window | Memory entry read after being written (reference bit set) |
| Page not referenced in window | Memory entry written once, never re-read (discard candidate) |
| Process completion | Task completion event (Stop hook) |

**The critical derivation**: Denning showed that setting τ = process execution time (the full run) produces the minimal set that should be kept in memory for program completion. For agent working memory, τ = task duration is exactly Q211's task scope. This is not a coincidence — it is the theoretically optimal scope derived from working set theory.

**Practical implication**: At task end, the working set W(t, τ) = all entries accessed at least once during the task. Entries not in the working set (written but never re-read) = cold pages = discard. Entries in the working set with high activation = "dirty pages with high importance" = promote to long-term.

### CLOCK Algorithm — Production Reference Bit Heuristic

CLOCK is the standard OS page replacement algorithm in production kernels (Linux, FreeBSD, macOS all use CLOCK variants). It provides a practical approximation to LRU using 1 bit per page:

**CLOCK mechanism**:
1. Each page has a reference bit R. R = 1 when page is accessed.
2. Clock hand sweeps pages in circular order.
3. On eviction pressure: if R=1, set R=0 (second chance) and advance. If R=0, evict this page.
4. A page gets a "second chance" before eviction.

**CLOCK applied to working memory entries**:

```
Entry state: (activation_score: f32, timestamp: u64, content: String, keywords: Vec<String>, referenced: bool)

On read: referenced = true
On write (new entry): referenced = false

At task end (Stop hook):
  Pass 1 — Identify discard candidates:
    entries where referenced = false → discard (never re-read = cold = no promotion value)
  Pass 2 — Select promotion candidates:
    entries where referenced = true → sort by activation_score × recency_weight → top-K → promote
```

**Why CLOCK over pure score×recency (Q211)**:

Q211's promotion criterion was "score threshold + minimum_read_count." CLOCK formalizes this with a binary reference bit — computationally cheaper than counting exact reads, and theoretically grounded in OS production evidence.

**Practical data**: In Linux, CLOCK reference bit clearing happens at page scan time. In working memory: reference bit is set on every read() call and cleared at task start. At task end, referenced=false entries are those that were "loaded" (observed and written) but never queried back — exactly the low-value observations that should be discarded.

**Improvement over Q211**: Q211 required a `minimum_read_count` parameter. CLOCK reference bit eliminates this parameter: the reference bit is a 1-bit minimum_read_count. Simpler, matches OS production practice, requires only 1 bool per entry.

### ARC T1/T2 Split — Multi-Access Distinction

ARC (Adaptive Replacement Cache) is used in ZFS's L1ARC and L2ARC caches. Its T1/T2 split directly maps to a quality distinction for working memory promotion candidates.

**ARC structure**:
- **T1**: Pages accessed exactly once recently (hot but unconfirmed)
- **T2**: Pages accessed more than once (repeatedly useful — confirmed value)
- **B1/B2**: Ghost lists of recently evicted T1/T2 pages (used for adaptive parameter tuning)

**ARC mapped to working memory**:

| ARC Component | Working Memory Equivalent |
|---|---|
| T1 (single-access) | Entries read exactly once after write (referenced=true, access_count=1) |
| T2 (multi-access) | Entries read multiple times (referenced=true, access_count≥2) |
| B1 ghost | Recently discarded entries (could be tracked for cold-start detection) |
| B2 ghost | Recently promoted entries (track to detect patterns in what gets promoted) |
| Eviction: T1 shrinks when T2 hits are high | Working memory: extend promotion budget when task produces many multi-read entries |
| Eviction: T2 shrinks when T1 hits are high | Working memory: conservative promotion when most entries are single-read |

**Promotion quality improvement**:

From Q211: top-10% promotion by score×recency regardless of access pattern.

**ARC-improved**: Promotion priority = T2 entries first (multi-access, confirmed useful), then T1 entries (single-access, uncertain) if promotion budget remains. Discard = entries never read (reference bit false). This matches real agent behavior: an observation that the agent returns to multiple times during a task is unambiguously more valuable than one it wrote and never queried.

**ZFS production evidence**: ZFS L2ARC (SSD-backed secondary cache) uses exactly this pattern — L1ARC (RAM) promotes to L2ARC (SSD) based on T2 multi-access classification. Pages that only hit T1 are not promoted to L2ARC. This is 10+ years of production evidence for T1/T2 as a promotion quality signal.

**Implementation**: extend working memory entry to include `access_count: u8` (1 byte, max 255). Set to 1 on write; increment on read; cap at 255. Promotion candidates: entries where `access_count >= 2` first, then `access_count == 1` if budget permits.

### LRU-K Adaptation — K-th Access as Promotion Signal

LRU-K evicts the page whose K-th most recent access is furthest in the past. For K=2: compare second-to-last access time across all pages. This is more stable than LRU-1 for bursty access patterns (pages that get a burst of K accesses in quick succession are not unfairly promoted).

**LRU-2 for working memory promotion**:

For K=2, the promotion score is based on the second access (not the most recent). This penalizes entries that had a burst of reads early in the task but were not consulted again — exactly the "agent reads context at task start, never returns to it" pattern.

**LRU-2 promotion score**:
```
score_k2 = activation_score × (1 / (current_time - second_access_time))
```

If there is no second access (access_count < 2), score_k2 = 0 (never promoted regardless of initial activation).

**Implementation cost**: requires storing up to K=2 access timestamps per entry. Memory cost: 2 × 8 bytes = 16 bytes per entry → at N=200: 3.2KB (fits in L1 cache). The LRU-K circuit breaker (access_count < 2 → score_k2 = 0) subsumes CLOCK reference bit and ARC T1/T2 into a single criterion.

### Pressure-Based Demotion: Not Applicable at N≤200

OS virtual memory uses memory pressure as the demotion trigger: when physical RAM fills, kswapd scans pages and evicts the coldest. For working memory at N≤200 entries:

- At N=200, 200 × 4KB (float32) = 800KB — fits in L3 cache (6MB). No memory pressure at task scale.
- At N=200, 200 × 128B (keyword-indexed, no embeddings) = 25KB — fits in L1 cache.
- **Conclusion**: Pressure-based demotion is not needed for working memory. The bounded N design (Q211) eliminates the need for a page reclaim daemon.
- **Trigger**: The demotion decision is made at task end (Stop hook), not in response to pressure. This is a design simplification compared to OS paging.

**If N grows beyond 200**: A bounded insertion policy (evict lowest score×recency when N_max reached) provides pressure-based demotion within the bounded working memory — exactly CLOCK/LRU behavior. The Q211 design already specifies this: "scored insertion" = evict the lowest-scored entry if N_max exceeded.

---

### Linux Active/Inactive LRU: Two-Level Refinement

Linux's page reclaim uses a two-level structure: **active list** (hot, recently accessed pages) and **inactive list** (cooling pages, candidates for eviction). Pages move from inactive → active on second access; from active → inactive when not accessed for a while.

**Agent memory mapping**:
- Active list = T2 multi-access entries (confirmed useful for current task)
- Inactive list = T1 single-access entries (observed but not confirmed useful)
- Promotion path: inactive → active = entry's access_count transitions from 1 to 2
- Eviction path: active → inactive → discard = entry not accessed for τ time units

This is structurally identical to ARC's T1/T2 split. Linux's two-level LRU provides the same signal as ARC's T1/T2 classification.

### Idea: clock-reference-bit-promotion-heuristic-for-working-memory-entries
- **Novelty**: 0.65 — CLOCK algorithm is well-established in OS. Applying the reference bit specifically to working memory **promotion** (not just eviction) — where reference=false means "discard without embedding" and reference=true means "candidate for embedding and long-term write" — is not documented in any agent memory system. The insight: a single bool per entry replaces Q211's `minimum_read_count` parameter with a theoretically grounded, O(1) production mechanism.
- **Evidence**: 0.85 — CLOCK is production-proven in Linux, FreeBSD, macOS, Windows kernels for 40+ years. The reference bit mechanism is in every OS textbook. Application to working memory promotion: the binary "was this observation queried back during the task?" signal is structurally equivalent to the CLOCK reference bit. Denning's working set theory confirms: pages not in W(t, τ) are cold and should not be promoted.
- **Feasibility**: 0.95 — Implementation: add 1 bool field `referenced: bool` to working memory Vec entry. Set `referenced = true` on every `read()` call. At task end (Stop hook): entries where `referenced = false` → discard; entries where `referenced = true` → promotion candidates sorted by activation_score×recency. Replaces the `minimum_read_count` parameter from Q211 with a theoretically grounded binary criterion. Pure Rust, zero external dependency.
- **Description**: CLOCK algorithm's reference bit applied to working memory: each entry has a `referenced: bool` initialized to false at write. Set to true on any read. At task end: referenced=false entries are discarded (equivalent to CLOCK eviction — never re-accessed during task = cold page). Referenced=true entries are promotion candidates for long-term embedding. This simplifies Q211's promotion criterion from (score_threshold AND minimum_read_count) to (referenced AND score_above_floor) — two parameters collapse to one threshold. Production OS evidence: Linux, macOS, FreeBSD all use CLOCK variants as the default page replacement algorithm.

### Idea: arc-t1-t2-access-count-promotion-priority-for-working-memory

This extends ARC from "which page to evict from cache" to "which entry is worth the 41ms embedding cost at task end."
- **Evidence**: 0.80 — ZFS L1ARC → L2ARC promotion uses T2 classification: only multi-access pages in L1ARC are promoted to L2ARC (SSD tier). This is production evidence across billions of ZFS deployments. The 10+ year ZFS track record validates T1/T2 as a promotion quality signal. Mapping to agent working memory: T2 entries were accessed multiple times during the task = the agent returned to them = high posterior confidence they are useful. - **Feasibility**: 0.75 — Implementation: add `access_count: u8` (1 byte) to entry. Increment on read, cap at 255. The 10% promotion budget from Q211 selects from T2 first; if fewer than 10% are T2, include highest-score T1 entries. Memory cost at N=200: 200 bytes for access_count array (negligible). Replaces the fixed 10% rate with access-pattern-adaptive promotion.
- **Description**: ARC T1/T2 classification applied to working memory promotion: T1 = accessed once (referenced; uncertain value); T2 = accessed ≥2 times (confirmed useful during task). Promotion priority: T2 entries (multi-access) first, T1 entries only if promotion budget permits after T2 is exhausted. This matches ZFS L1ARC→L2ARC semantics (multi-access pages in L1ARC are promoted to SSD-backed L2ARC; single-access pages are evicted to /dev/null). In working memory: multi-access entries are worth the 41ms embedding cost; single-access entries are uncertain; never-accessed entries are discarded. Result: promotion is adaptive to task content — tasks with many multi-access entries have higher promotion rates; tasks with few multi-access entries have lower rates, reducing unnecessary long-term write cost.

## Analytical Summary

**OS virtual memory theory directly validates Q211's working memory design at every level:**

1. **Denning's τ = task duration**: Working set theory confirms task scope is the optimal window parameter. Session scope is suboptimal (retains cold pages from prior tasks).

2. **CLOCK reference bit → simplified promotion criterion**: Replaces Q211's `minimum_read_count` with a theoretically grounded, O(1) binary check. Production-proven in every major OS kernel for 40 years.

3. **ARC T1/T2 → promotion priority**: ZFS L2ARC production evidence supports multi-access as the primary promotion signal. Entries accessed once = T1 (uncertain); entries accessed ≥2 times = T2 (confirmed useful). T2-first promotion reduces unnecessary embedding cost for incidental single-access observations.

4. **Pressure-based demotion not needed at N≤200**: The bounded design eliminates the need for a background reclaim daemon. The Stop hook fires at task end, providing a clean single-point eviction sweep.

**Synthesis**: The optimal working memory promotion algorithm is:

```
At task start: referenced = false, access_count = 0 for all entries

During task:
  write(content, keywords) → (score=1.0, referenced=false, access_count=0)
  read(query) → set referenced=true on matched entries; access_count += 1

At task end (Stop hook):
  CLOCK pass: discard all entries where referenced=false
  ARC pass: sort remaining by (access_count>=2, activation_score×recency)
  Promote top-10% (by ARC-sorted order) → batch embed → LMDB+HNSW write
  Clear working memory
```

This is the union of Q211's promotion design + CLOCK reference bit + ARC T1/T2 priority, all derived from 40+ years of OS production practice.
