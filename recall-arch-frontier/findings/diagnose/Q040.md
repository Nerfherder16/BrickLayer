# Finding: Q040 — Debiasing the Citation Feedback Loop

**Question**: Does the implicit citation feedback loop (Q009) measure genuine memory importance, or does it measure retrieval-rank position bias — and can these two signals be separated from Recall's existing access logs?
**Agent**: quantitative-analyst
**Question Type**: ADVERSARIAL
**Verdict**: BREAKTHROUGH
**Severity**: High
**Source field**: information retrieval, LLM evaluation, recommendation systems

---

## Adversarial Analysis

---

### Thread A: Defense — The citation signal measures genuine importance

**Core claim**: LLM citation is a selective act driven by semantic relevance, not a passive reflection of injection position. A memory cited in 80% of sessions where it appears has genuinely high signal value.

**Evidence**:

1. **LLMs are not passive readers**. Unlike human web search users who exhibit strong position bias (users click the first result at dramatically higher rates regardless of quality), LLMs attend across the full context window and generate responses by reasoning over all injected content. The attention mechanism does not mechanically favor rank-1 content — it weights by semantic coherence with the generation task.

2. **"Lost in the Middle" is context-window-filling, not small-K injection**. Liu et al. (Stanford, 2023) showed performance degradation when relevant content is buried in *long* contexts. Recall's injection is typically K=5 to K=10 memories. Cuconasu et al. (2025, arXiv:2505.15561) confirmed that positional bias in RAG is strongest when retrieval pipelines inject *distracting passages* — semantically similar but wrong content. When all injected memories are genuinely relevant to the user (which is the episodic-memory case), position bias is substantially weaker because the LLM has no reason to favor one over another based purely on order.

3. **Injection-rank correlation with importance is expected and correct**. If higher-importance memories (those previously cited many times) score higher cosine similarity with new queries — which is why they are retrieved at rank 1 — then rank-1 memories *should* be cited more. That correlation is evidence the system is working, not evidence of bias. The signal is: cosine similarity is a good proxy for relevance, and citation confirms it. The feedback loop is a refinement signal, not a noise source.

4. **Citation rate heterogeneity is real**. In a system where memories were uniformly ranked (randomized injection order), you would still expect that the memory containing Tim's CasaOS IP address gets cited whenever a homelab question is asked and not cited when a code question is asked. That differential rate is genuine importance signal, independent of injection rank.

**Best case scenario**: Over 100+ sessions, Recall's importance scores correctly identify the 20% of memories that are durably cross-session valuable — infrastructure facts, architectural decisions, long-standing preferences — versus episodic memories that were relevant once and rarely cited again. The citation loop is a self-correcting signal that routes importance scores toward the correct attractor.

---

### Thread B: Attack — The citation signal is a confounded artifact of the injection policy

**Core claim**: The citation feedback loop does not measure memory importance — it measures the *joint probability of being injected AND cited*, which is fundamentally different from importance. Every boosted score is an artifact of the injection mechanism.

**Evidence**:

1. **The zero-injection problem is categorical, not statistical**. A memory that never appears in the top-K for any query has zero citation probability by mathematical construction — not because it is unimportant, but because it is never injected. The feedback loop can only observe memories that have already crossed the retrieval threshold. This is not a small bias — it is a complete blind spot. Memories that start with low importance scores never get injected, never get cited, never get boosted, and progressively fall further behind. This is the classic *exposure bias* problem in recommender systems (Liang et al., 2023, arXiv:2305.05204; Huang et al., 2026, arXiv:2603.04986).

2. **Position bias in LLM generation is documented and non-trivial**. Multiple papers confirm that LLMs exhibit primacy and recency bias in long-context generation — information at the beginning and end of the context is more faithfully incorporated than information in the middle (Liu et al., 2023 "U-shaped faithfulness"; Wan et al., "On Positional Bias of Faithfulness for Long-form Summarization", arXiv:2410.23609). For Recall's K=5 injection, rank-1 memories are consistently injected first. Even moderate position bias (e.g., 15-20% higher citation rate for rank-1 vs rank-5 content of equal semantic value) would systematically over-boost rank-1 memories. Over hundreds of sessions, this compounds into a structurally biased importance distribution.

3. **The Matthew effect creates a closed self-reinforcing loop**. Once a memory accumulates high importance from early citations (possibly boosted by position bias), it scores higher cosine similarity due to the importance weighting in retrieval, which causes it to be injected at rank 1 again, which causes it to be cited again, which boosts importance further. Memories that start important become more important. Memories that start at medium importance stay there or decline, even if they contain genuinely valuable information. This is the "rich get richer" dynamic documented in recommendation system bias literature (arXiv:2305.18333 "Ranking with Popularity Bias: User Welfare under Self-Amplification Dynamics").

4. **The counterfactual is missing**. The citation signal answers: "Among memories that were injected, which were cited?" It never answers: "If we had injected a different memory, would it have been cited?" Without a counterfactual experiment — randomizing injection to include normally-low-importance memories and measuring their citation rate — every boosted importance score is observational, not causal. The field of counterfactual learning to rank (Joachims et al., IPS for click data) exists precisely because this distinction matters: logged interaction data systematically overestimates the quality of frequently-shown items and underestimates the quality of rarely-shown items.

5. **Semantic similarity conflation**. If a memory's text closely paraphrases a frequent user question (e.g., "Tim's CasaOS runs at 192.168.50.19"), its cosine similarity to any homelab query will be high regardless of whether the information is still relevant. Citation detection using fuzzy string match will flag it as "used" in any session where the response mentions the IP address — even if Claude derived that IP from another source entirely and the memory was merely adjacent context.

**Failure mode**: After 6 months of operation, Recall's importance distribution becomes bimodal — a small cluster of very-high-importance memories that are cited in nearly every session (because they are always injected at rank 1 and always pass the fuzzy citation threshold), and a large tail of medium-to-low importance memories that are chronically under-retrieved and under-cited despite containing genuinely valuable but less frequently accessed information. The system optimizes for *frequency of use*, not *value when used*.

## Synthesis

**What Thread A assumes that Thread B exposes**:

Thread A assumes that injection rank is uncorrelated with citation probability *conditional on relevance* — i.e., that two memories of equal semantic relevance to a query would be cited at equal rates regardless of their injection order. This is an empirical assumption, not an axiom. The LLM position bias literature directly challenges it, and the exposure bias literature makes the structural point that Recall's access logs cannot distinguish "this memory was not cited because it was irrelevant" from "this memory was not cited because it was never injected."

**When Thread A is right**:
- When K is small (K ≤ 5), memories are short, and the context window is nowhere near its limit — the "lost in the middle" effect is negligible, and citation differences between rank-1 and rank-5 memories are dominated by actual relevance differences.
- When the user's query distribution is diverse — covering many different topics — so no single memory gets systematically over-exposed.
- When the initial importance scores are reasonably calibrated (not random), so the injection policy already does a good job of surfacing genuinely relevant memories.

**When Thread B is right**:
- When Recall's injection is importance-weighted (higher-importance memories are always retrieved first), creating a structural coupling between importance rank and injection position.
- When the user's queries are topically narrow (e.g., 80% homelab questions), so a small set of homelab memories are injected in nearly every session regardless of whether they are the most relevant.
- When the fuzzy citation detector has low precision — flagging any memory whose content appears in the response even if Claude sourced that content independently.

**The architecture Thread B would accept — a debiased citation signal from existing logs**:

The key insight from the IPS (Inverse Propensity Scoring) literature in learning-to-rank: the citation signal can be debiased using only logged data, without a randomized experiment, if Recall logs the injection rank at retrieval time. The correction formula is:

```
debiased_citation_signal = citation_boolean / propensity(rank, cosine_similarity)
```

Where `propensity(rank, cosine_similarity)` is the estimated probability that a memory at that rank with that cosine similarity score *would* be cited even if it were not genuinely important — i.e., the baseline citation rate attributable to position and similarity alone.

Recall already logs (or can log cheaply):
- `retrieval_rank` — the position at which this memory was injected (rank 1, 2, ... K)
- `cosine_similarity` — the ANN similarity score at retrieval time
- `citation_boolean` — whether the memory's content appeared in the response
- `session_topic` — available from the query embedding or a lightweight classifier

From these four fields, Recall can fit a logistic regression model:

```
P(cited | rank, cosine_sim, session_topic) = sigmoid(β₀ + β₁·rank + β₂·cosine_sim + β₃·topic_embedding)
```

This model estimates the *expected* citation rate for a memory at that rank and similarity score. The debiased importance update is then:

```
# Raw signal: citation / expected_citation_rate
# Values > 1.0 mean "cited more than expected given rank and similarity" — genuine importance
# Values < 1.0 mean "cited less than expected" — may be contaminated by position noise

debiased_signal = citation_boolean / propensity_estimate

# Apply to importance update:
delta = 0.1 * debiased_signal * (1 - current_importance)   # for citation
importance_new = importance_old + delta
```

For memories that were never injected (zero-injection problem), the approach is different: Recall must implement *importance exploration* — occasionally injecting a randomly selected medium-importance memory instead of the top-K result (epsilon-greedy policy, ε=0.05 to 0.10). This is the only way to generate counterfactual data for under-exposed memories. The explored injections are logged with `is_exploration=True` and get a higher IPS weight in the debiased update since their propensity is known exactly (= ε).

**This architecture is implementable from Recall's existing logs with two additions**:
1. Log `retrieval_rank` and `cosine_similarity` per memory per session (minor schema change).
2. Implement a lightweight propensity model (logistic regression, fitted offline weekly on the last 500 sessions).
3. Apply the IPS-corrected update formula instead of the naive `importance += 0.1 × (1 - importance)`.

The Matthew effect is mitigated by the IPS correction — a rank-1 memory with high cosine similarity gets *discounted* for its citation because the model knows it was expected to be cited at that position. Only citations that exceed the positional baseline are credited as genuine importance evidence.

## Evidence

### LLM Position Bias in RAG

**Liu et al. (Stanford, 2023) — "Lost in the Middle: How Language Models Use Long Contexts"** (cs.stanford.edu/~nfliu/papers/lost-in-the-middle.arxiv2023.pdf): Foundational paper showing U-shaped performance — LLMs use information at the beginning and end of contexts most faithfully, with significant degradation for middle-positioned content. Effect strongest when context fills 25-50% of context window.

**Cuconasu et al. (May 2025, arXiv:2505.15561) — "Do RAG Systems Really Suffer From Positional Bias?"**: Re-examines the Liu result under realistic RAG conditions. Finds that >60% of queries have at least one highly distracting passage in the top-10 retrieved. Position bias interacts with distractor proximity — highly similar-but-wrong passages at rank 1 are problematic. When retrieved passages are all genuinely relevant (the episodic memory case), the effect is weaker but not zero.

**Wan et al. (arXiv:2410.23609) — "On Positional Bias of Faithfulness for Long-form Summarization"**: Confirms U-shaped faithfulness bias in summarization tasks. LLM-based faithfulness metrics are themselves sensitive to document order, introducing meta-measurement bias in any system that uses LLM-based citation detection.

**Zhang et al. (arXiv:2404.01430) — "Position-Aware Parameter Efficient Fine-Tuning"**: Traces position bias to inherent positional preferences in pre-training, not just context-window length. Prompt-based mitigation is insufficient; the bias is architectural.

**Veseli et al. (arXiv:2508.07479) — "Positional Biases Shift as Inputs Approach Context Window Limits"**: Finds LiM effect strongest at 0-50% context window fill. For Recall's small-K injection (K=5, memories typically 100-300 tokens), the bias is in the moderate range — not absent, but not the extreme degradation seen in million-token contexts.

### Exposure Bias and IPS in Recommender Systems

**Liang et al. (SIGIR 2023, arXiv:2305.05204) — "Popularity Debiasing from Exposure to Interaction in Collaborative Filtering"**: Formally proves that learning from implicit feedback without exposure correction leads to systematic overestimation of popular items' quality. The mathematical structure is isomorphic to the Recall citation problem: items that are always shown (memories always injected) accumulate artificially inflated interaction signals.

**Huang et al. (arXiv:2603.04986, March 2026) — "Debiasing Sequential Recommendation with Time-aware Inverse Propensity Scoring"**: Extends IPS to sequential / temporal settings. Directly applicable to Recall's session-sequential structure where memory importance evolves over time.

**Liang et al. (arXiv:2507.07909) — "Document Similarity Enhanced IPS Estimation for Unbiased Learning to Rank"**: Proposes incorporating document similarity into the propensity estimate — directly applicable to Recall's case where cosine similarity is a primary retrieval signal. Low-ranked documents similar to high-ranked ones are likely relevant and deserve IPS correction.

**Chen et al. (ACM TOIS, 2020) — "Bias and Debias in Recommender System: A Survey"**: Taxonomy of bias types. Exposure bias (never-shown items have zero signal), position bias (shown items benefit from rank position), and popularity bias (frequently-shown items accumulate signal artifacts) are all present in the Recall citation loop simultaneously.

### Matthew Effect in Algorithmic Systems

**"Ranking with Popularity Bias: User Welfare under Self-Amplification Dynamics" (arXiv:2305.18333)**: Models the feedback dynamics of popularity-biased ranking. Shows that without intervention, popularity-biased systems converge to fixed points where a small set of items capture nearly all exposure, independent of true item quality. The feedback loop amplification rate depends on the update step size — Recall's `0.1 × (1 - importance)` is a bounded update that slows amplification but does not eliminate the Matthew attractor.

## Ideas Extracted

### Idea: ips-corrected-citation-importance
- **Novelty**: 0.90 — No AI memory system implements IPS-corrected retrieval feedback. The combination of (logged injection rank + logged cosine similarity + logistic propensity model + IPS-corrected importance update) does not appear in any production or research system for episodic AI memory. IPS for learning-to-rank is mature in web search (Joachims et al.) but has never been applied to AI memory feedback loops.
- **Evidence**: 0.72 — IPS for debiasing implicit feedback is peer-reviewed and extensively validated in recommendation systems. The application to AI memory is novel but the mathematics transfers directly. The specific propensity model (logistic regression on rank × cosine_similarity × topic) is an engineering choice, not a theoretical claim.
- **Feasibility**: 0.68 — Requires: (1) schema change to log `retrieval_rank` and `cosine_sim` per memory per session (1 day), (2) a weekly offline propensity model fit on last 500 sessions (1-2 days), (3) updated importance formula in the feedback hook (1 day). Total: 1-2 weeks. The propensity model needs ~200+ sessions of data to fit reliably — system must run in naive mode initially and switch to IPS-corrected mode after sufficient data accumulates.
- **Description**: Extend Recall's session log to record `retrieval_rank` and `cosine_similarity` for each injected memory. After each session, run the citation detection as currently designed (Q009). Instead of applying `importance += 0.1 * citation * (1 - importance)` directly, divide the citation signal by `P(cited | rank, cosine_sim)` — the estimated probability of citation for a memory at that rank with that similarity score, fitted from historical data via logistic regression. A memory cited at rank 3 with cosine_sim=0.72 when the model predicts P(cited)=0.45 gets an IPS weight of 1/0.45 = 2.22 — effectively triple credit, because the citation exceeded positional expectation. A memory cited at rank 1 with cosine_sim=0.95 when the model predicts P(cited)=0.88 gets IPS weight 1/0.88 = 1.14 — barely any credit beyond what position already explains. This separates the genuine importance signal from the structural injection artifact.

### Idea: epsilon-greedy-memory-exploration
- **Novelty**: 0.80 — Exploration policies for memory retrieval (deliberately injecting non-top-K memories to generate counterfactual data) do not appear in any production AI memory system. The concept transfers directly from multi-armed bandit / exploration-exploitation literature.
- **Evidence**: 0.78 — Epsilon-greedy exploration is theoretically validated and widely deployed in recommendation systems for exactly the exposure bias problem described here. The regret bound is known. The trade-off (slight short-term quality degradation for long-term debiasing) is well-characterized.
- **Feasibility**: 0.82 — Trivial to implement: with probability ε (suggested 0.05), substitute one of the top-K memories with a randomly selected memory from the `medium_importance` tier (importance 0.3-0.6), log `is_exploration=True` for that slot, and apply IPS weight 1/ε to its citation signal in the update step. The 5% exploration rate means 1 in 20 sessions has one exploratory injection — Tim would likely never notice. Over 100 sessions, this generates ~5 counterfactual observations per memory in the medium tier.
- **Description**: At retrieval time in `recall-retrieve.js`, after the top-K memories are selected, check a random draw against ε=0.05. If triggered, replace the K-th (last) memory with a randomly selected memory from the medium-importance band. Tag the session log entry `is_exploration=True` for that memory UUID. This generates the counterfactual data needed to fit an unbiased propensity model and to detect whether medium-importance memories have suppressed citation rates due to chronic under-exposure.

### Idea: citation-signal-audit-from-existing-logs
- **Novelty**: 0.65 — Auditing existing logs for position-bias evidence using only rank × citation correlation is a standard diagnostic in IR, but applying it specifically to Recall's access logs to detect whether the citation signal is confounded is novel in the AI memory context.
- **Evidence**: 0.80 — The analytical method (Spearman rank correlation between injection rank and citation boolean, controlling for cosine similarity) is standard and well-validated. The result — either rank correlates with citation after controlling for similarity (position bias present) or it does not (signal is clean) — is definitive.
- **Feasibility**: 0.92 — Requires zero architectural changes. If Recall's session logs already record injection order and citation detection results, a 30-line Python script computes the Spearman correlation between `rank` and `cited` for each session, stratified by cosine similarity decile. The result tells Thread A vs Thread B who is right — empirically, from existing data, before building any debiasing machinery.
- **Description**: Before implementing IPS correction, run the audit: query Recall's PostgreSQL session log for all (memory_id, session_id, injection_rank, cosine_similarity, citation_boolean) tuples. Fit a logistic regression `P(cited | rank, cosine_sim)`. If the rank coefficient β₁ is statistically significant (p < 0.05) and practically meaningful (|β₁| > 0.15 on a standardized scale), position bias is confirmed and IPS correction is warranted. If β₁ is near zero, Thread A is vindicated and the naive citation update is acceptable. This is the minimum-viable diagnostic that should run before committing to the IPS architecture.

## Suggested Follow-ups

- **Implement the citation-signal-audit first** (Q040-audit): Run the Spearman/logistic regression diagnostic on existing Recall session logs before building IPS machinery. Result determines which thread wins empirically.
- **Q041 — Propensity model stability**: How many sessions does Recall need before the logistic propensity model is stable enough to trust? What are the consequences of applying an under-fitted propensity model?
- **Q042 — Fuzzy citation detection precision**: What is the false-positive rate of the fuzzy string matching citation detector (Q009 mechanism)? A detector that fires on any mention of a memory's topic (not just its specific content) will inflate citation rates for topic-rich memories, compounding the position bias problem.
- **Q043 — Exploration-exploitation trade-off for memory retrieval**: At what ε does epsilon-greedy exploration degrade session quality noticeably? Is there a smarter exploration policy (Thompson sampling, UCB) that generates counterfactual data with less quality cost?
- **Connect to Q009 stability concern**: Q009 flagged the need for a stability bound on the feedback loop. IPS correction directly addresses this — by discounting expected citations, the importance update rate for already-high-importance memories drops substantially, limiting the Matthew attractor.
