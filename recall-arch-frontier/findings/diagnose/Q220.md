## Neuroscience Mechanism

During slow-wave sleep (SWS/NREM), the hippocampus replays compressed representations of waking experiences to the neocortex via sharp-wave ripples (SWRs). These bursts couple with cortical slow oscillations (~0.75 Hz) and thalamic sleep spindles (7–14 Hz) — a three-way timing gate that controls which replays are transferred.

**Complementary Learning Systems (CLS) theory** (McClelland, McNaughton & O'Reilly 1995; updated 2016):
- **Hippocampus**: Fast, pattern-separated, episodic store. Encodes unique experiences with minimal interference. High write speed, low generalization.
- **Neocortex**: Slow, distributed, semantic store. Extracts statistical regularities across many exposures. Low write speed, high generalization.
- **Consolidation**: Repeated hippocampal replay during sleep transfers predictive structure from episodic → semantic, until neocortical retrieval no longer requires hippocampal reinstatement.

**Replay trigger**: Short-term synaptic depression destabilizes hippocampal attractors after waking input is removed. The system cycles through stored memories autonomously. NREM phase tightly couples hippocampal reinstatement with neocortical error-correction. REM phase decouples them, allowing neocortex to generalize independently.

**What gets consolidated vs. retained**: Fountas et al. (arXiv Mar 2026) provides the most rigorous formalization: minimize I(X;Z|Y) while preserving I(Y;Z). Discard sensory details that do not predict behavioral outcomes; retain predictive structure. Memories with high outcome-prediction value consolidate. Memories with only incidental sensory context (background, irrelevant co-occurrence) are compressed away.

## Sleep Consolidation vs. K-means / Centroid Approaches

**K-means / centroid**: Unsupervised distance minimization — groups inputs by geometric similarity in embedding space. Produces a centroid that averages all members — can produce a representation matching no actual memory.

**Sleep consolidation differs in three fundamental ways**:

1. **Outcome-conditional, not distance-conditional**: Groups memories by shared *predictive consequences*, not surface similarity. Two memories from different topics that predict the same outcome co-consolidate; similar memories with opposite outcomes are not merged.

2. **Error-driven, not centroid-averaging**: Contrastive Hebbian learning computes the difference between current neocortical prediction and hippocampal reinstatement, then corrects the error. Iterative refinement toward a better generative model — not averaging.

3. **Selective forgetting**: Deliberately discards non-predictive incidental features (minimize I(X;Z|Y)). K-means retains all information losslessly. Consolidation produces a *smaller, more predictive* representation, not a compressed-but-complete one.

**Implication for Recall 2.0**: A pure K-means pass on embeddings + centroid extraction produces mediocre semantics. The biologically-grounded approach requires:
- (a) Cluster by retrieval co-occurrence / behavioral outcome, not raw embedding distance
- (b) Use LLM-based error-corrective refinement (re-query the cluster, verify the consolidated form is correct) rather than averaging

## Recall 2.0 Background Consolidation Worker Design

### Biological → Technical Mapping

| Biological Property | Background Worker Equivalent |
|---|---|
| **Trigger: idle state, waking input removed** | Queue depth below threshold + no active user session + time since last write > N minutes (e.g., 30 min quiet window) |
| **Short-term synaptic depression → consolidation candidates** | TTL-based cooling: memory becomes a candidate only after quiet_since > 24h (not retrieved or updated recently) |
| **Selection: high activation frequency** | CO_RETRIEVED score > threshold OR retrieval_count > threshold in rolling 30-day window |
| **Selection: outcome-predictive** | Memory appears in retrieval sets that preceded user action events (PostToolUse writes indicate agent did something — preceding memories were predictive) |
| **Abstraction: N episodics → 1 semantic** | Cluster episodic embeddings by topic (cosine > 0.82 cluster threshold from Q215); LLM-summarize cluster into single semantic node; retain cluster member IDs as provenance pointers; assign SourceTrust=0.70 (SessionSummary tier) |
| **Guard: unique/identity items** | Exclude: Factual memory type (OD-22), UserVerified SourceTrust=0.90, content matching credential/key patterns, N=1 cluster singleton |
| **NREM phase (tight coupling)**: merge near-duplicates | Pass 1: merge entries with SimHash distance ≤ 6 AND cosine > 0.82 (near-duplicate dedup, extends Q171) |
| **REM phase (generalization)**: extract cross-cluster patterns | Pass 2: identify memories co-retrieved across multiple sessions → generate abstracted semantic summary |

### Algorithm Sketch

```
TRIGGER: idle_detector fires (no session for 30 min, queue depth < 5)

PASS 1 — Near-Duplicate Merge (NREM analog):
  candidates = memories WHERE:
    quiet_since > 24h
    AND (co_retrieved_count > 3 OR retrieval_count > 5)
    AND source_trust < 0.90  // don't touch UserVerified
    AND memory_type != Factual

  For each candidate:
    neighbors = HNSW search (cosine > 0.82)
    For each neighbor with SimHash_distance <= 6:
      → merge pair: keep highest-scored, discard duplicate (extend Q171 dedup)

PASS 2 — Episodic → Semantic Abstraction (REM analog):
  clusters = group candidates by cosine > 0.82 AND cluster_size >= 3

  For each cluster:
    → compute cluster centroid embedding
    → LLM summarize: "These N memories all relate to [topic]. Generate a single
       semantic memory that captures the predictive structure common to all of them."
    → write new semantic memory:
        embedding = centroid
        source_trust = 0.70 (SessionSummary tier)
        content = LLM-generated summary
        provenance_ids = [original memory IDs]
    → soft-delete originals (retain for 7 days, then hard-delete)

GUARD CHECKS (before each merge or abstract):
  - cluster contains N=1: skip
  - any member is UserVerified: skip entire cluster
  - any member matches credential/key patterns: skip
  - consolidated form re-queried with original cluster queries: if top-1 is no longer this cluster → abort and retain originals
```

## Ideas Extracted

### Idea: outcome-conditional-consolidation-co-retrieved-behavioral-signal-is-outcome-predictor
- **Novelty**: 0.80 — The connection between Fountas et al.'s outcome-conditional compression (I(X;Z|Y) minimization) and Recall 2.0's CO_RETRIEVED behavioral signal is novel: CO_RETRIEVED is already measuring outcome-prediction value. Memories with high CO_RETRIEVED were repeatedly retrieved before user actions — they are, by definition, outcome-predictive. Using CO_RETRIEVED as the primary consolidation selection criterion is the biologically-grounded choice, not retrieval frequency alone. This is not documented anywhere.
- **Evidence**: 0.80 — Fountas et al. (arXiv 2603.04688, Mar 2026): formal proof that optimal consolidation minimizes I(X;Z|Y) — discard features not predicting Y (behavioral outcomes). CO_RETRIEVED (Q206): behavioral score accumulates when memory is retrieved before agent actions (PostToolUse hooks). Direct equivalence: CO_RETRIEVED ≈ I(memory; agent_action). High CO_RETRIEVED memories should consolidate; low CO_RETRIEVED memories that are also low-recency should be discarded. Nalagatla & Grandhe (Nov 2025): predictive coding generative replay outperforms distance-based replay by 15.3% — same principle.
- **Feasibility**: 0.85 — Selection criterion: CO_RETRIEVED > 0.3 OR retrieval_count > 5 (rolling 30-day window). Both fields already tracked in Recall 2.0 memory records (Q206). Abstraction: LLM summarization of selected cluster. Guard: exclude Factual type + UserVerified SourceTrust. Trigger: idle_since > 30min (Tokio timer, no polling). Background task preempted by active sessions. Phase 3 CO_RETRIEVED sprint delivers the behavioral score; consolidation uses it in Phase 5.
- **Description**: Recall 2.0 background consolidation worker uses CO_RETRIEVED behavioral score as the primary consolidation selection criterion — directly implementing Fountas et al.'s outcome-conditional compression (I(X;Z|Y) minimization). Memories with CO_RETRIEVED > 0.3 (retrieved before agent actions) are outcome-predictive; they become abstraction candidates when they form topic clusters (cosine > 0.82, N ≥ 3). Low-CO_RETRIEVED, low-recency memories are discarded without abstraction (soft-delete, 7-day retention). This is biologically grounded: the brain does not consolidate everything — it consolidates what predicted useful behavior. K-means centroid averaging fails this test; CO_RETRIEVED-gated selection passes it.

### Idea: two-phase-nrem-rem-analog-dedup-then-abstract-consolidation-worker
- **Novelty**: 0.75 — The two-phase design (NREM analog: merge near-duplicates → REM analog: extract cross-cluster semantic abstractions) is not implemented in any production memory system (Q224 absence finding anticipated). The specific phase structure mirrors the biological NREM/REM sequence and provides distinct quality guarantees: Phase 1 (dedup merge) is lossless within the SimHash near-duplicate window; Phase 2 (abstraction) is lossy but guided by LLM. Running them in sequence produces both a leaner corpus (Phase 1) and richer semantic nodes (Phase 2).
- **Evidence**: 0.75 — C-HORSE (Singh et al. PNAS 2022): NREM = tight hippocampal-neocortical coupling (exact replay → error correction); REM = decoupled generalization (neocortical abstraction without hippocampal gating); the two-phase structure is biologically confirmed. Phase 1 extends Q171 SimHash dedup (already designed) to include temporal persistence logic (quiet_since > 24h). Phase 2 reuses Q215's contradiction detection threshold (cosine > 0.82) as cluster boundary. LLM summarization: Ollama qwen3:14b locally available (Tim's homelab, 100.70.195.84). Soft-delete with 7-day retention: reversible.
- **Feasibility**: 0.85 — Phase 1: HNSW search + SimHash check per candidate (same path as Q215 contradiction detection, reuse existing infrastructure); LMDB mark-for-delete on lower-scored duplicate. Phase 2: LLM call per cluster (Ollama qwen3:14b, 200-500ms/cluster); write new semantic node to LMDB+HNSW; mark originals soft-deleted. Background Tokio task: lowest priority, preempted by sessions. Total runtime: 5-30 seconds for 10-20 clusters. Reversibility: soft-delete with 7-day hard-delete lag allows user to review or undo. Phase 5 scope (after Phase 4 write-path defense).
- **Description**: Two-phase background consolidation worker for Recall 2.0: Phase 1 (NREM analog) merges near-duplicate memories with SimHash_distance ≤ 6 AND cosine > 0.82 AND quiet_since > 24h — lossless deduplication triggered at idle. Phase 2 (REM analog) identifies topic clusters (cosine > 0.82, N ≥ 3) among high-CO_RETRIEVED candidates and LLM-summarizes each cluster into a single semantic memory at SourceTrust=0.70 — lossy abstraction retaining provenance pointers to original members. Guards: exclude UserVerified SourceTrust, Factual memory type, credential patterns, N=1 singletons. Soft-delete with 7-day retention window. Total runtime 5-30s for typical 10-20 cluster runs, lowest-priority background task. Extends Q171 SimHash dedup and Q215 cosine threshold infrastructure with temporal persistence logic.

### Idea: idle-trigger-quiet-window-consolidation-not-scheduled-event-driven
- **Novelty**: 0.55 — The specific trigger design — idle detector (no session for 30 min, queue depth below threshold) rather than a cron schedule — mirrors the biological trigger (waking input removed → short-term depression → replay begins). Scheduled consolidation is the obvious implementation; event-driven idle detection is the biologically-correct design. The distinction matters: scheduled consolidation can run during active sessions (degrading performance); idle-triggered consolidation is guaranteed to run only when the system is not under load.
- **Evidence**: 0.75 — Biological: short-term synaptic depression destabilizes hippocampal attractors when waking input is removed — idle state is the trigger, not a timer. Recall 2.0: the Stop hook already detects session end; a 30-minute quiet timer after Stop hook fires (no new UserPromptSubmit in 30 min) is the idle condition. Queue depth < 5: ensures no pending writes are being processed. Tokio timer reset on any hook event. Background task preempt: any incoming request cancels the consolidation run and reschedules.
- **Feasibility**: 0.95 — Tokio timer after last hook event: 30-minute countdown, reset on any incoming request. Queue depth check: atomic counter in Rust. Consolidation task: spawned as lowest-priority async task via `tokio::task::spawn` with `tokio::task::yield_now()` throughout. Preemption: cancellation token on active session detection. Zero impact on request latency when user is active. Phase 5 scope.
- **Description**: Recall 2.0 consolidation worker triggers on idle detection (event-driven), not a cron schedule: 30-minute quiet window after last hook event (no UserPromptSubmit, PostToolUse, or Stop in 30 min) AND queue depth below threshold. Mirrors biological trigger (waking input removal → synaptic depression → replay begins). Advantages over scheduled consolidation: (1) guaranteed not to run during active sessions (no performance impact); (2) adapts to usage patterns (heavy users get consolidation less frequently; light users get it more); (3) natural backpressure when system is under load (quiet window never starts). Tokio timer reset on any hook event. Cancellation token fires if session starts mid-consolidation.

## Analytical Summary

**Neuroscience sleep consolidation provides a complete and biologically-validated design template for a Recall 2.0 background consolidation worker.** The three key insights that distinguish it from naive K-means/centroid approaches:

1. **Outcome-conditional selection**: Consolidate what CO_RETRIEVED identifies as outcome-predictive; discard what is merely semantically similar.
2. **Two-phase sequence**: NREM analog (lossless dedup) → REM analog (lossy abstraction). These are qualitatively different operations and must be run in sequence.
3. **Idle trigger**: Event-driven quiet window, not scheduled — guarantees consolidation never competes with active session load.

**The consolidation worker is Phase 5 scope** — after the write-path defense stack (Phase 4, Q215-Q217), it addresses the orthogonal problem of corpus size management. Q221 will model the crossover point at which consolidation becomes mandatory for maintaining retrieval quality.
