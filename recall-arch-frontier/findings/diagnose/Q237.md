## Research Question

Define the unit of memory using only physical requirements. Forbidden: document, record, row, entry, chunk, unit, segment, piece, text, string, blob, object, entity, fact, statement, sentence, paragraph, item, element, node, token, byte. What physical property makes two observations "the same memory" vs. "two different memories"?

---

## The Physical Question

What is the minimal indivisible thing that a memory system stores?

The forbidden vocabulary (document, record, sentence, fact) answers this at the **representation** layer — how we choose to encode observations. But the physical question is prior: what makes two observations "the same" in terms of what the system actually stores?

---

## The Physical Answer: Indistinguishability Under the Retrieval Operation

Two observations are "the same memory" if and only if **no retrieval operation can distinguish between them** — they surface together under every possible activation state.

Formally: observations O1 and O2 are the same memory if and only if for all possible current states S:

```
P(surface O1 | state S) = P(surface O2 | state S)
```

They are different memories if there exists at least one state S where the two surface probabilities differ.

**The minimal indivisible memory** is the smallest set of observations that cannot be split by any retrieval operation — the atomic unit under the equivalence relation defined by mutual surfacing probability.

---

## What This Implies

**Two observations merge into one memory** when they co-occur in every possible context — when there is no activation state that surfaces one without the other. Practically: when their information content is fully redundant given the other.

This is the physical definition underlying **deduplication**: two observations are the same memory when their mutual information I(O1; O2) approaches the entropy of each — when each fully predicts the other.

**Two observations are different memories** when there exists at least one activation state that surfaces one without the other. They carry non-redundant information about different aspects of the world.

---

## The Implication for Storage Granularity

Current practice: one sentence ≈ one stored observation (50-200 chars).

The physical analysis reveals this heuristic is approximately correct but not principled:

**Too coarse** (should split): A 200-char observation that asserts two independent facts about different entities. Example: "The Recall API is at 100.70.195.84:8200 and the dashboard uses port 3100." Under the physical definition, this is two observations (different entities, different predicates) — they can be surfaced independently.

**Too fine** (should merge): Two consecutive 50-char observations that together form a single assertion. Example: "Recall uses LMDB" followed immediately by "for persistence" — neither is independently surfaceable without the other.

**Principled boundary**: split at assertion boundaries (entity-predicate-value completeness), not at character count or sentence breaks.

---

## The Physical Unit

The physically minimal memory is an **assertion**: a complete claim about a single (entity, predicate, value) triple that is independently retrievable.

```
NOT a memory: "uses"                          → incomplete (not independently surfaceable)
NOT a memory: "LMDB for persistence and HNSW" → two assertions (split)
IS a memory:  "Recall 2.0 uses LMDB for persistence"    → complete assertion
IS a memory:  "Recall 2.0 uses HNSW for vector search"  → complete assertion
```

This is the physical equivalent of a **predicate logic atom**: Predicate(subject, object).

The entity-predicate-value triple is not a document format — it is the physical unit because it is the minimal structure that:
1. Can be independently activated (has a specific retrieval context)
2. Can be contradicted (as found in Q236)
3. Cannot be split without losing surfaceability

---

## Does the Physical Unit Match the Current Model?

The current Recall 2.0 model stores ~50-200 char text observations, which are approximately aligned with individual assertions. The physical analysis confirms this is approximately correct. The gaps:

1. **Multi-assertion observations** (should split): storage hooks currently concatenate multiple facts from a single edit into one observation. Physical requirement: split at assertion boundaries.

2. **Sub-assertion observations** (should merge): rare — the hooks mostly capture complete observations. But some very short observations ("LMDB") are not independently surfaceable.

**Practical implication**: the current granularity heuristic (sentence-level) is within 2× of the physical optimum (assertion-level). The gap is primarily in multi-assertion observations from verbose file edits.

## Verdict

**BREAKTHROUGH.** The physical unit of memory is the **assertion** — the minimal observation that is independently surfaceable under some retrieval activation state and cannot be split without losing that property. This is the entity-predicate-value triple from predicate logic, derived from physical requirements alone. The current sentence-level granularity heuristic is approximately correct but within 2× of the optimum — primarily failing on multi-assertion observations from verbose storage. The physical unit analysis directly reinforces Q236 (contradiction detection requires assertion-level granularity) and provides the principled storage boundary rule that the current system lacks: split at assertion completeness, not at character count or sentence breaks.
