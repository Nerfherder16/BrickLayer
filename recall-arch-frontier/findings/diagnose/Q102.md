### Section 5: Cypher Queries for Live Fanout Scan

All queries are designed to complete in under 100ms on Tim's graph at any scale through Month 24 (~11,300 nodes, ~19,200 edges). The `size()` function on relationship counts in Neo4j 4.x+ uses the relationship type index and runs in O(1) per node, giving O(N) total for a full-graph scan — approximately 1-5ms at 5,000-10,000 nodes.

**Query S6-GAUGE: Single-value health metric (run as Prometheus cron)**
```cypher
MATCH (n:Memory)
RETURN max(size((n)-[:CO_RETRIEVED]->())) AS max_fanout
```
Expected runtime: < 10ms at any scale through Month 36.

**Query S6-HISTOGRAM: Full degree distribution (run weekly for monitoring)**
```cypher
MATCH (n:Memory)
WITH size((n)-[:CO_RETRIEVED]->()) AS fanout
WHERE fanout > 0
WITH
  CASE
    WHEN fanout <= 5   THEN '01-05'
    WHEN fanout <= 20  THEN '06-20'
    WHEN fanout <= 50  THEN '21-50'
    WHEN fanout <= 100 THEN '51-100'
    WHEN fanout <= 158 THEN '101-158'
    WHEN fanout <= 200 THEN '159-200'
    WHEN fanout <= 500 THEN '201-500'
    ELSE               '500+'
  END AS bucket,
  fanout
RETURN bucket, count(*) AS node_count, max(fanout) AS max_in_bucket
ORDER BY bucket
```
Expected runtime: < 50ms at Month 12, < 100ms at Month 24.

**Query S6-THRESHOLDS: Threshold count summary (run as health check)**
```cypher
MATCH (n:Memory)
WITH size((n)-[:CO_RETRIEVED]->()) AS fanout
RETURN
  count(CASE WHEN fanout > 0   THEN 1 END) AS nodes_with_edges,
  count(CASE WHEN fanout > 20  THEN 1 END) AS nodes_minor_hub,
  count(CASE WHEN fanout > 50  THEN 1 END) AS nodes_moderate_hub,
  count(CASE WHEN fanout > 158 THEN 1 END) AS nodes_timeout_risk_warm,
  count(CASE WHEN fanout > 200 THEN 1 END) AS nodes_major_hub,
  max(fanout)                               AS max_fanout,
  avg(CASE WHEN fanout > 0 THEN toFloat(fanout) END) AS mean_nonzero_fanout
```
Expected runtime: < 30ms at Month 12.

**Query S6-TOPHUBS: Identity of top hub nodes (run on WARNING trigger)**
```cypher
MATCH (n:Memory)
WITH n, size((n)-[:CO_RETRIEVED]->()) AS fanout
WHERE fanout > 20
RETURN n.id AS memory_id,
       left(n.content, 80) AS preview,
       fanout
ORDER BY fanout DESC
LIMIT 20
```
This identifies which specific memories are hub candidates, enabling targeted epoch-filter activation (Q078 Option A) before the circuit breaker fires.
