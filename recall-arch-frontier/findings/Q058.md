# Finding: Q058 — IPS Variance Floor and the Crossover Conditions for Measurable P@3 Improvement

**Question**: What is the minimum measurable retrieval quality improvement from IPS importance correction — specifically, what is the smallest corpus size and session count at which the IPS propensity model produces statistically distinguishable precision@3 improvements over unweighted citation counts, and how do you measure it without ground-truth relevance judgments?
**Agent**: quantitative-analyst
**Question Type**: PHYSICS
**Verdict**: PROMISING
**Severity**: Medium
**Source field**: statistics, importance sampling, counterfactual learning to rank

---

## Operation Definition

- **Input**: N session logs, each containing (memory_id, injection_rank, cosine_similarity, citation_boolean, is_exploration) per injected slot. Epsilon-greedy policy: epsilon=0.05 exploration rate, so 5% of injection slots are random draws from the medium-importance band.
- **Output**: Two ranked lists for each query — (1) IPS-corrected importance ranking, (2) unweighted raw citation-count ranking — and a statistical test distinguishing their precision@3.
- **Guarantees required**: The test must be valid without external relevance labels. The only available signal is within-session retrieval frequency.
- **Key parameters**: epsilon=0.05, IPS weight for explored memory = 1/epsilon = 20, IPS weight for exploited memory = 1/(1-epsilon) = 1.0526, sessions/day = 5, K injection slots = 5.

## Theoretical Minimum

| Resource | Minimum | Calculation |
|---|---|---|
| Sessions for propensity model stability | 67 sessions | 30 citation events / (K * alpha * q) = 30 / (5 * 0.3 * 0.3) = 67; logistic regression with 3 parameters needs ~10 events/parameter |
| Sessions for first statistically valid test | 120 sessions | 30 exploration events / (K * epsilon) = 30 / (5 * 0.05) = 120; chi-squared test minimum cell count |

## IPS Variance Analysis

### The fundamental estimator

Let Y_t = 1 if memory m is cited in session t, 0 otherwise. Let P(A_t) be the propensity — the probability that memory m was injected in session t under the logging policy.

The IPS estimator of true importance q (probability of citation if injected at random) is:

```
q_hat_IPS = (1/K) * sum_{t=1}^{K} [ Y_t / P(A_t) ]
```

where K is the number of sessions in which memory m appeared. Under epsilon-greedy:
- Exploration events (5% of sessions): P(A_t) = epsilon = 0.05, IPS weight = 20
- Exploitation events (95% of sessions): P(A_t) = 1 - epsilon = 0.95, IPS weight = 1.0526

### Variance derivation

For a single event, since Y is Bernoulli with parameter q and Y^2 = Y:

```
Var(Y/p) = E[Y^2/p^2] - (E[Y/p])^2
         = q/p^2 - (q/p)^2
         = q*(1-q) / p^2
```

Per-event variance coefficients:
- Explored event (p = epsilon = 0.05): coefficient = 1/eps^2 = **400**
- Exploited event (p = 1-eps = 0.95): coefficient = 1/(1-eps)^2 = **1.108**
- Variance ratio: 400 / 1.108 = **361x**

The variance of the average IPS estimator over K appearances mixes the two event types. Since a fraction epsilon of appearances are explorations and (1-epsilon) are exploitations:

```
Var(q_hat_IPS) = (q*(1-q) / K) * [eps * (1/eps^2) + (1-eps) * (1/(1-eps)^2)]
              = (q*(1-q) / K) * [1/eps + 1/(1-eps)]
              = (q*(1-q) / K) * V_coeff
```

where **V_coeff = 1/epsilon + 1/(1-epsilon) = 20 + 1.0526 = 21.053** at epsilon=0.05.

This is the closed-form variance formula. The 1/epsilon term dominates: exploration events contribute 95% of total estimator variance despite being only 5% of events.

### Relative Standard Error as a function of K_appearances

```
RSE(q_hat_IPS) = std(q_hat_IPS) / q = sqrt((1-q)/q * V_coeff / K)
```

For q=0.3 (typical citation rate), RSE as a function of K appearances and N sessions (alpha=0.3):

| K appearances | N sessions | RSE | 90% CI width |
|---|---|---|---|
| 10 | 33 | 222% | ±1.33 |
| 50 | 167 | 99% | ±0.59 |
| 126 | 420 (Week 12) | 62% | ±0.37 |
| 500 | 1,667 | 31% | ±0.19 |
| 4,912 | 16,374 | 10% | ±0.06 |

RSE below 10% requires 16,374 sessions (9 years at 5/day). **This is not the relevant threshold.** The IPS estimator does not need to be individually precise per memory — it only needs to produce a ranking that is better than raw counts in aggregate.

### Why exploration events dominate and cannot be escaped

The high variance is a mathematical consequence of epsilon=0.05. The IPS weight for exploration events is 1/epsilon = 20. This is not a design flaw — it is the cost of unbiased correction. Reducing epsilon (say, to 0.01) would lower the variance coefficient but would also reduce the number of exploration events, yielding the same total variance per unit time. The variance per session is invariant to epsilon (the exploration-exploitation trade-off is symmetric in variance terms). The only way to reduce variance is more sessions or a larger corpus with higher alpha.

## Crossover Analysis

### The correct question: distinguishability vs. distinguishability of improvement

The task contains a subtle ambiguity. "Statistically distinguishable precision@3 improvements" can mean:

**Form A — Ranking divergence**: Does the IPS-ranked list differ from the raw-count-ranked list?
- First divergence: ~22 sessions after first exploration citation (mathematically trivial, statistically meaningless)
- Stable divergence (not noise): ~333 sessions for alpha=0.3 (5 exploration events per memory)

**Form B — Ranking quality improvement**: Is the IPS-ranked list measurably better than the raw-count list?
- This requires a measurement oracle. Without external labels, the only valid oracle is exploration-event citations (propensity known exactly = epsilon).

**Form C — Per-memory importance precision**: Is each IPS importance estimate individually reliable?
- Requires RSE < 10%: 16,374 sessions. Unreachable at 5/day.

The operationally correct form is **Form B**, and it has a tractable solution.

### Measuring improvement without ground truth: exploration events as oracle

When a memory is injected via exploration (probability = epsilon, exactly known), the citation event is **uncontaminated by position bias and importance-rank correlation**. The true citation probability is directly observable: P(cited | explored) = true_relevance. This is ground truth. It is generated by the system itself.

The measurement protocol:

1. Partition each session log into exploitation events and exploration events.
2. For each exploration event, record: (memory_id, was_cited, rank_under_IPS, rank_under_raw_count).
3. Compute P@3 for each ranking system **on exploration events only**:
   - IPS P@3 = fraction of exploration events where the memory ranked in top-3 by IPS was cited
   - Raw P@3 = fraction of exploration events where the memory ranked in top-3 by raw count was cited
4. Apply Wilcoxon signed-rank test (paired, non-parametric) on the per-session P@3 differences.

This is valid because the exploration oracle is unbiased. The exploitation events are used for fitting the propensity model; the exploration events are held out for evaluation.

### Session counts for this measurement

| Milestone | N sessions | Time at 5/day | Formula |
|---|---|---|---|
| Propensity model stable (30 citation events) | 67 | 13 days | 30 / (K * alpha * q) = 30 / (5 * 0.3 * 0.3) |
| Chi-squared test valid (30 exploration events) | 120 | 24 days | 30 / (K * epsilon) = 30 / (5 * 0.05) |
| Wilcoxon test valid (44 exploration comparisons) | 174 | 35 days = 5 weeks | n_comparisons = 44; N = 44 / (K * eps) |
| Power=0.80 at 15% P@3 improvement (44 comparisons, Wilcoxon) | 174 | 5 weeks | Derived from (z_alpha + z_beta)^2 / (2 * ES^2), ES = 0.15/0.5 |
| 5 exploration events per memory (finite-sample IPS validity, alpha=0.3) | 333 | 67 days = ~10 weeks | 5 / (alpha * epsilon) |

**The minimum corpus size for a valid measurement is 1 memory in the medium-importance band.** The minimum session count is ~174 sessions (5 weeks) for power=0.80 at a 15% improvement threshold.

### Crossover formula (general)

The minimum session count N* for detecting a P@3 improvement of size delta_p at power (1-beta) and significance alpha is:

```
N* = n_comparisons* / (K_slots * epsilon)

where n_comparisons* = (z_alpha + z_beta)^2 / (2 * ES^2)
      ES = delta_p / sqrt(p_null * (1 - p_null))
      p_null = 0.5 (null: both rankings equally good)
```

For the specific Recall parameters (K=5, epsilon=0.05, alpha=0.05, beta=0.20):

```
N*(delta_p) = (2.80)^2 / (2 * (delta_p / 0.5)^2) / 0.25
            = 15.68 / (8 * delta_p^2)
```

At delta_p = 0.10: N* = 196 sessions (39 days)
At delta_p = 0.15: N* = 87 sessions (17 days)
At delta_p = 0.20: N* = 49 sessions (10 days)

Note: these are sessions for the Wilcoxon test to be powered. The propensity model must already be stable (67 sessions), which is the binding early constraint.

### Comparison of ranking divergence thresholds (Form A)

For a 2-sigma detectable ranking swap between two specific memories with true importance gap delta:

```
N_swap(delta, alpha) = 8 * q * (1-q) * V_coeff / (delta^2 * alpha)
                     = 8 * 0.21 * 21.05 / (delta^2 * alpha)
                     = 35.4 / (delta^2 * alpha)
```

| delta (true importance gap) | N (alpha=0.3) | Months at 5/day |
|---|---|---|
| 0.05 | 47,158 | 315 months (26 years) |
| 0.10 | 11,789 | 79 months (6.5 years) |
| 0.15 | 5,240 | 35 months (3 years) |
| 0.20 | 2,947 | 20 months (1.7 years) |
| 0.30 | 1,310 | 9 months |

These thresholds apply to individual per-memory ranking reliability, not aggregate P@3 improvement. **The aggregate P@3 improvement test (Form B, 174 sessions) is achievable; individual per-memory ranking precision (Form A/C) is not achievable at sub-year timescales.**

### Is Month 3 in the Q025 build schedule premature?

At Week 12 (Month 3), the system has accumulated ~420 sessions:
- Propensity model stability: **YES** (stable after 67 sessions, Week 2)
- Chi-squared test of improvement: **YES** (valid from 120 sessions, 24 days)
- Wilcoxon test, power=0.80 at 15% improvement: **YES** (174 sessions, 35 days)
- Individual memory importance estimates reliable: **NO** (RSE = 62% at Week 12)
- IPS ranking demonstrably better than raw counts per-memory: **NO** (would require 11,789+ sessions)

**Month 3 is justified.** The propensity model trains in Week 1. The IPS correction starts producing a structurally correct signal (debiased in expectation) immediately at deployment. The evaluation framework (Wilcoxon on exploration events) becomes powered at Week 5. The system is in beneficial operation well before the Month 3 deployment target.

The caveat: "IPS produces statistically distinguishable P@3 improvements" in the strong per-memory precision sense requires years of data. In the aggregate P@3 signal sense, it requires 5 weeks. Month 3 is the right time to deploy the correction and begin the evaluation loop — it is not the time when the correction will be provably optimal.

## Ideas Extracted

### Idea: exploration-oracle-evaluation
- **Novelty**: 0.75 — The insight that epsilon-greedy exploration events are internally generated ground truth (propensity exactly known = epsilon) enabling powered statistical tests without external labels is not widely recognized. Most IPS literature assumes ground truth is unavailable; here it is generated for free by the exploration policy itself.
- **Evidence**: 0.80 — This is a direct application of the doubly-robust estimator framework from Dudik et al. (2011, "Doubly Robust Policy Evaluation and Learning") and the self-normalized IPS framework. The math is established; the application to AI memory retrieval without external labels is novel.
- **Feasibility**: 0.90 — Requires only that is_exploration=True be logged per slot. No external infrastructure. Evaluation runs as a weekly Python script over the session log. Implementable in 1 day.
- **Description**: Tag exploration injection events with `is_exploration=True` in the session log. Hold out all exploration events from propensity model training. Weekly, compute P@3 on exploration events under IPS ranking vs. raw-count ranking. Apply Wilcoxon signed-rank test. The test is powered at 174 sessions (5 weeks after exploration logging starts). This is the only statistically valid method for evaluating IPS improvement without external relevance labels.

### Idea: variance-aware-epsilon-schedule
- **Novelty**: 0.70 — Adapting epsilon over time based on IPS estimator variance (decreasing epsilon as confidence in propensity model increases) is a known technique in bandit literature but has not been applied to AI memory retrieval feedback.
- **Evidence**: 0.65 — Decaying exploration schedules (e.g., epsilon proportional to 1/sqrt(N)) are theoretically optimal for sublinear regret in UCB-style bandits. Directly applicable if the corpus is bounded.
- **Feasibility**: 0.72 — Requires a per-memory variance tracker in PostgreSQL. At deployment (Week 12), epsilon=0.05 is appropriate. Once memories accumulate >100 appearances, their IPS estimate is stable enough to reduce exploration to epsilon=0.01. Reduces quality degradation from exploration over time.
- **Description**: Compute the current RSE for each memory's IPS estimate. Memories with RSE < 50% have sufficient data — reduce their exploration probability to 0.01. Memories with RSE > 150% are high-priority exploration targets — maintain epsilon=0.05 or increase to 0.10. This focuses exploration budget where the information gain is highest (high-variance memories) and reduces the quality tax on well-characterized memories.

### Idea: session-count-stratified-propensity-model
- **Novelty**: 0.67 — Fitting separate propensity models for the early (N < 100 sessions), mid (100-500), and mature (>500) phases of system operation, because the propensity distribution shifts as the importance score distribution drifts from its initialization. Using a non-stationary propensity model that re-fits on a rolling window rather than the full history.
- **Evidence**: 0.60 — Time-varying propensity models are standard in causal inference for panel data but not commonly used in recommendation system IPS. The justification here is empirical: early sessions have less calibrated importance scores, so the rank-citation correlation is different from mature sessions.
- **Feasibility**: 0.80 — A rolling 100-session window for logistic regression fitting costs < 1ms per session. The model is refitted weekly (or after each batch of 20 sessions). No schema changes required beyond what Q040 already specifies.
- **Description**: The propensity model P(cited | rank, cosine_sim) should not be fitted on all historical sessions equally. Sessions from the first 100 (when importance scores are near their initial values) have a different rank-citation relationship than sessions from Month 6 (when importance scores have been updated many times). Fit the propensity model on a rolling window of the last 200-500 sessions to track the current operating point of the system.

## Key Finding

The IPS estimator variance at epsilon=0.05 is governed by the formula:

```
Var(q_hat_IPS) = q*(1-q) * (1/epsilon + 1/(1-epsilon)) / K_appearances
              = q*(1-q) * 21.05 / K_appearances
```

The variance is dominated by exploration events (contributing 95% of variance from 5% of events) because their IPS weight (20x) inflates the squared contribution by 400x. This is a **physics gap** — it cannot be reduced without either more sessions or higher epsilon.

However, this high per-memory variance does not prevent aggregate P@3 improvement from being measured. The key insight is that **epsilon-greedy exploration events constitute internally generated ground truth**: their propensity is known exactly (= epsilon), and their citations are uncontaminated by position bias. This enables a powered Wilcoxon test for P@3 improvement after only 174 sessions (5 weeks), without any external relevance labels.

The crossover conditions, in order of achievability:

1. **Propensity model deployable**: 67 sessions (~Week 2) — achievable before Month 3 target
2. **IPS correction mathematically sound from deployment**: immediate — the estimator is unbiased regardless of K
3. **P@3 improvement detectable (power=0.80, delta_p=0.15)**: ~87 sessions (17 days, ~Week 3)
4. **5 exploration events per memory (finite-sample IPS validity, alpha=0.3)**: 333 sessions (~10 weeks, Month 2.5)
5. **Individual per-memory ranking reliable (RSE < 50%)**: ~1,600 sessions (~6 months)
6. **Individual per-memory ranking precise (RSE < 10%)**: 16,374 sessions (9 years — never reached)

Month 3 deployment (Q025 schedule) is **not premature** for deploying the IPS correction mechanism. It is premature for claiming that individual importance estimates are reliable, but that claim is never reachable at Tim's session rate — it is the wrong measurement target. The correct measurement target is aggregate P@3 improvement via exploration-oracle evaluation, which is powered after 5 weeks.

The Q025 schedule has an implicit dependency gap: the exploration oracle evaluation described here requires `is_exploration=True` to be logged from Week 1 (when epsilon-greedy injection first activates). If this field is not logged until Month 3, the first 12 weeks of exploration data are lost and the evaluation clock resets. **The data collection prerequisite (logging is_exploration) has Week 1 urgency even though the IPS model itself is a Month 3 deliverable.**
