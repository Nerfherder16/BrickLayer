# Finding: Q014 — Taboo Architecture: Retrieval System from First Principles

**Question**: Q014 — Ignoring all existing tools in Recall's stack, design the retrieval system from physical first principles. Starting from: "I need to find the k most relevant text chunks from a set of N, given query Q, in under 50ms." What does the system look like?
**Agent**: quantitative-analyst
**Question Type**: TABOO
**Verdict**: BREAKTHROUGH
**Severity**: High

---

## First Principles Derivation

### 1. Physical Constraint: What Is "Relevant"?

Start with information theory. A message is informative to a receiver insofar as it reduces the receiver's uncertainty about something they care about. Shannon's mutual information I(X;Y) = H(X) - H(X|Y) captures this: the amount that knowing Y reduces uncertainty about X.

Applying this to retrieval: query Q is a compressed expression of an information need. Text chunk C is relevant to Q if knowing C reduces uncertainty about the meaning expressed by Q — that is, if I(Q; C) is large.

Now, what physical signal carries this mutual information?

Human language is constrained by the manifold of human thought. Not all combinations of words are equally probable — the distribution over word sequences is highly structured. Large neural language models trained on text have learned an approximation of P(word sequence), and a byproduct of this learning is that the internal representations (real-valued vectors in R^D) encode semantic relationships in their geometry. This is an empirical fact about how language models organize meaning, not an assumption.

The key physical signal is therefore: **two text sequences that share high mutual information will have nearby representations in the learned embedding space**. "Nearby" here is a geometric claim — distance in R^D is a proxy for the information-theoretic overlap between the represented meanings.

This is the physical foundation. Relevance = geometric proximity in a representation space where the geometry was shaped by the statistics of language.

However, this immediately reveals two assumptions embedded in the current framing:
1. The geometric proxy is a sufficient statistic for mutual information. (It is not — it compresses meaning, losing structure.)
2. All directions in the embedding space carry equal weight for a given query. (They do not — some dimensions encode topic, others encode sentiment, register, temporal context, etc.)

Both assumptions are unjustified from first principles. We'll return to them in section 6.

### 2. The Correct Data Structure (derived, not named)

**Physical requirement**: I have N points in a D-dimensional real-valued space (N up to 100,000; D = 1024). I need to find the k points closest to a query point, where "closest" is measured by a similarity function over R^D, in under 50ms total wall time.

**Derived from the physics of the space**:

First ask: what does the geometry of this space look like? With D=1024 dimensions and N=100,000 points, the space is extraordinarily sparse. The volume of a D-dimensional ball grows as r^D/Γ(D/2+1) — for D=1024, a ball of radius r_1 contains vanishingly little volume compared to a ball of radius r_1 + ε. This means points cluster strongly: the nearest neighbor of any point is typically much closer than most other points. The distribution of inter-point distances concentrates near a characteristic value, with a thin tail toward zero (the nearest neighbors).

This sparsity means: **most of the N points can be eliminated from consideration for any given query based on a coarse geometric test, without inspecting their full D-dimensional representation.**

What data structure has this property? Hierarchically nested regions of space, where each region contains a subset of points, and regions are organized so that a coarse check on a region's bounding volume allows pruning the entire subtree.

Concretely: partition the space into regions using representative anchor points. Each anchor point is the centroid of some cluster of nearby data points. To answer a query:
1. Find the nearest anchor(s) to the query — O(number_of_anchors) comparisons in full D dimensions, but number_of_anchors << N.
2. Search only the data points assigned to those anchor(s).

This is a recursive partitioning: anchors can themselves be clustered into super-anchors, etc. The structure that naturally emerges is a tree where:
- Each internal node holds a representative point and a bounding radius.
- Each leaf holds a small list of data points known to be geometrically close to each other.
- Navigation from root to relevant leaf costs O(log N) full-dimensional distance computations.
- Final exact search within the leaf costs O(leaf_size) full-dimensional distance computations.
- Total: O(log N + leaf_size) vs O(N) for exhaustive scan.

For N=100,000 and leaf_size~100, this is approximately 17 + 100 = 117 distance computations, vs 100,000. A 1,000x reduction.

**Why does this satisfy the 50ms constraint?**

Each distance computation in R^1024 requires 1024 multiplications and additions. On modern hardware (say 8 GFLOP/s single-threaded), 1024 floating-point ops takes ~128 nanoseconds. 100,000 such computations = 12.8ms just for arithmetic — before memory access. With D=1024 and float32, each vector is 4KB. 100,000 vectors = 400MB of memory reads. At memory bandwidth of ~40 GB/s, that's 10ms memory-bound time. Total exhaustive scan: ~23ms arithmetic + ~10ms memory = ~33ms just for distance computations, leaving 17ms for everything else. This is close to the 50ms budget but has no slack.

With hierarchical partitioning: ~117 distance computations = ~15 microseconds arithmetic, plus ~470KB memory reads for vectors = ~12 microseconds. Total distance computation: ~30 microseconds, leaving ~49.97ms for everything else. The budget is trivially satisfied.

The derived data structure has these structural properties:
- A multi-level tree where internal nodes are representative "anchor" points
- Each leaf contains a small, bounded set of data points
- Anchors are chosen to minimize the expected radius of the cluster they represent (minimizing false negatives during pruning)
- The tree is not perfectly balanced — it is proximity-balanced, meaning clusters reflect the actual geometry of the data, not equal partition sizes

### 3. The Correct Similarity Measure (derived from axioms)

Start from what we need the similarity function to express: "text A and text B have the same meaning as each other."

What axioms should this function satisfy?

**Axiom 1 — Symmetry**: sim(A, B) = sim(B, A). Meaning-overlap is symmetric. If A means what B means, B means what A means.

**Axiom 2 — Self-identity is maximal**: sim(A, A) ≥ sim(A, B) for all B ≠ A. A text is maximally similar to itself.

**Axiom 3 — Scale invariance**: The measure should not depend on the magnitude of the representation — only its direction. This is the critical axiom. Why? Because a text's meaning does not change if we scale its representation by a constant. The embedding of "the cat sat" and "the cat sat the cat sat" (repeated twice) might have different magnitudes but similar directions, and their meaning should score similarly similar to a query about cats.

Scale invariance means sim(λa, b) = sim(a, b) for any scalar λ > 0. The only functions on R^D satisfying this are functions of the angle between vectors, or equivalently, functions of the inner product of the normalized vectors.

**Axiom 4 — Smoothness**: sim should be a continuous function of its inputs. Small perturbations in representation space should produce small perturbations in similarity score.

The function that emerges from Axioms 1, 3, and 4 is the inner product of unit-normalized vectors: sim(a, b) = (a/|a|) · (b/|b|). This equals the cosine of the angle between the vectors, and ranges from -1 (opposite directions) to +1 (identical directions).

Note what I derived this from: axioms of meaning-equivalence, not from any named technique. The "normalize and take inner product" operation is the unique function satisfying these four axioms on R^D.

**Why scale invariance is the key axiom**: It immediately tells us that pre-normalizing all stored vectors at write time (not query time) eliminates a class of runtime computation. The similarity between stored vector v_i and query vector q becomes simply the inner product q_normalized · v_i_normalized — a linear operation, which is the cheapest possible operation on a GPU or SIMD CPU.

**The derived measure also has this property**: It is a linear map in q for fixed database. That means: sim(q, v_i) = q^T · V[:,i] where V is a matrix whose columns are the stored normalized vectors. The full set of similarities is a single matrix-vector product q^T V — computable via BLAS DGEMV in microseconds on modern hardware.

### 4. The Correct Index Structure (derived from geometry)

We established in section 2 that hierarchical proximity partitioning reduces search cost from O(N) to O(log N + leaf_size). Now derive the specific shape of the partitioning from the geometry.

**The geometry of the problem**: Our points live on the surface of a D-dimensional unit sphere (because we normalized them). A sphere in D dimensions has surface area proportional to r^(D-1). For D=1024, the surface is a 1023-dimensional manifold.

What is the best way to partition this manifold?

Consider a hyperplane through the origin. It divides the sphere's surface into two equal hemispheres. Any query point falls on one side, allowing us to prune half the database with one inner product computation (checking which side of the hyperplane the query lands on).

Recursively applying hyperplanes through the origin creates a binary tree. After log2(N) splits, each leaf contains O(1) points. The number of inner products needed to navigate: log2(N) = log2(100,000) ≈ 17. The number to search the leaf: ~1. Total: ~18 inner products.

But the optimal hyperplane at each level is not random — it should be the hyperplane that maximally separates the variance in the current subset. This is the direction of maximum spread: the first principal component of the local point cloud. This is derived from spectral graph theory: the Fiedler vector of the similarity graph among local points.

**Why multiple trees improve recall**: A single tree has "border effects" — points near the dividing hyperplane are as likely to be on either side. If the nearest neighbor is near a border, it may be in the other subtree, and a single-tree search would miss it.

Probabilistically: if the query's k-nearest neighbors are drawn from a distribution with some spread σ, the probability that all k are in the correct subtree of a single hyperplane is roughly (1 - 2σ/π). For k=10 and σ=0.1, this is (0.936)^10 ≈ 0.52 — we'd miss half the true top-k on average.

The solution: build multiple independent trees with different random hyperplane orientations. The probability of a point being missed by ALL M trees is (1 - p)^M. For p=0.5 and M=10 trees, this is (0.5)^10 ≈ 0.001. With M=10 trees, recall approaches 99.9%.

**The derived index structure**:
- M independent binary trees (M=5-20 depending on recall target)
- Each tree built by recursively splitting on random projections of the local principal component
- Navigation: O(M · log N) comparisons at query time
- Candidate set: union of M leaf sets, deduplicated
- Final ranking: exact similarity on the candidate set (~M · leaf_size points)

For M=10, N=100,000, leaf_size=100: 10 × 17 navigation + 1,000 exact computations = 1,170 inner products. At 128ns each: ~150 microseconds. Well within 50ms.

### 5. Combining Geometric + Temporal Signals

**The core problem**: Geometric proximity in embedding space captures meaning-similarity. Temporal proximity captures recency. These are measured in incompatible units (dimensionless angle vs. elapsed time in seconds). We cannot simply add them without a principled unit conversion.

**Derivation from decision theory**: What we actually want is not "most similar" but "most useful given the current context." Usefulness is a function of both relevance and recency. This is a multi-attribute decision problem, and the correct framework is expected utility.

Define utility U(chunk C | query Q, current time T) as the expected benefit of surfacing C when answering Q at time T.

U has two multiplicative components:
- Relevance factor R(C, Q) ∈ [0,1]: geometric similarity, derived in section 3
- Temporal decay factor D(C, T) ∈ [0,1]: a function of age

**Deriving the temporal decay**: What is the correct shape of D?

Empirical evidence from human memory (Ebbinghaus forgetting curve, 1885; replicated extensively in cognitive science): the probability that a human recalls a memory decreases as a power-law in time, not exponentially. The power-law form: D(age) = (1 + age/τ)^(-α) where τ is a characteristic timescale and α is the decay exponent.

The power-law is preferred over exponential decay for two physical reasons:
1. **Heavy-tailed**: Old memories that were accessed recently get a strong recency bonus (the power-law decays slower than exponential at long times).
2. **Scale-invariant**: The ratio D(2t)/D(t) depends on α only, not on the absolute time t. This means the same decay law applies whether the timescale is seconds or years — a desirable property for a system that stores memories with heterogeneous ages.

The combined score:

```
score(C, Q, T) = sim(C, Q)^β × D(age(C), T)^γ
```

The multiplicative (not additive) form is derived from expected utility theory for independent factors: if relevance and recency are independently beneficial, their contributions multiply. An additive form would allow a perfectly recent but completely irrelevant chunk to score high — which violates the intuition that both conditions are necessary.

**Setting the parameters**: β and γ control the relative weighting of semantic vs. temporal proximity. These should not be hardcoded — they should be learned from user feedback (which surfaced chunks led to useful completions?). This is a two-parameter regression problem with binary outcomes (useful/not useful), solvable with logistic regression over retrieved results.

**Unit reconciliation**: The multiplicative form sidesteps unit incompatibility entirely. sim ∈ [-1, 1] and D ∈ [0, 1] are both dimensionless ratios, so their product is dimensionless. No unit conversion is needed.

**Implementation note**: To avoid recomputing this for all N chunks at query time (O(N) operations), compute geometric candidates first using the index structure, then apply the temporal decay only to the candidate set (~1,000 points). This is justified because: if geometric similarity is near zero, the temporal factor cannot rescue the chunk regardless of recency. So geometric filtering first, temporal re-ranking second, is both efficient and approximately optimal.

### 6. What the Current Approach Gets Wrong

The current approach encodes text → dense vector → store → retrieve by geometric proximity. Three unjustified assumptions:

**Assumption 1: The embedding space is metrically uniform.**

The derived similarity measure (inner product of normalized vectors) treats all directions in R^1024 equally. But empirical evidence from embedding space geometry shows this is false: language model embeddings exhibit anisotropy — certain directions in the space are used far more than others (the "representation degeneration" problem, observed by Gao et al. 2019, Ethayarajh 2019). This means the effective dimensionality of the space is much lower than 1024, and the similarity measure overweights "popular" dimensions.

**Fix**: Whiten the embedding space before storing. Whitening (subtracting the mean and multiplying by the inverse square root of the covariance matrix) transforms the distribution to have identity covariance, making all directions equally informative. This is a one-time transformation computed over the stored corpus. After whitening, the inner product is a better proxy for semantic similarity.

**Assumption 2: The query and the chunk live in the same semantic space.**

The embedding model was trained to represent text as text. But a query is a question ("what happened with Tim's Proxmox config?") while a stored chunk is a statement ("Tim's Proxmox host was updated to VE 8.2 on March 1"). These are different speech acts, and embedding models encode speech act type in the vector direction. The geometric distance between a question and an answer is systematically larger than the distance between two statements with the same content.

**Fix**: Use an asymmetric similarity function: embed queries using a query-adapted transformation (a linear projection Q_proj applied to the query vector) and embed chunks using a chunk-adapted transformation (C_proj applied to stored vectors). Train Q_proj and C_proj jointly so that question-answer pairs score high. This is the bi-encoder pattern, derived from first principles as: we are computing a bilinear form q^T M c, where M = Q_proj^T C_proj is a learned matrix.

**Assumption 3: Geometric proximity is the only signal worth computing.**

The retrieval problem has structure that the embedding discards: temporal ordering, session co-occurrence, user-explicit associations. These are graph-theoretic signals, not geometric ones. Two chunks that always co-occur in useful sessions are related in a way that pure embedding geometry cannot capture — their relationship is expressed in the access pattern graph, not the content embedding space.

**Fix**: Augment geometric proximity with a graph proximity score. Build a co-occurrence graph over chunks (edges weighted by how often chunk A and chunk B were both surfaced in the same useful session). At query time, after geometric retrieval, propagate relevance scores through the graph using a few steps of personalized PageRank. This adds graph proximity to geometric proximity — the two signals are complementary and sum in a principled way because both are dimensionless scores in [0,1].

This is the most fundamental unjustified assumption: **that retrieval is a point-in-space problem rather than a trajectory-through-a-graph problem**. Human memory is associative (graph-structured), not just geometric. The current approach misses the associative structure entirely.

## Ideas Extracted (using real Recall stack components)

### Idea: whitened-embedding-store

- **Novelty**: 0.7 — Whitening of embedding spaces is well-known in NLP research (Mu & Viswanath 2018, "All-but-the-Top") but essentially no production AI memory system applies it to the stored vector corpus. Qdrant and similar tools simply store raw embeddings.
- **Evidence**: 0.8 — The anisotropy problem in language model embeddings is peer-reviewed and extensively replicated. Whitening demonstrably improves nearest-neighbor recall in the whitened space.
- **Feasibility**: 0.7 — Requires: (a) computing the mean and covariance matrix of all stored vectors (a one-time batch operation over the Qdrant vector store), (b) applying the whitening transformation to all stored vectors, (c) applying the same transformation to query vectors at runtime. The covariance matrix is 1024×1024 floats = 4MB. The whitening projection is a matrix multiply. The Recall embedding pipeline (Ollama qwen3-embedding → Qdrant) can be augmented with a whitening step in the FastAPI layer. Implementation: 1-2 weeks.
- **Description**: After collecting the initial corpus of N embeddings from qwen3-embedding, compute the empirical mean μ and covariance Σ over all stored vectors. Store the whitening matrix W = Σ^(-1/2). At write time, transform each new embedding v → W(v - μ) before storing. At query time, transform the query embedding similarly. This makes the effective similarity measure geometry-aware rather than geometry-blind. The whitening matrix needs periodic recomputation (e.g., nightly) as the corpus grows.

### Idea: bilinear-query-chunk-projection

- **Novelty**: 0.8 — Bi-encoder asymmetric similarity is used in some production search systems (DPR for open-domain QA), but no AI memory system (Recall, mem0, Zep) applies it to the hook-based memory retrieval use case where queries are natural language questions and stored items are observed facts/events.
- **Evidence**: 0.7 — Bi-encoder models (Karpukhin et al. 2020, "Dense Passage Retrieval") show measurable improvement in question-answer retrieval over symmetric similarity. The improvement on asymmetric tasks (question vs. statement) is well-established.
- **Feasibility**: 0.4 — Requires training a pair of linear projection matrices (Q_proj ∈ R^1024×512, C_proj ∈ R^1024×512) on Recall's actual query-retrieval feedback data. Need at minimum ~1,000 labeled (query, relevant chunk, irrelevant chunk) triples. At current Recall scale, this would take weeks of usage to accumulate. Could bootstrap with synthetic labels from Claude. Implementation: 2-4 weeks for the training pipeline + 1 week for integration.
- **Description**: Rather than storing the raw 1024-d embedding, store a projected 512-d representation using a learned chunk projection matrix. At query time, project the query with a separate learned query projection matrix. The similarity is the inner product of the two projected vectors. The projection matrices are trained to maximize similarity for (query, retrieved-and-used-chunk) pairs vs. (query, retrieved-but-ignored-chunk) pairs — a signal Recall already has if it logs which memories were surfaced but not referenced in the final response.

### Idea: power-law-temporal-reranker

- **Novelty**: 0.6 — Temporal decay in retrieval is known (BM25F includes recency signals, some production RAG systems use recency weighting), but the specific choice of power-law vs. exponential decay motivated by the Ebbinghaus forgetting curve is not standard in any AI memory system.
- **Evidence**: 0.85 — The Ebbinghaus forgetting curve as a power-law is one of the most replicated findings in cognitive psychology (Hermann Ebbinghaus 1885; replicated by Murre & Dros 2015 among many others). The power-law form (1 + t/τ)^(-α) is a better empirical fit than exponential to human forgetting.
- **Feasibility**: 0.9 — Extremely simple to implement in the existing Recall FastAPI layer. After geometric retrieval from Qdrant returns k candidates with timestamps, apply: score = geometric_sim^β × (1 + age_hours/τ)^(-α × γ). Parameters τ=168 (1 week), α=0.5 are reasonable defaults from Ebbinghaus literature. β and γ can start at 1.0 and be tuned. Requires adding a `created_at` timestamp to every stored memory payload (Qdrant payload field) — Recall may already store this. Implementation: 2-3 days.
- **Description**: Replace the current recency weighting (if any) with a multiplicative power-law decay applied to the geometric similarity scores of the Qdrant-retrieved candidates. The combined score is sim^β × (1 + age_hours/168)^(-0.5). Store the memory creation timestamp as a Qdrant payload field. At query time, retrieve the top 50 geometric candidates from Qdrant, then re-rank by the combined score, returning the top k=10. The power-law form naturally gives a "long tail" — old memories accessed recently (via association) get a recency bump without fully decaying.

### Idea: corpus-whitening-as-effective-dimensionality-probe

- **Novelty**: 0.75 — Using the eigenspectrum of the corpus covariance matrix as a diagnostic tool to understand what fraction of the 1024 dimensions are actually being used — and therefore what compression is safe — is not a documented practice in production AI memory systems.
- **Evidence**: 0.8 — The intrinsic dimensionality of language model embeddings has been studied (Aghajanyan et al. 2021 "Intrinsic Dimensionality Explains the Effectiveness of Language Model Fine-Tuning"), finding that effective dimensionality is often 10-100x lower than nominal dimensionality.
- **Feasibility**: 0.85 — One-time analysis: store all Recall embeddings as a numpy matrix, compute SVD, plot the eigenvalue spectrum. This tells us: if the top 64 eigenvalues explain 95% of variance, we can compress from 1024 to 64 dimensions with <5% information loss. This would reduce Qdrant storage by 16x and reduce similarity computation time by 16x. Implementation: 1 day for the diagnostic, 1 week for the compression pipeline if results justify it.
- **Description**: Extract all vectors currently stored in Qdrant, compute their empirical covariance matrix (1024×1024), and run eigendecomposition. Plot the eigenvalue spectrum (sorted descending). Identify the "elbow" — the point where additional eigenvectors explain negligible variance. If the elbow is at dimension d << 1024, project all stored vectors to R^d using the top d eigenvectors. This is a lossless compression in the information-theoretic sense (preserving 95%+ of variance). At d=64, similarity computation is 16x faster, enabling either faster retrieval or larger N within the same latency budget.

### Idea: co-occurrence-graph-reranker

- **Novelty**: 0.85 — Adding a session co-occurrence graph over memories as a second retrieval signal, combined multiplicatively with geometric similarity, is not implemented in any known AI memory system. The closest analogue is knowledge graph traversal in RAG pipelines, but that requires explicit entity linking.
- **Evidence**: 0.6 — Personalized PageRank for recommendation is well-established (Jeh & Widom 2003). Its application to AI memory retrieval is theoretical but grounded in the established effectiveness of co-occurrence signals in information retrieval (TF-IDF co-occurrence, word2vec skip-gram are both co-occurrence-based).
- **Feasibility**: 0.4 — Requires: (a) logging which memory IDs were retrieved together in each Recall session, (b) building a weighted co-occurrence graph in Neo4j (already in Recall's stack), (c) at query time, after Qdrant geometric retrieval, run 2-step personalized PageRank from the retrieved nodes to expand the candidate set with associatively-related memories. The PageRank step adds ~5-10ms per query. The graph population is passive (accumulates from session logs). Meaningful signal requires ~100+ sessions with co-retrieved memories. Implementation: 3-4 weeks.
- **Description**: Extend Recall's Neo4j graph to include a `CO_RETRIEVED_WITH` edge type between memory nodes, weighted by the number of sessions in which both were retrieved together. At query time: (1) retrieve top-20 geometric candidates from Qdrant, (2) use these as seed nodes for a 2-step weighted neighborhood expansion in Neo4j, (3) score expanded neighbors by combining geometric similarity (if available) with graph proximity (sum of incoming edge weights from seed nodes, normalized). Return the top-k from the union of seed and expanded sets. This captures the associative structure of Recall's own retrieval history, making the system learn from its own usage patterns.
