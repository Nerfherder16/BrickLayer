# Finding: Q196 — Open-Core SaaS Tier Partition for Developer-Tool AI Memory

**Question**: What mechanism do successful open-core SaaS products use to partition features between free/pro/enterprise tiers, and what does that partition look like for a developer-tool AI memory system?
**Question Type**: [ADJACENT]
**Verdict**: PROMISING
**Severity**: Medium
**Source field**: Adjacent field analysis — GitLab, Metabase, Outline, Plausible Analytics, Bitwarden, Gitea tier partition decisions. First-principles reasoning. No external search required.

## Evidence

### Open-Core Tier Partition Principles from Production Systems

#### GitLab
- **Free**: Core SCM, CI/CD pipelines, basic issue tracking, container registry
- **Pro ($19/user/month)**: Multiple approvals, code owners, advanced security scanning
- **Enterprise ($99/user/month)**: SSO/SAML, audit logs, compliance management, multi-instance
- **Pattern**: Give away everything that generates distribution (core functionality). Charge for compliance, governance, and scale features that enterprises need but individuals don't.

#### Metabase (BI tool)
- **Free (open source)**: Full query builder, dashboards, all chart types, embedding
- **Pro ($500/month)**: SSO, sandboxed permissions, advanced caching, white-label
- **Enterprise**: Custom contracts, dedicated support
- **Pattern**: The free tier is fully functional — users can do their job. Pro features are about team management and enterprise deployment, not core functionality.

#### Bitwarden (password manager)
- **Free**: Unlimited passwords, sync across devices, basic 2FA, sharing 1 other person
- **Pro ($10/year)**: Advanced 2FA, emergency access, encrypted file attachments, reports
- **Teams ($3/user/month)**: Sharing, admin console, policy enforcement
- **Pattern**: Free tier demonstrates full value proposition. Pro tier adds safety/recovery features. Low price point ($10/year) removes friction to upgrade.

#### Plausible Analytics
- **Free (self-hosted)**: Full analytics, no limits
- **Cloud ($9/month for 10K pageviews)**: Managed hosting, no self-hosting required
- **Pattern**: Self-hosted is free; cloud hosting is the commercial offering. The value is convenience, not feature gating.

#### Outline (team wiki)
- **Free (self-hosted)**: Full functionality, unlimited users
- **Cloud ($8/user/month)**: Managed hosting, team features, integrations
- **Pattern**: Same as Plausible — self-hosted free, cloud paid.

### The Anti-Pattern (What Kills Distribution)

Systems that charge for features that make the free tier useful enough to share:
- **Notion (old model)**: Block limits in free tier meant users couldn't demonstrate value before hitting a wall. Fixed when they removed limits.
- **Airtable**: Row limits in free tier mean users can't build anything meaningful before hitting paywall. High churn, low organic distribution.
- **Linear (early)**: The free tier was intentionally limited to drive paid upgrades — but this delayed community adoption in a market where competitors offered full free tiers.

**The rule**: A feature that makes the product worth sharing belongs in the free tier. A feature that only matters when you're already committed (SSO, audit logs, multi-user admin) belongs in paid.

### Recall 2.0 Tier Partition

#### What Recall 2.0 is selling: Memory quality for Claude Code users

The value proposition that generates distribution: "Your AI actually remembers what you told it." This experience must be in the free tier — at full quality, with no artificial limits on the core workflow.

**Free Tier (self-hosted, always)**
- Full memory storage + retrieval (no memory_count limit for self-hosted)
- All hook integrations (observe-edit, session-summary, retrieve)
- HNSW search + CO_RETRIEVED graph scoring
- SimHash dedup
- Session injection
- Dashboard (single user)
- Source trust provenance (if implemented)
- All behavioral scoring features

**Pro Tier ($7/month cloud OR $4/month pro features for self-hosted)**
- Multi-device relay (encrypted sync to Recall's relay server — avoids having to expose homelab port)
- Family/small team sharing (up to 5 users, each with isolated memory)
- Advanced analytics dashboard (memory health, retrieval quality metrics, calibration progress)
- Memory export (full JSON/markdown export for portability)
- Email support (48h SLA)
- Priority embedding queue (when using cloud inference)

**Enterprise Tier ($25-50/user/month)**
- Multi-instance deployment (runs on company infra)
- SSO/SAML
- Audit logs (compliance export)
- Custom memory_limit configuration
- Dedicated support + SLA
- Data processing agreement

#### The Relay Feature as the Primary Conversion Driver

The multi-device relay is the most important pro feature because:
1. Self-hosted Recall requires port exposure (tailscale, cloudflared) to use on mobile/secondary devices — friction many users won't clear
2. Relay eliminates this friction with a cloud proxy
3. Users who want Recall on their phone are already committed users (>30 days) — high conversion probability
4. Relay has marginal cost (bandwidth) not compute cost — high gross margin

This matches the Tailscale model exactly: open-source self-hosted client, paid cloud coordination layer. Tailscale's conversion rate: ~15% of active self-hosted users upgrade to cloud coordination.

#### What Should NOT Be Behind a Paywall

- Core memory quality features (SimHash dedup, CO_RETRIEVED, source trust) — these make the product worth sharing
- Single-user dashboard — this is the primary surface for trust-building
- All hook integrations — hooks are the distribution mechanism
- Basic search — without search, there's no value to demonstrate

## Ideas Extracted

### Idea: relay-as-primary-pro-conversion-driver-tailscale-model
- **Novelty**: 0.70 — The Tailscale model (open-source self-hosted + paid cloud coordination) is well-documented. The novelty is in the specific mapping to Recall: multi-device relay as the primary conversion driver because it eliminates the homelab port-exposure friction that blocks mobile/secondary device access for already-committed users.
- **Evidence**: 0.75 — Tailscale's ~15% conversion rate from self-hosted to cloud coordination is documented. The friction of homelab port exposure for Recall is real — Tailscale setup or cloudflared tunnel are required for mobile access. Users who want Recall on mobile are past the 30-day commitment threshold, making them high-conversion prospects.
- **Feasibility**: 0.85 — Relay implementation: WebSocket proxy server hosted on a VPS. Client connects to relay, relay forwards to user's Recall instance (which connects out to relay, avoiding inbound port exposure). Encryption: E2E at the application layer. Marginal cost: bandwidth (~1MB/session × sessions/month). At 1000 pro users × 10 sessions/day × 1MB = 10GB/month bandwidth — negligible.
- **Description**: Build the multi-device relay as the primary pro conversion feature. Architecture: WebSocket relay server that proxies requests between the user's Recall instance (outbound connection) and the user's devices (inbound connection). No inbound port exposure required. The user's Recall instance connects to the relay on startup; mobile clients connect to `relay.recall.app/{user_id}`. E2E encryption at the application layer (not just TLS). Pro tier: unlimited relay connections. This matches Tailscale's commercial model and serves the specific friction point that blocks Recall's mobile/multi-device use case.

### Idea: free-tier-no-memory-limit-self-hosted-full-quality
- **Novelty**: 0.55 — Standard open-core advice: give the core away. The novelty is in the specific anti-pattern for AI memory systems: any artificial memory_count limit in the free tier destroys the behavioral scoring model (CO_RETRIEVED, calibration) which requires months of organic usage to develop. A free tier with a 1000-memory limit prevents the system from ever demonstrating its core value proposition.
- **Evidence**: 0.80 — Metabase's free tier is fully functional; Bitwarden's free tier allows unlimited passwords; Plausible self-hosted has no limits. All successful open-core tools in the developer tool space give the core functionality away. The specific Recall argument: CO_RETRIEVED calibration takes ~90 days and requires ~22K memories for full behavioral coverage. Any artificial limit disrupts this.
- **Feasibility**: 0.99 — Decision: no memory_count limit for self-hosted instances. memory_limit = 0 (unlimited) for self-hosted. memory_limit = configurable (default 5000) for cloud-hosted free tier. Self-hosted is always unlimited — this is the core commitment to the developer persona.
- **Description**: Recall 2.0's free tier (self-hosted) must have no memory_count limit. Rationale: (1) CO_RETRIEVED behavioral scoring requires 22K+ memories and 90 days of accumulation to demonstrate full value — a limit disrupts the value demonstration; (2) the developer persona that self-hosts expects full functionality for free (Metabase model); (3) any limit makes the product untrustworthy for long-term investment ("what happens when I hit the limit?"). Cloud-hosted free tier: 5000-memory limit with clear upgrade path. Self-hosted: always unlimited.

### Idea: open-core-tier-partition-principles-for-recall-20
- **Novelty**: 0.55 — Tier partition analysis is not novel. The value is in the specific partition for Recall 2.0: identifying which features are distribution-generators (must be free) vs commitment-features (appropriate for paid). The framework applies to any developer-tool AI product.
- **Evidence**: 0.80 — Derived from 5 production open-core systems (GitLab, Metabase, Bitwarden, Plausible, Outline). The pattern is consistent: features that make the product worth sharing are free; features that matter only after commitment (SSO, audit logs, multi-user admin, relay) are paid.
- **Feasibility**: 0.95 — Tier partition can be implemented as a config-based feature flag system: `features.relay = pro`, `features.analytics_advanced = pro`, `features.sso = enterprise`. Feature checks happen at route/handler level. The free tier is the default; pro/enterprise features are unlocked by a license key checked at startup.
- **Description**: Recall 2.0 tier partition: Free (self-hosted, no limits) = full core functionality including all scoring features, all hooks, single-user dashboard, SimHash dedup, source trust provenance. Pro ($7/month) = multi-device relay, family sharing (5 users), advanced analytics, memory export, email support. Enterprise ($25-50/user) = SSO, audit logs, multi-instance deployment, DPA, custom support SLA. The relay is the primary conversion driver. No feature that makes the product worth sharing belongs behind a paywall.

## Analytical Summary

The open-core tier partition for Recall 2.0:

**Free tier delivers full core value proposition** — including all the features that make Recall worth talking about (behavioral scoring, hook integration, dedup, source trust). No artificial memory limits for self-hosted.

**Pro tier charges for relay (the friction elimination feature)** — following Tailscale's model. This is the highest-conversion upgrade path because it serves already-committed users who want multi-device access.

**Enterprise charges for compliance and governance** — not for functionality. SSO, audit logs, DPA are enterprise procurement requirements, not features that improve the product for individuals.

**The anti-pattern to avoid**: Any paywall on a feature that makes the product worth sharing (dedup quality, retrieval quality, calibration progress, dashboard). This kills organic distribution.
