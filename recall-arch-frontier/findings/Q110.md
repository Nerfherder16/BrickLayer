## Migration Plan

The plan is structured as five phases. Each phase has a defined rollback decision point. Total calendar time: 4–5 hours on a Saturday morning. Total data migration runtime: under 30 seconds for 800 edges.

### Phase Overview

```
Phase 0: Pre-flight (30 min)   → Rollback: trivial, nothing changed
Phase 1: Schema Setup (15 min) → Rollback: DROP TABLE co_retrievals
Phase 2: Data Migration (1 hr) → Rollback: truncate co_retrievals, verify Neo4j intact
Phase 3: Dual-Write (N sessions, async) → Rollback: disable PG writes via config flag
Phase 4: A/B Gate (15 min)    → Rollback: keep reading from Neo4j
Phase 5: Read Cutover (15 min) → Rollback: flip config flag, Neo4j read path re-enabled
Phase 6: Neo4j Removal (30 min) → Rollback: restore docker-compose backup, docker compose up
```

## Phase 1 — Schema Setup

**Rollback decision point**: `DROP TABLE co_retrievals; DROP INDEX idx_co_retrievals_fresh;`

```sql
-- Connect: docker exec -it recall-postgres psql -U recall -d recall

CREATE TABLE co_retrievals (
    from_memory_id UUID NOT NULL REFERENCES memories(id),
    to_memory_id   UUID NOT NULL REFERENCES memories(id),
    relationship_type VARCHAR(8) NOT NULL
        CHECK (relationship_type IN ('FRESH', 'STALE')),
    last_reinforced_epoch INTEGER NOT NULL,
    weight FLOAT NOT NULL DEFAULT 1.0,
    co_retrieval_count INTEGER NOT NULL DEFAULT 1,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    PRIMARY KEY (from_memory_id, to_memory_id)
);

-- Partial index: structural analog to Neo4j CO_RETRIEVED_FRESH chain.
-- Only FRESH rows stored in index — STALE rows are physically absent.
-- Use CONCURRENTLY to avoid table lock (safe on empty table too).
CREATE INDEX CONCURRENTLY idx_co_retrievals_fresh
    ON co_retrievals(from_memory_id)
    WHERE relationship_type = 'FRESH';

-- Supporting index for reverse-direction traversal
CREATE INDEX CONCURRENTLY idx_co_retrievals_to_fresh
    ON co_retrievals(to_memory_id)
    WHERE relationship_type = 'FRESH';

-- Verify
SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'co_retrievals';
-- Expected: 2 rows (PRIMARY KEY + idx_co_retrievals_fresh + idx_co_retrievals_to_fresh)
```

## Migration Script

Full Python script `migrate_neo4j_to_pg.py`. Place in the recall project root on CasaOS.

```python
#!/usr/bin/env python3
"""
migrate_neo4j_to_pg.py — One-time migration of CO_RETRIEVED edges from Neo4j to PostgreSQL.

Usage:
    python migrate_neo4j_to_pg.py --dry-run          # count check, no inserts
    python migrate_neo4j_to_pg.py                    # full migration
    python migrate_neo4j_to_pg.py --verify-only      # post-migration count check

Environment variables (or edit defaults below):
    NEO4J_URI       neo4j://localhost:7687
    NEO4J_USER      neo4j
    NEO4J_PASSWORD  <password>
    PG_DSN          postgresql://recall:<password>@localhost:5432/recall
"""

import argparse
import logging
import os
import sys
from datetime import datetime, timezone

from neo4j import GraphDatabase
import psycopg2
import psycopg2.extras

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s",
    handlers=[
        logging.StreamHandler(sys.stdout),
        logging.FileHandler(f"migration_{datetime.now(timezone.utc).strftime('%Y%m%d_%H%M%S')}.log"),
    ],
)
log = logging.getLogger(__name__)

# ---------------------------------------------------------------------------
# Configuration
# ---------------------------------------------------------------------------
NEO4J_URI = os.environ.get("NEO4J_URI", "neo4j://localhost:7687")
NEO4J_USER = os.environ.get("NEO4J_USER", "neo4j")
NEO4J_PASSWORD = os.environ.get("NEO4J_PASSWORD", "")
PG_DSN = os.environ.get("PG_DSN", "postgresql://recall:recall@localhost:5432/recall")

BATCH_SIZE = 200   # rows per PostgreSQL INSERT batch
LOG_INTERVAL = 100  # log progress every N edges


def export_from_neo4j(driver) -> list[dict]:
    """
    Export all CO_RETRIEVED_FRESH and CO_RETRIEVED_STALE edges from Neo4j.

    Returns a list of dicts with keys:
        from_content_hash, to_content_hash, rel_type,
        last_reinforced_epoch, co_retrieval_count, weight
    """
    cypher = """
    MATCH (a:Memory)-[r:CO_RETRIEVED_FRESH|CO_RETRIEVED_STALE]->(b:Memory)
    RETURN
        a.content_hash       AS from_content_hash,
        b.content_hash       AS to_content_hash,
        type(r)              AS rel_type,
        r.last_reinforced_epoch   AS last_reinforced_epoch,
        coalesce(r.co_retrieval_count, 1) AS co_retrieval_count,
        coalesce(r.weight, 1.0)           AS weight
    """
    edges = []
    with driver.session() as session:
        result = session.run(cypher)
        for record in result:
            edges.append({
                "from_content_hash": record["from_content_hash"],
                "to_content_hash": record["to_content_hash"],
                "rel_type": record["rel_type"],
                "last_reinforced_epoch": record["last_reinforced_epoch"],
                "co_retrieval_count": int(record["co_retrieval_count"]),
                "weight": float(record["weight"]),
            })
    return edges


def resolve_content_hashes(pg_conn, edges: list[dict]) -> tuple[list[dict], list[dict]]:
    """
    Resolve from_content_hash and to_content_hash to UUIDs from the memories table.

    Returns (resolved_edges, orphaned_edges).
    Orphaned edges are those where either content_hash has no matching memory row.
    """
    all_hashes = set()
    for e in edges:
        all_hashes.add(e["from_content_hash"])
        all_hashes.add(e["to_content_hash"])

    # Bulk lookup — single query for all hashes
    with pg_conn.cursor(cursor_factory=psycopg2.extras.DictCursor) as cur:
        cur.execute(
            "SELECT id, content_hash FROM memories WHERE content_hash = ANY(%s)",
            (list(all_hashes),),
        )
        rows = cur.fetchall()

    hash_to_id = {row["content_hash"]: str(row["id"]) for row in rows}

    resolved = []
    orphaned = []

    for e in edges:
        from_id = hash_to_id.get(e["from_content_hash"])
        to_id = hash_to_id.get(e["to_content_hash"])

        if from_id is None:
            orphaned.append({**e, "reason": f"from_content_hash '{e['from_content_hash']}' not found in memories"})
            continue
        if to_id is None:
            orphaned.append({**e, "reason": f"to_content_hash '{e['to_content_hash']}' not found in memories"})
            continue

        resolved.append({
            **e,
            "from_memory_id": from_id,
            "to_memory_id": to_id,
            "relationship_type": "FRESH" if e["rel_type"] == "CO_RETRIEVED_FRESH" else "STALE",
        })

    return resolved, orphaned


def insert_into_pg(pg_conn, resolved_edges: list[dict], dry_run: bool) -> int:
    """
    Batch-INSERT resolved edges into co_retrievals.

    Uses ON CONFLICT DO NOTHING to make re-runs idempotent (safe to re-run).
    Returns count of rows inserted.
    """
    if dry_run:
        log.info("[DRY RUN] Would insert %d edges into co_retrievals", len(resolved_edges))
        return 0

    inserted_total = 0

    with pg_conn.cursor() as cur:
        for batch_start in range(0, len(resolved_edges), BATCH_SIZE):
            batch = resolved_edges[batch_start : batch_start + BATCH_SIZE]

            # Build VALUES list
            values = [
                (
                    e["from_memory_id"],
                    e["to_memory_id"],
                    e["relationship_type"],
                    e["last_reinforced_epoch"],
                    e["weight"],
                    e["co_retrieval_count"],
                )
                for e in batch
            ]

            psycopg2.extras.execute_values(
                cur,
                """
                INSERT INTO co_retrievals
                    (from_memory_id, to_memory_id, relationship_type,
                     last_reinforced_epoch, weight, co_retrieval_count)
                VALUES %s
                ON CONFLICT (from_memory_id, to_memory_id) DO NOTHING
                """,
                values,
                template="(%s, %s, %s, %s, %s, %s)",
                page_size=BATCH_SIZE,
            )

            inserted_total += cur.rowcount
            if (batch_start // BATCH_SIZE + 1) % (LOG_INTERVAL // BATCH_SIZE + 1) == 0:
                log.info("  Inserted batch: %d / %d edges processed",
                         batch_start + len(batch), len(resolved_edges))

        pg_conn.commit()

    return inserted_total


def verify_counts(neo4j_edge_count: int, pg_conn) -> bool:
    """
    Verify that the PostgreSQL co_retrievals row count matches the Neo4j export count.

    Returns True if counts match.
    """
    with pg_conn.cursor() as cur:
        cur.execute("SELECT COUNT(*) FROM co_retrievals")
        pg_count = cur.fetchone()[0]

    log.info("Verification:")
    log.info("  Neo4j exported edges : %d", neo4j_edge_count)
    log.info("  PostgreSQL rows       : %d", pg_count)
    log.info("  Match                 : %s", "YES" if neo4j_edge_count == pg_count else "NO — MISMATCH")

    if neo4j_edge_count != pg_count:
        log.warning("Count mismatch. Check migration.log for orphaned edges.")
        return False
    return True


def check_orphaned_fk(pg_conn):
    """
    Confirm no co_retrievals row references a non-existent memory id.
    Should always be 0 given the FK constraint, but verify explicitly.
    """
    with pg_conn.cursor() as cur:
        cur.execute("""
            SELECT COUNT(*) FROM co_retrievals cr
            LEFT JOIN memories m ON cr.from_memory_id = m.id
            WHERE m.id IS NULL
        """)
        bad_from = cur.fetchone()[0]

        cur.execute("""
            SELECT COUNT(*) FROM co_retrievals cr
            LEFT JOIN memories m ON cr.to_memory_id = m.id
            WHERE m.id IS NULL
        """)
        bad_to = cur.fetchone()[0]

    if bad_from == 0 and bad_to == 0:
        log.info("FK integrity check: PASS (0 orphaned from_memory_id, 0 orphaned to_memory_id)")
    else:
        log.error("FK integrity check: FAIL (bad_from=%d, bad_to=%d)", bad_from, bad_to)
    return bad_from == 0 and bad_to == 0


def main():
    parser = argparse.ArgumentParser(description="Migrate Neo4j CO_RETRIEVED edges to PostgreSQL co_retrievals")
    parser.add_argument("--dry-run", action="store_true",
                        help="Export from Neo4j and count but do not insert into PostgreSQL")
    parser.add_argument("--verify-only", action="store_true",
                        help="Skip migration, only run count verification (post-migration check)")
    args = parser.parse_args()

    log.info("=== Neo4j → PostgreSQL CO_RETRIEVED migration ===")
    log.info("Mode: %s", "DRY RUN" if args.dry_run else ("VERIFY ONLY" if args.verify_only else "LIVE"))

    # --- Connect to Neo4j ---
    log.info("Connecting to Neo4j at %s ...", NEO4J_URI)
    driver = GraphDatabase.driver(NEO4J_URI, auth=(NEO4J_USER, NEO4J_PASSWORD))
    try:
        driver.verify_connectivity()
        log.info("Neo4j connection: OK")
    except Exception as exc:
        log.error("Neo4j connection failed: %s", exc)
        sys.exit(1)

    # --- Connect to PostgreSQL ---
    log.info("Connecting to PostgreSQL ...")
    try:
        pg_conn = psycopg2.connect(PG_DSN)
        log.info("PostgreSQL connection: OK")
    except Exception as exc:
        log.error("PostgreSQL connection failed: %s", exc)
        driver.close()
        sys.exit(1)

    # --- Verify-only mode ---
    if args.verify_only:
        log.info("Running verify-only check ...")
        edges = export_from_neo4j(driver)
        neo4j_count = len(edges)
        verify_counts(neo4j_count, pg_conn)
        check_orphaned_fk(pg_conn)
        driver.close()
        pg_conn.close()
        return

    # --- Export from Neo4j ---
    log.info("Exporting CO_RETRIEVED_FRESH and CO_RETRIEVED_STALE edges from Neo4j ...")
    edges = export_from_neo4j(driver)
    neo4j_edge_count = len(edges)
    log.info("Exported %d edges from Neo4j", neo4j_edge_count)

    if neo4j_edge_count == 0:
        log.warning("No edges exported from Neo4j. Verify Neo4j has CO_RETRIEVED_FRESH/STALE relationships.")

    # --- Resolve content_hashes to UUIDs ---
    log.info("Resolving content_hashes to PostgreSQL memory UUIDs ...")
    resolved, orphaned = resolve_content_hashes(pg_conn, edges)
    log.info("Resolved: %d edges", len(resolved))

    if orphaned:
        log.warning("Orphaned edges (content_hash not found in memories): %d", len(orphaned))
        for o in orphaned:
            log.warning("  ORPHAN: from=%s to=%s reason=%s",
                        o["from_content_hash"], o["to_content_hash"], o["reason"])
        log.warning("Orphaned edges will be SKIPPED. This is expected if memories were deleted.")

    # --- Insert into PostgreSQL ---
    log.info("Inserting %d resolved edges into co_retrievals (batch_size=%d) ...",
             len(resolved), BATCH_SIZE)
    inserted = insert_into_pg(pg_conn, resolved, dry_run=args.dry_run)

    if not args.dry_run:
        log.info("Inserted %d rows into co_retrievals", inserted)

        # --- Verify counts ---
        count_ok = verify_counts(neo4j_edge_count, pg_conn)
        fk_ok = check_orphaned_fk(pg_conn)

        if count_ok and fk_ok:
            log.info("=== MIGRATION COMPLETE — all checks passed ===")
        else:
            # Note orphans reduce the expected count — adjust expectation
            expected = len(resolved)
            log.warning("=== MIGRATION COMPLETED WITH WARNINGS ===")
            log.warning("Expected %d inserts (neo4j_total=%d minus %d orphans)",
                        expected, neo4j_edge_count, len(orphaned))
            log.warning("Review migration_*.log for details before proceeding.")

    driver.close()
    pg_conn.close()


if __name__ == "__main__":
    main()
```

## Phase 3 — Dual-Write Bridge Code

During the dual-write period the application writes to **both** Neo4j and PostgreSQL on every session-end reinforcement event. This validates that the PostgreSQL write path is correct before switching the read path.

### Configuration flag

Add to `config.py` (or environment variable):

```python
# config.py
import os

class Config:
    # Dual-write: write to both Neo4j and PostgreSQL
    # Set to True during the transition period.
    # Set to False after Neo4j is removed.
    DUAL_WRITE_ENABLED: bool = os.getenv("RECALL_DUAL_WRITE", "false").lower() == "true"

    # Read path: "neo4j" (default) | "postgres" (after cutover)
    CO_RETRIEVAL_READ_SOURCE: str = os.getenv("RECALL_READ_SOURCE", "neo4j")
```

### Session-end reinforcement hook — dual-write version

This replaces the current session-end Neo4j-only write. Both the `neo4j_path` and `pg_path` must succeed for the reinforcement to be considered complete. If either fails, log the error but do not block the session from ending.

```python
# recall_api/hooks/session_end.py  (or equivalent reinforcement location)

import logging
from typing import Optional
from neo4j import AsyncSession as Neo4jSession
from asyncpg import Connection as PgConnection
from config import Config

log = logging.getLogger(__name__)


async def reinforce_co_retrieval(
    a_content_hash: str,
    b_content_hash: str,
    current_epoch: int,
    neo4j_session: Optional[Neo4jSession],
    pg_conn: Optional[PgConnection],
) -> None:
    """
    Write a co-retrieval edge to Neo4j (primary) and PostgreSQL (dual-write).

    During dual-write period: both writes execute.
    After cutover: only PostgreSQL write executes (neo4j_session is None or DUAL_WRITE_ENABLED=False).
    """

    # --- Neo4j write (primary, existing path) ---
    if neo4j_session is not None and Config.DUAL_WRITE_ENABLED:
        try:
            await neo4j_session.run(
                """
                MATCH (a:Memory {content_hash: $a_hash})
                MATCH (b:Memory {content_hash: $b_hash})
                MERGE (a)-[r:CO_RETRIEVED_FRESH]->(b)
                ON CREATE SET
                    r.last_reinforced_epoch = $epoch,
                    r.co_retrieval_count = 1,
                    r.weight = 1.0
                ON MATCH SET
                    r.last_reinforced_epoch = $epoch,
                    r.co_retrieval_count = r.co_retrieval_count + 1
                """,
                a_hash=a_content_hash,
                b_hash=b_content_hash,
                epoch=current_epoch,
            )
        except Exception as exc:
            log.error("Neo4j reinforcement failed: %s (non-blocking during dual-write)", exc)

    # --- PostgreSQL write (new path) ---
    if pg_conn is not None:
        try:
            # Resolve content_hashes to UUIDs (cached in practice via Redis or in-memory dict)
            row = await pg_conn.fetchrow(
                """
                WITH a AS (SELECT id FROM memories WHERE content_hash = $1),
                     b AS (SELECT id FROM memories WHERE content_hash = $2)
                SELECT a.id AS a_id, b.id AS b_id FROM a, b
                """,
                a_content_hash,
                b_content_hash,
            )

            if row is None or row["a_id"] is None or row["b_id"] is None:
                log.warning(
                    "PG dual-write skip: content_hash not found in memories "
                    "(a=%s, b=%s)", a_content_hash, b_content_hash
                )
                return

            await pg_conn.execute(
                """
                INSERT INTO co_retrievals
                    (from_memory_id, to_memory_id, relationship_type,
                     last_reinforced_epoch, weight, co_retrieval_count)
                VALUES ($1, $2, 'FRESH', $3, 1.0, 1)
                ON CONFLICT (from_memory_id, to_memory_id) DO UPDATE
                    SET relationship_type      = 'FRESH',
                        last_reinforced_epoch  = EXCLUDED.last_reinforced_epoch,
                        weight                 = co_retrievals.weight + 0.1,
                        co_retrieval_count     = co_retrievals.co_retrieval_count + 1,
                        updated_at             = now()
                """,
                row["a_id"],
                row["b_id"],
                current_epoch,
            )
        except Exception as exc:
            log.error("PostgreSQL dual-write reinforcement failed: %s (non-blocking)", exc)
```

### Background reclassification job — dual-write version

The reclassification job (Q096, Q104) promotes FRESH→STALE for aged edges. During dual-write it must run on both stores.

```python
# recall_api/jobs/reclassify.py

async def reclassify_stale_edges(
    current_epoch: int,
    tau_edge: int,
    neo4j_session,
    pg_conn,
) -> dict:
    """
    Promote FRESH→STALE for edges whose last_reinforced_epoch < current_epoch - tau_edge.
    Runs on both Neo4j and PostgreSQL during dual-write period.
    """
    results = {}

    # --- Neo4j reclassification (existing Q096 pattern) ---
    if neo4j_session is not None and Config.DUAL_WRITE_ENABLED:
        threshold = current_epoch - tau_edge
        result = await neo4j_session.run(
            """
            MATCH (a:Memory)-[r:CO_RETRIEVED_FRESH]->(b:Memory)
            WHERE r.last_reinforced_epoch < $threshold
            WITH a, b, r,
                 r.last_reinforced_epoch AS epoch,
                 r.co_retrieval_count    AS count,
                 coalesce(r.weight, 1.0) AS w
            CREATE (a)-[new_r:CO_RETRIEVED_STALE]->(b)
            SET new_r.last_reinforced_epoch = epoch,
                new_r.co_retrieval_count = count,
                new_r.weight = w
            DELETE r
            RETURN count(*) AS reclassified
            """,
            threshold=threshold,
        )
        record = await result.single()
        results["neo4j_reclassified"] = record["reclassified"] if record else 0

    # --- PostgreSQL reclassification (single atomic UPDATE) ---
    if pg_conn is not None:
        pg_result = await pg_conn.execute(
            """
            UPDATE co_retrievals
            SET    relationship_type = 'STALE',
                   updated_at        = now()
            WHERE  relationship_type = 'FRESH'
              AND  last_reinforced_epoch < $1
            """,
            current_epoch - tau_edge,
        )
        # asyncpg returns "UPDATE N" as string
        results["pg_reclassified"] = int(pg_result.split()[-1]) if pg_result else 0

    return results
```

### How many sessions before switching the read path?

**Recommended N: 10 production sessions** (approximately 1–2 days of normal Tim usage). The criterion is not just N sessions but passing the A/B validation gate below.

Rationale: 10 sessions generates enough new edge reinforcements (Q091: ~45 edges/session) to confirm that the UPSERT path is working correctly on both new and reinforce-existing code paths. At 10 sessions the PostgreSQL co_retrievals table will have accumulated ~450 edge-write events and the counts will have diverged detectably from the initial migration snapshot if the dual-write is broken.

**Minimum acceptance for switching read path**:
- Zero PostgreSQL dual-write errors in logs for the last 5 sessions
- co_retrievals row count increases monotonically session-over-session
- A/B validation gate passes (see Phase 4)

## Phase 4 — A/B Validation Gate

Before switching reads from Neo4j to PostgreSQL, run both BFS queries against the same seed set and compare top-10 results.

### Validation script

```python
#!/usr/bin/env python3
"""
ab_validate_co_retrievals.py — Compare Neo4j BFS vs PostgreSQL CTE on same seeds.

Usage:
    python ab_validate_co_retrievals.py --seeds 5
    # Picks 5 random seed hashes from memories, runs both queries, compares top-10.

Accept threshold: Jaccard overlap of top-10 content_hashes >= 0.80.
"""

import argparse
import logging
import os
import random
import sys

from neo4j import GraphDatabase
import psycopg2
import psycopg2.extras

log = logging.getLogger(__name__)
logging.basicConfig(level=logging.INFO, format="%(levelname)s %(message)s")

NEO4J_URI = os.environ.get("NEO4J_URI", "neo4j://localhost:7687")
NEO4J_USER = os.environ.get("NEO4J_USER", "neo4j")
NEO4J_PASSWORD = os.environ.get("NEO4J_PASSWORD", "")
PG_DSN = os.environ.get("PG_DSN", "postgresql://recall:recall@localhost:5432/recall")


def get_random_seed_hashes(pg_conn, n: int) -> list[str]:
    """Pick N content_hashes that have at least 1 FRESH co-retrieval edge."""
    with pg_conn.cursor() as cur:
        cur.execute("""
            SELECT DISTINCT m.content_hash
            FROM co_retrievals cr
            JOIN memories m ON cr.from_memory_id = m.id
            WHERE cr.relationship_type = 'FRESH'
            ORDER BY random()
            LIMIT %s
        """, (n,))
        return [row[0] for row in cur.fetchall()]


def query_neo4j_bfs(driver, seed_hashes: list[str]) -> list[str]:
    """
    Run depth-2 BFS on Neo4j CO_RETRIEVED_FRESH, return top-10 content_hashes by co_retrieval_count.
    """
    cypher = """
    UNWIND $seeds AS seed_hash
    MATCH (s:Memory {content_hash: seed_hash})-[:CO_RETRIEVED_FRESH]->(d1:Memory)
    OPTIONAL MATCH (d1)-[:CO_RETRIEVED_FRESH]->(d2:Memory)
    WHERE NOT d2.content_hash IN $seeds
    WITH collect(DISTINCT d1.content_hash) + collect(DISTINCT d2.content_hash) AS neighbors
    UNWIND neighbors AS n
    RETURN n AS content_hash, count(*) AS hit_count
    ORDER BY hit_count DESC
    LIMIT 10
    """
    with driver.session() as session:
        result = session.run(cypher, seeds=seed_hashes)
        return [record["content_hash"] for record in result if record["content_hash"] is not None]


def query_pg_bfs(pg_conn, seed_hashes: list[str]) -> list[str]:
    """
    Run depth-2 CTE BFS on PostgreSQL co_retrievals FRESH, return top-10 content_hashes.
    Uses the Q108 CTE pattern with LIMIT guards.
    """
    sql = """
    WITH seeds AS (
        SELECT id, content_hash FROM memories WHERE content_hash = ANY(%s)
    ),
    depth1 AS (
        SELECT cr.to_memory_id AS memory_id, cr.co_retrieval_count, 1 AS depth
        FROM co_retrievals cr
        JOIN seeds s ON cr.from_memory_id = s.id
        WHERE cr.relationship_type = 'FRESH'
        LIMIT 50
    ),
    depth2 AS (
        SELECT cr.to_memory_id AS memory_id, cr.co_retrieval_count, 2 AS depth
        FROM co_retrievals cr
        JOIN depth1 d1 ON cr.from_memory_id = d1.memory_id
        WHERE cr.relationship_type = 'FRESH'
          AND cr.to_memory_id NOT IN (SELECT id FROM seeds)
        LIMIT 50
    ),
    combined AS (
        SELECT memory_id, SUM(co_retrieval_count) AS total_count
        FROM (SELECT * FROM depth1 UNION ALL SELECT * FROM depth2) u
        GROUP BY memory_id
    )
    SELECT m.content_hash, c.total_count
    FROM combined c
    JOIN memories m ON c.memory_id = m.id
    ORDER BY c.total_count DESC
    LIMIT 10
    """
    with pg_conn.cursor(cursor_factory=psycopg2.extras.DictCursor) as cur:
        cur.execute(sql, (seed_hashes,))
        return [row["content_hash"] for row in cur.fetchall() if row["content_hash"] is not None]


def compute_overlap(neo4j_results: list[str], pg_results: list[str]) -> float:
    """Jaccard overlap of two ranked lists (treating as sets)."""
    if not neo4j_results and not pg_results:
        return 1.0
    if not neo4j_results or not pg_results:
        return 0.0
    s1 = set(neo4j_results)
    s2 = set(pg_results)
    return len(s1 & s2) / len(s1 | s2)


def main():
    parser = argparse.ArgumentParser(description="A/B validate Neo4j vs PostgreSQL co_retrieval BFS")
    parser.add_argument("--seeds", type=int, default=5, help="Number of seed hashes to test")
    parser.add_argument("--threshold", type=float, default=0.80, help="Minimum Jaccard overlap to accept")
    args = parser.parse_args()

    driver = GraphDatabase.driver(NEO4J_URI, auth=(NEO4J_USER, NEO4J_PASSWORD))
    pg_conn = psycopg2.connect(PG_DSN)

    seed_hashes = get_random_seed_hashes(pg_conn, args.seeds)
    if not seed_hashes:
        log.error("No seeds found with FRESH co-retrieval edges. Run migration first.")
        sys.exit(1)

    log.info("Testing with %d seeds: %s", len(seed_hashes), seed_hashes[:3])

    neo4j_results = query_neo4j_bfs(driver, seed_hashes)
    pg_results = query_pg_bfs(pg_conn, seed_hashes)

    overlap = compute_overlap(neo4j_results, pg_results)

    log.info("Neo4j top-10 : %s", neo4j_results)
    log.info("PG top-10    : %s", pg_results)
    log.info("Jaccard overlap: %.2f (threshold: %.2f)", overlap, args.threshold)

    if overlap >= args.threshold:
        log.info("RESULT: PASS — overlap %.2f >= threshold %.2f. Safe to switch read path.", overlap, args.threshold)
        sys.exit(0)
    else:
        log.error("RESULT: FAIL — overlap %.2f < threshold %.2f. Do NOT switch read path.", overlap, args.threshold)
        log.error("Investigate: dual-write log errors, orphaned edges, reclassification divergence.")
        sys.exit(1)


if __name__ == "__main__":
    main()
```

### A/B acceptance criteria

| Metric | Accept | Reject |
|--------|--------|--------|
| Jaccard overlap (top-10 sets) | ≥ 0.80 | < 0.80 |
| Neo4j result count | ≥ 5 (graph not empty) | < 5 (degenerate) |
| PG result count | ≥ 5 | < 5 |
| Query latency ratio PG/Neo4j | ≤ 2.0 | > 5.0 (unexpected regression) |

**Why 80% and not 100%**: Ordering divergence is expected. PostgreSQL ranks by `co_retrieval_count` aggregated across both depths; Neo4j Cypher ranks by `count(*)` which counts path hits, not raw co_retrieval_count. The semantic intent is the same but integer accumulation differs by path multiplicity. 80% Jaccard (8 of 10 matching) indicates structural agreement; 100% would falsely reject correct migrations due to tie-breaking differences.

```python
# recall_api/retrieval.py

async def get_co_retrieved_memories(
    seed_hashes: list[str],
    pg_conn,
    neo4j_session,
    depth: int = 2,
    limit: int = 50,
) -> list[dict]:
    """
    BFS retrieval of co-retrieved memories. Source controlled by RECALL_READ_SOURCE config.
    """
    if Config.CO_RETRIEVAL_READ_SOURCE == "postgres":
        return await _pg_bfs(seed_hashes, pg_conn, depth=depth, limit=limit)
    else:
        return await _neo4j_bfs(seed_hashes, neo4j_session, depth=depth, limit=limit)


async def _pg_bfs(seed_hashes: list[str], pg_conn, depth: int, limit: int) -> list[dict]:
    """Depth-2 CTE BFS on co_retrievals. Q108 pattern."""
    sql = """
    WITH seeds AS (
        SELECT id FROM memories WHERE content_hash = ANY($1)
    ),
    depth1 AS (
        SELECT cr.to_memory_id AS memory_id,
               cr.weight,
               cr.co_retrieval_count,
               1 AS depth
        FROM co_retrievals cr
        JOIN seeds s ON cr.from_memory_id = s.id
        WHERE cr.relationship_type = 'FRESH'
        LIMIT 50
    ),
    depth2 AS (
        SELECT cr.to_memory_id AS memory_id,
               cr.weight * d1.weight AS weight,
               cr.co_retrieval_count,
               2 AS depth
        FROM co_retrievals cr
        JOIN depth1 d1 ON cr.from_memory_id = d1.memory_id
        WHERE cr.relationship_type = 'FRESH'
          AND cr.to_memory_id NOT IN (SELECT id FROM seeds)
        LIMIT 50
    )
    SELECT m.content_hash, m.content, MAX(c.weight) AS weight, MIN(c.depth) AS depth
    FROM (SELECT * FROM depth1 UNION ALL SELECT * FROM depth2) c
    JOIN memories m ON c.memory_id = m.id
    GROUP BY m.content_hash, m.content
    ORDER BY weight DESC
    LIMIT $2
    """
    rows = await pg_conn.fetch(sql, seed_hashes, limit)
    return [dict(row) for row in rows]
```

### Step 6b — Redefine S6 health signal

Before removing Neo4j from docker-compose.yml, the S6 health gauge (Q102) must be redefined in terms of PostgreSQL. The existing Cypher `S6-GAUGE` query (degree distribution) becomes a PostgreSQL degree scan:

```sql
-- S6-GAUGE equivalent: co-retrieval edge degree distribution
-- Health: P95 fresh degree <= 50 (Q094 circuit breaker threshold)
SELECT
    percentile_cont(0.50) WITHIN GROUP (ORDER BY fresh_degree) AS p50_fresh_degree,
    percentile_cont(0.95) WITHIN GROUP (ORDER BY fresh_degree) AS p95_fresh_degree,
    percentile_cont(0.99) WITHIN GROUP (ORDER BY fresh_degree) AS p99_fresh_degree,
    COUNT(*) AS node_count,
    SUM(fresh_degree) AS total_fresh_edges
FROM (
    SELECT from_memory_id, COUNT(*) AS fresh_degree
    FROM co_retrievals
    WHERE relationship_type = 'FRESH'
    GROUP BY from_memory_id
) degree_dist;

-- S6-TOPHUBS equivalent: top-10 hub nodes by fresh degree
SELECT m.content_hash, COUNT(*) AS fresh_degree
FROM co_retrievals cr
JOIN memories m ON cr.from_memory_id = m.id
WHERE cr.relationship_type = 'FRESH'
GROUP BY m.content_hash
ORDER BY fresh_degree DESC
LIMIT 10;

-- S6-HISTOGRAM equivalent: degree histogram (0-5, 6-20, 21-50, 51+)
SELECT
    CASE
        WHEN fresh_degree BETWEEN 0 AND 5  THEN '0-5'
        WHEN fresh_degree BETWEEN 6 AND 20 THEN '6-20'
        WHEN fresh_degree BETWEEN 21 AND 50 THEN '21-50'
        ELSE '51+'
    END AS degree_bucket,
    COUNT(*) AS node_count
FROM (
    SELECT from_memory_id, COUNT(*) AS fresh_degree
    FROM co_retrievals
    WHERE relationship_type = 'FRESH'
    GROUP BY from_memory_id
) degree_dist
GROUP BY degree_bucket
ORDER BY degree_bucket;
```

## Ideas Extracted

### Idea: config-flag-dual-write-pattern
- **Novelty**: 0.55
- **Evidence**: 0.90
- **Feasibility**: 0.95
- **Description**: Database migration dual-write pattern controlled by a single environment variable (`RECALL_DUAL_WRITE`) with a separate read-source flag (`RECALL_READ_SOURCE`). Writes go to both old and new store during the transition period; reads switch independently of writes. The rollback action for any post-cutover failure is a single env var change + container restart — no data restoration required. This is a well-established pattern (Stripe, GitHub "dark write" pattern) applied concretely to the Recall reinforcement hook.

### Idea: jaccard-gate-schema-migration
- **Novelty**: 0.70
- **Evidence**: 0.72
- **Feasibility**: 0.88
- **Description**: Use Jaccard overlap of top-10 BFS results as a migration acceptance gate before switching the read path. Threshold of 0.80 allows for legitimate ranking divergence (tie-breaking differences, path multiplicity counting differences) while detecting structural failures (missing edges, orphaned nodes, reclassification divergence). This is a semantic correctness check — not just a row count check — that catches cases where counts match but the BFS traversal returns a different neighbor set. Applicable to any migration between two graph stores or graph-equivalent relational tables.

### Idea: pg-degree-scan-s6-replacement
- **Novelty**: 0.65
- **Evidence**: 0.80
- **Feasibility**: 0.92
- **Description**: The Neo4j S6 health signal (degree distribution, hub detection via Cypher) is replaced by a PostgreSQL window function degree scan on the co_retrievals table. The PostgreSQL version produces identical metrics (p50/p95/p99 fresh degree, top-10 hubs, degree histogram) with no graph database overhead. Running at session-end, the degree scan takes < 5ms at 5,000 edges (table scan with GROUP BY, no index needed for aggregate). The S6 numeric thresholds from Q102 remain unchanged — only the query source changes.

### Idea: partial-index-orphan-detection
- **Novelty**: 0.68
- **Evidence**: 0.75
- **Feasibility**: 0.95
- **Description**: During migration, edges whose `content_hash` has no matching row in the `memories` table are "orphaned" — they reference memories that were deleted or never imported. The migration script resolves all content_hashes in a single bulk query (`WHERE content_hash = ANY($1)`), identifies orphans by set difference, logs them with their reason, and skips them rather than failing. The count mismatch between Neo4j export and PostgreSQL inserts (neo4j_edge_count != pg_inserted_count) is then explained precisely by the orphan log. This separation of "orphan logging" from "migration failure" prevents a brittle all-or-nothing migration that would fail if even one deleted memory left a dangling edge.

### Idea: migration-idempotent-on-conflict-do-nothing
- **Novelty**: 0.60
- **Evidence**: 0.88
- **Feasibility**: 0.97
- **Description**: Using `ON CONFLICT (from_memory_id, to_memory_id) DO NOTHING` in the migration INSERT makes the migration script idempotent — re-running it after a partial failure is safe. Any edges already inserted are skipped; only missing edges are inserted. This eliminates the need for a pre-migration TRUNCATE and allows the migration to be interrupted and resumed at any point without data corruption. The inserted count returned by `cur.rowcount` after `DO NOTHING` reflects only newly inserted rows, providing an accurate "inserted this run" count distinct from "total rows in table."
