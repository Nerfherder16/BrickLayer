## Part 1: Mathematical Derivation

### 1.1 Eigendecomposition of the Embedding Covariance

Let X be the matrix of N stored embeddings, each of dimension d = 384 (qwen3-embedding:0.6b).
Let μ = (1/N) Σᵢ xᵢ be the mean embedding (a d-dimensional vector).
Let Σ = (1/N) Σᵢ (xᵢ - μ)(xᵢ - μ)^T be the d×d covariance matrix.

Σ has eigendecomposition: Σ = V Λ V^T

Where:
- V = [v₁ | v₂ | ... | v_d] is the d×d matrix of eigenvectors (columns = principal components), ordered by descending eigenvalue
- Λ = diag(λ₁, λ₂, ..., λ_d) with λ₁ ≥ λ₂ ≥ ... ≥ λ_d ≥ 0

The eigenvalue λᵢ is the variance of the data projected onto the i-th principal component vᵢ.

### 1.2 PCA Rotation (Full Rank)

The full PCA-rotated embedding is:

```
z_pca = V^T (x - μ)         [d-dimensional, same information content]
```

This is an orthogonal rotation. Key properties:
- Pairwise Euclidean distances are PRESERVED exactly: ||z_pca_a - z_pca_b|| = ||x_a - x_b||
- Cosine similarities are NOT preserved (because norms change after rotation when vectors are not zero-mean — but after centering by μ, rotation preserves inner products, so cosine similarities of the centered embeddings ARE preserved)
- The d components of z_pca are UNCORRELATED (because Var(z_pca) = V^T Σ V = Λ, which is diagonal)
- But they do NOT have unit variance — component i has variance λᵢ

### 1.3 Truncated PCA to k Dimensions

Define V_k = first k columns of V (the k eigenvectors with largest eigenvalues).

```
z_pca_k = V_k^T (x - μ)    [k-dimensional, k << d]
```

For Recall with k = 50, d = 384:

```
z_pca_50 ∈ R^50
```

The retained variance fraction is: (Σᵢ₌₁ᵏ λᵢ) / (Σᵢ₌₁ᵈ λᵢ)

If the intrinsic dimensionality of qwen3-embedding:0.6b embeddings is ~25–50 (per Aghajanyan et al. 2021, consistent with Q013 findings), this truncation retains ~95% of the total explained variance.

CRITICAL PROPERTY: The k components of z_pca_k are uncorrelated but STILL have unequal variances (component i has variance λᵢ). The high-eigenvalue components dominate cosine similarity in the compressed space, exactly as they did in the full 384-dim space. Truncated PCA does NOT correct anisotropy — it just moves the problem to a smaller space.

### 1.4 Full Whitening Transform

The whitening transform normalizes each component to unit variance:

```
z_white = Λ^(-1/2) V^T (x - μ)
```

Where Λ^(-1/2) = diag(λ₁^(-1/2), λ₂^(-1/2), ..., λ_d^(-1/2)).

This is equivalent to Σ^(-1/2) (x - μ) because:

```
Σ^(-1/2) = V Λ^(-1/2) V^T
```

So: Σ^(-1/2)(x - μ) = V Λ^(-1/2) V^T (x - μ) = ... wait, let me re-expand:

The whitening matrix W = Λ^(-1/2) V^T transforms x as: W(x - μ).

Verify: Var(z_white) = W Σ W^T = Λ^(-1/2) V^T (V Λ V^T) V Λ^(-1/2) = Λ^(-1/2) Λ Λ^(-1/2) = I

After whitening, the covariance is the identity matrix. All components have variance 1. The embedding space is isotropic.

KEY RESULT: Cosine similarity on whitened embeddings equals Mahalanobis distance on the original embeddings:

```
cos(z_white_a, z_white_b) = <W(a-μ), W(b-μ)> / (||W(a-μ)|| ||W(b-μ)||)
                           = Mahalanobis_inner_product(a, b; Σ) / (||a-μ||_Σ ||b-μ||_Σ)
```

This is the core insight from arXiv:2103.15316.

### 1.5 The Composition: What Is "PCA-50 After Whitening"?

Two candidate pipelines differ critically:

**Pipeline A: Whiten-first, then truncate**

```
Step 1: Compute full whitening → z_white = Λ^(-1/2) V^T (x - μ)    [384-dim, isotropic]
Step 2: Truncate to first 50 dims → z_A = [z_white]_{1:50}           [50-dim]
```

The retained dimensions are the 50 whitened principal components. Since whitening has already normalized variance to 1 per component, ALL components have equal variance after step 1. Truncating to 50 therefore throws away 334 components each carrying equal (unit) variance. This does NOT preferentially retain the high-variance components. In fact, since whitening inverts the eigenvalue scaling, the truncated components after whitening correspond to the 50 LARGEST eigenvalues of the original space — but they are now normalized to the same scale. The information distribution is uniform across the 50 retained dimensions.

Pipeline A retains 50 out of 384 components, each normalized. The truncation IS harmful because after whitening, all components have the same variance so there is no low-information tail to discard safely.

**Pipeline B: Truncate PCA first, then normalize (Whitened-PCA)**

```
Step 1: Truncate to top-50 PCA components → z_pca = V_{50}^T (x - μ)    [50-dim, anisotropic]
Step 2: Normalize each component to unit variance → z_B = Λ_{50}^(-1/2) z_pca  [50-dim, isotropic]
```

Pipeline B explicitly:
1. Discards the 334 lowest-variance dimensions (which carry ~5% of explained variance for Recall's embeddings, per Q013 intrinsic dimensionality analysis)
2. Then normalizes the 50 retained dimensions to unit variance

Pipeline B is the CORRECT operation. It is the standard "whitened PCA" or "ZCA in the PCA basis truncated to k dimensions." The result has:
- 50 dimensions (the 334 discarded components carry the ~5% noise tail)
- Isotropic covariance (each component has unit variance)
- Cosine similarity in this 50-dim space approximates Mahalanobis distance in the original 384-dim space, computed with the rank-50 approximation of Σ

**Why Pipeline A is wrong**: Whitening to unit variance first destroys the variance ordering that tells you which components to keep. After whitening, all 384 components look equivalent. Truncating to 50 is then arbitrary — you're discarding components that now carry equal information, not low-information tails.

**Why Pipeline B is correct**: Truncate first (discard noise) then normalize (correct anisotropy within the signal subspace). PCA compression and whitening are separable: PCA handles dimensionality, whitening handles isotropy within whatever dimensionality you've chosen.

### 1.6 Fused Single-Matrix Implementation

The fused transform T that implements Pipeline B as a single matrix multiplication:

```
T = Λ_{50}^(-1/2) V_{50}^T     [50 × 384 matrix]
```

Where:
- V_{50} = [v₁ | v₂ | ... | v_{50}] is the 384×50 matrix of the top-50 eigenvectors
- Λ_{50} = diag(λ₁, λ₂, ..., λ₅₀) is the 50×50 diagonal eigenvalue matrix

Application:
```
z = T (x - μ)        [50-dim result]
```

This is a single matrix-vector multiply. Cost: 50 × 384 = 19,200 multiply-add operations.

Compare to the un-fused two-step:
- Step 1: 50 × 384 = 19,200 ops (V_{50}^T times (x-μ))
- Step 2: 50 × 1 = 50 ops (diagonal Λ^(-1/2) scaling)
- Total: ~19,250 ops — negligibly more

The fused form saves the intermediate 50-dim allocation. Both forms have the same arithmetic complexity. The fused form is the correct implementation to ship.

Storage of T: 50 × 384 float32 = 76,800 floats = 307 KB. Trivial.

### 1.7 Does Whitening Make PCA Redundant, or Vice Versa?

**Does full whitening (384-dim) make PCA compression unnecessary?**

No. Full whitening produces a 384-dim isotropic vector. It does not compress storage. Qdrant still stores 384 float32 values (1,536 bytes) per embedding. ANN search complexity is unchanged. The query latency gain from PCA (smaller vectors = faster HNSW traversal) is absent. Full whitening alone solves anisotropy but not storage or speed.

**Does truncated PCA (50-dim) make whitening unnecessary?**

No. Truncated PCA to 50 dimensions without whitening produces a 50-dim ANISOTROPIC vector. Component 1 has variance λ₁ (e.g., 15.2) and component 50 has variance λ₅₀ (e.g., 0.3). Cosine similarity in this space is still dominated by the high-variance components. The anisotropy problem persists at reduced scale. The retrieval precision improvement from whitening is lost.

**Conclusion**: The two transforms address different problems and are NOT redundant. The correct implementation is the fused transform T = Λ_{50}^(-1/2) V_{50}^T applied as a single step, which combines both benefits.

## Part 2: Impact on Q013's 0.92 Deduplication Threshold

### 2.1 How the Threshold Was Calibrated

Q013's write-time deduplication threshold (cosine ≥ 0.92) was calibrated in the ORIGINAL unwhitened 384-dim embedding space. It represents the cosine similarity above which two memories are considered paraphrases of the same fact.

### 2.2 What Happens to Cosine Similarity After the Fused Transform

Let a and b be two original embeddings. After applying T = Λ_{50}^(-1/2) V_{50}^T (and centering by μ):

```
a' = T(a - μ),  b' = T(b - μ)
```

The cosine similarity in the transformed space is:

```
cos(a', b') = <a', b'> / (||a'|| ||b'||)
```

This is NOT equal to the original cosine similarity cos(a, b). It is a rank-50 Mahalanobis-like inner product normalized by the Mahalanobis norm. The transformation changes BOTH the numerator and the denominator.

**Effect on paraphrase pairs (originally at cosine ~ 0.93–0.98):**

Paraphrases cluster tightly in the original space because they share almost all semantic content. In the original anisotropic space, their high cosine similarity is partly due to both embeddings projecting strongly onto the same high-variance dimensions. After whitening, those high-variance dimensions are scaled DOWN to unit variance, reducing their contribution to the inner product.

The net effect on paraphrase cosine similarity in the whitened-PCA space:
- The relative ordering of pairs is preserved (whitening is a monotone transform for true paraphrases)
- The absolute cosine similarity values DECREASE for most pairs because the inflated contribution of high-variance dimensions is corrected
- Typical decrease: whitening-paper experiments report paraphrase cosine similarity decreasing from ~0.92–0.96 range in original space to ~0.85–0.93 range in whitened space

**Effect on non-paraphrase pairs (originally at cosine ~ 0.70–0.88):**

Non-paraphrase pairs that appear similar due to shared high-variance dimensions (the anisotropy problem) see a LARGER decrease in cosine similarity after whitening. This is the intended benefit: whitening separates paraphrases from non-paraphrases more cleanly.

### 2.3 The Threshold Recalibration Problem

The 0.92 threshold was chosen in the original space to separate:
- Paraphrase pairs (keep, merge at write time): cosine ≥ 0.92
- Distinct-but-related pairs (keep both): cosine < 0.92

After applying the fused whitened-PCA transform, the distribution of cosine similarities shifts:
- Paraphrase pairs: shift to ~0.85–0.93 (lower on average)
- Distinct-but-related pairs: shift to ~0.60–0.82 (lower by more on average)

The gap between the two distributions typically INCREASES after whitening (this is the point), but the absolute position of the threshold that separates them SHIFTS DOWNWARD.

**Estimated recalibrated threshold: 0.82–0.88**

The exact recalibration requires empirical measurement on Recall's actual embedding corpus. The procedure is:

1. Export a sample of N = 500 memory pairs manually labeled as "paraphrase" vs. "distinct"
2. Apply the fused transform T to all embeddings
3. Measure cosine similarity distribution in the transformed space for each label
4. Find the threshold that maximizes F1 (precision/recall tradeoff) for the paraphrase classification task
5. Set this as the new write-time deduplication threshold

Without this calibration step, applying the whitened-PCA transform while keeping threshold = 0.92 will MISS paraphrases (because their cosine similarity has shifted below 0.92 in the transformed space). This is a correctness bug, not just a precision issue.

### 2.4 The Threshold Cannot Be Analytically Derived — It Must Be Measured

There is no closed-form expression for how a specific cosine threshold transforms under the whitened-PCA map. The shift depends on:
- The specific eigenvalue spectrum of Recall's embedding corpus
- The distribution of paraphrase pair embedding differences in the original space
- The fraction of the paraphrase signal that falls in the retained top-50 dimensions

The best available estimate is 0.82–0.88, but this must be confirmed empirically before deploying the fused transform.

**Implementation note**: The deduplication check occurs in the FastAPI layer before the Qdrant write. If the fused transform is applied at write time (both to stored embeddings and to the pre-write similarity check query), the threshold must already be recalibrated. The calibration step is therefore a HARD PREREQUISITE for correct deployment.

## Part 3: Implementation Recommendation

### 3.1 Correct Build Sequence

**Phase 0 (Prerequisite): Measure the embedding corpus**
- Export all current embeddings from Qdrant (or a 2,000-embedding sample)
- Compute μ (mean) and Σ (covariance) via numpy
- Compute eigendecomposition: eigenvalues, eigenvectors
- Plot the scree curve (explained variance vs. k) to confirm the ~95% knee at k = 40–60
- This phase produces: μ, V_{50}, Λ_{50}, T = Λ_{50}^(-1/2) V_{50}^T
- Estimated effort: 1 day

**Phase 1: Calibrate the deduplication threshold in the transformed space**
- Sample 200–500 memory pairs covering a range of cosine similarities (0.75–0.99)
- Apply T to both members of each pair
- Compute cosine similarity in the transformed 50-dim space
- Label pairs as paraphrase / distinct (manually or via LLM)
- Find the optimal threshold (expected: 0.82–0.88)
- Estimated effort: 1–2 days (including LLM labeling)

**Phase 2: Ship the fused transform**
- Store T (307 KB), μ (1.5 KB), and the new threshold as static configuration in the FastAPI service
- Modify the embedding pipeline: after Ollama returns a 384-dim vector, apply z = T(x - μ) before any Qdrant operation
- Modify the Qdrant collection to accept 50-dim vectors (requires creating a new collection; migrate old embeddings)
- Modify the pre-write deduplication check to use the new threshold
- Estimated effort: 3–5 days

**Phase 3: Periodic recomputation**
- Recompute T whenever the embedding corpus grows by 20%+ or monthly
- Store T versioned; re-transform old embeddings when T changes (or re-embed from PostgreSQL text store)
- Estimated effort: 1 day for the recompute job

**Total estimated effort: 2–3 weeks including calibration**

### 3.2 Fused vs. Separate Implementation

**Verdict: Implement as a single fused transform. Do not implement whitening and PCA as separate steps.**

Rationale:
1. The fused matrix T = Λ_{50}^(-1/2) V_{50}^T is a single 50×384 matrix multiply — same cost as truncated PCA alone
2. Separate implementation doubles the transform complexity, adds an intermediate 384-dim allocation, and introduces ordering bugs if applied in the wrong sequence (Pipeline A vs. B)
3. The separate implementation provides no additional flexibility — the composition is mathematically fixed
4. Maintaining two separate transforms requires two sets of periodic recompute jobs that must stay synchronized

### 3.3 Does This Replace the 384-dim Whitening from Q008?

Yes, with a caveat. The fused whitened-PCA transform replaces the pure 384-dim whitening idea (recall-embedding-whitening from Q008) with a strictly superior alternative:

- Same: corrects anisotropy (cosine similarity becomes Mahalanobis-equivalent, approximately)
- Same: 5–15% retrieval precision improvement expected
- Better: 82x storage reduction (384-dim float32 → 50-dim float32, or 50-dim int8 for further compression)
- Better: Faster ANN search in Qdrant (smaller vectors)
- Better: Lower Qdrant memory footprint
- Caveat: The Mahalanobis equivalence is only approximate (rank-50 approximation vs. full-rank). The dropped 334 dimensions carry ~5% of variance; this is acceptable given the intrinsic dimensionality finding from Q013.

The full 384-dim whitening without truncation should NOT be implemented — it provides no storage or speed benefit.

### 3.4 Does This Replace the Truncated PCA from Q013?

Yes. The truncated PCA idea (recall-embedding-pca-compression from Q013) proposed PCA-50 WITHOUT whitening. That idea should be upgraded to the fused whitened-PCA transform. The storage reduction is identical (384→50 dims). The retrieval improvement is additive (PCA would provide compression but not anisotropy correction; the fused transform provides both).

### 3.5 Qdrant Integration Note

Qdrant does not need to know about the transform. The FastAPI layer applies T before every Qdrant insert and before every Qdrant search. Qdrant receives 50-dim unit-normalized vectors and uses standard cosine similarity. From Qdrant's perspective, nothing has changed except the vector dimension.

This is the correct architecture: the transform lives in the application layer, not the vector database. It can be updated without Qdrant downtime (create new collection, migrate, swap).

- **Evidence**: 0.75 — the direction of threshold shift after whitening is analytically established; the empirical measurement procedure is standard in classification system design

Store the threshold as a named configuration parameter, not a magic number in code. This prevents the correctness bug where the 0.92 threshold silently stops working after embedding space transformation.

## Suggested Follow-ups

1. **Empirical scree plot for qwen3-embedding:0.6b on Recall's corpus**: Export 2,000 embeddings, compute PCA, plot eigenvalues. Confirm that k=50 retains ≥ 95% of explained variance. If the knee is at k=30, use k=30 (additional compression with equal quality). If the knee is at k=80, use k=80 (more dimensions needed). The k=50 figure from Q013 is theoretically motivated but should be confirmed on the actual model.

2. **Calibrate the deduplication threshold empirically**: Apply the fused transform to a sample of Recall's embeddings and measure how the cosine similarity distribution changes for known-paraphrase and known-distinct pairs. The 0.82–0.88 range is an estimate; the exact value must be measured.

3. **Measure whether the fused transform improves retrieval on Recall's actual query distribution**: Run 50 representative historical queries against both the original 384-dim index and the transformed 50-dim index. Compare top-5 recall@5 for known-relevant memories. This is the empirical validation of the 5–15% gain claimed in Q008.

4. **Q030 linkage (embedding model upgrade)**: If Recall moves to a higher-dimensional embedding model (e.g., qwen3-embedding:2b at 1024 dims), the fused transform matrix must be recomputed from scratch. The transform is model-specific and corpus-specific. This is not a weakness but a maintenance reality that should be documented as part of the model upgrade procedure.

5. **Investigate int8 quantization on top of the fused transform**: After applying T to get 50-dim float32 vectors, Qdrant's scalar quantization (int8) can reduce each vector from 200 bytes to 50 bytes (4x further). In the whitened space, all 50 components have variance 1 (by construction), which means int8 quantization with a fixed [-4σ, +4σ] = [-4, +4] range covers 99.99% of values without clipping. Whitening makes int8 quantization maximally efficient — the uniform variance means no component is wasted. The combined effect: 384 float32 (1,536 bytes) → 50 int8 (50 bytes) = 30x reduction with ~1% quality loss. This is better than the 82x stated in Q013 because that estimate compared float32→float32; int8 adds another 4x.
