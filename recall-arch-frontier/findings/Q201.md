## Evidence

### The Only Permitted Starting Point

> A query arrives as a pattern of activations. A corpus of stored experiences exists. The task is to find which stored experiences are most relevant to the query.

No other assumptions. No tooling. Just physics.

### Derivation from Physical Requirements

**Step 1: What is "relevance"?**

Relevance between two patterns is the degree to which they activate the same response. Two identical patterns are perfectly relevant. Two random patterns have no predictable relationship.

Physically: relevance is the amplitude of the cross-correlation between two waveforms. When query pattern Q and stored pattern P have high amplitude at the same dimensions, the cross-correlation is large. When they have independent amplitude distributions, the cross-correlation is small.

This is not a design choice — it is the physical definition of pattern overlap. Any retrieval system that selects "relevant" experiences must compute something equivalent to this amplitude overlap.

**Step 2: What physical structure can store and retrieve pattern overlaps?**

Consider a physical medium M that can accumulate patterns. When experience P_i is stored, M changes state. After storing all N experiences, M reflects the superposition of all N pattern contributions.

For M to enable retrieval (given query Q, find which P_i best overlaps with Q), M must satisfy one property: **multiplying M by Q must produce an output whose amplitude at dimension d is proportional to the overlap between Q and the stored experiences at dimension d.**

The only linear operation that satisfies this is matrix multiplication where M = sum of outer products of stored experiences with themselves:
```
M = sum over all i of: P_i ⊗ P_i^T   (outer product accumulation)
```

Retrieval: output = M × Q
The output amplitude at each stored experience location is proportional to the cross-correlation between Q and that experience.

This substrate has been derived from physics alone. It requires:
- No search algorithm
- No index structure
- No comparison operations
- No stored pointers

The stored medium M is the complete retrieval apparatus. The query operation is a single linear transformation.

**Step 3: What are the physical properties of this substrate?**

*Storage capacity*: For D-dimensional patterns, M is a D×D accumulation. Classical capacity: M can store approximately 0.14×D patterns before they interfere destructively. With modern extensions (exponential activation functions), capacity scales as log(D) × D²/2. For D=1024: classical capacity ~143 patterns, modern capacity ~millions.

*Retrieval mechanism*: Given query Q, compute M×Q. Apply a winner-take-all nonlinearity to extract the single highest-activated pattern. The output is the best-match experience, or the top-K experiences by activation amplitude.

*Write cost*: Storing a new experience requires accumulating its outer product into M. Cost: D² operations per write. For D=1024: ~1M operations per write (~0.5ms on N100 with SIMD).

*Read cost*: Given query Q, compute M×Q. Cost: D² operations per retrieval. For D=1024: ~1M operations (~0.5ms on N100 with SIMD). This is a single matrix-vector multiplication — cache-friendly sequential memory access.

*Scale property*: Read cost is independent of corpus size N. Whether 22K or 22M experiences are stored, retrieval takes the same time (one D×D matrix multiplication). The write cost is independent of N too.

### The Physical Surprise

Standard production retrieval systems have O(N×D) read cost (linear scan) or O(log N × D) read cost (navigable graph approximation). This substrate has **O(D²) read cost independent of N**.

For the crossover: at N < D², this substrate is slower than linear scan. At N > D², it becomes faster. For D=1024: crossover at N=1,048,576 (1M experiences). At Tim's 22K corpus, the substrate has higher read cost than linear scan.

However, the write mechanism compounds: as N grows past D², the matrix M becomes increasingly saturated. Modern exponential activation (Ramsauer 2020) extends capacity, but at the cost of requiring exact attention computation (softmax over all N stored patterns), which recovers the O(N×D) complexity.

### The Fundamental Discovery

**Every production AI memory retrieval system is an approximation of this physically-derived substrate.**

- ANN search (navigable graph approximation): approximates the outer product substrate by navigating a sparse graph of most-similar patterns — trading exact retrieval for O(D × log N) traversal
- Linear scan: is the exact substrate computation M×Q unrolled across all N stored experiences sequentially
- Locality-sensitive partitioning: partitions the outer product matrix by projecting patterns onto random planes (which are exactly the band projections in SimHash)

The reason production systems don't use the exact substrate (the accumulated outer product matrix M) is capacity: at D=1024, M is 1024×1024×4 bytes = 4MB. This is cache-resident. But storing 22K patterns in M requires 22K writes of 1M operations each — total write cost: 22 billion operations for the initial corpus. Prohibitive.

The practical lesson: the derived substrate is the theoretically correct architecture. Production systems make a trade-off: they substitute the exact substrate for an approximation that reduces write cost (storing pointers to patterns rather than accumulating outer products) at the expense of exact retrieval.

### Mapping to Recall 2.0

The Recall 2.0 Hopfield component (from Q192) is precisely this substrate:
- It accumulates pattern co-occurrence weights (approximating outer products at lower dimension)
- Retrieval is a relaxation operation (approximately M×Q)
- The behavioral CO_RETRIEVED graph is a sparse approximation of the full outer product matrix M

The Q201 derivation validates the Hopfield inclusion in Recall 2.0's design from first principles: it is the physically correct substrate for associative retrieval. The HNSW component is the practical approximation that makes retrieval tractable at N=22K scale, where the exact substrate is too expensive to write.

**The two-component architecture (Hopfield for behavioral graph + HNSW for semantic search) is the natural consequence of first-principles derivation:**
- HNSW handles the N-scale regime (22K patterns, fast approximate retrieval)
- Hopfield handles the behavioral regime (sparse co-occurrence matrix, approximate accumulation)

Neither is redundant. They approximate different aspects of the exact correlation matrix.

## Analytical Summary

**The physically-derived substrate is the accumulated correlation matrix** (sum of outer products of stored patterns). This substrate has O(D²) read cost independent of N, but prohibitive write cost at large N (22B operations for Tim's 22K corpus).

**Production retrieval systems are approximations of this substrate.** HNSW navigates a sparse graph approximation; SimHash projects onto random planes; linear scan computes the exact read while avoiding write accumulation. Each trades write tractability for read approximation.

**Recall 2.0's design is validated from first principles.** HNSW + Hopfield approximates the correlation matrix from two directions: HNSW for semantic similarity (high-N, approximate) and Hopfield for behavioral co-occurrence (sparse, accumulative). The Q201 derivation is consistent with the Q192 design but arrives at it through a different path.

**The Q201 finding does not change the build plan.** It provides theoretical confidence that the HNSW+Hopfield architecture is not an ad-hoc choice but a physical consequence of the retrieval problem's structure.
