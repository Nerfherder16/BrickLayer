## Change 1: `src/core/embeddings.py` — Add `simhash()` function

Insert immediately after `content_hash()` at line 267.

**Algorithm**: 64-bit SimHash via token unigrams + bigrams using SHA-256 per feature.
- Tokenize: lowercase, split on non-alphanumeric, remove stopwords shorter than 2 chars
- Feature set: unigrams + bigrams from token list
- Per feature: `v = int(sha256(feature).hexdigest(), 16)` → 64 bits → vote ±1 into 64 accumulators
- Final: bit[i] = 1 if accumulator[i] > 0 else 0 → pack to int

```python
# ---- Diff: src/core/embeddings.py ----
# After line 267 (end of content_hash function), insert:

def simhash(text: str) -> int:
    """
    Compute a 64-bit SimHash fingerprint for near-duplicate detection.

    Uses token unigrams + bigrams with SHA-256 bit weighting.
    Two texts with Hamming distance <= 8 on this hash are near-duplicates.
    No external dependencies — stdlib hashlib only.

    Returns:
        int: unsigned 64-bit integer fingerprint
    """
    import re

    # Normalize: lowercase, tokenize on non-alphanumeric
    normalized = text.lower()
    tokens = [t for t in re.split(r"[^a-z0-9]+", normalized) if len(t) >= 2]

    # Build feature set: unigrams + bigrams
    features: list[str] = tokens[:]
    for i in range(len(tokens) - 1):
        features.append(tokens[i] + "_" + tokens[i + 1])

    if not features:
        # Empty or all-stopword text: hash the raw normalized bytes
        raw = hashlib.sha256(normalized.encode()).digest()
        val = int.from_bytes(raw[:8], "big")
        return val & 0xFFFFFFFFFFFFFFFF

    # 64-element accumulator: each bit position votes ±1
    v = [0] * 64
    for feature in features:
        h = int(hashlib.sha256(feature.encode()).hexdigest(), 16)
        for i in range(64):
            if (h >> i) & 1:
                v[i] += 1
            else:
                v[i] -= 1

    # Pack: bit[i]=1 if v[i]>0
    fingerprint = 0
    for i in range(64):
        if v[i] > 0:
            fingerprint |= 1 << i

    return fingerprint & 0xFFFFFFFFFFFFFFFF
```

## Change 4: `src/storage/qdrant.py` — `find_by_simhash_bands()` Method

Add this method immediately after `find_by_content_hash()` (after line 478).

**Band decomposition for k=8 Hamming tolerance**:
- 64-bit fingerprint split into 8 bands of 8 bits each
- If Hamming(fp1, fp2) <= 8, at least one 8-bit band must be identical (pigeonhole: 8 errors across 8 bands → at least one band survives)
- Per band: extract 8-bit value, query `Range(gte=band_val, lte=band_val)` on `simhash` after masking
- Actually: since Qdrant stores the full 64-bit integer, we cannot mask individual bands server-side
- **Correct approach**: store 8 separate `simhash_b0`..`simhash_b7` integer payload fields (8-bit each), query with exact MatchValue on each band independently using `should` (OR) filter, then post-filter results by computing actual Hamming distance client-side

```python
# ---- Diff: src/storage/qdrant.py ----
# Insert after find_by_content_hash() (after line 478):

    @staticmethod
    def _simhash_bands(fp: int) -> list[int]:
        """Extract 8 bands of 8 bits each from a 64-bit SimHash."""
        return [(fp >> (i * 8)) & 0xFF for i in range(8)]

    @staticmethod
    def _hamming64(a: int, b: int) -> int:
        """Count differing bits between two 64-bit integers."""
        return bin(a ^ b).count("1")

    async def find_by_simhash_bands(
        self,
        fingerprint: int,
        user_id: int | None = None,
        hamming_threshold: int = 8,
    ) -> str | None:
        """
        Find a near-duplicate memory via SimHash band lookup.

        Uses 8-band decomposition: if Hamming(fp_stored, fp_query) <= 8,
        at least one 8-bit band is identical. Queries each band independently
        (OR logic), then post-filters by exact Hamming distance.

        Returns memory_id of first near-duplicate found, or None.
        """
        bands = self._simhash_bands(fingerprint)
        user_conds = self._user_conditions(user_id)

        # Build OR filter: match any band exactly
        band_conditions = [
            Filter(
                must=[
                    FieldCondition(
                        key=f"simhash_b{i}",
                        match=MatchValue(value=bands[i]),
                    ),
                ]
                + user_conds
            )
            for i in range(8)
        ]

        # Qdrant scroll with should (OR) across band conditions
        # Fetch up to 50 candidates; post-filter by Hamming distance
        candidates, _ = await self.client.scroll(
            collection_name=self.collection,
            scroll_filter=Filter(should=band_conditions),
            limit=50,
            with_payload=["simhash"],
        )

        for point in candidates:
            stored_fp = point.payload.get("simhash", 0) if point.payload else 0
            if stored_fp and self._hamming64(stored_fp, fingerprint) <= hamming_threshold:
                return str(point.id)

        return None
```

**Note on band storage**: This approach requires storing 8 additional integer fields (`simhash_b0`..`simhash_b7`) alongside the full `simhash` integer. Add them to the payload dict and index list. See the refined payload schema below.

## Latency Impact Analysis

| Stage | Without SimHash | With SimHash |
|-------|----------------|--------------|
| Exact dedup hit | 2ms (skip embed) | 2ms (unchanged) |
| Near-dedup hit (new) | 200ms (embed) | 2.5ms (simhash + band query, skip embed) |
| Novel content | 200ms (embed) | 200.5ms (+0.5ms simhash overhead) |

For the D_near >= 0.5% near-duplicate rate (Q152 break-even), SimHash saves ~197ms per near-duplicate write. At 10 writes/sec with 0.5% near-dedup rate, that is ~0.05 * 197ms = ~10ms average latency reduction, negligible. At 5% near-dedup rate (realistic for Recall's repetitive session summaries), saving is ~100ms average. The simhash overhead on novel content (0.5ms) is never material.

## Backfill Script (Existing Corpus)

Per Q154: scroll + upsert at 500/batch ≈ 30s for current corpus.

```python
# scripts/backfill_simhash.py
import asyncio
from src.core.embeddings import simhash
from src.storage.qdrant import QdrantStore

async def backfill(store: QdrantStore, batch_size: int = 500):
    offset = None
    total = 0
    while True:
        points, next_offset = await store.client.scroll(
            collection_name=store.collection,
            offset=offset,
            limit=batch_size,
            with_payload=["content", "simhash"],
            with_vectors=False,
        )
        if not points:
            break

        updates = []
        for p in points:
            if not p.payload:
                continue
            if p.payload.get("simhash", 0) != 0:
                continue  # already has fingerprint
            content = p.payload.get("content", "")
            fp = simhash(content)
            bands = [(fp >> (i * 8)) & 0xFF for i in range(8)]
            updates.append((str(p.id), fp, bands))

        if updates:
            from qdrant_client.models import PointIdsList
            for (pid, fp, bands) in updates:
                payload = {"simhash": fp}
                for i, b in enumerate(bands):
                    payload[f"simhash_b{i}"] = b
                await store.client.set_payload(
                    collection_name=store.collection,
                    payload=payload,
                    points=[pid],
                )
            total += len(updates)
            print(f"Backfilled {total} points...")

        offset = next_offset
        if offset is None:
            break

    print(f"Backfill complete: {total} points updated")
```

## Risk Flags

- **False positive risk**: Hamming k=8 on 64-bit hash → false positive rate ≈ C(64,8)/2^64 ≈ 4.4e9/1.8e19 ≈ negligible (~2.4e-10 per pair). Safe.
- **Band collision cardinality**: 8-bit bands have 256 values. At 50K memories with 256 band values, average 195 memories per band bucket — the 50-candidate scroll limit is appropriate but may miss matches at very high corpus size. At 500K memories, increase scroll limit to 200 or switch to 6-band 10-bit decomposition.
- **Qdrant integer range**: Qdrant integer payload uses signed 64-bit. SimHash values up to 2^63-1 are safe; values >= 2^63 will overflow to negative on storage. Solution: mask to signed range: `simhash_val = fp if fp < (1 << 63) else fp - (1 << 64)`. Add this to both `simhash()` return and the store call.
- **Episodic skip is correct**: Session summaries store the same session facts repeatedly with slight wording changes — exactly the near-duplicate pattern. But each is a distinct temporal snapshot and must not be deduplicated. The `_simhash_skip_types` guard is essential.
