## Absence Classification

**Classification**: TRUE ABSENCE

The word `simhash` does not appear in a single file anywhere in the Recall codebase. The payload fields specified in Q155 (`simhash`, `simhash_b0`..`simhash_b7`) are not written by `qdrant.py store()`. No scroll query filtering on those fields exists. No SimHash library is imported. The calibration script from Q160 (`simhash_calibration.py`) is described as a prerequisite but is equally absent from both the Recall source tree and the autosearch research directory — it was specced, not shipped.

**Why the gap exists**: Q155 produced a complete write-path specification — 9 named payload fields, their types, and the intent — but the research loop concluded before the implementation step. Q160 noted that calibration should precede implementation. Neither calibration nor implementation occurred. The existing dedup mechanism (cosine similarity via full vector search at threshold 0.92) is functional enough to suppress exact and near-exact rewrites during normal operation, which reduced the perceived urgency of the SimHash fast path.

## What Exists (Semantic Dedup — the current mechanism)

**Location**: `src/api/routes/memory.py`, lines 231-275 (`store_memory()` endpoint body)

The current write-path dedup works as follows:

1. After embedding the incoming content and passing write-guard injection checks, the store handler calls `qdrant.search(query_vector=embedding, limit=1)` — a full ANN vector search returning the single nearest existing memory.
2. If the top result's cosine similarity score exceeds `semantic_dedup_threshold` (0.92, configured in `src/core/config.py`), the store is rejected and the existing memory's ID is returned with `created=False`.
3. If the new memory's `importance` exceeds the existing match's importance, the existing record is promoted before the early return.
4. Episodic and `session_summary` types are explicitly exempt from dedup.

The same pattern is replicated in `src/workers/observer.py` (line 178) and `src/workers/signals.py` (lines 286-296) for the worker-initiated write paths.

**Cost of the current approach**: Every write incurs a full ANN search (Qdrant HNSW traversal). At the current corpus size this is inexpensive, but the SimHash motivation was to short-circuit this search for budget-sensitive or high-throughput scenarios using an O(1) payload filter before the O(log N) HNSW walk.

## What's Missing (SimHash Read Path)

The following code units are absent and would need to be created to complete the Q155 spec:

1. **SimHash computation on write** — a function that takes the embedding or raw content text and produces a 64-bit SimHash integer, then splits it into 8 × 8-bit band integers. Would live in `src/core/embeddings.py` or a new `src/core/simhash.py`.

2. **Payload field writes in `qdrant.py store()`** — 9 additional fields added to the `PointStruct` payload dict at lines 220-250: `simhash` (int64), `simhash_b0` through `simhash_b7` (uint8 each).

3. **`find_by_simhash_bands()` method in `QdrantStore`** — a method that:
   - Accepts the 8 band integers for a new memory
   - Issues a Qdrant `scroll()` call with a `Filter(should=[FieldCondition(key="simhash_b{i}", match=MatchValue(value=band_i)) for i in range(8)])` OR filter
   - Returns candidate memory IDs and their full SimHash values

4. **Hamming-distance post-filter** — a utility that takes the candidate IDs from step 3, fetches their `simhash` payload field, computes Hamming distance against the query SimHash, and keeps only candidates with distance <= D_near (the threshold Q160's calibration script was to determine empirically).

5. **Call site in `store_memory()`** — replacing or prepending the current `qdrant.search()` dedup check (lines 239-275) with a `find_by_simhash_bands()` call that short-circuits before the ANN search when a near-duplicate is found via Hamming distance.

6. **Backfill migration** — existing memories in Qdrant have no SimHash payload fields. A one-time migration worker would need to fetch all points, compute SimHash for each, and patch the payload.

## Key Findings

1. The word `simhash` has zero occurrences in the entire Recall repository — neither in source code, comments, configuration, nor scripts.

2. `qdrant.py store()` writes exactly 20 payload fields; none are `simhash`, `simhash_b0`, or any band field. The Q155 schema is completely unimplemented.

3. The existing dedup mechanism is a single-call cosine ANN search at threshold 0.92, implemented in three separate locations (`memory.py`, `observer.py`, `signals.py`) — no shared abstraction, no SimHash fast path.

4. The calibration script described in Q160 (`simhash_calibration.py`) is also absent — it was never created as a file in the Recall tree or the autosearch research directory. Prerequisite and implementation are both missing.

5. The absence is not a latent bug — the system works correctly for dedup. The SimHash read path is a performance optimization (O(1) band filter vs O(log N) HNSW) that was designed but never built.

6. Implementation is self-contained: no changes to existing Qdrant collection schema configuration are required because Qdrant accepts arbitrary payload fields without pre-declaration. The risk surface is the backfill migration for the existing corpus.

## Ideas Extracted

### Idea: simhash-read-path-implementation
- **Novelty**: 0.65 — The mechanism is fully specced (Q155); the gap is implementation, not design. Band LSH for near-duplicate detection is standard practice; the novelty is in applying it as a pre-filter in a self-hosted memory system.
- **Evidence**: 0.90 — Exhaustive grep across all 85 Python source files in `src/`. Zero false negatives possible: `simhash` is a distinctive string with no aliasing risk.
- **Feasibility**: 0.95 — Q155 specced the schema; Q160 specced the calibration methodology; the Qdrant payload filter API is available and well-documented. No external dependencies beyond adding a SimHash library (e.g., `datasketch` or a 40-line pure-Python implementation). The only friction is the corpus backfill migration.
- **Description**: Implement the four missing units in sequence: (a) `src/core/simhash.py` with `compute_simhash(text: str) -> tuple[int, list[int]]` returning the 64-bit hash and 8 band ints; (b) add the 9 payload fields to `qdrant.py store()`; (c) add `QdrantStore.find_by_simhash_bands()` using Qdrant's `scroll()` with a `should` OR filter; (d) prepend the band-filter call to `store_memory()` before the existing ANN dedup check. Run Q160's calibration script on the live corpus first to determine D_near before wiring the Hamming threshold into config.
