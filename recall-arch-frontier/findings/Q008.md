## Mechanism Under Investigation

A retrieval scoring function that accounts for the DISTRIBUTION of embeddings across the index — specifically: Mahalanobis distance (cosine similarity normalized by the covariance structure of the embedding space), Jensen-Shannon divergence (information-theoretic divergence between query and candidate distributions), or Wasserstein/earth-mover's distance — as opposed to raw cosine similarity between two point embeddings.

## Absence Classification

**Classification**: PARTIAL ABSENCE

**Explanation**: The mechanism exists in academic literature across multiple formulations (whitening/Mahalanobis: 2021–2025; Wasserstein: 2018–2023; manifold-aware graph metrics: 2025; hyperbolic distance: 2017–2025). The most developed is the whitening/Mahalanobis line of work, which is mathematically equivalent to normalizing cosine similarity by the covariance of the embedding space. However:

1. **Zero production vector databases ship this as a retrieval mode.** Qdrant, Weaviate, Pinecone, Milvus, Chroma, pgvector all support only cosine, euclidean, dot product (and manhattan in Qdrant). A concrete DIEM feature request to Qdrant was rejected in May 2025 on grounds of maintenance cost and lack of real-world validation.

2. **Zero production AI memory systems (MemGPT, mem0, Zep) apply distribution-aware metrics.** All delegate to cosine similarity via their underlying vector stores.

3. **The most actionable validated approach is whitening.** The 2021 whitening paper (arXiv:2103.15316) demonstrated that applying a whitening transform (centering + covariance normalization) to BERT embeddings consistently improved semantic similarity performance by 1–3 points on 7 benchmarks. Whitening is mathematically equivalent to Mahalanobis distance: computing cosine similarity on whitened vectors = computing Mahalanobis distance on original vectors with the precomputed covariance matrix.

4. **Why it doesn't exist in production**: Three barriers identified:
   - **Maintenance cost**: Every production vector DB must implement the metric across CPU/GPU/quantization combinations. A custom matrix transform does not fit neatly into existing ANN index structures.
   - **Covariance estimation requires the full corpus**: The whitening matrix must be estimated from the entire embedding index and updated as the index changes. This is a non-trivial infrastructure problem for most deployments.
   - **Perceived marginal gain vs. engineering cost**: The 1–3 point improvement on STS benchmarks is real but modest enough that most teams prefer fine-tuning the embedding model instead. The 26% OOD gain from MA-DPR is more compelling but requires a graph index structure.
   - **No one has closed the loop for a specific embedding model**: The gain from whitening is model-specific. A whitening transform for qwen3-embedding:0.6b would need to be estimated from a corpus representative of Recall's actual memory contents — this is a self-hosted system problem that no general-purpose library has solved.

## Evidence

### System: Qdrant (current)
Supports: cosine, dot product, euclidean, manhattan. A feature request (issue #4963) to add DIEM (Dimension Insensitive Euclidean Metric) was opened and rejected by the team in May 2025. The team's stated reasons: "distance metrics require implementations across multiple cpu architecture X quantization type combinations" and "no popular embedding models currently use DIEM." This is the clearest documented case of a distribution-aware metric being proposed for a production vector DB and rejected.

### Paper: MA-DPR (arXiv:2509.13562, September 2025)
Uses graph shortest-path distance on a k-nearest neighbor graph — a manifold-aware approach that is not Mahalanobis but similarly distribution-aware. Reports 26% improvement over cosine on out-of-distribution passage retrieval benchmarks (ANTIQUE, NFCorpus, SciDocs). Comparable performance to cosine on in-distribution (MS MARCO). Code reportedly available on GitHub but not integrated into any vector database. This is the most recent and strongest evidence that cosine similarity underperforms on OOD retrieval — directly relevant to Recall's episodic memory retrieval, which by definition is always "out of distribution" from the embedding model's training data.

### Paper: Whitening Sentence Representations (arXiv:2103.15316, 2021)
Proposes applying a whitening transform (zero-mean, identity covariance) to sentence embeddings before computing cosine similarity. This is mathematically equivalent to Mahalanobis distance on the original embeddings. Shows 1–3 point improvements on 7 STS benchmarks. Also achieves dimensionality reduction to ~1/3 original size with maintained performance. No production adoption found. The key insight: whitening is a precomputed linear transform applied at index build time — queries can then use standard cosine similarity against the transformed vectors.

### Paper: HyperbolicRAG (2024–2025)
Implements hyperbolic (Poincaré) distance in a graph-based RAG retrieval framework. Uses a depth-aware representation learner. Research prototype only; no shipping product found.

### Wasserstein distance (ICLR 2018, 2019)
Two ICLR papers proposed learning Wasserstein embeddings for retrieval. Fundamental barrier: Wasserstein distance requires O(n²)–O(n³) computation. The 2023 EMNLP linear-time approximation exists but still adds meaningful overhead vs. cosine. No production retrieval system found using this.

### Jensen-Shannon divergence
Used in a 2025 RAG paper for context attribution analysis (not retrieval scoring). Used for query hardness estimation. No production retrieval system found applying JSD as the primary retrieval distance function.

## Ideas Extracted

### Idea: recall-embedding-whitening
- **Novelty**: 0.85 — well-established in NLP literature but zero production AI memory systems have implemented it; the specific application to a self-hosted episodic memory store is novel
- **Evidence**: 0.75 — whitening improvements are consistently replicated across papers (2021–2025); the MA-DPR OOD finding (26% gain) provides strong motivation for memory retrieval specifically
- **Feasibility**: 0.80 — implementable with a single precomputed matrix multiply; can be done as a preprocessing step at Qdrant insert time without modifying Qdrant itself
- **Description**: Estimate the covariance matrix of Recall's existing embedding index (qwen3-embedding:0.6b vectors in Qdrant). Compute the whitening transform W = Σ^(-1/2) offline. At index build time, transform each embedding: e' = W @ e. At query time, transform the query embedding identically: q' = W @ q. Then use standard cosine similarity on the transformed vectors — this is mathematically equivalent to Mahalanobis distance on the originals. The whitening matrix must be recomputed periodically as the index grows. Estimated implementation: 2–3 days. Estimated impact: 5–15% improvement in retrieval precision for memories that are semantically adjacent but not the closest cosine neighbors (the anisotropy correction specifically helps with memories that cluster in dense regions of the embedding space — which is exactly where Recall stores frequently-accessed project memories).

### Idea: recall-ood-manifold-retrieval
- **Novelty**: 0.75 — MA-DPR (Sep 2025) is the closest prior work; applying it to episodic memory retrieval rather than passage retrieval is a novel context
- **Evidence**: 0.65 — MA-DPR results are strong (26% OOD gain) but the paper is very recent and uses a graph-based approach that requires maintaining a KNN graph over the embedding index
- **Feasibility**: 0.45 — requires maintaining a KNN graph index alongside Qdrant's HNSW index; significant infrastructure overhead; the graph must be updated as memories are added
- **Description**: Build a k-nearest neighbor graph over Recall's embedding index. Define retrieval distance as the shortest path between query embedding and candidate embeddings in this graph, rather than raw cosine similarity. This would correct for the manifold structure of the embedding space — memories that are semantically related but in different local neighborhoods of the embedding space would be reachable via graph traversal that raw cosine similarity misses. Higher implementation complexity than whitening; warranted only if whitening is insufficient.

### Idea: recall-corpus-specific-covariance-reranking
- **Novelty**: 0.80 — novel combination of standard reranking with covariance-based score adjustment specific to memory retrieval
- **Evidence**: 0.60 — theoretical motivation is strong; no direct empirical validation for this specific formulation in a memory context
- **Feasibility**: 0.85 — lightweight: applies at reranking time after cosine retrieval, not at indexing time; no Qdrant changes required
- **Description**: Retrieve the top-K candidates by cosine similarity (existing path). For each candidate, compute a density penalty: how many other embeddings are in the local neighborhood of this candidate? Dense-neighborhood candidates get a penalty (cosine similarity is likely inflated by anisotropy). Sparse-neighborhood candidates get a bonus. Adjust cosine score by this density term. This is a soft approximation of Mahalanobis distance that can be implemented as a post-retrieval reranking pass without modifying the index. Can be computed using Qdrant's existing search API by issuing a neighborhood count query for each candidate.

## Suggested Follow-ups

1. **Estimate Recall's embedding anisotropy empirically**: Pull 1,000 random embeddings from the Qdrant index and compute the spectral decomposition of their covariance matrix. If the top 10 principal components explain >80% of variance (typical for transformer embeddings), the anisotropy is severe enough to justify whitening.

2. **Prototype whitening on a static snapshot**: Export the current Qdrant embedding index, compute the whitening transform offline, re-index the transformed embeddings, and run a comparative retrieval evaluation on 50 representative queries. This is a weekend-scale experiment.

3. **Check BEIR results for qwen3-embedding:0.6b specifically**: The anisotropy severity varies significantly by model. If qwen3-embedding:0.6b is already trained with whitening-aware contrastive objectives (WhitenedCSE-style), the gain from post-hoc whitening would be smaller.

4. **Track MA-DPR code release**: The September 2025 paper claims code availability. If the KNN-graph shortest-path implementation is released, evaluate it against whitening for Recall's specific query distribution.

5. **Q018 adversarial linkage**: This finding strengthens the "attacker" position in Q018 (is cosine similarity fundamentally wrong for AI memory?). The OOD retrieval failure mode (26% gap) is directly relevant — Recall's queries are always OOD relative to the embedding model's training distribution.
