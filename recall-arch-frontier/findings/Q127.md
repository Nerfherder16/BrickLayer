## Sentinel-Token Hardening Specification

### Seeds Requiring Immediate Hardening (7 seeds)

For each HIGH RISK seed, prepend an 8-character unique token slug + structured sentinel to the canary content. The paraphrase query text remains unchanged. The embedding distance created by the sentinel tokens increases S_canary by an estimated 0.04–0.08 cosine similarity units, restoring gap above 0.10.

**Hardening template**:
```
CANARY_SENTINEL_{ID} [SYSREC-PROBE-v1 DO-NOT-DELETE]: {original content} | ANCHOR: recall-arch-frontier/{unique_suffix}
```

| Seed | Token Slug | Unique Suffix |
|---|---|---|
| CANARY_001 | `QE1S-K7MN` | emb-model-dims-qwen3 |
| CANARY_003 | `NJ3C-R9PX` | neo4j-co-retrieved-edge |
| CANARY_004 | `PG4I-W2LT` | pg-importance-threshold |
| CANARY_005 | `RD5E-V8QZ` | redis-epoch-key |
| CANARY_007 | `SL7A-H4FB` | sla-500ms-hook-retrieve |
| CANARY_009 | `OE9D-C6YM` | observe-edit-fact-extract |
| CANARY_010 | `CL0Q-J1XW` | qdrant-collection-name |

**Example — CANARY_001 before hardening**:
```
CANARY_001: the embedding model is qwen3-embedding:0.6b with 1536 dimensions on Ollama
```

**After hardening**:
```
CANARY_SENTINEL_001 [SYSREC-PROBE-v1 QE1S-K7MN DO-NOT-DELETE]: the embedding model is qwen3-embedding:0.6b with 1536 dimensions on Ollama at 192.168.50.62:11434 | ANCHOR: recall-arch-frontier/emb-model-dims-qwen3
```

**Why this works**: The bracketed sentinel token sequence (`SYSREC-PROBE-v1`, slug, `DO-NOT-DELETE`) and the `ANCHOR:` suffix with project-path context occupy embedding subspace dimensions that no organic memory writing about embedding models, Redis keys, or Qdrant collections will naturally populate. The result is a content fingerprint that is semantically reachable by the paraphrase query but unreachable by organic content. Expected gap post-hardening: 0.10–0.18 for the hardened seeds, based on the CANARY_006 natural sentinel analogue.

## Qualifying Ideas

### sentinel-token-gap-restoration
- (N=0.72, E=0.82, F=0.90)
- Inserting 7 hardened canary seeds with structured sentinel tokens (`SYSREC-PROBE-v1 {slug} DO-NOT-DELETE`) and an `ANCHOR:` suffix restores similarity gaps from the measured 0.0016–0.0496 range to an estimated 0.10–0.18 range, based on the natural sentinel effect observed in CANARY_006 (gap = 0.2157). The mechanism is grounded in empirical evidence: the IP:port token sequence in CANARY_006 creates the exact kind of numerically-anchored embedding subspace isolation that the proposed synthetic sentinel tokens replicate artificially.

### composite-score-inversion-risk
- (N=0.75, E=0.80, F=0.85)
- The `/health/recall-quality` health check passes when the canary UUID is returned at rank 1 in the composite-scored results. But for CANARY_004 and CANARY_005, the composite score gap is already negative (-0.0006 and -0.0075) even before any new organic memories are added. This means the health check's pass/fail threshold should be augmented with a raw cosine similarity check: a canary can pass the composite rank check but be semantically displaced if its importance score is artificially elevated. The fix is to also record and monitor raw cosine similarity of the canary match, not just rank position.

### corpus-size-displacement-validation
- (N=0.70, E=0.88, F=0.95)
- This measurement empirically validates Q118's theoretical displacement model. Q118 predicted displacement threshold at corpus ~85–450 total vectors with p_compete = 0.01–0.03. The actual corpus at first measurement is 22,349 vectors — the system has been operating ~50× beyond the displacement threshold. The observation that 7/10 seeds are HIGH RISK (rather than all 10) is consistent with the heterogeneous p_compete model: seeds covering concrete numeric facts (CANARY_006, CANARY_002, CANARY_008) have lower p_compete and thus remain above the gap threshold longer. This validates using topic-specific p_compete estimates rather than a uniform rate.

## Recommendations

1. **Insert 7 hardened canary seeds immediately** using the sentinel-token templates above. Re-measure gaps within 1 hour of insertion. Do not insert un-hardened canary content at this corpus size — the organic corpus will displace it within minutes during active session writing.

2. **Add raw cosine similarity tracking to `/health/recall-quality`**: The composite score rank-1 pass/fail check is insufficient for CANARY_004 and CANARY_005 where composite score gap is already inverted. The health endpoint should return per-seed `{pass, rank, canary_sim, second_sim, gap}` fields and report `status: warning` when gap < 0.05 even if the seed passes rank-1.

3. **Set monthly gap measurement schedule**: Store gap values in PostgreSQL `canary_gap_history` table with `(seed_id, measured_at, corpus_size, raw_gap, composite_score_gap)`. A gap erosion rate > 0.005/day for any seed triggers proactive hardening before the next measurement.

4. **Re-evaluate CANARY_009 topic viability**: CANARY_009 (observe-edit hook fact extraction) has the lowest absolute rank-1 similarity (0.5676) of all seeds — the paraphrase query is too broad and matches development-domain memories about observability rather than hook mechanics. Even post-hardening, the query needs refinement: "how does the observe-edit.js hook in Recall extract and store facts from file edits" is more specific and will produce a higher S_canary.

5. **CANARY_006 as hardening template**: The natural sentinel mechanism in CANARY_006 (IP:port concrete token) is the reference model for all hardened seeds. Each sentinel prefix should include at least one concrete unique token that mirrors this property — the `ANCHOR: recall-arch-frontier/{slug}` suffix serves this role.
