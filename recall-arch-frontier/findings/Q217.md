## Detection Mechanism Taxonomy

### Class 1: Statistical / Distributional (Centroid Distance, DRS)
Compares incoming embedding vectors against the established distribution of the collection. Methods: Euclidean/Mahalanobis centroid distance, z-score on cosine similarity to cluster means, Directional Relative Shifts (DRS).

- **DRS** (RAGuard, Oct 2025): Specifically targets directions where attacks must concentrate — tests variance shift along the eigenvectors of minimum variance of the clean distribution. More sensitive than naive centroid distance because adversarial embeddings must move toward query-aligned directions.
- **Cost**: O(d) per write for centroid distance (~3μs at d=768). DRS requires incremental PCA: O(k·d) per write where k=tracked directions.
- **Limitation**: Defeated by adaptive attacks that regularize against DRS signal (demonstrated in same paper).

### Class 2: Embedding-Space Outlier Detection (Isolation Forest, LOF, OCSVM)
Unsupervised algorithms that score incoming vectors against a fitted model of the normal distribution.

- **Isolation Forest**: O(log n) inference (100 trees, 256 subsamples). 0.1–1ms per vector at d=768. Distribution-assumption-free. Best tradeoff for write-time gating.
- **LOF (Local Outlier Factor)**: O(n·k) inference — prohibitive for synchronous write gating at collection sizes >1,000 entries. Usable only for async batch auditing.
- **One-Class SVM**: Requires kernel computation at inference — similar order to LOF for high-d data. Inferior to IF on speed at this dimensionality.
- **Comparative result** (ScienceDirect 2024): Isolation Forest slightly outperforms LOF in AUC and execution time for high-dimensional embedding anomaly detection.
- **Theoretical limit** (arXiv 2401.15883): A sufficiently resourced adversary can craft embeddings that are statistically indistinguishable from benign samples in the penultimate representation layer. This sets an upper bound — no purely embedding-space detector can provide strong guarantees against adaptive adversaries.

### Class 3: Perplexity / Linguistic Heuristic (RAGuard)
Operates on raw text before embedding. Uses a small LM (GPT-2 class) for chunk-wise perplexity scoring. Poisoned documents exhibit bimodal perplexity: unusually low-perplexity trigger phrases embedded in normal-perplexity prose.

- **RAGuard result**: DACC 92.4–99.9%, FPR 0.2–10% (typically <6%), FNR 0.1–8.4%, output accuracy 97–100% post-filtering.
- **Cost**: ~20–50ms per chunk with GPT-2 locally (~500MB model). Compatible with async write-path (not synchronous gating).
- **Limitation**: Requires maintaining a local small LM — adds infrastructure dependency to Recall 2.0.

### Class 4: Behavioral / Write-Pattern Anomaly Detection
Monitors write metadata rather than content. Pure count/rate statistics; sub-millisecond; no model required.

| Signal | Normal Pattern | Poisoning Signature |
|---|---|---|
| Write rate | Session-driven, irregular | Burst: 10+ writes in 60s |
| Intra-burst similarity | Mixed topic distribution | Mean pairwise cosine >0.85 |
| Query-write alignment | Weak correlation | Incoming write cosine >0.92 against query centroid |
| Retrieval frequency | Even distribution | Single entry retrieved on diverse queries ("retrieval magnetism") |
| Entry age distribution | Gradually aging | Many near-simultaneous timestamps |
| Write-to-retrieval ratio | Low (reads >> writes) | High (attacker writes, may not retrieve) |

### Class 5: Provenance / Cryptographic Attestation
Tags each write with source identity; signs legitimate entries at write time. Unsigned/low-provenance entries suppressed at retrieval.

- **MemoryGraft proposal** (Dec 2024): Constitutional Consistency Reranking + Cryptographic Provenance Attestation — theoretical, not implemented.
- **Relevance**: Recall 2.0's hook-type SourceTrust (Q200) is a simplified provenance model — already partially addresses this class.

## Minimum Viable Detector for Recall 2.0

### Two-Stage Synchronous Write Guard (No LLM Calls)

**Total latency: 1–6ms synchronous. Memory state: ~150KB. Catches: burst campaigns, narrow-topic injection clusters, adversarially query-aligned embeddings, embedding-space outliers.**

**Stage 1: Behavioral Gate (<1ms)**

Three signals computed from a sliding window of recent writes:

1. **Burst rate EMA**: Write rate exponential moving average vs. μ + 3σ baseline. Flag if exceeded.
2. **Intra-burst cosine**: Mean pairwise cosine similarity of last 10 writes. Flag if >0.85 (narrow-topic injection).
3. **Query-alignment score**: Cosine similarity of incoming embedding vs. centroid of last 20 query embeddings. Flag if >0.92 (adversarially query-optimized write).

State: 150KB for 50 writes at d=768. Latency: <1ms total.

**Stage 2: Isolation Forest Gate (0.1–5ms)**

Isolation Forest (100 trees, 256 subsamples) fitted on last 2,000 collection entries. Per-write inference: O(log n) ≈ 0.1–1ms at d=768 on CPU. Flag if anomaly score < −0.1 (configurable contamination=0.01 → ~1% FPR on clean writes). Async refit every 500 writes.

**On flag**: Quarantine to staging collection (same pattern as Q215 contradiction_queue, Q216 instruction_injection_queue). Tag with anomaly metadata. Async perplexity scoring via GPT-2-class model if second-signal confirmation needed.

**Catches:**
- Burst injection campaigns (Stage 1 rate gate)
- Narrow-topic injection clusters (Stage 1 cosine gate)
- Adversarially query-aligned embeddings (Stage 1 alignment gate)
- General embedding-space outliers (Stage 2 IF gate)

**Does NOT catch:**
- Slow, patient poisoning campaigns spread over days/weeks (rate gate ineffective)
- Embedding-indistinguishable attacks (arXiv 2401.15883 theoretical limit)
- MINJA-style semantic deception (no imperative syntax, semantically benign records — confirmed by Q216)

## Production Deployment Gap

**No production vector database implements any of these detection mechanisms as a built-in write-time feature.**

Confirmed absent in: Qdrant, Weaviate, Pinecone, Chroma, Milvus (all 2024–2025 feature surveys). The prompt.security PoC attack achieved 80% success rate "without triggering alerts" — explicitly noting "structural blindness in current RAG architectures." Production focus remains on: access control, encryption, multi-tenancy. Write-time content validation is absent from all.

## Integration with Recall 2.0 Defense Stack

The two-stage detector extends the four-layer write-path stack from Q216:

```
Content arrives at write endpoint
  ↓
Layer 0: Behavioral gate (Stage 1, Q217) — rate/burst/query-alignment (<1ms)
  ↓ (if clean)
Layer 0b: Isolation Forest gate (Stage 2, Q217) — embedding-space outlier (0.1-5ms)
  ↓ (if clean)
Layer 1: Instruction-pattern heuristic (Q216) — explicit injection syntax (<0.5ms)
  ↓ (if clean)
Layer 2: Contradiction detection (Q215) — hallucination vs. established truth (+0.3-0.5ms)
  ↓ (if clean)
Layer 3: SimHash dedup (Q171) — near-duplicate writes (<0.1ms)
  ↓ (if clean)
Layer 4: SourceTrust assignment (Q200) — trust score by hook provenance
  ↓
Store with LMDB+HNSW
```

**Total overhead: +1.8–7.5ms per write. Budget: 186ms (Q197). Headroom: 97%.** This is the complete write-path defense architecture for Recall 2.0.

## Ideas Extracted

### Idea: isolation-forest-write-time-embedding-anomaly-no-llm-1ms-o-log-n
- **Novelty**: 0.70 — Isolation Forest (scikit-learn) as a write-time embedding anomaly detector for an AI memory system is not documented in any production system and is absent from all surveyed production vector databases. The specific application to the Recall 2.0 write path — with async refit, 256-subsample efficiency at d=768, and integration as Layer 0b of the defense stack — is a new operational design. The O(log n) inference property is critical: it makes synchronous write-time scoring feasible where LOF (O(n·k)) is not.
- **Evidence**: 0.80 — IF O(log n) complexity: scikit-learn documentation confirmed. IF vs LOF comparative accuracy (ScienceDirect 2024): IF slightly outperforms LOF in AUC and execution time for high-d embeddings. MemoryGraft (Dec 2024) demonstrates 47.9% PRP with only 10 poisoned records — the threat requires only small injections to be effective, making per-write scoring necessary. RAGuard (Oct 2025) validates the principle of write-time filtering achieving DACC 92–99.9%, but uses perplexity (requires small LM); IF replaces this with a model-free alternative.
- **Feasibility**: 0.90 — Implementation: scikit-learn IsolationForest, 100 trees, max_samples=256, contamination=0.01. Per-write inference: 0.1–1ms at d=768. Refit: every 500 writes, async background thread, warm_start=True for incremental tree addition. State: 2,000-entry fit buffer ~5.9MB at d=768. Integration: single function call in the Recall 2.0 write handler before storage. No external service. No LLM. Phase 4 scope.
- **Description**: Isolation Forest scoring at write time for Recall 2.0 agent memory writes. Fits on last 2,000 collection entries; per-write inference is O(log n) ≈ 0.1–1ms at d=768 on CPU. Anomaly score < −0.1 (1% FPR at contamination=0.01) triggers quarantine. Async refit every 500 writes. No LLM required. Outperforms LOF (O(n·k), prohibitive synchronously) and OCSVM at this dimensionality. No production vector database currently implements this. Catches general embedding-space outliers and adversarially query-aligned embeddings not caught by behavioral signals. Does not defeat embedding-indistinguishable adaptive attacks (theoretical limit, arXiv 2401.15883).

### Idea: behavioral-write-burst-intra-cosine-query-alignment-sub-ms-three-signals
- **Novelty**: 0.65 — Three specific behavioral signals for write-time poisoning detection (burst rate EMA, intra-burst cosine similarity >0.85, query-alignment cosine >0.92) derived from the ML security literature's characterization of poisoning campaign signatures. No production system implements any of these. The query-alignment signal is particularly novel: normal agent writes have weak correlation with query embeddings; adversarially crafted writes have high cosine with query centroids (they must, to be retrieved). This inverts the typical assumption that high write-query similarity is good (it is suspicious).
- **Evidence**: 0.75 — MemoryGraft (Dec 2024) confirms "retrieval magnetism" signal: poisoned entries activate on narrow query ranges. Christian Schneider (2025) characterizes anomalously high retrieval frequency across diverse queries as a key adversarial signal. Burst injection signature: prompt.security PoC and AgentPoison (NeurIPS 2024) both use concentrated write campaigns. Intra-burst cosine >0.85 threshold: derived from Q171's 0.72 near-duplicate threshold (same-topic but different-claim entries will have lower cosine than near-duplicates but higher than normal memory diversity). All three signals are sub-millisecond by construction.
- **Feasibility**: 0.95 — Pure Python/Rust arithmetic on a deque of (timestamp, embedding) tuples. State: 50 entries × d=768 float32 = 150KB. No external service. No model. Rate EMA: 2 floats + Welford online update. Intra-burst cosine: O(N²·d) for N=10 last writes = ~0.4ms. Query-alignment: O(d) dot product = ~3μs. Combined: <1ms. Implementation: single function called from write handler before Isolation Forest gate. Stacks cleanly with existing write path. Phase 4 scope alongside other write-time defenses.
- **Description**: Three behavioral anomaly signals for write-time poisoning detection, all sub-millisecond, no model required: (1) burst rate EMA — flag write rate >μ+3σ over 60s window; (2) intra-burst cosine — mean pairwise cosine of last 10 writes >0.85 indicates narrow-topic injection cluster; (3) query-alignment score — incoming write cosine >0.92 against centroid of last 20 queries indicates adversarially query-optimized write. 150KB state. Combined: <1ms synchronous. Catches burst injection campaigns missed by embedding-space detectors. Does not catch patient attacks (spread over days) or semantically benign MINJA-style injection.

### Idea: embedding-indistinguishability-theoretical-upper-bound-layered-defense-required
- **Novelty**: 0.60 — The formal result that a sophisticated adversary can defeat any single-layer embedding-space detector (arXiv 2401.15883) is established in the literature but has not been explicitly connected to AI memory system design. The implication for Recall 2.0's architecture is: no individual detection layer provides strong security guarantees; the 5-layer stack (behavioral + IF + instruction heuristic + contradiction detection + SimHash + SourceTrust) is not overkill — it is the minimum required to address distinct attack classes. Each layer catches attacks that other layers miss.
- **Evidence**: 0.85 — Embedding indistinguishability formalized in arXiv 2401.15883 (Jan 2024): proves that backdoored samples can be crafted statistically indistinguishable from benign in embedding space, setting a fundamental upper bound on detector performance. RAGuard (Oct 2025) confirms: DRS-evading adaptive attacks demonstrated within the same paper that proposes DRS. MINJA (Mar 2025): bypasses all moderation including embedding-space detection. Combined: three independent sources confirm no single-layer approach is sufficient.
- **Feasibility**: 0.99 — Theoretical result; no implementation required. Implication: Recall 2.0 Phase 4 should implement all write-path layers (Q217 behavioral + IF + Q216 instruction heuristic + Q215 contradiction detection) rather than selecting one. Total combined overhead: +1.8–7.5ms within 186ms budget. Defense-in-depth architecture is the only available response to adaptive adversaries. Flag MINJA semantic deception as acknowledged unresolved gap in Recall 2.0 threat model documentation.
- **Description**: Formal result (arXiv 2401.15883) establishes that sufficiently resourced adversaries can craft embedding vectors statistically indistinguishable from benign collection entries, defeating purely embedding-space detectors. Combined with MINJA's query-only semantic injection (Q216), this sets a hard theoretical limit on any single-layer write-time defense. Architecture implication for Recall 2.0: all five layers of the write-path defense stack (behavioral gate + Isolation Forest + instruction heuristic + contradiction detection + SimHash + SourceTrust) are necessary and complementary. Each layer catches distinct attack classes that other layers miss. The combined stack addresses all documented attack classes except: (1) slow patient campaigns spread over days (no burst signal), (2) adaptive embedding-indistinguishable attacks (theoretical limit), (3) MINJA semantic deception (acknowledged gap). These three unresolved vectors define the remaining frontier of Recall 2.0's adversarial threat model.

## Analytical Summary

**Data poisoning detection in ML retrieval systems has produced viable mechanisms in 2024–2025 that are directly applicable to Recall 2.0.** The production-to-research gap is total: no vector database ships these mechanisms. The minimum viable detector (behavioral gate + Isolation Forest, 1–6ms, no LLM) is feasible within Recall 2.0's write budget and stacks cleanly with Q215 and Q216 defenses.

**The Isolation Forest approach is the right algorithm for synchronous write-time scoring.** O(log n) inference makes it uniquely suited to sub-5ms write gates. LOF and OCSVM are prohibitive synchronously; perplexity scoring requires a local LM. IF with periodic async refit provides continuous coverage with minimal infrastructure addition.

**A theoretical limit exists**: adaptive adversaries can defeat any single embedding-space layer (arXiv 2401.15883). Layered detection is required. The combined 5-layer stack provides defense-in-depth against all documented attack classes, with three acknowledged unresolved gaps (patient campaigns, adaptive embedding attacks, MINJA).
