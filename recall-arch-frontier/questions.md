### [ADVERSARIAL] Q031: Does the retrieval-frequency IDF penalty (Q013) work at cross-purposes with the implicit citation feedback loop (Q009), or do they compose into a coherent importance signal?
**Status**: DONE
**Priority**: High
**Rationale**: Q013 proposes a Zipf-aware IDF penalty: `adjusted_score = cosine × importance × log(N / retrieval_count)`, which progressively down-weights memories retrieved frequently. Q009 proposes implicit citation feedback: memories whose content appears in the LLM response get `importance += 0.1 × (1 - importance)` boosts. These two mechanisms act on the same importance field but from opposing directions for high-frequency, high-citation memories. A memory about a core infrastructure fact (e.g., CasaOS IP address) would be: (a) retrieved in nearly every session (high retrieval_count → IDF penalty forces score down), AND (b) cited in nearly every response (high citation rate → feedback loop pushes importance up). The IDF signal says "this is noise, demote it." The citation signal says "this is useful, promote it." Which signal should dominate? Are they measuring the same underlying quality, or orthogonal properties (ubiquity vs. utility)? An adversarial framing is needed because the wrong answer permanently corrupts the importance field — the very signal both mechanisms depend on.
**Defender claim**: The IDF penalty is correct to dominate. A memory retrieved in 95% of sessions is definitionally omnipresent background knowledge that does not help the LLM discriminate between contexts. Its high citation rate is precisely because it is always injected, not because it is uniquely valuable to any given session. The IDF penalty should suppress its retrieval rank so that session-specific memories can surface in top-K. Citation feedback should be disabled or heavily discounted for memories above a retrieval-frequency threshold.
**Attacker claim**: The citation feedback signal is more reliable because it measures actual utility — did the LLM use this information? A memory cited in 90% of sessions where it is injected is genuinely valuable, not noise. The IDF penalty mistakes high-value universal facts for low-value filler. The correct response to ubiquitous useful memories is to move them to the always-injected hot tier (Q003's LLPC analog), not to suppress their retrieval rank.
