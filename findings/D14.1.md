### `_get_next_wave_index()` тАФ lines 218тАУ223

```python
def _get_next_wave_index(questions_text: str) -> int:
    """Find the next sequential wave index for QG questions."""
    matches = re.findall(r"## QG(\d+)\.\d+", questions_text)
    if not matches:
        return 1
    return max(int(m) for m in matches) + 1
```

The regex is `## QG(\d+)\.\d+`. It matches only headers of the form `## QGN.N`.

BL 2.0 headers use domain-prefixed IDs: `## D1.1`, `## F2.3`, `## A14.3`, `## V5.1`. None
contain the literal substring `QG`. On any questions.md containing only BL 2.0 questions,
`matches` is an empty list, `not matches` is `True`, and the function returns `1`
unconditionally. Every successive invocation of `generate_goal_questions()` against a BL 2.0
question bank reads the same empty match set and assigns wave index 1, producing duplicate IDs
(`QG1.1`, `QG1.2`, тАж) on every run after the first.

- **Line 242**: `if "## QG" not in block: continue` тАФ this is a hard gate on the literal

under normal operation the parser is internally consistent тАФ the LLM outputs QG headers and the
guard accepts them. The guard would silently drop output if the LLM deviated to BL 2.0 format,
or if this function were ever reused on a BL 2.0 questions.md directly.

## Analysis

**`_get_next_wave_index()` тАФ confirmed defect, guaranteed production impact.**

Causal chain:
1. BL 2.0 campaign accumulates questions.md with headers `## D1.1`, `## F2.1`, `## A14.3`.
2. User runs `generate_goal_questions()` (line 254).
3. Line 299 reads existing_text from questions.md.
4. Line 301: `wave_idx = _get_next_wave_index(existing_text)` тАФ returns `1`.
5. Line 306 renumbers all LLM output blocks to `QG1.1`, `QG1.2`, тАж
6. Blocks appended to questions.md as `QG1.x`.
7. Second invocation: line 299 reads updated questions.md тАФ now contains `## QG1.1`, `## QG1.3`.
8. `_get_next_wave_index` finds those QG headers, returns `2`. Correct on second run.

**The collision window is the first invocation only.** If the user runs goal-mode twice
in the same session without an intervening file write completing (e.g., two concurrent calls,
or a dry-run followed by a real run using a cached text snapshot), both calls read a QG-free
file and both return wave index 1 тАФ producing duplicate QG1.x IDs.

More practically: on the very first goal campaign run against a BL 2.0 file that has never
had QG questions, the wave is 1. This is not wrong per se тАФ QG1.x is a valid first wave.
The genuine defect is that if the campaign designer expects goal wave numbering to reflect the
total question wave count in the file (e.g., a file already on D14.x should start goal waves
at QG15), the function cannot do that because it is blind to non-QG headers.

**`_parse_goal_questions()` тАФ correct for its stated input, but QG-locked.**

The guard at line 242 enforces the QG namespace on all goal output. This is consistent with
the prompt but is a hardcoded constraint that would silently discard BL 2.0-format blocks
without any error. The `_QUESTION_BLOCK_HEADER` compiled regex on line 33 is unused and
suggests this was once planned to be more general.

Root cause of the wave-index defect: `_get_next_wave_index()` line 220, regex
`r"## QG(\d+)\.\d+"` тАФ anchored to the QG prefix, invisible to BL 2.0 domain headers.

## Fix Specification

- **File**: `C:\Users\trg16\Dev\autosearch\bl\goal.py`
- **Line**: 220
- **Change**: Extend `_get_next_wave_index` to also count existing QG waves by scanning the
  maximum numeric suffix of any `## QGN.` header, and separately count all distinct BL 2.0
  wave numerics (the number part of `## D14.1`, `## F2.3`, etc.), then return
  `max(qg_max, bl2_max) + 1`. Minimal diff:

  ```python
  # Before (line 220):
  matches = re.findall(r"## QG(\d+)\.\d+", questions_text)
  if not matches:
      return 1
  return max(int(m) for m in matches) + 1

  # After:
  qg_matches = [int(m) for m in re.findall(r"## QG(\d+)\.\d+", questions_text)]
  bl2_matches = [int(m) for m in re.findall(r"## [DFAV](\d+)\.\d+", questions_text)]
  all_waves = qg_matches + bl2_matches
  if not all_waves:
      return 1
  return max(all_waves) + 1
  ```

  This keeps the function's contract intact for QG-only campaigns while making it wave-aware
  for BL 2.0 campaigns. The renumber substitution at line 306
  (`re.sub(r"## QG\d+\.(\d+)", f"## QG{wave_idx}.{i}", block)`) is unaffected тАФ it only
  rewrites QG headers in the LLM output and does not need to change.
