**Summary:**

**Risk 1 — Fixed** (`commit 584e000`): `warm_injection_embeddings()` used `import logging` / `logging.getLogger()` but called the logger with structlog-style kwargs (`error=str(exc)`, `count=N`). The stdlib `Logger._log()` rejects keyword arguments, so the `except` block itself raised `TypeError` — defeating the documented fail-silently contract. Fixed by replacing with `structlog.get_logger()`, consistent with every other module in the codebase. Two previously-omitted security tests (`test_embed_failure_does_not_raise`, `test_redis_failure_does_not_raise`) now pass. 55/55 total.

**Risk 2 — Reported** (`security_finding_q3_3_toctou.md`): The Q3.3 TOCTOU finding is a **false positive**. There is no Redis SETNX anywhere in `write_guard.py`. `WriteGuard.check()` is a stateless, purely synchronous function — no shared mutable state, no locks needed. The concurrent test suite confirms deterministic behaviour under load. The finding may have been intended for a different module; the existing `POST /admin/consolidate` Redis guard (`6ed5bcf`) is the correct target if concurrent write locking is the concern.
