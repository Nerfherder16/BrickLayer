## Mitigation Recommendation

The 11,926 Neo4j nodes with importance=0.0 indicate the Neo4j write path for importance updates is broken or was never initialized for pre-existing records. The reconcile worker (added in Q9.5.3 fix) will sync these on its first Sunday run via `neo4j.update_importance()`. However, the root cause — why Neo4j importance is 0.0 for 58% of nodes while Qdrant has the real values — should be investigated in the write path.
