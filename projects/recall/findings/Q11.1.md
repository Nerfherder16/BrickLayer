## Summary

## Root Cause (from Q10.2)

The write path stores importance in Qdrant payload and Neo4j node properties independently. If the Neo4j write fails silently (transient connection error, timeout), Qdrant gets the real value but Neo4j defaults to `0.0`. The reconcile worker exists but runs weekly (Sunday 05:30 UTC) — the accumulated backlog was 6 weeks of drift.

## Remediation

**Immediate**: Manual repair pass via `/admin/reconcile?repair=true` — zero data loss, all values sourced from Qdrant (authoritative).

**Ongoing**: Weekly cron already configured (`ReconcileWorker` in `src/workers/reconcile.py`, committed Q9.5.3). Next Sunday run will catch any new drift.

Repair completed without errors, mismatch count reduced from 11,924 to 1 (99.99% recovery rate). HEALTHY.
