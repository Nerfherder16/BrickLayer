# Finding: Q5.4 — embed_batch silent failure and zero-vector propagation

**Question**: embed_batch() catches all exceptions and returns zero-vectors silently. Callers receive [] or zero vectors with no signal that embedding failed — these bad vectors get stored in Qdrant and poison future retrieval results.
**Verdict**: WARNING
**Severity**: Medium
**Mode**: quality
**Target**: src/core/embeddings.py + tests/core/test_embeddings_cache.py

## Summary

embed_batch() line 232-233: `except Exception: results[orig_i] = []` — silent, no logging. The outer batch failure IS logged (line 224-227, logger.warning("embed_batch_failed")), but the sequential fallback's inner loop catches individual embed() failures silently and returns empty lists. Callers cannot distinguish "embedding succeeded and returned []" from "embedding failed and returned []". The batch failure IS observable; the per-item sequential fallback failure is NOT. Verdict: WARNING (not FAILURE — batch path is logged; only the fallback within the fallback is silent).
