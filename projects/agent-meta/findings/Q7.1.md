# Finding: Q7.1 — hypothesis-generator uses domain "{project}-autoresearch" while all BL 2.0 agents use "{project}-bricklayer"; Recall queries return empty

**Question**: Does hypothesis-generator's Recall query use the correct domain string for BL 2.0 campaigns?
**Verdict**: WARNING
**Severity**: High

## Evidence

### hypothesis-generator.md Recall queries (5 queries at session start)

```python
recall_search(query="failure boundary sensitivity leverage",
    domain="{project}-autoresearch", tags=["agent:quantitative-analyst"])
recall_search(query="legal constraint regulatory risk",
    domain="{project}-autoresearch", tags=["agent:regulatory-researcher"])
recall_search(query="market analogue failure trigger",
    domain="{project}-autoresearch", tags=["agent:competitive-analyst"])
recall_search(query="cross-domain dependency critical path",
    domain="{project}-autoresearch", tags=["agent:synthesizer"])
recall_search(query="regression baseline performance degradation",
    domain="{project}-autoresearch", tags=["agent:benchmark-engineer"])
```

**Domain**: `{project}-autoresearch` — 5 occurrences

### hypothesis-generator.md Recall store (after generating questions)

```python
recall_store(
    domain="{project}-autoresearch",
    tags=["bricklayer", "autoresearch", "agent:hypothesis-generator", "type:wave-summary"],
)
```

**Store domain**: `{project}-autoresearch`

### All other BL 2.0 agents — domain convention

| Agent | Domain used |
|-------|------------|
| Mortar (checkpoint store) | `{project}-bricklayer` |
| Mortar (campaign-start query) | `{project}-bricklayer` |
| agent-auditor (query + store) | `{project}-bricklayer` |
| planner (query + store) | `{project}-bricklayer` |
| peer-reviewer (OVERRIDE store) | `{project}-bricklayer` |
| peer-reviewer (CONFIRMED store) | `{project}-bricklayer` |
| **hypothesis-generator (queries)** | **`{project}-autoresearch`** ← MISMATCH |
| **hypothesis-generator (store)** | **`{project}-autoresearch`** ← MISMATCH |

### Impact

When hypothesis-generator runs its 5 startup Recall queries with `domain="{project}-autoresearch"`:
- Mortar's wave checkpoints (stored in `{project}-bricklayer`) → **not found**
- agent-auditor reports (stored in `{project}-bricklayer`) → **not found**
- peer-reviewer OVERRIDEs (stored in `{project}-bricklayer`) → **not found**
- Any agent's stores from the current campaign → **not found**

All 5 queries return empty. hypothesis-generator proceeds without any cross-agent memory context — it reads only the filesystem (findings directory and results.tsv).

Additionally, hypothesis-generator's own wave-summary is stored in `{project}-autoresearch`. On the next invocation, its store-query would still return nothing (because it searches `{project}-autoresearch` with `agent:quantitative-analyst` tags — it doesn't query its own prior wave summaries). But if it DID query its own domain, it would find the summaries.

The more critical issue: if a user runs a second campaign session after the first, hypothesis-generator gains no context about what Wave 1 covered — it can't avoid redundant questions. The duplicate-prevention mechanism (query for prior wave summaries) is broken by the domain mismatch.

### Root cause

The BrickLayer project was originally named "autoresearch" (the tool was called `autoresearch/`). hypothesis-generator.md was written when the domain convention was `{project}-autoresearch`. When the project was rebranded to BrickLayer 2.0 and all agents updated to use `{project}-bricklayer`, hypothesis-generator.md was not updated.

This is consistent with the root cause identified in Q1.1-WM4: agents developed independently without a shared interface contract — in this case, the shared interface is the Recall domain naming convention.

## Mitigation Recommendation

1. **Update hypothesis-generator.md**: Replace all 6 occurrences of `{project}-autoresearch` with `{project}-bricklayer`.

2. **Add a Wave N self-query**: At startup, query `tags=["agent:hypothesis-generator"]` in addition to the per-specialist queries:
   ```python
   recall_search(query="wave questions generated coverage",
       domain="{project}-bricklayer", tags=["agent:hypothesis-generator"])
   ```
   This retrieves prior wave summaries and prevents redundant question generation.

3. **Add domain constant to constants.py**: `RECALL_DOMAIN_SUFFIX = "bricklayer"` so all agents reference a single source of truth for the domain suffix rather than hardcoding it.
