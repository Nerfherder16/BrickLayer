# Finding: Q1.1-FU3 — BL 2.0 specialist agents exist but are unreachable via Mortar's routing table

**Question**: Do design-reviewer, evolve-optimizer, health-monitor, and cascade-analyst exist in the template agents directory? If not, Mortar would call undefined agents even with correct routing.
**Verdict**: WARNING
**Severity**: High

## Evidence

All 5 BL 2.0 specialist agents corresponding to modes missing from Mortar's routing table are present in the template:

```
template/.claude/agents/
  cascade-analyst.md    ← Predict mode
  design-reviewer.md    ← Validate mode
  evolve-optimizer.md   ← Evolve mode
  health-monitor.md     ← Monitor mode
  research-analyst.md   ← Research/Frontier mode (no dedicated frontier-analyst.md)
```

Additionally, BL 1.x agents `regulatory-researcher.md` and `competitive-analyst.md` are also present — they are referenced in Mortar's routing table for the `research` mode. This creates a mixed fleet: both BL 1.x and BL 2.0 research agents exist side-by-side, but Mortar only routes to the BL 1.x versions.

### What this means

The agents are **ready** — the gap is entirely in Mortar's routing table and field name mismatch (from Q1.1-FU1). If both were fixed:
- `Validate` → `design-reviewer.md` (exists)
- `Evolve` → `evolve-optimizer.md` (exists)
- `Monitor` → `health-monitor.md` (exists)
- `Predict` → `cascade-analyst.md` (exists)
- `Frontier` → `research-analyst.md` (exists — serves double duty, no dedicated frontier-analyst)
- `Research` → `research-analyst.md` (BL 2.0) or `regulatory-researcher.md` + `competitive-analyst.md` (BL 1.x)

### Missing agent

No `frontier-analyst.md` exists. The Frontier mode would need to route to `research-analyst.md` as a proxy, which may not be documented anywhere.

### Summary

This is a pure configuration gap, not a missing code gap. All infrastructure exists; Mortar's routing table is simply 3+ versions behind the agent fleet it orchestrates.

## Mitigation Recommendation

1. **Update mortar.md routing table** — add all 5 missing mode routes pointing to the correct existing agents.
2. **Document the Frontier → research-analyst proxy** in both mortar.md and question-designer-bl2.md.
3. **Consider retiring regulatory-researcher and competitive-analyst** from the routing table in favor of the unified `research-analyst` — or keep both and let the `**Agent**:` field field override on a per-question basis.
4. **Add `frontier-analyst.md`** as a thin wrapper around research-analyst with Frontier-specific instructions, to complete the BL 2.0 mode coverage.
