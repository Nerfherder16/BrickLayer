# Finding: Q2.3 — agent-auditor's definitive verdict whitelist excludes 14 of 26 valid verdicts; code-reviewer and peer-reviewer would always score UNDERPERFORMING

**Question**: Does agent-auditor's scoring methodology match the definitive rate thresholds in constants.py?
**Verdict**: WARNING
**Severity**: High

## Evidence

### Threshold comparison

| Metric | constants.py | agent-auditor | Match? |
|--------|-------------|----------------|--------|
| MIN_DEFINITIVE_RATE | 0.80 | 0.80+ = HEALTHY | ✓ |
| MIN_FIX_SPEC_COMPLETENESS | 0.80 | 0.80+ = HEALTHY | ✓ |
| MIN_EVIDENCE_DEPTH | 0.80 | 80%+ = HEALTHY | ✓ |

The numeric thresholds align exactly. The hypothesis of a "double standard" on thresholds is **not confirmed**.

### The real gap: definitive verdict whitelist is incomplete

agent-auditor's definitive rate formula:

```
definitive = DIAGNOSIS_COMPLETE + COMPLIANT + NON_COMPLIANT + FIXED + HEALTHY + CALIBRATED
rate = definitive / total_questions
```

This is a **whitelist** of 6 verdicts. All other verdicts count as non-definitive.

constants.py defines 26 VALID_VERDICTS. Mapping each to agent-auditor's scoring:

| Verdict | In agent-auditor definitive set? | Expected? |
|---------|----------------------------------|-----------|
| DIAGNOSIS_COMPLETE | ✓ YES | ✓ |

**14 verdicts** that represent clear conclusions are excluded from agent-auditor's definitive set.

### Impact: affected agents

Agents whose primary verdicts are NOT in the definitive whitelist:

| Agent | Primary verdicts | Would score |
|-------|-----------------|-------------|
| code-reviewer | APPROVED, NEEDS_REVISION, BLOCKED | 0% definitive → UNDERPERFORMING |
| peer-reviewer | CONFIRMED, CONCERNS, OVERRIDE, INCONCLUSIVE | 0% for CONFIRMED/CONCERNS → UNDERPERFORMING |
| forge-check | FLEET_COMPLETE, GAPS_FOUND | 0% definitive → UNDERPERFORMING |
| agent-auditor | FLEET_HEALTHY, FLEET_WARNING, FLEET_UNDERPERFORMING | 0% definitive → UNDERPERFORMING |

**All four agents would be incorrectly rated as UNDERPERFORMING by agent-auditor**, triggering "Escalate to Overseer immediately" for every audit. The fleet would appear broken when it is functioning correctly.

### No structural coupling between agent-auditor and constants.py

agent-auditor.md hardcodes its thresholds and verdict whitelist. It does not import or read constants.py. If MIN_DEFINITIVE_RATE changes in constants.py, agent-auditor retains its hardcoded 0.80. Both happen to agree now, but divergence is possible and would be invisible.

constants.py does not define which verdicts are "definitive" — it only defines the minimum rate. The operational definition of "definitive" lives exclusively in agent-auditor.md.

## Summary

| Issue | Severity |
|-------|----------|
| Numeric thresholds match (0.80) | ✓ No issue |
| Definitive whitelist excludes 14 of 26 valid verdicts | High — 4 agents always misclassified |
| code-reviewer, peer-reviewer, forge-check, agent-auditor all score UNDERPERFORMING | High — false alarms saturate Overseer |
| No structural coupling: constants.py changes don't propagate to agent-auditor | Medium — silent divergence risk |

## Mitigation Recommendation

1. **Rewrite the definitive rate formula**: Change from whitelist to exclusion model:
   ```
   non_definitive = INCONCLUSIVE + RE_QUEUED
   definitive = total - non_definitive
   rate = definitive / total
   ```
   Or equivalently: count everything except INCONCLUSIVE and RE_QUEUED as definitive.

2. **Add to constants.py**: Define `NON_DEFINITIVE_VERDICTS = ["INCONCLUSIVE", "RE_QUEUED"]` so agent-auditor can import rather than hardcode.

3. **Test with code-reviewer findings**: After fixing the formula, verify code-reviewer scores HEALTHY on its APPROVED/NEEDS_REVISION/BLOCKED verdicts.
