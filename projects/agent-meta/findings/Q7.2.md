# Finding: Q7.2 — overseer.md exists but is not wired into the loop; agent-auditor escalates to it but Mortar never calls it; interface requires agent_db.json that no agent writes

**Question**: Does agent-auditor's "Escalate to Overseer immediately" recommendation have a reachable target agent?
**Verdict**: WARNING
**Severity**: Medium

## Evidence

### overseer.md — exists in template

`template/.claude/agents/overseer.md` exists. The hypothesis that it doesn't exist is **not confirmed**. overseer.md describes a meta-agent that:
- Reads `agent_db.json` (required input)
- Reads `agents_dir`, `findings_dir`, `project_brief`
- Identifies underperforming agents and rewrites their `.md` files
- Creates new agents when FORGE_NEEDED.md shows missing capabilities

### Interface mismatch: agent-auditor writes AUDIT_REPORT.md, overseer reads agent_db.json

**agent-auditor.md** escalation path:
```
"Any dimension UNDERPERFORMING | UNDERPERFORMING | Escalate to Overseer immediately"
```
agent-auditor writes `{agents_dir}/AUDIT_REPORT.md` — this is its output artifact.

**overseer.md** required inputs:
```
- `agent_db_json` — path to agent_db.json with scores and verdict histories
- `agents_dir`
- `findings_dir`
- `project_brief`
```

overseer requires `agent_db.json`. No agent in the fleet writes `agent_db.json`. The "Escalate to Overseer" instruction in agent-auditor means: invoke overseer with `agent_db_json=...` but the required file doesn't exist.

Mortar's sentinel table (from mortar.md):

| Every N questions | Action |
|-------------------|--------|
| Every 5 | forge-check |
| Every 10 | agent-auditor |
| After every finding | peer-reviewer |

**overseer is not in the sentinel table**. Mortar never calls overseer. agent-auditor.md says "Overseer reads [AUDIT_REPORT.md] on its scheduled invocation" — but there is no schedule.

### overseer.md is not in constants.py REQUIRED_AGENTS

```python
REQUIRED_AGENTS = [
    "mortar", "question-designer", "hypothesis-generator", "diagnose-analyst",
    "fix-implementer", "synthesizer", "planner", "code-reviewer",
    "peer-reviewer", "forge-check", "agent-auditor",
]
```

overseer is absent from REQUIRED_AGENTS. forge-check's phantom agent check would NOT flag overseer as missing, even if it were deleted.

### End-to-end escalation path

1. agent-auditor identifies agent X as UNDERPERFORMING
2. agent-auditor writes AUDIT_REPORT.md: "Recommended action: ESCALATE_TO_OVERSEER"
3. Mortar reads AUDIT_REPORT.md at next wave-start
4. Mortar has no instruction for "ESCALATE_TO_OVERSEER" — it checks for FLEET_HEALTHY/FLEET_WARNING/FLEET_UNDERPERFORMING
5. **Escalation silently terminates**: Mortar doesn't know to call overseer; overseer has no schedule; no agent calls overseer autonomously

### Summary

| Issue | Severity |
|-------|----------|
| overseer.md exists (hypothesis wrong) | ✓ No issue |
| overseer requires agent_db.json, no agent writes it | Medium |
| overseer not in Mortar's sentinel table | Medium — never invoked |
| overseer not in REQUIRED_AGENTS | Low — could be deleted silently |
| ESCALATE_TO_OVERSEER not handled by Mortar | Medium — escalations silently dropped |

## Mitigation Recommendation

1. **Add overseer to Mortar's wave sentinels**: On FLEET_UNDERPERFORMING from agent-auditor, invoke overseer with `agents_dir`, `findings_dir`, `project_brief`, and the path to AUDIT_REPORT.md.

2. **agent-auditor should write agent_db.json** (or overseer should accept AUDIT_REPORT.md as a substitute): Align the two agents' interface. Either agent-auditor outputs the format overseer expects, or overseer is updated to read AUDIT_REPORT.md.

3. **Add overseer to REQUIRED_AGENTS in constants.py** if it's a production-critical path.
