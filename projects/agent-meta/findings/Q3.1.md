# Finding: Q3.1 — simulate.py has a hollow-section bypass: section headers satisfy content checks, scoring a structurally broken agent as WARNING not FAILING

**Question**: Does the simulate.py fleet scorer accurately identify FAILING agents, or does it score too leniently?
**Verdict**: WARNING
**Severity**: Medium

## Evidence

### Scoring deduction table (from score_agent function)

| Check | Condition | Deduction |
|-------|-----------|-----------|
| Description length | < 50 chars | -15 |
| Missing `## Inputs` section | header not found | -10 |
| Missing `## Output contract` section | header not found | -10 |
| Missing `## Recall` section | header not found | -10 |
| No output contract in content | `"output contract" not in content.lower()` | -20 |
| No recall calls | no `recall_store` or `recall_search` | -10 |
| No valid verdicts | no VALID_VERDICTS found | -15 |
| Bad tag format | no `tags=["bricklayer"` | -5 |

FAILING threshold: score < 60. WARNING: 60–79. HEALTHY: ≥ 80.

### Hypothesis check: "no output contract (-20), no recall (-10), no verdicts (-15) = 55 = WARNING"

The hypothesis's arithmetic is wrong: 55 < 60 → **FAILING** by the code's own threshold. An agent stripped of output contract, recall, and verdicts scores 55 — which IS FAILING, not WARNING. The specific claimed failure mode does not exist.

### Actual lenient path: hollow section headers bypass the content check

The output contract check is:
```python
if "output contract" not in content.lower():
    score -= 20
```

And the section check is:
```python
for required in REQUIRED_SECTIONS:
    req_lower = required.lower()
    if not any(s.startswith(req_lower) for s in sections):
        score -= 10
```

REQUIRED_SECTIONS includes `"## Output contract"`.

**The content check `"output contract" not in content.lower()` is satisfied by the section HEADER `## Output contract` itself** — the literal string "output contract" appears in the header text. So:

1. An agent with `## Output contract` as a header but **no actual content** below it passes the -20 check.
2. It also passes the section header -10 check.
3. Together, a hollow `## Output contract` section bypasses 30 points of potential deductions.

**Concrete worst-case for a "hollow stub" agent:**
```markdown
---
name: broken-agent
description: This agent does something useful, has 55 chars in description here.
---
## Inputs
## Output contract
## Recall
```

Scoring:
- Description 55 chars ≥ 50: no -15
- `## Inputs` header found: no -10
- `## Output contract` header found: no -10
- `## Recall` header found: no -10
- "output contract" in content (it's the header): no -20
- No `recall_store`/`recall_search`: **-10**
- No valid verdicts: **-15**
- No bricklayer tags: **-5**

**Score = 100 - 30 = 70 → WARNING**

A stub agent with empty section bodies, no recall calls, no verdicts, and no tags scores **WARNING** — it is functionally useless but escapes the FAILING threshold.

### Recall and verdict checks can also be trivially gamed

- Verdict check: `re.findall(r"\b(" + "|".join(VALID_VERDICTS) + r")\b", content)` — any mention of a verdict word in any context counts. An agent saying "this agent never produces HEALTHY verdicts" would count as having valid verdicts.
- Recall check: presence of `recall_store` or `recall_search` anywhere in the file. An agent with a commented-out recall call in a code example passes.

### Summary table

| Scenario | Expected | Actual score | Verdict |
|----------|----------|-------------|---------|
| No output contract, no recall, no verdicts | FAILING | 55 | **FAILING** ✓ (hypothesis wrong) |
| Hollow section headers + no recall + no verdicts + no tags | FAILING | 70 | **WARNING** ✗ (lenient) |
| Mentioned verdict in negative sentence + hollow sections | FAILING | 85 | **HEALTHY** ✗ (very lenient) |

## Conclusion

The hypothesis's specific arithmetic error (55=WARNING) is incorrect — 55 IS FAILING. However, a different and real lenient path exists: hollow section headers satisfy content checks, allowing a structurally broken agent to score 70 (WARNING) instead of FAILING. An agent with every required section empty except for its header passes 30 points worth of checks it should fail.

## Mitigation Recommendation

1. **Check section content, not just section presence**: After finding a section header, verify the content below it is non-empty (> 10 chars). Empty sections should apply the same penalty as missing sections.

2. **Require recall calls to appear below the `## Recall` section**: Don't allow recall calls embedded in code examples or comments to satisfy the check.

3. **Restrict verdict matching to the taxonomy/output contract section**: Only count verdicts that appear in a table or explicit list within the Output contract or Verdicts section.
