# Finding: Q1.1-FU4 — research-analyst.md has no Frontier-mode instructions; epistemologically incompatible with Frontier questions

**Question**: Does research-analyst.md have Frontier-mode instructions, or is it purely a Research/regulatory agent?
**Verdict**: WARNING
**Severity**: Medium

## Evidence

research-analyst.md (from description field):
> "Gathers evidence to stress-test a hypothesis or assumption against real-world data. Use for all Research mode questions (ID prefix R) that challenge business, technical, market, or regulatory assumptions."

### What research-analyst.md does

1. **Stress-tests existing assumptions** — its entire reasoning model starts from "what belief is the project relying on?" and tries to refute it
2. **Uses evidence gathering** (market data, regulatory text, analogues, datasets)
3. **Verdicts**: HEALTHY | WARNING | FAILURE | INCONCLUSIVE — all oriented toward pass/fail for an existing claim
4. **Threshold application**: compares evidence against constants.py thresholds — requires a defined threshold to check against

### What Frontier mode requires

From question-designer-bl2.md:
> "Remove constraints: 'What if this didn't have X limitation?' Draw analogies: 'What has succeeded in domain Y that could apply here?' Explore the maximum ambition: 'What is the most capable version of this?'"

Frontier example question:
> "FR1.1: What would a zero-latency memory retrieval architecture look like, and what prerequisites does the current system lack?"

### The incompatibility

| Dimension | research-analyst | Frontier mode needs |
|-----------|-----------------|---------------------|
| Epistemology | Falsification (refute or confirm claims) | Exploration (generate possibilities) |
| Starting condition | Existing assumption to test | No prior constraint |
| Evidence type | Empirical data, citations, thresholds | Analogies, thought experiments, architectural imagination |
| Verdict type | HEALTHY/WARNING/FAILURE/INCONCLUSIVE | No standard pass/fail — produces a possibility map |
| Output | Quantified finding with evidence citations | Conceptual exploration with speculative scenarios |

A research-analyst asked a Frontier question would try to find evidence to confirm/deny it and likely return INCONCLUSIVE (no empirical evidence exists for speculative possibilities). This is the wrong operation.

### No Frontier agent exists

There is no `frontier-analyst.md` in the template agents directory. The question-designer-bl2.md references `frontier-analyst` in its `**Method**:` enum but this agent does not exist.

## Mitigation Recommendation

1. **Create `frontier-analyst.md`** as a new agent (not a wrapper around research-analyst) with:
   - Epistemology: exploration and possibility generation, not falsification
   - Method: analogy mapping, constraint lifting, architectural imagination
   - Output format: possibility map with feasibility estimates, not HEALTHY/WARNING verdicts
   - Verdicts could be: FRONTIER_VIABLE, FRONTIER_BLOCKED, FRONTIER_PARTIAL (new taxonomy needed)
2. **Add routing entry** in mortar.md: `frontier` → `frontier-analyst`
3. **Interim workaround**: Document in mortar.md that Frontier questions should use research-analyst with explicit instruction to switch from falsification to exploratory mode — but flag this as degraded capability
