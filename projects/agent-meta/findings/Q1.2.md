# Finding: Q1.2 — All agents referenced in program.md have .md definitions; but agent-meta project lacks local .claude/agents/ directory for path resolution

**Question**: Do all agents referenced in program.md have corresponding .md definition files?
**Verdict**: WARNING
**Severity**: Medium

## Evidence

### Agent reference audit

Agents referenced in `template/program.md`:
- `.claude/agents/agent-auditor.md` — explicit path reference
- `.claude/agents/forge-check.md` — explicit path reference
- `.claude/agents/peer-reviewer.md` — explicit path reference
- `quantitative-analyst` — name reference
- `regulatory-researcher` — name reference
- `competitive-analyst` — name reference
- `benchmark-engineer` — name reference
- `hypothesis-generator` — name reference
- `synthesizer` — name reference
- `question-designer` — name reference

**All 10 have corresponding .md files** in `template/.claude/agents/`. No phantom agents.

### Path resolution concern

`program.md` specifies agent paths as `.claude/agents/{agent-name}.md` — relative to the project directory. This resolution works for:
- `projects/bl2/` — has its own `.claude/agents/` directory with all agent files ✓
- `template/` — has `.claude/agents/` directory ✓

**Does NOT work for:**
- `projects/agent-meta/` — **no `.claude/agents/` directory exists**

Verification: `ls projects/agent-meta/.claude/` → "No .claude/ directory in agent-meta project"

When program.md tells an agent "Act as forge-check per .claude/agents/forge-check.md", Claude Code resolves `.claude/agents/` relative to the current working directory. In the agent-meta session (cwd = `projects/agent-meta/`), the path `projects/agent-meta/.claude/agents/forge-check.md` does not exist.

### How this was mitigated in practice

The research loop launched for this campaign used full absolute paths in agent invocations (e.g., `C:/Users/trg16/Dev/Bricklayer2.0/template/.claude/agents/forge-check.md`) rather than the relative paths in program.md. This worked but required manual path correction.

### Impact assessment

For any project that:
1. Uses `program.md` as-is (without path correction)
2. Does NOT have a local `.claude/agents/` directory

All agent invocations in the background-agent `Spawn` instructions would fail to resolve the agent definition files, causing those agents to run as generic Claude behavior with no specialist prompt.

## Mitigation Recommendation

1. **For agent-meta specifically**: Create `.claude/agents/` symlinks or copies from `template/.claude/agents/` — OR update program.md to use absolute paths for this project.
2. **For the template system**: Add a check in project initialization (QUICKSTART.md step 6: "Copy agents: `cp -r ../template/.claude/agents/ .claude/agents/`") — this is already documented in CLAUDE.md master context, but agent-meta never ran this step.
3. **Long-term**: The template should auto-populate `.claude/agents/` during project initialization, or program.md should use a path variable (e.g., `$AGENTS_DIR`) rather than a hardcoded relative path.
