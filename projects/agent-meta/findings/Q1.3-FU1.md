# Finding: Q1.3-FU1 — Neither QUICKSTART.md nor CLAUDE.md mentions the planner; planner is undiscoverable in the documented workflow

**Question**: Does QUICKSTART.md or CLAUDE.md document the correct workflow for calling planner before question-designer? If not, is planner effectively an undiscoverable optional step?
**Verdict**: WARNING
**Severity**: Medium

## Evidence

### QUICKSTART.md — Step 5 (new project workflow)

```
### 5. Generate the question bank
Open Claude Code in the project directory and invoke the question-designer agent:
  Read project-brief.md, all files in docs/, constants.py, and simulate.py.
  Check for conflicts between sources. Then generate the initial question bank
  in questions.md following the format in program.md.
```

**planner is not mentioned.** Step 5 jumps directly to question-designer.

### QUICKSTART.md — Agent Quick Reference table

| Agent | When to invoke |
|-------|---------------|
| question-designer | Once at project init — generates questions.md |
| quantitative-analyst | D1/D5/D6 simulation questions |
| regulatory-researcher | D2 legal/compliance questions |
| competitive-analyst | D3 market/analogues questions |
| benchmark-engineer | When simulate.py calls a live service |
| hypothesis-generator | When questions.md is exhausted, or every 5 questions |
| synthesizer | At session end, before running analyze.py |

**planner is not in the table.**

### CLAUDE.md (Bricklayer2.0 master context) — Agent Reference table

```
| question-designer | .claude/agents/question-designer.md | Once at init — generates questions.md |
| quantitative-analyst | ...
| regulatory-researcher | ...
| competitive-analyst | ...
| benchmark-engineer | ...
| hypothesis-generator | ...
| synthesizer | ...
```

**planner is not in the CLAUDE.md agent reference either.**

### planner.md's own self-description

```
---
name: planner
description: Pre-campaign strategic planner. Runs once at campaign init — reads
project-brief.md and docs/, ranks research domains by risk, produces a campaign
targeting brief for question-designer, and estimates wave count. Call before
question-designer on any new project.
---
```

planner.md explicitly says "Call before question-designer on any new project." This is the correct intended workflow — but this instruction only appears in planner.md itself, not in any documentation the user would read during onboarding.

### Discoverability analysis

A user starting a new BrickLayer project would:
1. Read QUICKSTART.md — no mention of planner
2. Read CLAUDE.md — no mention of planner
3. Copy template, edit project-brief.md, invoke question-designer (Step 5)

The only way to discover planner is to:
- Read all agent .md files in `.claude/agents/` (not documented as required)
- Already know about it from prior experience
- Encounter a mention in synthesis.md of a prior campaign that used it

### Confirmation of Q1.3 root cause

This confirms the finding from Q1.3: the planner→question-designer interface is severed at both the technical level (question-designer doesn't read CAMPAIGN_PLAN.md) AND the documentation level (users are never told to run planner first). The planner is doubly dead: its output is unconsumed and its existence is undocumented.

## Impact

In practice: 100% of campaigns that follow the documented workflow skip the planner. Domain risk ranking, known landmines identification, and wave structure recommendations are never produced. question-designer independently reads all project files and makes its own prioritization — without the benefit of a dedicated risk-ranking analysis.

For simple projects this is a minor gap. For complex projects with many domains and prior campaigns, the planner's landmine detection (recurring failure patterns from prior Recall queries) would meaningfully reduce re-investigation of already-settled questions.

## Mitigation Recommendation

1. **Add planner to QUICKSTART.md Step 5** (insert before question-designer):
   ```
   ### 5a. (Optional but recommended) Run the planner
   Act as the planner agent defined in .claude/agents/planner.md.
   Read project-brief.md, all docs/, constants.py, and simulate.py.
   Write CAMPAIGN_PLAN.md.
   ```

2. **Add planner to QUICKSTART.md's Agent Quick Reference table**:
   ```
   | planner | Once at project init, BEFORE question-designer |
   ```

3. **Add planner to CLAUDE.md agent reference table** with the same note.

4. **Mark as optional but time-saving** in documentation — planner adds ~2 minutes of context but may save multiple waves of questions by flagging known landmines.
