# Q10.2 — _bounded_glob symlink safety

**Verdict**: HEALTHY
**Commit**: wave-10

## Question
Can a symlink loop in a project directory cause `_bounded_glob` to recurse infinitely?

## Finding
`_bounded_glob` uses `os.walk(root)` with no `followlinks` argument. Python's `os.walk` defaults
to `followlinks=False`, meaning symlinked directories are yielded once as directory entries but
are never descended into. A symlink pointing to a parent cannot cause an infinite loop.

```python
def _bounded_glob(root: Path, pattern: str, max_depth: int = 4) -> bool:
    for dirpath, dirnames, filenames in os.walk(root):  # followlinks=False (default)
        ...
```

This is correct and safe behavior. Additionally, the depth guard (`dirnames.clear()` at
`depth >= max_depth`) provides a second layer of protection against any future change that
might enable symlink following.

## Verification
```
grep -n "followlinks" onboard.py  -> (no output — followlinks never set to True)
```
`os.walk` documentation confirms: "If optional argument topdown is True and followlinks is
False (the default), symlinks to directories are not followed."
