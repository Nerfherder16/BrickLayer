# Q10.1 — template/questions.md parses correctly in table format

**Verdict**: HEALTHY
**Commit**: wave-10

## Question
Does a new project copied from the template have its questions visible in the dashboard?

## Finding
`template/questions.md` uses the legacy table format (`| ID | Status | Question |` rows under
`## Domain N — ...` headers). After the Q5.7 fix, `parse_questions()` supports both block and
table formats. Called with `parse_questions(Path('template'))`, all 17 placeholder questions are
returned with correct ids (1.1–6.3), PENDING status, and non-empty titles.

The dashboard will display all questions for any new project copied from the template immediately,
with no extra setup step required.

## Verification
```
parse_questions(Path('template')) -> 17 questions
  1.1 PENDING At what [parameter X] does the primary metric drop below...
  1.2 PENDING What [volume / adoption / usage] is needed for...
  1.3 PENDING If [critical dependency] exits, does the system collapse?...
  ...
```

Note: The earlier probe erroneously passed the file path (`Path('template/questions.md')`) instead
of the project dir (`Path('template')`), producing 0. This was a test error, not a code bug.
`parse_questions` correctly expects a project directory, not a file path.
