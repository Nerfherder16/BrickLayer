# Q6.8 — Add API key warning in recall/simulate.py

**Verdict**: HEALTHY
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`recall/simulate.py` had no warning when `BASE_URL` pointed at a live service but `API_KEY` was still the default placeholder `"recall-admin-key-change-me"`. A misconfigured project.json would silently send the wrong key and get 401s with no guidance.

**Fix applied** (after `init_project()` call in `__main__` block):

```python
_DEFAULT_KEY = "recall-admin-key-change-me"
if BASE_URL not in ("none", "None", "") and API_KEY == _DEFAULT_KEY:
    print(
        "Warning: using default API key — set api_key in project.json before targeting a live service.",
        file=sys.stderr,
    )
```

The check runs after `init_project()` so it reflects the resolved values from `project.json`, not the module-level defaults. The bricklayer project has `target_live_url: none` so the warning never fires during normal campaign runs.

## Risk

None. The warning is printed to stderr only, does not interrupt execution, and only fires when both conditions are true simultaneously.
