# Q6.7 — Fix dashboard start.sh 0.0.0.0 binding

**Verdict**: HEALTHY
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`dashboard/start.sh` line 16 previously hardcoded `--host 0.0.0.0`, making the dashboard accessible to all LAN clients regardless of intent.

**Fix applied:**

```bash
HOST=${DASHBOARD_HOST:-127.0.0.1}
cd "$SCRIPT_DIR/backend" && AUTOSEARCH_PROJECT="$PROJECT" uvicorn main:app --host "$HOST" --port 8100 --reload &
```

Default is now `127.0.0.1`. To expose on LAN intentionally:
```bash
DASHBOARD_HOST=0.0.0.0 bash start.sh /path/to/project
```

## Evidence

```
$ grep "HOST\|uvicorn" dashboard/start.sh
HOST=${DASHBOARD_HOST:-127.0.0.1}
cd "$SCRIPT_DIR/backend" && AUTOSEARCH_PROJECT="$PROJECT" uvicorn main:app --host "$HOST" --port 8100 --reload &
```

## Risk

None. The DASHBOARD_HOST env var provides full opt-in LAN exposure when needed. Combined with the CORS restriction from Q4.1, the dashboard is now localhost-only by default.
