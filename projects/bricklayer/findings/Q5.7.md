# Q5.7 — Fix dashboard parse_questions + add_question format mismatch

**Verdict**: HEALTHY
**Severity**: Fixed

## What Was Found

`parse_questions()` only handled the legacy pipe-table format (`| ID | Status | Question |`).
Current `questions.md` uses the block format (`## Q{N}.{M} [CATEGORY] Title`), causing
`/api/questions` to return 0 questions for all current projects.

`add_question()` generated IDs as `len(existing_ids) + 1`, which collides after any
deletion or gap in the question sequence. It also wrote pipe-table rows into block-format files.

## Fix Applied

- `parse_questions()` now auto-detects format: if any `## Q{N}.{M}` header is present, block
  parsing is used; otherwise legacy table parsing runs as fallback.
- Block parser extracts id, title, category, domain, mode, hypothesis, and status
  (`**Status**:` field; defaults to PENDING if absent).
- `add_question()` now writes block format entries with `**Mode**: custom`, `**Target**: TBD`,
  `**Hypothesis**:`, and `**Status**: PENDING`.
- ID generation changed from `len(existing_ids) + 1` to `max(existing_numeric_suffixes) + 1`,
  preventing collisions after deletions.
