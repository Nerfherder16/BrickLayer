# Q2.2 — Silent swallow in bricklayer_launcher load_projects

**Verdict**: FAILURE
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`load_projects()` in `bricklayer_launcher.pyw` line 53:

```python
try:
    cfg = json.loads(cfg_path.read_text(encoding="utf-8"))
    cfg["_dir"] = str(d)
    projects.append(cfg)
except Exception:
    pass   # ← project silently disappears from UI
```

A corrupt or partially-written `project.json` causes the project to vanish from the launcher with no error message, no log, and no indication to the user that anything went wrong. The user sees fewer projects than expected with no explanation.

## Evidence

```
$ grep -n "except Exception" bricklayer_launcher.pyw
53:            except Exception:
```

Context confirms the `pass` is the only handler — no `messagebox`, `print`, or `logging` call follows.

## Risk

Medium. The launcher is a UI tool so silent failures are especially bad — the user may spend time debugging why their project doesn't appear, not knowing the JSON is corrupt. Write failures to `project.json` (e.g., from a crashed `onboard.py` run) would trigger this silently.

## Recommended Action

Q4.2 dispatches security-hardener to replace `pass` with at minimum a `print(f"Warning: could not load {cfg_path}: {e}", file=sys.stderr)` or a tkinter `messagebox.showwarning()`.
