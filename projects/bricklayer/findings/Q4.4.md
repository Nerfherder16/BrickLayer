# Q4.4 — Type Annotation Migration (Optional → X|None)

**Question**: Does `dashboard/backend/main.py` use legacy `typing.Optional` that should be migrated to the modern `X | None` union syntax?

**Verdict**: HEALTHY

**Severity**: Low (style/modernization — no runtime impact)

---

## What Was Found

`dashboard/backend/main.py` imported `from typing import Optional` and used `Optional[str]` in 8 locations:
- `get_project_path(project: Optional[str] = None)`
- `get_status(project: Optional[str] = Query(None))`
- `get_questions(project: Optional[str] = Query(None))`
- `add_question(..., project: Optional[str] = Query(None))`
- `get_findings(project: Optional[str] = Query(None))`
- `get_finding(..., project: Optional[str] = Query(None))`
- `correct_finding(..., project: Optional[str] = Query(None))`
- `get_results(project: Optional[str] = Query(None))`

The three helper functions (`parse_questions`, `parse_results`, `parse_findings_index`) already had return type annotations and required no changes.

---

## Changes Applied

- Removed `from typing import Optional` import line
- Replaced all 8 `Optional[str]` usages with `str | None`
- No logic changes, no reformatting

---
