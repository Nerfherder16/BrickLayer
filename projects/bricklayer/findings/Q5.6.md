# Q5.6 — Template baseline returns HEALTHY after revenue model fix

**Verdict**: HEALTHY
**Commit**: 2b27996

## Question
Does the unmodified template return `verdict: HEALTHY` on baseline parameters?

## Finding
The original placeholder `revenue = monthly_volume * 0.001` produced ~$175/mo revenue
against $30,000/mo ops cost, causing treasury collapse and `verdict: FAILURE` at month 1
(primary_metric: -0.34, below FAILURE_THRESHOLD = 6).

## Fix Applied
Replaced the placeholder with an ops-cost-anchored formula:
```python
ops_cost = get_monthly_ops_cost(units)
revenue = ops_cost * 1.5  # TODO: replace with project-specific revenue model
                           # This default ensures the baseline is HEALTHY before customization
```

`ops_cost` is now computed before `revenue` (the original ordering would have raised
`F821 undefined name` at runtime — caught by lint hook).

## Verification
```
primary_metric: 15.25
verdict: HEALTHY
```

## Impact
New projects copied from the template will see a HEALTHY baseline out of the box.
Developers are prompted by the TODO comment to replace the formula with their actual
revenue model before running stress tests.
