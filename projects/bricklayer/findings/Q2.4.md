# Q2.4 — Missing type annotations in dashboard backend

**Verdict**: INCONCLUSIVE
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`mypy` is not installed in the active Python environment:

```
$ python -m mypy dashboard/backend/main.py --ignore-missing-imports
No module named mypy
```

Static type checking cannot be performed without installing mypy. However, from manual inspection:

- `dashboard/backend/main.py` line 7: `from typing import Optional` (legacy Python 3.9 style)
- Several helper functions (`parse_questions`, `parse_results`, `parse_findings_index`) lack explicit return type annotations
- `Optional[str]` is used throughout instead of the modern `str | None` syntax

## Evidence


## Risk

Low-medium. Missing annotations won't cause runtime failures, but they make the codebase harder to maintain and prevent static analysis from catching type errors. The legacy `Optional` import is cosmetic but signals the code hasn't been updated to Python 3.10+ style.

## Recommended Action

Install mypy (`pip install mypy`) and re-run Q2.4, or proceed directly with Q4.4 which migrates to modern type annotations. The agent fix is valid regardless of whether mypy can currently run.
