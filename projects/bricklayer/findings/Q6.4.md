# Q6.4 — Dashboard backend still binds to 0.0.0.0

**Verdict**: WARNING
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`dashboard/start.sh` line 16:
```bash
uvicorn main:app --host 0.0.0.0 --port 8100 --reload
```

The Q4.1 fix restricted CORS to `localhost:3100`, which prevents browser-based cross-origin attacks. However, direct non-browser requests (curl, httpx, scripts) from any LAN host can still reach all API endpoints without CORS restriction, including the mutation routes:
- `POST /api/questions`
- `POST /api/findings/{id}/correct`

The path traversal and CORS fixes (Q4.1) significantly reduced the attack surface, but the 0.0.0.0 binding means the API is still LAN-accessible to non-browser clients.

## Risk

Low for a homelab tool. The remaining risk requires a LAN attacker who knows the port (8100) and the API structure. The path traversal guards prevent file escapes. The CORS restriction prevents browser-based CSRF.

## Recommended Action

Change `start.sh` to `--host 127.0.0.1` by default, and add a `--lan` flag for intentional LAN exposure:
```bash
HOST=${DASHBOARD_HOST:-127.0.0.1}
uvicorn main:app --host $HOST --port 8100 --reload
```
