## Notable Implementation Detail

`template/simulate.py` reassigns `sys.stdout` at module level via
`sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding="utf-8")`.
This corrupts pytest's stdout capture object and causes an `I/O operation on closed file`
crash during teardown. The test helper `_import_simulate()` patches `sys.stdout` to a
throwaway `BytesIO`-backed wrapper before import, then restores the real stdout immediately
after — making the module-level reassignment a no-op from pytest's perspective.
This issue should be noted as a secondary finding: simulate.py's stdout replacement
is incompatible with any test runner that wraps stdout.
