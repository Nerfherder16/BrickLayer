# Q8.2 — bricklayer_launcher.pyw stderr visibility

**Verdict**: HEALTHY
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

The `.pyw` extension means `pythonw.exe` (no console window) — a concern because stderr from in-process code would be silently dropped. However, `_open_onboard()` in `bricklayer_launcher.pyw` launches `onboard.py` in a **new terminal window**:

```python
# Windows Terminal path
subprocess.Popen(f'wt.exe new-tab -- python "{onboard}"', shell=True)

# Fallback: cmd.exe with /k (keep open after completion)
subprocess.Popen(f'start cmd.exe /k "python \\"{onboard}\\""', shell=True)
```

The new terminal window has its own console so stderr is fully visible. The Q4.2 warnings introduced in `onboard.py` will surface normally.

## Risk

None. The launcher correctly spawns onboard.py in a real console window.
