# Q2.3 — Hardcoded absolute path in dashboard get_projects

**Verdict**: FAILURE
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`dashboard/backend/main.py` line 286:

```python
def get_projects():
    base = Path("C:/Users/trg16/Dev/autosearch")
```

Also line 28 (env var with hardcoded default):
```python
os.getenv("AUTOSEARCH_PROJECT", "C:/Users/trg16/Dev/autosearch/adbp")
```

The `get_projects()` base path has **no environment variable fallback at all** — it is a bare hardcoded string. On any other machine, or if the repo is moved or cloned to a different path, this endpoint returns an empty list with no error.

## Evidence

```
$ grep -n "C:/Users/trg16" dashboard/backend/main.py

```

## Risk

High for portability. The dashboard is completely non-functional on any machine other than Tim's dev box. Any attempt to share, deploy, or move the repo breaks the `/api/projects` endpoint silently.

## Recommended Action

Q4.1 dispatches security-hardener to replace line 286 with:
```python
base = Path(os.getenv("AUTOSEARCH_BASE", str(Path(__file__).parent.parent.parent)))
```
and update line 28's default to use the same env var.
