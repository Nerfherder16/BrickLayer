# Q4.2 — Security: Silent Exception Swallows Fixed

**Verdict**: HEALTHY
**Agent**: security-hardener
**Commit**: b2a18a3
**Date**: 2026-03-13

---

## Summary

Four silent exception swallows were identified and corrected across two files. All were bare
`except Exception: pass` patterns that hid errors from the operator. No business logic was
changed. One benign clipboard swallow (onboard.py line 621) was intentionally left untouched
as documented in the task spec.

---

## Fixes Applied

### onboard.py — Line 241 (CRITICAL)

**Before:**
```python
except Exception:
    stack.append("Node.js")   # appended even when JSON parse failed
```

**After:**
```python
except Exception as e:
    print(f"Warning: could not parse package.json: {e}", file=sys.stderr)
    # Do not append Node.js — detection failed
```

**Impact**: Previously, a corrupt or unreadable `package.json` caused "Node.js" to be
silently appended to the detected stack — producing a wrong stack label with no indication
of failure. Now the error is surfaced to stderr and the append is skipped.

---

### onboard.py — Line 253

**Before:**
```python
except Exception:
    pass
```

**After:**
```python
except Exception as e:
    print(f"Warning: could not parse Cargo.toml: {e}", file=sys.stderr)
```

**Impact**: `Cargo.toml` read failures were silently swallowed. Framework sub-detection
(Anchor, Solana) would be skipped with no diagnostic output.

---

### onboard.py — Line 343

**Before:**
```python
except Exception:
    pass
```

**After:**
```python
except Exception as e:
    print(f"Warning: could not read doc file {doc.name}: {e}", file=sys.stderr)
```

**Impact**: Doc file read failures during Scout prompt construction were silently skipped.
Now the operator sees which file could not be read.

---

### bricklayer_launcher.pyw — Line 53

**Before:**
```python
except Exception:
    pass
```

**After:**
```python
except Exception as e:
    import sys
    print(f"Warning: could not load project from {cfg_path}: {e}", file=sys.stderr)
```

**Impact**: A corrupt `project.json` was silently dropped from the launcher UI with no
feedback. Now the path and error are printed to stderr so the operator can diagnose it.

## Not Changed

- **onboard.py line 621** (clipboard): `except Exception: pass` retained intentionally.
  Clipboard operations legitimately fail on headless systems and do not affect program
  correctness. This is a well-understood benign swallow.
