# Q6.6 — Fix unbounded rglob calls in detect_stack

**Verdict**: HEALTHY
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`detect_stack()` in `onboard.py` previously used 4 unbounded `rglob()` calls with no depth limit and no exclusion of `.git`, `node_modules`, `vendor`, `.venv`. On a large repo each call walked the entire filesystem tree.

**Fix applied:**

Added `_EXCLUDE_DIRS` constant and `_bounded_glob()` helper before `detect_stack()`:

```python
_EXCLUDE_DIRS = {".git", "node_modules", "vendor", ".venv", "__pycache__", ".mypy_cache"}

def _bounded_glob(root: Path, pattern: str, max_depth: int = 4) -> bool:
    for dirpath, dirnames, filenames in os.walk(root):
        depth = len(Path(dirpath).relative_to(root).parts)
        if depth >= max_depth:
            dirnames.clear()
        else:
            dirnames[:] = [d for d in dirnames if d not in _EXCLUDE_DIRS]
        for fname in filenames:
            if Path(fname).match(pattern):
                return True
    return False
```

All 4 rglob calls replaced:
- `any(repo_path.rglob("build.gradle.kts"))` → `_bounded_glob(repo_path, "build.gradle.kts")`
- `any(repo_path.rglob("*.kt"))` → `_bounded_glob(repo_path, "*.kt")`
- `any(repo_path.rglob("AndroidManifest.xml"))` → `_bounded_glob(repo_path, "AndroidManifest.xml")`
- `any(repo_path.rglob("*.swift"))` → `_bounded_glob(repo_path, "*.swift")`
- `any(repo_path.rglob("*.tf"))` → `_bounded_glob(repo_path, "*.tf")`
