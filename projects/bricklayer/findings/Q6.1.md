# Q6.1 — Hardcoded API_KEY in recall/simulate.py

**Verdict**: WARNING
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`recall/simulate.py` line 56:
```python
API_KEY = "recall-admin-key-change-me"
```

The key is overridden from `project.json` when an `api_key` field is present (line 112: `cfg.get("api_key", API_KEY)`). Projects with no live API (`target_live_url: none`) never send this key. However, the placeholder is committed to source control and would be used verbatim if a project.json omits the `api_key` field and the target URL is a live service.

## Risk

Low for current projects (bricklayer has no live target). Medium if a future project with a live API forgets to set `api_key` in project.json — the wrong key is sent silently with no error until the first HTTP 401.

## Recommended Action

- Add a warning when `BASE_URL != "none"` and `API_KEY == "recall-admin-key-change-me"`: `print("Warning: using default API key — set api_key in project.json", file=sys.stderr)`
- Alternatively, require `api_key` in project.json for performance/live-service modes.
