# Q5.1 — prepare.md runner path is wrong

**Verdict**: FAILURE
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

`projects/bricklayer/prepare.md` instructs the user to run:
```bash
python simulate.py --project bricklayer --list
```
from `C:/Users/trg16/Dev/autosearch`. That file does not exist at the root. The actual runner is `recall/simulate.py`.

```
$ ls C:/Users/trg16/Dev/autosearch/simulate.py
ls: cannot access 'simulate.py': No such file or directory

$ ls C:/Users/trg16/Dev/autosearch/recall/simulate.py
C:/Users/trg16/Dev/autosearch/recall/simulate.py  ✓
```

`prepare.md` lines 87–89:
```
python simulate.py --project bricklayer --list
python simulate.py --project bricklayer --question Q1.1
python simulate.py --project bricklayer --campaign
```

All three commands fail on a fresh session with `python: can't open file 'simulate.py': [Errno 2] No such file or directory`.

## Root Cause

`prepare.md` was written assuming a root-level dispatcher would be created. The dispatcher was never built. The working runner lives at `recall/simulate.py` but `prepare.md` was never updated.

## Risk

High. Every new BrickLayer session starts by reading `prepare.md`. The first command a new user tries immediately fails with a confusing error. This session ran all questions manually as a workaround.

## Recommended Action

Fix `prepare.md` to reference `recall/simulate.py`:
```bash
python recall/simulate.py --project bricklayer --list
```
Or create a thin root-level shim `simulate.py` that delegates to `recall/simulate.py`.
