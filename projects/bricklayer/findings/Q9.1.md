# Q9.1 — onboard.py does not copy analyze.py to new projects

**Verdict**: FAILURE
**Commit**: (fixed by Q9.3)

## Question
Does `onboard.py` copy `analyze.py` and create `reports/` for new projects?

## Finding
`onboard.py` created `findings/`, `docs/`, `results.tsv`, `prepare.md`, and `questions.md` during project
init but never copied `template/analyze.py` to the new project directory, and never created `reports/`.
Any project onboarded via the launcher or `onboard.py` CLI would fail at `python analyze.py` with a
`FileNotFoundError` because the file simply didn't exist.

## Verification
Before fix — project dir contents had no `analyze.py`:
```
findings/  docs/  results.tsv  prepare.md  project-brief.md  questions.md  constants.py  simulate.py
```
`grep -n "analyze\|reports" onboard.py` returned only comment lines.

## Impact
All projects created through the normal onboarding workflow lacked PDF report generation. The gap was
masked for bricklayer because it was created manually (not via onboard.py).
