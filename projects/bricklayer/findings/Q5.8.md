# Q5.8 — Fix docs prompt injection delimiters in run_scout

**Verdict**: HEALTHY
**Severity**: Fixed

## What Was Found

`run_scout()` in `onboard.py` injected docs file content directly into the Claude prompt
with no delimiters:

```python
{f"**Supporting docs content**:{docs_content}" if docs_content else "..."}
```

A malicious or accidentally crafted doc file could inject instructions that override
Scout's behavior or exfiltrate other project files, since the content was indistinguishable
from the surrounding prompt instructions.

## Fix Applied

Wrapped docs content in XML tags:

```python
{f"<docs>{docs_content}\n</docs>" if docs_content else "**Docs folder**: empty — scan the codebase only"}
```

XML delimiters clearly separate untrusted doc content from the instruction context,
preventing injected text from being interpreted as instructions.

## Verification

```
grep -A2 "docs_content" onboard.py | grep "<docs>"
```

Line 361 of `onboard.py` now contains `<docs>{docs_content}\n</docs>`.
