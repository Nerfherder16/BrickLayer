# Q1.2 — Template simulate.py runs to HEALTHY verdict

**Verdict**: FAILURE
**Timestamp**: 2026-03-13T00:00:00Z

## Finding

The unmodified `template/simulate.py` produces `verdict: FAILURE` instead of `verdict: HEALTHY`. The root cause is an unfilled TODO in the revenue formula:

```python
revenue = monthly_volume * 0.001  # TODO: replace with real revenue model
```

With baseline parameters (500 units × 350 volume × 0.001 = $175/mo revenue vs $30,000/mo ops cost), the treasury burns at ~$29,825/mo and goes negative around month 68. The final primary metric is -0.34, which falls below `FAILURE_THRESHOLD = 6`.

## Evidence

```
$ python template/simulate.py 2>&1 | grep -E "^verdict:|^primary_metric:"
primary_metric: -0.34
verdict: FAILURE
```

## Root Cause

`template/simulate.py` was shipped with a placeholder revenue model. The scenario parameters are calibrated for HEALTHY but the revenue multiplier (`* 0.001`) was never replaced. A project copied from this template would immediately return FAILURE before the agent makes any changes, making it impossible to distinguish a real failure from the scaffolding failure.

## Risk

High. Any new project started from this template has a broken baseline — the agent loop starts from a FAILURE state, which invalidates all stress-test comparisons. A new developer using the template would assume they've broken something when the template itself is broken.

## Recommended Action

Replace `revenue = monthly_volume * 0.001` in `template/simulate.py` with a self-sustaining formula (e.g., `revenue = ops_cost * 1.2`) for the baseline HEALTHY case, with a comment directing project developers to replace it with the real model.
