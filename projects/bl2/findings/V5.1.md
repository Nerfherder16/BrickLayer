## Known Campaign-Specific Discrepancy

The manually-maintained results.tsv in this self-audit campaign uses the BL 1.x format (`commit\tquestion_id\tverdict\t...`), while `update_results_tsv()` writes `qid\tverdict\t...`. `get_question_status()` matches `parts[0] == qid`, so it reads `update_results_tsv()`-written rows correctly. The manual rows in this campaign's TSV produce PENDING for all questions — functionally acceptable because status was tracked via questions.md for this manual campaign. In a live BL 2.0 run driven by `campaign.py`, `update_results_tsv()` writes the correct format.
