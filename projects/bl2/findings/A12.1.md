# Finding: A12.1 — C-04 follow-up trigger excludes NON_COMPLIANT

**Question**: C-04 adaptive follow-up trigger excludes NON_COMPLIANT
**Verdict**: NON_COMPLIANT
**Severity**: High
**Mode**: code_audit
**Operational Mode**: audit

## Audit Check

Verified the exact guard condition in `campaign.py` at line 106 (C-04 adaptive follow-up
trigger) and the secondary guard inside `followup.py` at line 251 (`generate_followup`
public API). Both independently check that verdict is in `("FAILURE", "WARNING")` before
generating drill-down sub-questions. `NON_COMPLIANT` is not present in either allowlist.

## Evidence

`campaign.py` line 106:
```python
# C-04: adaptive follow-up drill-down on FAILURE/WARNING
if result.get("verdict") in ("FAILURE", "WARNING"):
    from bl.followup import generate_followup
    followup_ids = generate_followup(question, result, cfg.questions_md)
```

`followup.py` line 250-252:
```python
verdict = result.get("verdict", "")
if verdict not in ("FAILURE", "WARNING"):
    return []
```

The guard appears in both places. Even if the outer `campaign.py` check were fixed to
include `NON_COMPLIANT`, `generate_followup` would still reject it at line 251 with an
early return of `[]`. Both layers must be updated.

The docstring for `generate_followup` (line 239) also explicitly states:
> "Generate drill-down sub-questions after a FAILURE or WARNING verdict."

confirming that `NON_COMPLIANT` was never considered when the function was written.

## Impact

BL 2.0 audit-mode agents return `NON_COMPLIANT` for compliance violations — this is the
primary verdict type for the entire audit campaign (A-series questions). Because neither
`campaign.py` nor `followup.py` recognises `NON_COMPLIANT` as a drill-down trigger:

1. No sub-questions are ever generated for any audit finding, regardless of severity.
2. C-04 drill-down is completely non-functional for audit mode.
3. Violations that need deeper investigation (e.g., partial compliance, ambiguous scope)
   cannot be automatically followed up on.
4. The campaign silently skips follow-up without any warning — `result["followup_questions"]`
   is never set, and no error is logged.

## Verdict: NON_COMPLIANT

The C-04 adaptive follow-up mechanism is broken for audit-mode campaigns. `NON_COMPLIANT`
must be added to the verdict allowlist in both `campaign.py:106` and `followup.py:251` for
drill-down to function in audit mode.
