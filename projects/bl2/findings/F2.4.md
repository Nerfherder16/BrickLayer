# Finding: F2.4 — Replace verdict-diff condition with identity check in campaign.py

**Question**: Fix D9: `if healed_result.get("verdict") != result.get("verdict")` in `bl/campaign.py` silently discarded heal loop results when the verdict was unchanged, losing accumulated diagnostic context.
**Verdict**: FIXED
**Severity**: High
**Mode**: fix
**Source finding**: D9

## Changes Made

Changed `if healed_result.get("verdict") != result.get("verdict"):` to `if healed_result is not result:` in `bl/campaign.py` lines 136-138.

## Correctness Proof

`run_heal_loop()` returns the same `initial_result` object (by reference) in two cases:
1. `_is_enabled()` is False → returns `initial_result` at line 142
2. Verdict is not FAILURE/DIAGNOSIS_COMPLETE → returns `initial_result` at line 151

In all other cases (loop ran and produced output), a different dict object is returned (`current_result` or `fix_result`). The `is not` identity check correctly distinguishes "loop ran" from "loop skipped" without relying on verdict comparison.

## Verification

Static analysis of `bl/healloop.py` lines 141-159 confirms `initial_result` is returned by reference (not copied) when skipped. After the fix, any heal loop run that produces output will always propagate to campaign.py's result, preserving enriched context in session-context.md and the final JSON output.
