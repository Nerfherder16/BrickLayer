# Finding: A4 — Recall bridge graceful-fail (checklist item A4)

**Question**: Every httpx call in `bl/recall_bridge.py` must be wrapped in try/except. No bare httpx call should exist without exception handling.
**Verdict**: COMPLIANT
**Severity**: Info
**Mode**: agent

## Evidence

`bl/recall_bridge.py` contains 3 httpx calls:

1. `_is_available()` line 30:
   ```python
   try:
       r = httpx.get(f"{RECALL_BASE}/health", timeout=2.0)
       return r.status_code == 200
   except Exception:
       return False
   ```
   ✓ Wrapped in try/except Exception

2. `store_finding()` line 84:
   ```python
   try:
       r = httpx.post(f"{RECALL_BASE}/memories", json=payload, timeout=RECALL_TIMEOUT)
       return r.status_code in (200, 201)
   except Exception:
       return False
   ```
   ✓ Wrapped in try/except Exception

3. `search_before_question()` line 119:
   ```python
   try:
       r = httpx.get(f"{RECALL_BASE}/memories/search", params=params, timeout=RECALL_TIMEOUT)
       if r.status_code != 200:
           return ""
       ...
   except Exception:
       return ""
   ```
   ✓ Wrapped in try/except Exception

All 3 httpx calls have exception handling. The top-level import of httpx is also guarded:
```python
try:
    import httpx
    _HTTPX_AVAILABLE = True
except ImportError:
    _HTTPX_AVAILABLE = False
```

## Mitigation Recommendation

No action required. All httpx calls are properly wrapped.
