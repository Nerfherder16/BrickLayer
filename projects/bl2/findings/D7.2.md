# Finding: D7.2 — _is_leaf_id() treats all BL 2.0 question IDs as leaf nodes

**Question**: Does _is_leaf_id() return True for all BL 2.0 question IDs, blocking C-04 adaptive drill-down for the entire BL 2.0 campaign?
**Verdict**: FAILURE
**Severity**: Medium
**Mode**: diagnose
**Operational Mode**: diagnose

## Root Cause

`bl/followup.py` `_is_leaf_id()` lines 37-48 (before fix):

```python
stripped = qid
if stripped.startswith("QG"):
    stripped = stripped[2:]
elif stripped.startswith("Q"):
    stripped = stripped[1:]
else:
    # Unknown format — treat as leaf to be safe
    return True        # ← ALL BL 2.0 IDs hit this branch

parts = stripped.split(".")
return len(parts) >= 3
```

BL 2.0 question IDs ("D1", "D5.1", "F4.3", "A6.1", "M2.1", "V3.1"...) don't start with Q or QG. Every BL 2.0 ID hits the `else: return True` branch.

## Impact

`generate_followup()` calls `_is_leaf_id(qid)` before attempting drill-down. If True, it immediately returns without generating sub-questions. Since all BL 2.0 IDs return True:

- FAILURE verdict on D1 → `_is_leaf_id("D1")` → True → no followup generated
- WARNING verdict on D7 → `_is_leaf_id("D7")` → True → no followup generated
- C-04 adaptive follow-up has been 100% non-functional for this entire BL 2.0 campaign

## Fix Specification

- **File**: `bl/followup.py`, `_is_leaf_id()`
- **Change**: Replace `else: return True` with BL 2.0 dot-count heuristic: `parts = qid.split("."); return len(parts) >= 3`
- **Verification**: `_is_leaf_id("D5.1")` → False; `_is_leaf_id("D5.1.1")` → True
