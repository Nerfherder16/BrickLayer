## Change Made

### bl/synthesizer.py — parse_recommendation()

```python
# Before:
def parse_recommendation(synthesis_text: str) -> str:
    for line in synthesis_text.splitlines():
        upper = line.upper()
        if "STOP" in upper:
            return "STOP"
        if "PIVOT" in upper:
            return "PIVOT"
        if "CONTINUE" in upper:
            return "CONTINUE"
    return "CONTINUE"

# After:
def parse_recommendation(synthesis_text: str) -> str:
    lines = synthesis_text.splitlines()

    # F10.2: locate the Recommended Next Action section, scan only lines after it
    section_start = None
    for i, line in enumerate(lines):
        if "recommended next action" in line.lower():
            section_start = i + 1
            break

    scan_lines = lines[section_start:] if section_start is not None else lines

    for line in scan_lines:
        upper = line.upper()
        if "STOP" in upper:
            return "STOP"
        if "PIVOT" in upper:
            return "PIVOT"
        if "CONTINUE" in upper:
            return "CONTINUE"
    return "CONTINUE"
```

```
synthesis = """

# After fix:

```

## Fallback Behavior

If "## Recommended Next Action" section header is absent (e.g., malformed synthesis), `section_start` remains `None` and `scan_lines` falls back to all lines. This preserves the original behavior for synthesis output that doesn't follow the expected format.

## Impact

Campaigns with dead ends documented in synthesis will no longer be prematurely terminated. The STOP signal is now only triggered when the LLM explicitly writes STOP/PIVOT/CONTINUE in the designated recommendation section.
