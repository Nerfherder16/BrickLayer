# Finding: A20.1 — QuestionQueue STATUS_COLORS and filter dropdown missing HEAL_EXHAUSTED and all BL 2.0 terminal statuses

**Question**: Does the dashboard QuestionQueue component display HEAL_EXHAUSTED questions with a distinct visual status — or does it render them with default fallback styling?
**Verdict**: NON_COMPLIANT
**Severity**: Low

## Evidence

**`STATUS_COLORS` map** (`dashboard/frontend/src/components/QuestionQueue.tsx` lines 9-14):
```typescript
const STATUS_COLORS: Record<string, string> = {
  PENDING: "bg-[#374151] text-[#9ca3af]",
  DONE: "bg-[#064e3b] text-[#34d399]",
  INCONCLUSIVE: "bg-[#451a03] text-[#f59e0b]",
  IN_PROGRESS: "bg-[#1e3a5f] text-[#38bdf8]",
};
```

**Status badge render** (line 194):
```typescript
className={`... ${STATUS_COLORS[q.status] ?? "bg-[#374151] text-[#9ca3af]"}`}
```

Fallback is the same color as `PENDING` (gray) — HEAL_EXHAUSTED questions are visually indistinguishable from PENDING questions.

**Filter dropdown** (lines 152-156):
```tsx
<option value="PENDING">PENDING</option>
<option value="DONE">DONE</option>
<option value="INCONCLUSIVE">INCONCLUSIVE</option>
<option value="IN_PROGRESS">IN_PROGRESS</option>
```
No HEAL_EXHAUSTED option — operators cannot filter to show only exhausted questions.

**Missing BL 2.0 statuses** (all absent from STATUS_COLORS and filter):
- `HEAL_EXHAUSTED` — requires human intervention (highest priority gap)
- `FIXED`, `FIX_FAILED`, `DIAGNOSIS_COMPLETE`, `COMPLIANT`, `NON_COMPLIANT` — all BL 2.0 terminal statuses
- `PROMISING`, `BLOCKED`, `PENDING_EXTERNAL`

## Fix Specification

**Target file**: `dashboard/frontend/src/components/QuestionQueue.tsx`
**Target location**: Lines 9-14 (STATUS_COLORS) and lines 152-156 (filter dropdown)

**Concrete edit**:
```typescript
// STATUS_COLORS — add BL 2.0 statuses
const STATUS_COLORS: Record<string, string> = {
  PENDING: "bg-[#374151] text-[#9ca3af]",
  DONE: "bg-[#064e3b] text-[#34d399]",
  INCONCLUSIVE: "bg-[#451a03] text-[#f59e0b]",
  IN_PROGRESS: "bg-[#1e3a5f] text-[#38bdf8]",

  FIXED: "bg-[#064e3b] text-[#34d399]",                // same as DONE (success)
  FIX_FAILED: "bg-[#4c0519] text-[#f43f5e]",           // rose-red — active failure
  DIAGNOSIS_COMPLETE: "bg-[#1e3a5f] text-[#38bdf8]",   // blue — ready for fix
  COMPLIANT: "bg-[#064e3b] text-[#34d399]",            // green — pass
  NON_COMPLIANT: "bg-[#451a03] text-[#f59e0b]",        // amber — needs fix
  BLOCKED: "bg-[#4a1d96] text-[#c4b5fd]",              // purple — blocked
};

// Filter dropdown — add HEAL_EXHAUSTED and common BL 2.0 statuses
<option value="HEAL_EXHAUSTED">HEAL_EXHAUSTED</option>

<option value="FIX_FAILED">FIX_FAILED</option>
<option value="DIAGNOSIS_COMPLETE">DIAGNOSIS_COMPLETE</option>
```
