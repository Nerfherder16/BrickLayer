# Finding: A5 — Session context append-only (checklist item A5)

**Question**: `session-context.md` must only be opened with `"a"` (append) mode in `bl/campaign.py`.
**Verdict**: COMPLIANT
**Severity**: Info
**Mode**: agent

## Evidence

`bl/campaign.py` line 142:
```python
with open(session_ctx_path, "a", encoding="utf-8") as f:
    f.write(insight_line)
```

This is the only `open()` call for `session-context.md` (or `session_ctx_path`) in `bl/campaign.py`. Mode is `"a"` (append).

`session_ctx_path` is defined at line 61:
```python
session_ctx_path = cfg.project_root / "session-context.md"
```

The read of session-context.md at lines 62-64 uses:
```python
text = session_ctx_path.read_text(encoding="utf-8")
```

`read_text()` does not open the file with write mode — it's read-only. Only the `open(..., "a")` write happens, and it's append-only.

## Mitigation Recommendation

No action required.
