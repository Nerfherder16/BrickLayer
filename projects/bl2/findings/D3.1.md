# Finding: D3.1 — Print statement at healloop.py used max_cycles inconsistently after F2.6 fix

**Question**: After F2.6, `_append_heal_note()` was updated to use `last_cycle`, but the companion `print()` at line 338 still used `max_cycles`, creating inconsistent output.
**Verdict**: FAILURE
**Severity**: Low
**Mode**: diagnose
**Motivated by**: F2.6 FIXED

## Evidence

```
grep -n "max_cycles\|last_cycle" bl/healloop.py
→ line 334: f"Self-healing exhausted {last_cycle} cycle(s)..."  ← EXHAUSTED note (correct)
→ line 338: f"[heal-loop] {original_qid} exhausted {max_cycles} cycle(s)..."  ← stderr print (stale)
```

On early break (e.g., cycle 1), the EXHAUSTED note says "1 cycle(s)" but stderr prints "3 cycle(s)".

## Fix Applied

Changed line 338 to use `last_cycle` instead of `max_cycles`. Now both the finding note and the stderr print report the same actual exit cycle.
