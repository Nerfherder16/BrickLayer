register("agent", run_agent)

## Rationale

All BL 2.0 code audit questions (diagnose-analyst, compliance-auditor) use an LLM agent to read and analyze source code. `run_agent` is the correct runner — it spawns the configured agent from `question["agent_name"]`. The "code_audit" label is semantic (describes what the agent does) not mechanical (the execution mechanism). Aliasing to `run_agent` makes both labels functionally equivalent.
