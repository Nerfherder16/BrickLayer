# Finding: D4.3 â€” _synthetic_question() inherits question_type from original, triggering C-30 on heal findings

**Question**: Does `_synthetic_question()` inherit `question_type` from the original question, causing C-30 logic to cap confidence at "medium" for synthetic diagnose/fix findings?
**Verdict**: FAILURE
**Severity**: Low
**Mode**: diagnose
**Type**: CODE-AUDIT

## Root Cause

`bl/healloop.py` `_synthetic_question()` creates synthetic questions via:

```python
q = dict(original_question)  # shallow copy â€” inherits ALL fields
```

This inherits `question_type` from the original question. `parse_questions()` derives `question_type` from the bracket tag:
- `[CORRECTNESS]` â†’ `code_audit` (in `_CODE_AUDIT_TAGS`)
- `[FIX]` â†’ `code_audit`
- `[COMPLIANCE]` â†’ `code_audit`
- `[BEHAVIORAL]` â†’ `behavioral`

So for any code_audit original question, `diag_q["question_type"] == "code_audit"` and `fix_q["question_type"] == "code_audit"`.

## C-30 Impact on Synthetic Findings

`write_finding()` applies C-30 logic when `question_type == "code_audit"`:

```python
if question_type == "code_audit":
    # Cap confidence at medium
    current_conf = result.get("confidence", "")
    if current_conf == "high":
        result = dict(result)
        result["confidence"] = "medium"
    # Downgrade HEALTHY â†’ WARNING
    if result.get("verdict") == "HEALTHY":
        ...
```

For a synthetic `diag_q` with inherited `question_type="code_audit"`:
- `diagnose-analyst` returns `DIAGNOSIS_COMPLETE` with `confidence="high"`
- C-30 caps confidence at "medium" silently â€” the written finding understates the analyst's actual confidence
- The HEALTHYâ†’WARNING downgrade does not affect diag_q (DC_COMPLETE is not HEALTHY) or fix_q (FIXED/FIX_FAILED, not HEALTHY)

## Severity Assessment

The confidence cap on synthetic diagnose findings is incorrect but non-blocking:
- It doesn't prevent DIAGNOSIS_COMPLETE from being written
- It doesn't prevent the heal loop from proceeding to the fix phase
- It makes the written finding appear less confident than the diagnose-analyst intended

The HEALTHYâ†’WARNING downgrade is not triggered for normal heal agents (they don't return HEALTHY).

## Fix Specification

**File**: `bl/healloop.py`, `_synthetic_question()` function

**Change**: After `q = dict(original_question)`, reset `question_type` based on the synthetic question's role:

```python
q = dict(original_question)
# Synthetic questions are not code_audit questions â€” reset question_type
# to avoid C-30 confidence cap and HEALTHYâ†’WARNING downgrade
if "diagnose" in agent_name:
    q["question_type"] = "behavioral"  # diagnose-analyst returns DC_COMPLETE, not code_audit verdict
else:
    q["question_type"] = "behavioral"  # fix-implementer returns FIXED/FIX_FAILED, not code_audit verdict
```

Or more simply: `q["question_type"] = "behavioral"` â€” synthetic heal questions are always behavioral (live agent execution with real verification, not code_audit).
