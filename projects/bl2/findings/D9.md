# Finding: D9 — campaign.py heal loop result replacement is conditional — may discard enriched context

**Question**: In `bl/campaign.py` lines 129-138, the heal loop result replaces the original result only `if healed_result.get("verdict") != result.get("verdict")`. If the initial verdict is FAILURE and heal loop returns FAILURE (due to D6 mutation), or if initial is DIAGNOSIS_COMPLETE and fix-implementer is missing (loop returns DIAGNOSIS_COMPLETE), the condition is False and the heal loop's accumulated context is silently discarded.
**Verdict**: FAILURE
**Severity**: High
**Mode**: agent

## Evidence

`bl/campaign.py` lines 136-138:
```python
healed_result = run_heal_loop(question, result, heal_finding_path)
if healed_result.get("verdict") != result.get("verdict"):
    result = healed_result
```

Two concrete failure scenarios:

**Scenario A (interacts with D6)**: Initial result is FAILURE. Heal loop runs: diagnose → DIAGNOSIS_COMPLETE, fix → FIX_FAILED. At line 313, `current_result["verdict"]` is mutated to "FAILURE". `run_heal_loop()` returns `current_result` with `verdict = "FAILURE"`. Condition: `"FAILURE" != "FAILURE"` → False. `healed_result` discarded. Campaign output shows only the original FAILURE, with no record of the diagnose-analyst's root cause analysis. This combines with D6 to create a compounding failure.

**Scenario B (independent of D6)**: Initial result is DIAGNOSIS_COMPLETE (from first run). Heal loop enters DIAGNOSIS_COMPLETE branch, attempts to run fix-implementer, but `fix-implementer.md` is missing → breaks at line 253. Returns `current_result` which is `diag_result` (the DIAGNOSIS_COMPLETE output from an already-run diagnose). Condition: `"DIAGNOSIS_COMPLETE" != "DIAGNOSIS_COMPLETE"` → False. Discarded.

In both scenarios, the heal loop DID write intermediate findings to disk (via `write_finding()`) and updated results.tsv. But the returned result dict — used for session-context.md updates (line 141), the JSON output (line 154), and campaign-level reporting — is the stale original, not the enriched heal loop output.

## Mitigation Recommendation

Replace the verdict-diff check with a more meaningful condition that preserves heal loop context:

```python
healed_result = run_heal_loop(question, result, heal_finding_path)
# Always update if heal loop ran (returned different dict object)
# or if the verdict changed — not just on verdict change
if healed_result is not result:
    result = healed_result
```

Since `run_heal_loop()` returns `initial_result` unchanged when disabled or when verdict is not FAILURE/DIAGNOSIS_COMPLETE, the `is not` check correctly identifies when the loop actually ran.

## Suggested Follow-ups
- Does the session-context.md accumulator (line 141) capture the original stale verdict or the healed verdict in the cases where healed_result is discarded?
