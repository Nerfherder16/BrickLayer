# Finding: D3 — Heal loop synthetic question ID overwritten before agent sees it

**Question**: In `bl/healloop.py`, `_synthetic_question()` generates an ID like `{qid}_heal{cycle}_diagnose` (line 75), which is passed to `_run_heal_agent()`. But after `run_agent()` returns, the ID is overwritten to `{qid}_heal{cycle}_diag` (line 218) before `write_finding()`. The agent itself sees the `_diagnose` ID in its prompt/context, but the finding is written with the `_diag` ID.
**Verdict**: FAILURE
**Severity**: Medium
**Mode**: agent

## Evidence

`bl/healloop.py` line 75 (`_synthetic_question()`):
```python
q["id"] = f"{original_question['id']}_heal{cycle}_{agent_name.split('-')[0]}"
```

For `agent_name="diagnose-analyst"`, `split('-')[0]` = `"diagnose"`, so the ID is `{qid}_heal{cycle}_diagnose`.

This ID is what `run_agent()` sees when it builds the prompt (the question dict passed to it has `id = _diagnose`).

Then at line 218, AFTER `_run_heal_agent("diagnose-analyst", diag_q)` returns:
```python
diag_q["id"] = f"{original_qid}_heal{cycle}_diag"
write_finding(diag_q, diag_result)
update_results_tsv(diag_q["id"], ...)
```

The finding is written with `_heal{cycle}_diag` but the agent was invoked with `_heal{cycle}_diagnose`.

The project-brief confirms the intended IDs are `{qid}_heal{n}_diag` and `{qid}_heal{n}_fix` (line: "Writes intermediate findings `{qid}_heal{n}_diag.md`"). So the intended final ID is `_diag`, and the `_diagnose` ID in `_synthetic_question()` is the bug. If the diagnose-analyst agent quotes its own question ID in its output (e.g., in a DIAGNOSIS_COMPLETE Fix Spec referencing "this question"), it will reference `D1_heal1_diagnose` which does not exist in findings/ or results.tsv.

For `fix-implementer`, line 75 produces `{qid}_heal{cycle}_fix` (since `"fix-implementer".split('-')[0]` = `"fix"`), and line 275 sets `fix_q["id"] = f"{original_qid}_heal{cycle}_fix"` — these MATCH. The fix-implementer path has no ID mismatch.

## Mitigation Recommendation

Fix `_synthetic_question()` to use the short form immediately:
```python
short_type = "diag" if "diagnose" in agent_name else "fix"
q["id"] = f"{original_question['id']}_heal{cycle}_{short_type}"
```

This eliminates the post-facto ID overwrite at line 218 and ensures the agent's context matches the written finding ID.
