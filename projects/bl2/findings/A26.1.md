## Audit Findings

### Primary Issue: Wave 1 Question Blocks Not Found (Score = 0.00)

`_score_question_designer()` in `bl/crucible.py` returns score **0.00** for the bl2 project with the message "Only 0 Wave 1 questions found (need >= 5)". This is a separate structural failure upstream of the has_hypothesis and has_thresholds checks being audited.

**Root cause**: The Wave 1 block detection at line 250 uses:
```python
blocks = re.split(r"(?=^## \w+\d+\.\d+)", wave1_text, flags=re.MULTILINE)
blocks = [b.strip() for b in blocks if re.match(r"## \w+\d+\.\d+", b.strip())]
```

The regex `\w+\d+\.\d+` requires a decimal point (e.g., `## D1.1`). The bl2 Wave 1 questions use the format `## D1`, `## D2`, ..., `## D10`, `## A1`–`## A10` — no decimal suffix. The 22 Wave 1 blocks are entirely invisible to the scorer.

**Confirmed by direct analysis**: Using a corrected regex `^## [A-Z]\d+(?:\.\d+)?[ \[]`:
- Wave 1 blocks found: **22** (satisfying the ≥5 threshold)
- Unique prefixes: `{D, A}` (missing F, V, M prefixes → domains_score = 2/5 = 0.40)

### Finding 1: has_hypothesis — NON_COMPLIANT

**Line 270** (`bl/crucible.py`):

The bare string `"Hypothesis:"` is NOT a substring of `"**Hypothesis**:"` — the asterisks precede the colon, breaking the match. For the 22 Wave 1 bl2 blocks:
- `"Hypothesis:"` matches: **0/22 = 0.000**
- `"**Hypothesis**:"` matches: **22/22 = 1.000**

All 22 Wave 1 questions use the BL 2.0 bold format exclusively. The has_hypothesis score is structurally suppressed to 0.00 for this campaign. Weight: 0.20 — total impact: **-0.20** from maximum achievable score.

## Fix Specification

### Fix 1 (Critical): Wave 1 block detection regex — `bl/crucible.py` line 250–251
```python
# Change from:
blocks = re.split(r"(?=^## \w+\d+\.\d+)", wave1_text, flags=re.MULTILINE)
blocks = [b.strip() for b in blocks if re.match(r"## \w+\d+\.\d+", b.strip())]

# Change to:
blocks = re.split(r"(?=^## [A-Z]\d+(?:\.\d+)?[ \[])", wave1_text, flags=re.MULTILINE)
blocks = [b.strip() for b in blocks if re.match(r"## [A-Z]\d+(?:\.\d+)?", b.strip())]
```
Also update line 262 domains regex:
```python
# Change from:
unique_prefixes = set(re.findall(r"^## ([A-Z])\d+\.\d+", wave1_text, re.MULTILINE))
# Change to:
unique_prefixes = set(re.findall(r"^## ([A-Z])\d+", wave1_text, re.MULTILINE))
```

### Fix 2 (High): has_hypothesis bare string check — `bl/crucible.py` line 270
```python
# Change from:
has_hypothesis = sum(1 for b in blocks if "Hypothesis:" in b) / len(blocks)
# Change to:
has_hypothesis = sum(1 for b in blocks if re.search(r"\*\*Hypothesis\*\*:|Hypothesis:", b)) / len(blocks)
```

### Fix 3 (Medium): has_thresholds — accept BL 2.0 format — `bl/crucible.py` lines 266–268
```python
# Change from:
has_thresholds = sum(
    1 for b in blocks if "FAILURE:" in b and "HEALTHY:" in b
) / len(blocks)
# Change to:
has_thresholds = sum(
    1 for b in blocks if (
        ("FAILURE:" in b and "HEALTHY:" in b)
        or "**Verdict threshold**:" in b
        or "**Success criterion**:" in b
    )
) / len(blocks)
```
