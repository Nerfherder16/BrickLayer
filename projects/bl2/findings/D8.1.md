# Finding: D8.1 — _inject_override_questions() glob("Q*.md") excludes all BL 2.0 findings

**Question**: Does _inject_override_questions() use glob("Q*.md"), making peer review OVERRIDE injection non-functional for all BL 2.0 findings?
**Verdict**: FAILURE
**Severity**: High
**Mode**: diagnose
**Operational Mode**: diagnose

## Root Cause

`bl/campaign.py` `_inject_override_questions()` line 396:

```python
for finding_file in sorted(cfg.findings_dir.glob("Q*.md")):
```

BL 2.0 findings are named by question ID: `D5.1.md`, `F7.1.md`, `A7.1.md`, `V8.1.md`, etc. None of these start with "Q". The glob `Q*.md` only matches BL 1.x question findings (`Q2.4.md`, `Q13.1.md`, etc.).

## Secondary Issue: Re-exam ID Format

The injected re-exam block (line 409) uses:

```python
## {qid}.R [CORRECTNESS] Re-examine {qid}
```

For a BL 2.0 parent qid "D5.1", the injected ID would be "D5.1.R". Check against block_pattern `[\w][\w.-]*`:
- "D" ✓ (starts with \w)
- "5.1.R" — "R" is \w, "." is in `[\w.-]` ✓

"D5.1.R" matches block_pattern. The ID format itself is valid for BL 2.0 parse_questions().

However, `_is_leaf_id("D5.1.R")` → `parts = "D5.1.R".split(".")` → 3 parts → True (leaf). Re-exam questions would be correctly treated as leaves (non-drillable). ✓

## Impact

Every peer review OVERRIDE on a BL 2.0 finding is silently dropped. The `print(f"[campaign] OVERRIDE in {finding_file.name}...")` line never executes for any BL 2.0 finding. No re-exam questions are ever injected for BL 2.0 campaigns.

In this campaign: Waves 1-7 contain 56 questions, all BL 2.0 format (D*, F*, A*, V*). If any peer reviewer returned OVERRIDE on any finding, the re-exam mechanism has been completely non-functional.

## Fix Specification

- **File**: `bl/campaign.py`, `_inject_override_questions()`, line 396
- **Change**: Replace `glob("Q*.md")` with `glob("*.md")`
- **Verification**: Create a test finding `D_test.md` with `## Peer Review\n\n**Verdict**: OVERRIDE` in findings/. Run `_inject_override_questions()`. Confirm a D_test.R question is injected into questions.md.
