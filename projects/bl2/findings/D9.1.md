# Finding: D9.1 — hypothesis.py _QUESTION_BLOCK_HEADER regex only matches Q-prefixed BL 1.x IDs

**Question**: Does _QUESTION_BLOCK_HEADER use a Q-only regex, making wave detection and ID extraction non-functional for BL 2.0 questions.md?
**Verdict**: FAILURE
**Severity**: High
**Mode**: diagnose
**Operational Mode**: diagnose

## Root Cause

`bl/hypothesis.py` line 21-23:

```python
_QUESTION_BLOCK_HEADER = re.compile(
    r"^## (Q\d+\.\d+[\w.]*)\s+\[(\w+)\]\s+(.+)$", re.MULTILINE
)
```

The ID pattern `Q\d+\.\d+[\w.]*` requires the ID to start with literal "Q" followed by digits. BL 2.0 IDs (D8.1, F5.1, A7.1, V6.1, M2.1) start with non-Q letters and never match.

## Impact Trace

### _get_wave_number() for BL 2.0 questions.md

```python
matches = _QUESTION_BLOCK_HEADER.findall(questions_text)
# → [] (no Q-prefixed headers in BL 2.0 questions.md)
if not matches:
    return 1  # ← always returns 1 for BL 2.0 campaigns
```

### _get_existing_ids() for BL 2.0 questions.md

```python
return {m[0] for m in _QUESTION_BLOCK_HEADER.findall(questions_text)}
# → set() (empty — no BL 2.0 IDs found)
```

### generate_hypotheses() consequence

```python
next_wave = wave or (_get_wave_number(questions_text) + 1)
# → wave=None, _get_wave_number()=1 → next_wave=2
```

Generated prompt asks LLM to produce "Wave 2 questions" with IDs "Q2.1, Q2.2...". For a campaign currently on Wave 8, this produces:
1. Wrong wave number in prompt (Wave 2 instead of Wave 9)
2. Wrong ID format (Q2.1 instead of D9.1)
3. Existing ID deduplication fails (empty set → can't check for duplicates)
4. LLM gets confusing context — "Wave 2" when findings show Wave 7+ results

### Wave extraction for BL 2.0 IDs (after fix)

For ID "D8.1": `qid.split(".")[0]` → "D8" → `int("D8"[1:])` → `int("8")` → 8 ✓
For ID "D8.1.R": `"D8"[1:]` → "8" → 8 ✓
For "D10.1": `"D10"[1:]` → "10" → 10 ✓
The `[1:]` slice works because all BL 2.0 ID prefixes are exactly 1 character (D, F, A, V, M).

## Fix Specification

- **File**: `bl/hypothesis.py`, `_QUESTION_BLOCK_HEADER`
- **Change**: Replace `Q\d+\.\d+[\w.]*` with `[\w][\w.-]*` to match all BL 2.0 and BL 1.x IDs
- **Verification**: `_QUESTION_BLOCK_HEADER.search("## D8.1 [CORRECTNESS] title")` matches; `_get_wave_number()` returns 8 for current questions.md; `_get_existing_ids()` returns all 62+ question IDs
