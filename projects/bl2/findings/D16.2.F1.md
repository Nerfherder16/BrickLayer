# Finding: D16.2.F1 — _inject_override_questions() has no mode filter — re-exam questions hardcode mode:agent for code_audit findings

**Question**: Does `_inject_override_questions()` in `bl/campaign.py` lack a mode filter, causing it to inject re-exam questions that hardcode `**Mode**: agent` for code_audit OVERRIDE findings, producing re-exams with prose-only test commands that cannot be resolved?
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: High

## Evidence

**`_inject_override_questions()` lines 395-431**: The function iterates ALL `*.md` files in `findings/` with NO mode filter:

```python
for finding_file in sorted(cfg.findings_dir.glob("*.md")):
    content = finding_file.read_text(encoding="utf-8")
    if not override_pattern.search(content):
        continue
    qid = finding_file.stem
    # No lookup of original question mode here
```

The injected `reexam_block` at lines 410-421 hardcodes `**Mode**: agent`:
```python
reexam_block = f"""
## {qid}.R [CORRECTNESS] Re-examine {qid}
**Mode**: agent
**Status**: PENDING
**Test**: Re-run the original test command from {qid} and confirm the concern raised...
"""
```

**The function never looks up the original question's mode from questions.md.** It uses only `qid` (the filename stem) to generate the re-exam block.

**Consequence chain for code_audit questions**:
1. code_audit finding D16.x gets an OVERRIDE peer review section (from a peer-reviewer that couldn't execute the prose test)
2. `_inject_override_questions()` scans findings → finds OVERRIDE → injects `D16.x.R` with `**Mode**: agent`
3. Next campaign run: `D16.x.R` with mode=agent and `**Test**: Re-run the original test command from D16.x...` is in pending list
4. Campaign routes to `run_agent()` with the question
5. The agent reads "Re-run the original test command from D16.x" → reads D16.x test field → test field says "Read bl/healloop.py lines 288-343 and trace..."
6. The agent is told to "run" a prose instruction as a test command → produces INCONCLUSIVE or FAILURE
7. If the agent produces FAILURE → follow-up drill-down spawns → another D16.x.R.F question → cascade of unresolvable re-exams

**Interaction with D16.2 fix**: After D16.2 is fixed (peer-reviewer spawn guarded for code_audit), new OVERRIDE verdicts from code_audit peer-reviewers stop being generated. But `_inject_override_questions()` still scans ALL historical findings — any code_audit finding that received an OVERRIDE before the D16.2 fix would continue to re-inject re-exam questions on every campaign run until `reexam_marker` guard at line 407 prevents duplicate injection.

## Fix Specification

**Target file**: `bl/campaign.py`
**Target location**: `_inject_override_questions()` lines 398-408 — before the `reexam_block` generation
**Concrete edit**: After extracting `qid = finding_file.stem`, look up the original question's mode from questions.md and skip code_audit findings:
```python
qid = finding_file.stem
# Skip code_audit mode findings — their "test" fields are prose, not executable commands
original_question = get_question_by_id(parse_questions(), qid)
if original_question and original_question.get("mode") == "code_audit":
    continue
```
Alternatively, inject with the original mode instead of hardcoding `agent`:
```python
original_mode = original_question.get("mode", "agent") if original_question else "agent"
# Use original_mode in reexam_block instead of "agent"
```
**Verification command**: `grep -n "Mode.*agent\|mode.*code_audit\|get_question_by_id" bl/campaign.py` — confirm `_inject_override_questions()` either skips code_audit findings or uses the original mode.

## Mitigation Recommendation

1. Add mode lookup to `_inject_override_questions()` — skip findings for code_audit questions (or inherit the original mode).
2. For code_audit OVERRIDE verdicts, consider injecting a different re-exam type that asks the diagnose-analyst to re-read the code rather than re-run a test command.
3. Scan questions.md for any existing `.R` re-exam questions derived from code_audit findings (identifiable by their test field referencing prose code-read instructions) and remove or convert them.
