# Finding: D6.1 — F5.1 regression: "code_audit" body Mode dispatches to unregistered runner

**Question**: Does F5.1's body **Mode** field priority cause Wave 4-5 questions with `**Mode**: code_audit` to dispatch to an unregistered runner, producing INCONCLUSIVE?
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: High
**Mode**: diagnose
**Operational Mode**: diagnose

## Root Cause

F5.1 changed `question["mode"]` from `mode_raw` (bracket tag) to `fields.get("mode", mode_raw)` (body **Mode** field). For 19 Wave 4-5 questions with `**Mode**: code_audit` in their body, this sets `question["mode"] = "code_audit"`.

`run_question()` dispatches on `question["mode"]`. Registered runners:
```python
['agent', 'correctness', 'http', 'performance', 'quality', 'static', 'subprocess']
```

"code_audit" is absent. `run_question()` falls through to the INCONCLUSIVE path:
```python
result = {
    "verdict": "INCONCLUSIVE",
    "summary": f"Unknown mode 'code_audit' — no runner registered",
    ...
}
```

## Evidence

```
grep -n "code_audit" bl/runners/__init__.py
→ (no output, exit code 1 — not registered)

grep -c '**Mode**: code_audit' projects/bl2/questions.md
→ 19  (Wave 4-5 questions: D4.1, D4.2, D4.3, F4.1, F4.2, F4.3, F4.4, D5.1, A5.1, F5.1, V5.1, and 8 more)
```

Before F5.1, these questions used the bracket tag as fallback:
- D4.1 [CORRECTNESS] → mode_raw = "correctness" → registered → `run_correctness()` (wrong runner, but registered)
- F4.1 [FIX] → mode_raw = "fix" → unregistered → INCONCLUSIVE (already broken)

After F5.1:
- D4.1 [CORRECTNESS] with `**Mode**: code_audit` → mode = "code_audit" → unregistered → INCONCLUSIVE

## Fix Specification

- **File**: `bl/runners/__init__.py`, `_register_builtins()`
- **Change**: Add `register("code_audit", run_agent)` after `register("agent", run_agent)`
- **Rationale**: All "code_audit" questions in BL 2.0 use a diagnose-analyst or compliance-auditor LLM agent to read source code — `run_agent` is the correct runner. "code_audit" is a semantic label for what the agent does, not a different execution mechanism.
- **Verification**: `registered_modes()` includes "code_audit"; `run_question({"mode": "code_audit", ...})` dispatches to `run_agent`
