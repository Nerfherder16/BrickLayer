# Finding: D14.3 — synthesizer.py writes synthesis.md to project root — callers expect findings/synthesis.md

**Question**: Does synthesizer.py write synthesis.md to the correct path?
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: High
**Mode**: code_audit
**Operational Mode**: diagnose

## Root Cause

**`synthesizer.py` line 226**:
```python
synthesis_path = project_dir / "synthesis.md"
```

Writes to `{project_root}/synthesis.md` (project root level).

**`campaign.py` lines 731-732**:
```python
synthesis_path = cfg.project_root / "findings" / "synthesis.md"
```

The skill-forge background agent is passed `{project_root}/findings/synthesis.md` — a path that is **never written**.

## Impact

- `synthesize()` call site at campaign.py lines 714-715 reads back via the returned `Path` object, so **that read is correct** — it uses the actual written path.
- But the skill-forge agent (called after) independently constructs its own path using `findings/synthesis.md` and **always operates on a nonexistent file**.
- Every synthesis overwrites the same root-level `synthesis.md` while `findings/synthesis.md` is never created.
- Skill-forge produces empty or error output every wave.

## Fix Specification

**File**: `C:\Users\trg16\Dev\autosearch\bl\synthesizer.py`
**Line**: 226

Change from:
```python
synthesis_path = project_dir / "synthesis.md"
```

Change to:
```python
synthesis_path = project_dir / "findings" / "synthesis.md"
```

The `findings/` directory is guaranteed to exist (`init_project()` calls `mkdir` on it). No other change needed — `campaign.py` lines 714-715 read via the returned `Path` object, so they follow the fix automatically.

**Verification**:
```python
# After fix: run any wave to completion and assert:
assert (project_root / "findings" / "synthesis.md").exists()
assert not (project_root / "synthesis.md").exists()
```

**Risk**: Low. Only one call site reads via the returned Path object (campaign.py 714-715), and that follows automatically. No external scripts reference the old root-level path in the codebase.
