# Finding: D2 — _NON_FAILURE_VERDICTS missing DEGRADED, ALERT, UNKNOWN, BLOCKED

**Question**: `_NON_FAILURE_VERDICTS` in `bl/findings.py` is missing `DEGRADED`, `ALERT`, `UNKNOWN`, and `BLOCKED` — all four are listed in `constants.py` `NON_FAILURE_VERDICTS` but absent from the code's frozenset.
**Verdict**: FAILURE
**Severity**: High
**Mode**: agent

## Evidence

`constants.py` `NON_FAILURE_VERDICTS` lists 18 verdicts:
```
HEALTHY, WARNING, DIAGNOSIS_COMPLETE, FIXED, CALIBRATED, COMPLIANT,
PARTIAL, NOT_APPLICABLE, IMPROVEMENT, OK, DEGRADED, DEGRADED_TRENDING,
ALERT, UNKNOWN, PROMISING, BLOCKED, WEAK, SUBJECTIVE
```

`bl/findings.py` `_NON_FAILURE_VERDICTS` frozenset contains 19 verdicts:
```
HEALTHY, WARNING, DIAGNOSIS_COMPLETE, PENDING_EXTERNAL, PROMISING,
WEAK, CALIBRATED, FIXED, COMPLIANT, PARTIAL, NOT_APPLICABLE,
IMPROVEMENT, OK, POSSIBLE, UNLIKELY, DEGRADED_TRENDING, SUBJECTIVE,
NOT_MEASURABLE, UNCALIBRATED
```

Missing from `_NON_FAILURE_VERDICTS` vs constants.py:
- `DEGRADED` — Monitor mode verdict, should NOT be classified as a failure
- `ALERT` — Monitor mode verdict, should NOT be classified as a failure
- `UNKNOWN` — Monitor mode verdict, should NOT be classified as a failure
- `BLOCKED` — Frontier mode verdict, should NOT be classified as a failure

Impact: `classify_failure_type()` checks `if verdict in _NON_FAILURE_VERDICTS: return None`. These 4 missing verdicts will proceed into the failure classification logic and receive a failure_type label (e.g., `logic`, `unknown`), producing incorrect `failure_type` and inflated severity in findings. Monitor-mode campaigns (DEGRADED, ALERT) and Frontier-mode campaigns (BLOCKED) will have their verdicts misclassified as failures.

Secondary: `_NON_FAILURE_VERDICTS` also contains `POSSIBLE`, `UNLIKELY`, `NOT_MEASURABLE`, `UNCALIBRATED`, `PENDING_EXTERNAL` — verdicts not listed in constants.py `NON_FAILURE_VERDICTS`. This may be intentional but creates a discrepancy between the spec contract and the code.

## Mitigation Recommendation

Add DEGRADED, ALERT, UNKNOWN, BLOCKED to `_NON_FAILURE_VERDICTS` in `bl/findings.py`. Consider importing `NON_FAILURE_VERDICTS` from constants.py directly to prevent future drift:

```python
from bl.constants import NON_FAILURE_VERDICTS as _NON_FAILURE_VERDICTS_LIST
_NON_FAILURE_VERDICTS = frozenset(_NON_FAILURE_VERDICTS_LIST)
```
