# Finding: D25.1 — has_test=0.00 and has_hypothesis=0.00 confirmed: check_block uses bare BL 1.x strings; BL 2.0 uses bold **Hypothesis**: and **Method**: — "Hypothesis:" is not a substring of "**Hypothesis**:"

**Question**: Why are has_test=0.00 and has_hypothesis=0.00 — do check_block fields use BL 1.x bare strings while BL 2.0 questions use bold markdown headers?
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: Low

## Evidence

**`check_block()` in `_score_hypothesis_generator`** (`bl/crucible.py` lines 188–197):
```python
def check_block(b: str) -> dict[str, float]:
    return {
        "has_status": 1.0 if re.search(r"PENDING|DONE|Status", b) else 0.0,
        "has_derived_from": 1.0 if ("Derived from" in b or "Motivated by" in b) else 0.0,
        "has_test": 1.0 if re.search(r"Test:|pytest|Simulation path", b) else 0.0,
        "has_verdict_threshold": 1.0 if ("FAILURE:" in b and "HEALTHY:" in b) else 0.0,
        "has_hypothesis": 1.0 if "Hypothesis:" in b else 0.0,
    }
```

**Field name frequency in `projects/bl2/questions.md`**:

| Check | Scorer expects | Count in questions.md | BL 2.0 field | Count |
|-------|--------------|----------------------|-------------|-------|
| has_test | `"Test:"` | 2 (in D25.1 description) | `**Method**:` | 59 |
| has_hypothesis | `"Hypothesis:"` | 2 (in D25.1 description) | `**Hypothesis**:` | 136 |

**Root cause 1 — has_test (weight=0.20)**:
`re.search(r"Test:|pytest|Simulation path", b)` matches BL 1.x question format (`Test: python simulate.py ...`). BL 2.0 questions use `**Method**:` — a bold header describing the investigation method. No BL 2.0 question contains "Test:", "pytest", or "Simulation path".

**Root cause 2 — has_hypothesis (weight=0.10)**:
`"Hypothesis:" in b` checks for the bare unbolded string. BL 2.0 questions use `**Hypothesis**:`. The string `"**Hypothesis**:"` does NOT contain the substring `"Hypothesis:"` — after "Hypothesis" comes `**:` (two asterisks then colon), not a bare colon. The has_hypothesis check always returns 0.00 for BL 2.0 blocks.

**Combined weight impact**: 0.20 + 0.10 = 0.30 of total score weight is zeroed by these two mismatches. Post-F24.1 score is ~0.37; fixing both would raise it to ~0.67.

**has_verdict_threshold (weight=0.25, rate=0.20)**: BL 2.0 questions use `FAILURE:` and `HEALTHY:` labels in their "Verdict threshold" section. ~20% of questions have these labels. The remaining 80% use "Success criterion:" format without FAILURE:/HEALTHY: labels — this is a partial mismatch also worth noting but lower impact.

## Fix Specification

**Target file**: `bl/crucible.py`
**Target location**: `_score_hypothesis_generator()` → `check_block()` (lines 188–197)
**Concrete edit**:
```python
# Before:

...

```
