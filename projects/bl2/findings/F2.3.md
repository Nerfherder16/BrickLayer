# Finding: F2.3 — Use dict copy instead of alias for fix_result in healloop.py

**Question**: Fix D6: `current_result = fix_result` at line 309 was an alias, causing `current_result["verdict"] = "FAILURE"` to mutate `fix_result` in place. `run_heal_loop()` returned `verdict="FAILURE"` instead of `"FIX_FAILED"`, compounding with D9 to silently discard the heal loop's context.
**Verdict**: FIXED
**Severity**: High
**Mode**: fix
**Source finding**: D6

## Changes Made

Changed `current_result = fix_result` to `current_result = dict(fix_result)` (shallow copy) in `bl/healloop.py`. This preserves `fix_result["verdict"] = "FIX_FAILED"` for the returned dict, while the copy's verdict is correctly set to `"FAILURE"` for re-entry into the next diagnose cycle.

## Verification

Static analysis: `fix_result["verdict"]` is now only read after this point, not mutated. `current_result["verdict"] = "FAILURE"` operates on the copy. `return current_result` at the end of the loop returns the copy with `"FAILURE"` (which is correct for campaign.py's result propagation after F2.4 is applied), while `fix_result` retains `"FIX_FAILED"` for correct results.tsv recording.
