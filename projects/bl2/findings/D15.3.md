# Finding: D15.3 — crucible.py _score_synthesizer() and _score_quantitative_analyst() use Q-prefix-only patterns

**Question**: Do crucible.py scorer functions correctly detect BL 2.0 finding files and ID references?
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: High
**Mode**: code_audit
**Operational Mode**: diagnose

## Root Cause

### Bug 1 — `_score_synthesizer()` line 300

```python
re.findall(r'Q\d+\.\d+', synthesis_text)
```

BL 2.0 synthesis.md cites `D/F/A/V`-prefixed IDs (e.g., `D14.1`, `F12.3`, `A13.1`). These never match `Q\d+\.\d+`. The check carries weight 0.30, capping the synthesizer's maximum achievable score at **0.70** on any BL 2.0 campaign.

### Bug 2 — `_score_quantitative_analyst()` line 328

```python
findings_dir.glob('Q*.md')
```

BL 2.0 findings are named `D14.1.md`, `F12.3.md`, `A13.1.md` etc. — none match `Q*.md`. `perf_files` is always empty, the scorer hits the early return, and records **score 0.0** regardless of finding file quality.

## Impact

Both agents are structurally scored below their promote thresholds in every BL 2.0 campaign despite producing correct output.

## Fix Specification

**File**: `C:\Users\trg16\Dev\autosearch\bl\crucible.py`

**Line 300**: Replace `r'Q\d+\.\d+'` with `r'\b[A-Z]\d+\.\d+'`

**Line 328**: Replace `findings_dir.glob('Q*.md')` with:
```python
(f for f in findings_dir.glob('*.md') if f.name != 'synthesis.md')
```

The existing content filter on line 333 already restricts `perf_files` to performance-domain findings.

**Risk**: Low. Synthesizer regex is looser but synthesis.md text is unlikely to contain non-finding strings shaped like `X##.##`. BL 1.x campaigns unaffected — `Q*.md` files are included under `*.md`.
