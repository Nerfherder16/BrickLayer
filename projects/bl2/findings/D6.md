# Finding: D6 — Heal loop fix_result mutation via current_result alias

**Question**: In `bl/healloop.py` line 309, `current_result = fix_result` creates an alias (not a copy). Then line 313, `current_result["verdict"] = "FAILURE"` mutates the original `fix_result` dict. The `fix_result` dict returned by `run_heal_loop()` will show "FAILURE" instead of "FIX_FAILED" when the loop exhausts without a fix.
**Verdict**: FAILURE
**Severity**: High
**Mode**: agent

## Evidence

`bl/healloop.py` lines 301-313:
```python
# FIX_FAILED — loop back for next diagnose cycle
_append_heal_note(...)
current_result = fix_result             # ← alias, not a copy
# The FIX_FAILED finding contains the Root Cause Update
current_finding_id = fix_q["id"]
current_result["verdict"] = "FAILURE"  # ← mutates fix_result directly
```

Python dict assignment creates a reference. `current_result["verdict"] = "FAILURE"` modifies `fix_result["verdict"]` in place.

`write_finding(fix_q, fix_result)` and `update_results_tsv(fix_q["id"], fix_result["verdict"], ...)` are called BEFORE the mutation (lines 276-279), so `results.tsv` and the finding file correctly record `FIX_FAILED`. However:

1. `return current_result` at line 334 returns the mutated dict with `verdict = "FAILURE"`.
2. In `campaign.py` lines 136-138:
   ```python
   healed_result = run_heal_loop(question, result, heal_finding_path)
   if healed_result.get("verdict") != result.get("verdict"):
       result = healed_result
   ```
   If the original `result.get("verdict")` was `"FAILURE"` and `healed_result.get("verdict")` is also `"FAILURE"` (from mutation), the condition is `False` and `healed_result` is discarded. The campaign proceeds as if no healing was attempted, losing the heal loop's accumulated summary context and the DIAGNOSIS_COMPLETE note.

3. The final JSON output printed at line 154 reflects the original (unhealed) result, even though the heal loop ran and potentially produced useful diagnostic context.

## Mitigation Recommendation

Use an explicit copy instead of an alias:
```python
current_result = dict(fix_result)       # shallow copy
current_result["verdict"] = "FAILURE"   # no longer mutates fix_result
```

This preserves `fix_result["verdict"] = "FIX_FAILED"` for the returned dict, so campaign.py can correctly detect `healed_result.verdict != result.verdict` and update the result.
