# Finding: D8.2 — _mark_question_done() _PRESERVE_AS_IS missing FAILURE and NON_COMPLIANT

**Question**: Does _mark_question_done() write status "DONE" for FAILURE and NON_COMPLIANT verdicts, erasing diagnostic visibility in questions.md?
**Verdict**: FAILURE
**Severity**: Medium
**Mode**: diagnose
**Operational Mode**: diagnose

## Root Cause

`bl/findings.py` `_mark_question_done()` lines 502-512:

```python
_PRESERVE_AS_IS = frozenset(
    {
        "INCONCLUSIVE",
        "DIAGNOSIS_COMPLETE",
        "PENDING_EXTERNAL",
        "FIXED",
        "FIX_FAILED",
        "BLOCKED",
    }
)
new_status = verdict if verdict in _PRESERVE_AS_IS else "DONE"
```

Missing from `_PRESERVE_AS_IS`:
- `"FAILURE"` — critical — questions with active bugs written as DONE
- `"NON_COMPLIANT"` — important — compliance violations written as DONE
- `"WARNING"` — medium — degraded but non-critical state written as DONE
- `"REGRESSION"` — important — regressions written as DONE
- `"ALERT"` — important (monitor mode) — alert state written as DONE

## Verdict Routing Trace

| Verdict | In _PRESERVE_AS_IS? | Written status |
|---------|---------------------|----------------|
| FAILURE | No | DONE ← bug |
| NON_COMPLIANT | No | DONE ← bug |
| WARNING | No | DONE ← visibility loss |
| REGRESSION | No | DONE ← bug |
| ALERT | No | DONE ← bug |
| INCONCLUSIVE | Yes | INCONCLUSIVE ✓ |
| DIAGNOSIS_COMPLETE | Yes | DIAGNOSIS_COMPLETE ✓ |
| FIXED | Yes | FIXED ✓ |
| FIX_FAILED | Yes | FIX_FAILED ✓ |

## Impact in This Campaign

In this campaign, manually maintained statuses (FAILURE, DONE, DIAGNOSIS_COMPLETE) were set by hand. But when the BL 2.0 engine runs questions autonomously via `update_results_tsv()` → `_mark_question_done()`:
- A question returning FAILURE gets status "DONE" in questions.md
- `get_next_pending()` correctly skips it (only processes PENDING)
- But a human inspecting questions.md sees "DONE" — indistinguishable from a passing result
- The heal loop reads from `get_question_status()` (results.tsv), not questions.md status — so heal loop is unaffected
- The visibility bug reduces the value of questions.md as a campaign dashboard

## Fix Specification

- **File**: `bl/findings.py`, `_mark_question_done()`, `_PRESERVE_AS_IS` frozenset
- **Change**: Add `"FAILURE"`, `"NON_COMPLIANT"`, `"WARNING"`, `"REGRESSION"`, `"ALERT"` to `_PRESERVE_AS_IS`
- **Verification**: `verdict="FAILURE"` → questions.md block shows `**Status**: FAILURE`; `verdict="NON_COMPLIANT"` → `**Status**: NON_COMPLIANT`; `verdict="HEALTHY"` → `**Status**: DONE` (unchanged)
