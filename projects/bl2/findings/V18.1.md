# Finding: V18.1 — _score_diagnose_analyst dc_rate denominator includes BL 1.x rows; metric is diluted but structurally correct as a campaign-wide history metric

**Question**: Does `_score_diagnose_analyst` correctly scope dc_rate to diagnose-analyst rows, or does it count verdicts from all agents in results.tsv?
**Verdict**: NON_COMPLIANT
**Severity**: Low

## Evidence

**Current results.tsv analysis**:
```
all_diag rows (DIAGNOSIS_COMPLETE|HEALTHY|FAILURE|INCONCLUSIVE): 45
diag_rows (DC+HEALTHY only): 23
dc_rate = 23/45 = 0.511
```

**`_score_diagnose_analyst` dc_rate logic** (`bl/crucible.py` lines 383-393):
```python
diag_rows = [
    ln for ln in lines if "\tDIAGNOSIS_COMPLETE\t" in ln or "\tHEALTHY\t" in ln
]
all_diag = [
    ln for ln in lines
    if re.search(r"\t(DIAGNOSIS_COMPLETE|HEALTHY|FAILURE|INCONCLUSIVE)\t", ln)
]
dc_rate = len(diag_rows) / len(all_diag) if all_diag else 0.0
```

**Root cause**: `all_diag` pulls ALL rows matching these four verdict strings. In the BL 2.0 campaign, BL 1.x Wave 1–15 D-prefix questions returned FAILURE/INCONCLUSIVE verdicts (investigated by quantitative-analyst, not diagnose-analyst). These 22+ FAILURE rows enter `all_diag` and inflate the denominator, dragging dc_rate from ~0.85 (BL 2.0 only) to ~0.51 (campaign-wide).

**What the metric SHOULD measure**: diagnose-analyst success rate = (DIAGNOSIS_COMPLETE + HEALTHY) / total from BL 2.0 D-prefix questions only.

**The 40 fix_rows** in results.tsv are all F-prefix (F2.1–F2.6, F-mid.1–F17.1) — these match `\t(FIXED|FIX_FAILED)\t` and do NOT enter `all_diag`. The contamination is solely from old BL 1.x D-prefix FAILURE rows.

## Fix Specification

**Target file**: `bl/crucible.py`
**Target location**: `_score_diagnose_analyst()` lines 383-393
**Concrete edit**: Scope `all_diag` to rows containing "DIAGNOSIS_COMPLETE" anywhere (a proxy for BL 2.0 D-prefix questions), OR add a question-ID prefix filter `parts[0].startswith("D")` combined with an exclusion of old-format rows (no `N/A` run_id prefix):
```python
# Scope to BL 2.0 diagnose questions: rows where qid starts with D and verdict is from the diagnose lifecycle
def _is_bl2_diag_row(ln):
    parts = ln.split("\t")
    # New format: N/A | qid | verdict | ...
    if len(parts) >= 3 and parts[0] == "N/A":
        return parts[1].startswith("D") and re.search(
            r"^(DIAGNOSIS_COMPLETE|HEALTHY|FAILURE|INCONCLUSIVE)$", parts[2]
        )
    return False
diag_rows = [ln for ln in lines if _is_bl2_diag_row(ln) and
             ("\tDIAGNOSIS_COMPLETE\t" in ln or "\tHEALTHY\t" in ln)]
all_diag = [ln for ln in lines if _is_bl2_diag_row(ln)]
```
**Verification command**: Run `run_all_benchmarks()` — `diagnose-analyst` score should rise from ~0.33 to ~0.65+ when BL 1.x rows are excluded.
