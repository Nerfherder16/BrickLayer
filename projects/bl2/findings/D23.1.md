# Finding: D23.1 — has_derived_from=0.00 root cause confirmed: _score_hypothesis_generator checks "Derived from" (BL 1.x field) but BL 2.0 questions use "**Motivated by**"

**Question**: Why is hypothesis-generator has_derived_from=0.00? Does `_score_hypothesis_generator` check `"Derived from"` while BL 2.0 questions use `**Motivated by**`?
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: Medium

## Evidence

**`_score_hypothesis_generator()` check_block function** (`bl/crucible.py` line 188–197):
```python
def check_block(b: str) -> dict[str, float]:
    return {
        "has_status": 1.0 if re.search(r"PENDING|DONE|Status", b) else 0.0,
        "has_derived_from": 1.0 if "Derived from" in b else 0.0,
        "has_test": 1.0 if re.search(r"Test:|pytest|Simulation path", b) else 0.0,
        "has_verdict_threshold": 1.0
        if ("FAILURE:" in b and "HEALTHY:" in b)
        else 0.0,
        "has_hypothesis": 1.0 if "Hypothesis:" in b else 0.0,
    }
```

The scorer checks for the literal string `"Derived from"` — the BL 1.x lineage field format.

**BL 2.0 question format** (from `projects/bl2/questions.md` — Wave 16+ questions):
```
**Motivated by**: D15.3 — ...
```

BL 2.0 questions use `**Motivated by**` as the lineage field. The literal string `"Derived from"` never appears in any BL 2.0 question block.

**Impact of mismatch**:
- `has_derived_from` weight = 0.35 (highest single weight in the scorer)
- With has_derived_from=0.00 across all BL 2.0 Wave 2+ questions, the 0.35 weight contributes nothing
- Maximum attainable score without has_derived_from: 0.10+0.20+0.25+0.10 = 0.65 weighted, vs possible 1.00
- Confirmed symptom: hypothesis-generator score ≈ 0.1511 (observed in bl2 benchmarks)

**Root cause**: `check_block()` was written for BL 1.x question format and never updated for BL 2.0's field renaming from `Derived from:` to `**Motivated by**:`.

## Fix Specification

**Target file**: `bl/crucible.py`
**Target location**: `_score_hypothesis_generator()` → `check_block()` function (line 191)
**Concrete edit**:
```python
# Before (BL 1.x field name only):
"has_derived_from": 1.0 if "Derived from" in b else 0.0,

# After (supports both BL 1.x and BL 2.0 field names):
"has_derived_from": 1.0 if ("Derived from" in b or "Motivated by" in b) else 0.0,
```

**Verification**: After fix, re-run benchmarks. `has_derived_from` should increase from 0.00 to near 1.00 for BL 2.0 campaigns (all Wave 2+ questions contain `**Motivated by**`). Overall hypothesis-generator score should rise from ~0.15 to ~0.45+.
