# Finding: D18.2 — fix_rows in _score_fix_implementer is de facto scoped to fix questions; no cross-agent contamination in current campaign

**Question**: Does `_score_fix_implementer` scope fix_rows to F-prefix questions only, or do verdicts from other agents inflate/deflate the FIXED/FIX_FAILED ratio?
**Verdict**: HEALTHY
**Severity**: Info

## Evidence

**`_score_fix_implementer` fix_rows filter** (`bl/crucible.py` lines 425-429):
```python
fix_rows = [ln for ln in lines if re.search(r"\t(FIXED|FIX_FAILED)\t", ln)]
```
No question_id prefix filter. Matches any row with FIXED or FIX_FAILED verdict.

**Actual fix_rows content** from current results.tsv (40 rows):
```
col0=F2.1, col1=FIXED (old format — F-prefix fix question)
col0=F2.2, col1=FIXED
...
col0=F-mid.1, col1=FIXED (new format)
col0=F-mid.2, col1=FIXED
col0=F-mid.3, col1=FIXED
col0=D-mid.4, col1=FIXED   ← D-prefix fix finding (Wave-mid dual-mode question)
col0=F-mid.5, col1=FIXED
col0=F17.1,   col1=FIXED
```

**Key observation**: All 40 `fix_rows` are from F-prefix questions (BL 1.x Wave 2) or F/D-wave fix questions (Wave-mid). **No non-fix agent** (diagnose-analyst, compliance-auditor, design-reviewer) produces FIXED or FIX_FAILED verdicts in this campaign.

The one potential exception — `D-mid.4` — is a diagnose question whose finding was marked FIXED (the fix was applied in-session as part of the diagnose question), but the verdict was written as `FIXED` to results.tsv. This is a corner case where a diagnose question doubles as its own fix question. It does not represent contamination from a different agent.

**Fragility note**: In a future campaign where a diagnose question intentionally produces FIXED (e.g., a D-prefix question that self-fixes), these rows would enter fix_rows and slightly dilute fixed_rate. The fragility is theoretical for standard campaign flows where only F-prefix questions produce FIXED.

## Summary

fix_rows is de facto scoped to fix-type questions. No cross-agent contamination in the current campaign. The scorer is functionally correct. The theoretical fragility (D-prefix FIXED rows) is a design consideration for future campaigns, not a current defect.
