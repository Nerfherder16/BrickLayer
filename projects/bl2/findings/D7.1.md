# Finding: D7.1 — _verdict_from_agent_output() checks changes_committed before self_verdict in else-branch

**Question**: Does the else-branch of _verdict_from_agent_output() return HEALTHY on changes_committed > 0 before checking self_verdict, masking BL 2.0 agent verdicts like FIXED?
**Verdict**: DIAGNOSIS_COMPLETE
**Severity**: High
**Mode**: diagnose
**Operational Mode**: diagnose

## Root Cause

`bl/runners/agent.py` `_verdict_from_agent_output()` lines 122-126 (before fix):

```python
else:
    if output.get("changes_committed", 0) > 0:
        return "HEALTHY"          # ← fires before self_verdict for ALL non-hardcoded agents

self_verdict = output.get("verdict", "").upper()
if self_verdict in _ALL_VERDICTS:
    return self_verdict
```

For any agent not named security-hardener/test-writer/type-strictener/perf-optimizer (which includes all BL 2.0 agents: diagnose-analyst, fix-implementer, compliance-auditor, design-reviewer, etc.), the flow is:

1. Check `changes_committed > 0` → if True, return HEALTHY immediately
2. Check `self_verdict` → only reached if changes_committed is 0 or absent

## Impact Analysis

| Agent | Output | Before F7.1 | After F7.1 |
|-------|--------|-------------|------------|
| fix-implementer | {"verdict": "FIXED", "changes_committed": 2} | HEALTHY (wrong) | FIXED |
| fix-implementer | {"verdict": "FIX_FAILED", "changes_committed": 0} | FIX_FAILED (accidental correct) | FIX_FAILED |
| diagnose-analyst | {"verdict": "DIAGNOSIS_COMPLETE"} | DIAGNOSIS_COMPLETE (no changes_committed) | DIAGNOSIS_COMPLETE |
| legacy-agent | {"changes_committed": 1} | HEALTHY | HEALTHY (heuristic preserved) |

The most critical failure: fix-implementer successfully commits a fix, outputs `{"verdict": "FIXED", "changes_committed": 2}`, campaign records HEALTHY. The heal loop sees HEALTHY (not FIXED), misclassifies the outcome. `_NON_FAILURE_VERDICTS` determines whether the loop terminates — HEALTHY is in that set, so the loop does terminate, but the finding is mislabeled HEALTHY instead of FIXED.

## Fix Specification

- **File**: `bl/runners/agent.py`, `_verdict_from_agent_output()`, else-branch
- **Change**: Move `self_verdict` check before `changes_committed` heuristic
- **Verification**: `_verdict_from_agent_output("fix-implementer", {"verdict": "FIXED", "changes_committed": 2})` → FIXED
