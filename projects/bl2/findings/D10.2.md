# Finding: D10.2 — synthesizer.py parse_recommendation() false-triggers STOP on Dead Ends prose

**Question**: Does parse_recommendation() in synthesizer.py incorrectly return STOP when Dead Ends prose contains "stop"?
**Verdict**: FAILURE
**Severity**: High
**Mode**: code_audit
**Operational Mode**: diagnose

## Root Cause

### parse_recommendation() — full-text first-match scan

```python
def parse_recommendation(synthesis_text: str) -> str:
    for line in synthesis_text.splitlines():
        upper = line.upper()
        if "STOP" in upper:      # ← checked FIRST
            return "STOP"
        if "PIVOT" in upper:
            return "PIVOT"
        if "CONTINUE" in upper:
            return "CONTINUE"
    return "CONTINUE"
```

The function scans ALL lines of synthesis_text from top to bottom and returns on first match.

### The prompt template plants the trigger

```python
## Dead Ends
List paths that have been exhausted with no findings worth pursuing further:
- [what was tested]: [Q IDs] found nothing actionable — stop probing here
```

Line 167 of synthesizer.py instructs Claude to write "**stop** probing here" at the end of each Dead Ends bullet. When Claude follows this instruction, the synthesis output contains lines like:

```
- auth endpoint rate limiting: Q3.5 found nothing actionable — stop probing here
```

### Failure trace

Synthesis text structure (in order):
1. `# Campaign Synthesis — Wave N`
2. `## Core Hypothesis Verdict` — paragraph (may contain "continue")
3. `## Validated Bets` — list
4. `## Dead Ends`
5. `  - some test: Qx.y found nothing actionable — stop probing here`  ← **STOP triggered here**
6. `## Unvalidated Bets`
7. `## Recommended Next Action`
8. `CONTINUE`  ← never reached

When any dead end exists, `parse_recommendation()` returns `"STOP"` before reaching the actual recommendation.

### Additional trigger vectors

- Line 180-181 of prompt: `STOP — core hypothesis confirmed or refuted...` — if Claude reproduces this explanation in output, it also triggers
- Any narrative prose discussing "stopping" in Validated Bets or Core Hypothesis sections also triggers

### Impact

Any campaign with at least one Dead End in its synthesis will be prematurely terminated with "STOP" regardless of the actual recommendation. Since campaigns naturally accumulate dead ends over multiple waves, this bug escalates — later waves are increasingly likely to produce false STOP signals.

The synthesizer is called from the BL 2.0 overseer/orchestrator to decide whether to continue. A false STOP terminates the campaign entirely.

## Verdict: FAILURE

parse_recommendation() scans all lines for "STOP" before reaching the designated recommendation section. The synthesis prompt template guarantees "stop probing here" appears in Dead Ends, causing false STOP returns whenever any dead end is documented. Fix target: F10.2.
