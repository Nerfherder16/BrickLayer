# Finding: D17.1 — All remaining background-agent spawns in run_campaign() are mode-insensitive; no additional guards needed

**Question**: Do any background-agent spawns other than peer-reviewer in run_campaign() (campaign.py) misfire for code_audit questions due to missing mode guards?
**Verdict**: HEALTHY
**Severity**: Info

## Evidence

**_spawn_agent_background call sites inside the per-question loop** (`for i, question in enumerate(pending, 1):`):

| Trigger | Agent | Mode guard | Mode-sensitive? |
|---------|-------|------------|----------------|
| Every question | peer-reviewer | ✅ `if question.get("mode") not in ("code_audit",):` (F-mid.2) | YES — guarded |
| `questions_done % 5 == 0` | forge-check | None | **NO** — inventories agent fleet; reads findings; never executes the question's test field |
| `questions_done % 10 == 0` | agent-auditor | None | **NO** — reads agents/findings/results for fleet health report; never touches test fields |
| `questions_done % 10 == 0` (conditional) | overseer | None | **NO** — reviews underperforming agent instruction files; never executes question tests |

**Analysis per agent:**

**forge-check**: Context string is `"Inventory the agent fleet, scan the 5 most recent findings, check all PENDING questions for missing agents."` — this is a fleet inventory action. The agent reads agent definition files and finding files, not the current question's test field. Fires every 5 questions regardless of mode. Mode-insensitive: the fleet check is equally valid for code_audit and agent questions.

**agent-auditor**: Context string is `"Read all agents, findings, and results. Write fleet health report."` — this reads findings/*.md and results.tsv for health scoring. Never attempts to execute test commands. Mode-insensitive.

**overseer**: Triggered only if `agent_db.get_underperformers()` returns non-empty. Context includes `underperforming agents` list and instructions to improve their instruction files. Mode-insensitive.

**All post-loop spawns** (synthesizer-bl2, mcp-advisor, skill-forge, and others at lines 675-776): These fire after the question loop terminates entirely. They operate on the full findings corpus, not individual question test fields. All mode-insensitive.

## Summary

The F-mid.2 fix (peer-reviewer guard) is the only mode guard needed. The remaining background-agent spawns (forge-check, agent-auditor, overseer, synthesizer-bl2, mcp-advisor, skill-forge) all perform fleet management or synthesis operations that are independent of whether the questions were code_audit or agent mode. No additional guards required.
