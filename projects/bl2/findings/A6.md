# Finding: A6 — operational_mode defaults to "diagnose" (checklist item A6)

**Question**: `bl/questions.py` `parse_questions()` must set `operational_mode` to `"diagnose"` as default when the field is absent from a question block.
**Verdict**: COMPLIANT
**Severity**: Info
**Mode**: agent

## Evidence

`bl/questions.py` line 76:
```python
"operational_mode": fields.get("operational_mode", "diagnose"),
```

Default value is `"diagnose"` when `**Operational Mode**:` field is absent from the question block.

This satisfies the BL 2.0 backwards-compatibility contract: existing BL 1.x questions without an `Operational Mode` field will default to `"diagnose"` mode, which loads `modes/diagnose.md` as mode context.

## Mitigation Recommendation

No action required.
