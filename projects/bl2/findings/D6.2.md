# Finding: D6.2 — results.tsv format mismatch: get_question_status() returns PENDING for all rows

**Question**: Does the manually-maintained results.tsv use a format that get_question_status() cannot parse, causing all questions to appear PENDING?
**Verdict**: FAILURE
**Severity**: Medium
**Mode**: diagnose
**Operational Mode**: diagnose

## Root Cause

Two conflicting TSV formats:

**BL 1.x manual format** (current results.tsv header):
```
commit	question_id	verdict	primary_metric	key_finding	scenario_name
N/A	D1	FAILURE	N/A	...	Code audit: ...
```
Column 0 = "N/A" (commit placeholder)

**BL 2.0 engine format** (written by `update_results_tsv()`):
```
question_id	verdict	failure_type	summary	timestamp
D1	FAILURE		...	2026-03-16T...
```
Column 0 = question_id

`get_question_status(qid)` checks:
```python
if parts and parts[0] == qid:
    return parts[1].strip()
```

For BL 1.x format rows: `parts[0] = "N/A"` ≠ `"D1"` → no match → returns PENDING.

## Evidence

```
python -c "...get_question_status('D1')..."
→ PENDING

python -c "...get_question_status('F4.3')..."
→ PENDING

python -c "...get_question_status('V5.1')..."
→ PENDING
```

All 44 rows have "N/A" as column 0, so no question is ever found → every question is PENDING.

## Impact

In this self-audit campaign, status tracking via the BL engine is non-functional. `get_next_pending()` would return D1 on every invocation, causing infinite re-execution of D1. The campaign has been running manually (me answering questions directly in this session), so the functional impact has been zero for the current campaign run. However, if `campaign.py` were invoked to run this question bank, it would re-run D1 indefinitely.

## Fix Specification

- **File**: `projects/bl2/results.tsv`
- **Change**: Rewrite TSV in BL 2.0 engine format: header = `question_id\tverdict\tfailure_type\tsummary\ttimestamp`; each row = `{qid}\t{verdict}\t\t{key_finding_truncated}\t2026-03-16T00:00:00Z`
- **Verification**: `get_question_status("D1")` returns "FAILURE"; `get_question_status("F4.3")` returns "FIXED"; all answered questions show non-PENDING status
