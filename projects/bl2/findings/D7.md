# Finding: D7 — _STORE_VERDICTS in recall_bridge.py defined inside function body

**Question**: In `bl/recall_bridge.py`, `_STORE_VERDICTS` is defined as a local set inside `store_finding()` rather than as a module-level frozenset.
**Verdict**: WARNING
**Severity**: Medium
**Mode**: agent

## Evidence

`bl/recall_bridge.py` lines 52-68:
```python
def store_finding(question: dict, result: dict, project: str = "") -> bool:
    ...
    # Only store significant verdicts
    _STORE_VERDICTS = {          # ← local set, not module-level frozenset
        "FAILURE",
        "WARNING",
        "DIAGNOSIS_COMPLETE",
        "FIXED",
        "FIX_FAILED",
        "PROMISING",
        "BLOCKED",
        "IMMINENT",
        "PROBABLE",
        "IMPROVEMENT",
        "REGRESSION",
        "NON_COMPLIANT",
        "PARTIAL",
        "ALERT",
        "DEGRADED",
    }
    if verdict not in _STORE_VERDICTS:
        return False
```

This is a regular mutable `set` (not `frozenset`) created fresh on every `store_finding()` call. Functionally correct — the 15 verdicts are the right ones to store. However:

1. **No contract enforcement**: There is no corresponding constant in `constants.py` defining which verdicts should be stored. Drift from the intended list is invisible.
2. **Not importable for testing**: Unit tests cannot import `_STORE_VERDICTS` to assert its contents without calling `store_finding()`.
3. **Micro-inefficiency**: A new set object is created on every call (minor; set construction is O(n) but n=15).
4. **Naming convention violation**: Module-private constants conventionally use the `_ALL_CAPS` name at module level.

The set contains 15 verdicts. No obviously missing verdicts detected — FIX_FAILED is absent but that may be intentional (FIX_FAILED is terminal-negative and storing it might be useful for debugging).

## Mitigation Recommendation

Move `_STORE_VERDICTS` to module level as a frozenset:
```python
_STORE_VERDICTS: frozenset[str] = frozenset({
    "FAILURE", "WARNING", "DIAGNOSIS_COMPLETE",
    "FIXED", "FIX_FAILED", "PROMISING", "BLOCKED",
    "IMMINENT", "PROBABLE", "IMPROVEMENT", "REGRESSION",
    "NON_COMPLIANT", "PARTIAL", "ALERT", "DEGRADED",
})
```

Optionally add a corresponding `STORE_VERDICTS` constant to `constants.py` to create a testable contract.
