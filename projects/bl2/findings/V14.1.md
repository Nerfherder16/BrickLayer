`quality.py` is a pure math function (`estimate_remediation_feasibility()`). `action_type` is compared only against `"amnesty"` — a domain-neutral remediation type. No verdict vocabulary, no ID parsing. Works identically under BL 1.x and BL 2.0.

**Note**: `corpus_size` defaults to `1`, which produces inflated `max_delta` if a caller omits it but passes `n_affected > 0`. Not a BL version issue — pre-existing API design; not a regression risk.
