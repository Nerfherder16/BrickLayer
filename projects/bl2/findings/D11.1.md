# Finding: D11.1 — crucible.py scoring functions return 0.0 for all BL 2.0 campaigns

**Question**: Do crucible.py scoring functions produce meaningful scores for BL 2.0 question banks?
**Verdict**: FAILURE
**Severity**: Medium
**Mode**: code_audit
**Operational Mode**: diagnose

## Root Cause

### _score_hypothesis_generator() — "## Wave N" header gate

```python
wave_match = re.search(r"## Wave [2-9]\d*", text)
if not wave_match:
    return AgentScore(
        "hypothesis-generator", 0.0, {}, "No Wave 2+ sections found in questions.md"
    )
```

BL 2.0 questions.md does NOT have "## Wave N" section headers — wave information is encoded in question ID prefixes (D7.1, F8.1, etc.). This check always fails → function returns 0.0 before any per-question scoring.

### _score_hypothesis_generator() — Q-only block split (line 172-173)

```python
blocks = re.split(r"(?=^## Q[2-9]\d*\.\d+)", wave2_text, flags=re.MULTILINE)
blocks = [b.strip() for b in blocks if re.match(r"## Q[2-9]\d*\.\d+", b.strip())]
```

Even if the Wave header check passed, only Q-prefixed IDs would be counted.

### _score_question_designer() — same two bugs

```python
wave2_match = re.search(r"^## Wave [2-9]", text, re.MULTILINE)
wave1_text = text[: wave2_match.start()] if wave2_match else text  # → full text if no match

blocks = re.split(r"(?=^## Q\d+\.\d+)", wave1_text, flags=re.MULTILINE)
blocks = [b.strip() for b in blocks if re.match(r"## Q\d+\.\d+", b.strip())]

if len(blocks) < 5:
    return AgentScore(
        "question-designer", 0.0, {}, f"Only {len(blocks)} Wave 1 questions found (need >= 5)"
    )
```

With BL 2.0 IDs (D1.1, D1.2, ...), zero Q-prefix blocks are found → returns 0.0.

### Impact

`run_all_benchmarks()` calls all scorers and `get_all_statuses()` aggregates results. For any BL 2.0 campaign:
- hypothesis-generator score = 0.0 (false underperformer)
- question-designer score = 0.0 (false underperformer)
- Overseer may flag these agents as failing and trigger unnecessary repair/replacement

The crucible benchmark results are completely meaningless for BL 2.0 campaigns.

## Verdict: FAILURE

Both scoring functions return 0.0 for BL 2.0 campaigns due to (1) missing "## Wave N" section headers that BL 2.0 doesn't use, and (2) Q-only block detection regex that excludes all BL 2.0 question IDs. Fix target: F11.1.
