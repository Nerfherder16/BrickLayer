## Analytical Derivation

### Recalibrated formula chain at DN

```
base_accuracy     = 0.65 * 0.875 + 0.35 * 0.50            = 0.74375  (fixed)
novelty_mult(DN)  = piecewise:
                    DN ≤ 0.30:  1.0 - DN * 0.10
                    DN ≤ 0.60:  0.97 - (DN - 0.30) * 0.35
                    DN > 0.60:  0.865 - (DN - 0.60) * 0.55
accuracy(DN)      = base_accuracy * novelty_mult(DN)
drift(DN)         = 1.0 - accuracy(DN)
novelty_discount  = max(0.05, 1.0 - DN * 0.90)            [Change 2]
peer_correction   = 1.0 * drift(DN) * 0.40 * novelty_discount  [Change 1]
effective_accuracy(DN) = min(0.98, accuracy(DN) + peer_correction(DN))
```

### Cliff location formula

Campaign yield (expectation, 4-wave) = effective_accuracy(DN) × mean_uniqueness

```
Wave uniqueness: [1.00, 0.85, 0.70, 0.55] (WAVE_SATURATION_RATE=0.15, 7 q/wave)
Mean uniqueness over 4 waves = (1.00 + 0.85 + 0.70 + 0.55) / 4 = 0.7750

Cliff condition: effective_accuracy(DN) × 0.7750 < CAMPAIGN_YIELD_WARNING (0.45)
→ effective_accuracy(DN) < 0.45 / 0.7750 = 0.5806
```

## Root Cause: Three Compounding Factors

### Factor 1 — High specialist-weighted base accuracy (dominant)

The recalibration reduced generalist accuracy from 0.625 → 0.50 (Change 3), but left specialist accuracy at 0.875. With AGENT_SPECIALIZATION_RATIO=0.65:

```
base_accuracy = 0.65 * 0.875 + 0.35 * 0.50 = 0.74375
(pre-recal:   = 0.65 * 0.875 + 0.35 * 0.625 = 0.78750)
```

The 5.5% reduction in base_accuracy is partially compensated by the mixed fleet. Specialists still anchor the fleet accuracy high.

### Factor 2 — Novelty discount slope 0.90 leaves substantial peer correction at moderate DN (primary mechanism)

Change 2 steepened the novelty_discount slope from 0.60 → 0.90 and lowered the floor from 0.20 → 0.05. However at DN=0.70:

```
Old formula: max(0.20, 1 - 0.70 * 0.60) = max(0.20, 0.580) = 0.580
New formula: max(0.05, 1 - 0.70 * 0.90) = max(0.05, 0.370) = 0.370
```

The new formula reduces the discount at DN=0.70 from 0.580 to 0.370 — a meaningful reduction, but at slope=0.90 the discount only reaches the floor (0.05) at DN = (1 - 0.05) / 0.90 = **1.056** (beyond the DN=1.0 scale). So at every test point DN ≤ 1.0, the discount formula is still in its linear regime, providing real peer correction.

Peer correction at DN=0.70: `1.0 * 0.3976 * 0.40 * 0.370 = 0.0588`
This lifts effective_accuracy from accuracy=0.6024 to 0.6613 — a 9.8% boost that pushes the campaign yield firmly above threshold.

### Factor 3 — Wave uniqueness saturation is the dominant yield driver at nominal DN (structural)

At nominal (DN=0.35), Wave 4 already has uniqueness=0.55, producing per-wave yield of only 0.55 × 0.79 = 0.43. The wave_novelty_floor trigger (0.15) is far below this, so saturation governs yield before accuracy collapse does. For the accuracy-driven cliff to emerge, effective_accuracy must drop below 0.5806 — requiring the novelty penalty to compound with substantially reduced peer correction.

**The cliff displacement is a calibration artifact, not a structural feature.** The three recalibrations collectively lowered effective_accuracy by ~0.07 across the DN range, but the new floor of the effective_accuracy curve (determined by the slowness of `_novelty_penalty()` slope at DN > 0.60: -0.55 per DN unit) still produces effective_accuracy ≥ 0.58 until DN ≈ 0.86-0.95.

### Why Q6.1 predicted the cliff at 0.65-0.75

Q6.1's prediction was made against the original (pre-recalibration) parameters. Under the original `max(0.20, 1-DN*0.60)` formula, the discount reaches its floor of 0.20 at DN = (1-0.20)/0.60 = **1.333** — also never hitting the floor in the DN=[0,1] range. However, the original peer correction rate (0.55) and higher generalist accuracy (0.625) produced a higher starting effective_accuracy and a different cliff location. Q6.1's prediction was not based on post-recalibration algebra.

## Proposed Fix: Steepen Novelty Discount Slope to k=1.20

**Single-line change to `_peer_correction()` in `simulate.py`:**

```python
# Current (Change 2 recalibration):
novelty_discount = max(0.05, 1.0 - DOMAIN_NOVELTY * 0.90)

# Proposed:
novelty_discount = max(0.05, 1.0 - DOMAIN_NOVELTY * 1.20)
```

This slope reaches the floor at DN = (1 - 0.05) / 1.20 = **0.792**, meaning peer correction is effectively floored before the expected cliff range.

### Structural note: Hard minimum cliff with discount-only fix

The minimum achievable cliff via novelty_discount tuning alone is DN ≈ 0.753, where accuracy(DN) alone = 0.5806 (required EA). At DN < 0.753, peer correction is irrelevant (accuracy already below threshold). The proposed k=1.20 puts the cliff at 0.780, which is 0.027 above the hard minimum. Further steepening (k > 1.20) yields diminishing returns — the cliff converges to DN=0.774 (floor=0.05 limit) or DN=0.753 (floor=0.00 limit).
