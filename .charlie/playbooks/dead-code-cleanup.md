# Playbook: Dead Code Cleanup

## Trigger

- **Schedule:** Every Monday at 09:00 (cron: `0 9 * * 1`)
- **Agent:** `economizer`
- **Conductor ID:** `dead-code-cleanup`

## Purpose

Automatically identify and remove unused code (dead code) from the codebase on a weekly
cadence. Reduces technical debt and keeps the codebase lean.

## Inputs

- **Scope:** Entire codebase rooted at the current working directory
- **File patterns:** `**/*.py`, `**/*.ts`, `**/*.js` (excludes `node_modules/`, `.venv/`, `__pycache__/`)
- **Reference:** `masonry/agent_registry.yml` — agent inventory for cross-reference

## Expected Output

The economizer agent must emit a structured findings list, one entry per dead code candidate:

```
FILE:LINE  reason  symbol_name
```

Example:
```
masonry/src/utils/legacy.py:42  unused_import  numpy
masonry/src/routing/old_router.py:17  unreachable_function  _legacy_route
```

## Success Criteria

| Condition | Action |
|-----------|--------|
| Fewer than 3 findings | Log "No action needed" and exit |
| 3 or more findings | Create a pull request (see PR format below) |

## PR Format

- **Title:** `chore: remove dead code (weekly cleanup)`
- **Branch:** `charlie/dead-code-YYYY-MM-DD`
- **Body:**

```markdown
## Weekly Dead Code Cleanup

Automated cleanup triggered by Charlie conductor (Monday 09:00).

### Removed items

| File | Line | Symbol | Reason |
|------|------|--------|--------|
| ... | ... | ... | ... |

_Generated by the `economizer` agent._
```

## Safety Rules

The economizer agent MUST NOT delete or modify:

1. **Test files** — any file matching `tests/**`, `test_*.py`, `*_test.py`, `*.test.ts`
2. **Public API symbols** — functions or classes exported in `__init__.py` or `index.ts` at
   package root level
3. **`constants.py`** — project invariants, never auto-edited
4. **`simulate.py`** — simulation entry point, protected
5. Any file listed under `docs/` or `project-brief.md`

If a candidate violates a safety rule, skip it and log a warning — do not include it in the PR.

## Rollback

If the PR build fails after merging:

1. Revert the merge commit: `git revert -m 1 <merge_sha>`
2. Open a follow-up issue linking to the failure log
3. Disable the `dead-code-cleanup` schedule entry in `conductor.json` until resolved
