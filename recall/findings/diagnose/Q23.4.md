## Finding 3: New Bug — Active Memory Neo4j Sync Gap

The 50 visible mismatches represent a **different issue** from Q21.1: active memories where Neo4j importance was not properly initialized at creation.

**Pattern**:
- Memory created recently (2026-03-15T05:33)
- Qdrant: importance = 0.7 (correct initial importance)
- Neo4j: importance = 0.0 (not initialized, or failed to sync at creation)
- `superseded_by: None` — these are active memories not superseded ones

**Impact on graph traversal**: `find_related()` uses `importance` from Neo4j for ranking. If 50 active memories have `neo4j_importance=0.0`, they'll rank at the bottom of graph traversal results even though they have correct Qdrant importance of 0.7+. This is a real retrieval quality issue.

**Root cause hypothesis**: The memory creation path may fail to initialize Neo4j `importance` if Neo4j node creation is partially failing or if the importance field isn't passed to the Neo4j write. Could be related to the recent system restarts (Q22.9 outage window).
