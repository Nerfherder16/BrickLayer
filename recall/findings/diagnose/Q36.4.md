## Analysis

### Why Processed + Skipped != Total Memories

`scroll_all()` (used by the amnesty endpoint) filters to non-superseded memories only (`superseded_by IS NULL`). Of the 21,555 total Qdrant points, ~15,429 have `superseded_by` set and are excluded. The amnesty operates on the active-memory pool of ~6,100-6,200 memories only.

This is correct behavior: superseded memories are dead-end versions and should not be rehabilitated.

### Amnesty Endpoint Logic (confirmed from source)

The endpoint scans all non-superseded memories and:
- **Skips** (importance >= 0.4): already healthy — most of the 1,750 skipped
- **Skips** (pinned or permanent): already protected
- **Skips** (ephemeral): intentionally short-lived
- **Processes** (durable/None, importance < 0.4): checks if boost formula produces higher value
- **Boosts** only if `new_importance > old_importance`
