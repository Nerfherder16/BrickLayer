## Risk Assessment

**Risk: over-inflation of legitimately floor-clamped memories (initial_importance = 0.05)**

Branch 2 applies `max(importance, 0.2)` to all durable/permanent memories regardless of their `initial_importance`. A memory written with `initial_importance = 0.05` (low-priority, intentionally unimportant) that naturally decayed to floor would be boosted to 0.2 by fixed rehabilitate — an unwarranted inflation by 4× floor value.

- **Rehabilitate branch 2**: boosts to min 0.2, regardless of initial_importance
- **Amnesty**: boosts to min 0.3, regardless of initial_importance

The over-inflation risk exists in both endpoints. Amnesty carries a larger magnitude risk (0.3 vs 0.2) and already ran. Fixed rehabilitate's risk is the same class, smaller magnitude.

Branch 1 risk: a memory with initial_importance=0.05 but ac>=3 gets boosted to max(0.025, 0.3) = 0.3. This is arguably correct — repeated access revealed utility not predicted at write time.

**Mitigation gap**: neither rehabilitate nor amnesty checks `initial_importance <= 0.05` before boosting. A targeted fix could add `if initial_importance is not None and initial_importance <= 0.05: skip` to branch 2, but this was not done and mirrors an existing gap in amnesty.

**Risk severity: LOW** — magnitude is 0.05→0.2 (rehabilitate) vs 0.05→0.3 (amnesty already applied). The window for newly-created low-importance memories to be incorrectly boosted is narrow (they must decay to below 0.05 first, which takes significant time for low but non-zero importance values).
