## Finding 2: Hygiene Is Active-Only Soft-Delete

**scroll_hygiene_candidates conditions** (`qdrant.py:718-730`):
```python
must = [
    FieldCondition(key="pinned", match=MatchValue(value="false")),
    FieldCondition(key="importance", range=Range(lt=0.3)),
    FieldCondition(key="access_count", range=Range(lte=0)),
    IsNullCondition(is_null=PayloadField(key="superseded_by")),  ← active-only
    FieldCondition(key="created_at", range=DatetimeRange(lt=cutoff_iso)),
]
```

Hygiene ONLY targets active memories (`IS NULL on superseded_by`). The ~15,099 consolidation-superseded memories are **explicitly excluded** from hygiene.

**Archive mechanism** (hygiene.py:47):
```python
await qdrant.update_payload(memory_id, {"superseded_by": "auto-archive"})
```
This is a soft-delete — sets `superseded_by="auto-archive"` in Qdrant. The memory stays in Qdrant (as a superseded record) and the Neo4j node is NOT deleted. No DETACH DELETE.
