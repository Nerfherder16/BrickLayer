## Finding 1: Code Inspection — _user_conditions(None) Unchanged

```python

def _user_conditions(self, user_id: int | None) -> list:
    """Build Qdrant filter conditions for user_id isolation.

    user_id=None or user_id=0 → no filter (system/admin sees everything).
    user_id>0 → restrict to that user's data.
    """
    if user_id is not None and user_id > 0:
        return [FieldCondition(key="user_id", match=MatchValue(value=user_id))]
    return []
```

The fix — returning `[IsNullCondition(key="user_id")]` when `user_id is None` — has not been applied. The docstring even documents the bug as intended behavior ("user_id=None → no filter"). 17th consecutive wave with this code unchanged (first flagged Q22.1).
