# Finding: Q25.4 — Admin purge endpoint targets ACTIVE low-quality memories only (include_superseded=False default); WARNING; 1,081 active memories purge-eligible (importance≤0.15, age≥7d, access=0); consolidation-superseded GC gap persists — no purge mechanism for superseded memories exists

**Question**: Manual GC first batch — use the admin purge endpoint to delete a sample of 30+ day old superseded memories and verify the operation is safe.
**Verdict**: WARNING — The `/admin/memory/purge` endpoint does NOT target consolidation-superseded memories. It uses `qdrant.scroll_all()` with default `include_superseded=False`, which applies an `IS NULL on superseded_by` filter. Consolidation-superseded memories (15,099) are explicitly excluded from its scope. The endpoint is designed for cleaning up low-quality ACTIVE memories, not GC for superseded ones. The Q24.6 GC gap (no DETACH DELETE for superseded memories) remains fully open.
**Severity**: WARNING — The purge endpoint is useful for active-memory cleanup (1,081 eligible) but the superseded-memory GC gap from Q24.6 is unaddressed by any available endpoint or cron.

---

## Finding 1: Purge Endpoint Code Inspection — Active-Only Scope

**Location**: `src/api/routes/admin.py:1755`

```python
memories = await qdrant.scroll_all()   ← no include_superseded argument
```

**`scroll_all()` default**:
```python
async def scroll_all(
    self,
    include_superseded: bool = False,   ← defaults to active-only
    ...
):
    if not include_superseded:
        conditions.append(
            IsNullCondition(
                is_null=PayloadField(key="superseded_by"),
            )
        )
```

The purge endpoint calls `scroll_all()` without arguments, which defaults to `include_superseded=False`. This means the IS NULL filter is applied and ALL 15,099 consolidation-superseded memories are invisible to the purge logic. Purge only processes active memories.

```
POST /admin/memory/purge?max_importance=0.15&min_age_days=7&dry_run=true

```

1,081 active memories are eligible for purge under default settings (importance ≤ 0.15, age ≥ 7 days, access_count = 0, not pinned, not permanent). Sample domains: development, infrastructure, networking, general, security — primarily observer-created low-value notes.

The purge deletion logic (when `dry_run=false`) correctly does DETACH DELETE in both Qdrant and Neo4j:
```python
await qdrant.delete(c["id"])
async with neo4j.driver.session() as session:
    await session.run("MATCH (m:Memory {id: $id}) DETACH DELETE m", id=c["id"])
```
This is the correct GC operation, but applied to active memories only.

## Finding 3: Superseded Memory GC Gap — No Available Mechanism

| GC Mechanism | Targets | Status |
|-------------|---------|--------|
| `/admin/memory/purge` | Active low-quality (include_superseded=False) | Available — does NOT target superseded |
| `/admin/memory/purge-domain` | All memories in a domain | Available — domain-scoped |
| Hygiene cron (4am daily) | Active memories with IS NULL superseded_by | Running — 0 archived; age threshold not met |
| Direct Qdrant+Neo4j deletion | Targeted by ID | Available manually; no endpoint for bulk superseded GC by criteria |
| Scheduled DETACH DELETE cron | Superseded memories > 30 days | **NOT IMPLEMENTED** |

The 15,099 consolidation-superseded memories (estimated 11,000+ aged > 30 days) have no automated or semi-automated cleanup path. The only option is building a new endpoint or cron that calls `scroll_all(include_superseded=True)`, filters for `superseded_by IS NOT NULL AND invalid_at < cutoff`, and performs individual DETACH DELETE for each eligible memory.

**Verdict: WARNING** — The `/admin/memory/purge` endpoint targets active low-quality memories (1,081 eligible) and would safely delete both Qdrant and Neo4j records. However, it does NOT address the Q24.6 GC gap: consolidation-superseded memories (15,099) are excluded by the `include_superseded=False` default in `scroll_all()`. The Q21.3 / Q24.6 GC cron recommendation remains fully unimplemented. A GC mechanism for superseded memories requires a new endpoint or cron that passes `include_superseded=True` and filters by `invalid_at < (now - 30d)`.
