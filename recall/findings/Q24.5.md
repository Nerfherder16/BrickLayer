## Finding 1: Code Inspection — user_id Absent from Memory() Constructor

**Location**: `src/core/consolidation.py:235-249`

```python
merged = Memory(
    content=merged_content,
    content_hash=content_hash(merged_content),
    memory_type=cluster[0].memory_type,   # ← copies from source
    source=MemorySource.CONSOLIDATION,
    domain=cluster[0].domain,             # ← copies from source
    tags=list(all_tags),
    importance=min(1.0, avg_importance + 0.1),
    stability=min(1.0, max(m.stability for m in cluster) + 0.2),
    confidence=max_confidence,
    access_count=total_access,
    parent_ids=[m.id for m in cluster],
    durability=best_durability,
    initial_importance=min(1.0, avg_importance + 0.1),
    # ← NO user_id= argument
)
```

The memory_type, domain, and tags are copied from the source cluster members, but `user_id` is not. The Memory model defaults `user_id` to None when not provided.

**The fix**: Add `user_id=cluster[0].user_id` (assuming same-user consolidation) or `user_id=cluster[0].user_id if len({m.user_id for m in cluster}) == 1 else None` (multi-user safety).

## Finding 2: Implication for User Attribution

Every consolidation event (~796/day) creates a merged memory with user_id=None:
- The merged memory is NOT counted in the original user's memory count
- If all originals for a user are consolidated, that user "disappears" from `get_distinct_user_ids()`
- The merged memory becomes system-owned (user_id=None)
- The user's data is preserved in content but attribution is lost
