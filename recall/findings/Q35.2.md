# Q35.2 — Hygiene Cron Task: Is It Registered and Running?

**Status: WARNING**

## Question

Is the hygiene cron task registered in the `cron_jobs` list? Has it ever run?

## Findings

### Registered in cron_jobs

`run_hygiene` is present in `WorkerSettings.cron_jobs` in `src/workers/main.py:389–393`:

```python
# Auto-archive stale memories daily at 4:00 AM
cron(
    run_hygiene,
    hour=4,
    minute=0,
),
```

Scheduled: daily at 04:00 UTC, no weekday restriction. Also appears in `WorkerSettings.functions` list (line 339).

### What hygiene does

`src/workers/hygiene.py` — soft-archives memories matching ALL of:
- `access_count == 0`
- `importance < 0.3`
- `created_at < now - 30 days`
- NOT pinned, NOT permanent durability

Criteria are applied server-side via `qdrant.scroll_hygiene_candidates(cutoff_iso)`. On success it logs `action=hygiene_run` to the audit table with `archived` and `candidates_scanned` counts.

### Audit Log

```json
{
  "entries": [
    {
      "id": 379766,
      "timestamp": "2026-03-15T14:47:35.212441+00:00",
      "action": "hygiene_run",
      "memory_id": "system",
      "actor": "hygiene",
      "details": {"archived": 0, "candidates_scanned": 0}
    }
  ],
  "count": 1
}
```

One run confirmed — today, 2026-03-15 at 14:47 UTC. However:
- `candidates_scanned: 0` — zero memories were evaluated
- `archived: 0` — zero memories were archived

This suggests `scroll_hygiene_candidates` returned an empty result set. Either no memories currently meet all hygiene criteria (possible — the system is young and most memories have been accessed), or there is a defect in the Qdrant scroll filter that causes it to return nothing.

### Concern

A single run with 0 candidates scanned on a system with thousands of stored memories is suspicious. The hygiene criteria require `access_count == 0`, `importance < 0.3`, and `age > 30 days`. If most memories have at least one access event this result is plausible. However it warrants verification that `scroll_hygiene_candidates` is applying the filter correctly rather than short-circuiting.

## Verdict

**WARNING** — registered and ran once, but the single run archived 0 memories with 0 candidates scanned. The task is functional in terms of registration and execution, but the zero-candidate result needs explanation before hygiene can be considered operationally confirmed.

## Recommended Next Step

Verify `qdrant.scroll_hygiene_candidates()` filter logic to confirm it correctly applies all three criteria (access_count, importance, age). Cross-check against actual memory population age distribution.
