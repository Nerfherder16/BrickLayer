## Finding 4: llm.py generate() signature

`src/core/llm.py` lines 51–60:

```python
async def generate(
    self,
    prompt: str,
    *,
    model: str | None = None,
    temperature: float = 0.3,
    format_json: bool = False,
    system: str | None = None,
    timeout: float | None = None,
) -> str:
```

The `timeout` parameter is passed directly to `httpx` at line 99:
```python
response = await self.client.post(
    f"{self.settings.ollama_host}/api/generate",
    json=body,
    timeout=timeout,
)
```

`httpx.TimeoutException` is caught and re-raised as `LLMError` with the effective timeout value logged. When `timeout=None` is passed (the default), httpx falls back to the client-level timeout (`self.settings.signal_detection_timeout`), but all callsites now supply an explicit value so this fallback is never reached in production paths.
