### Why So High?

The mark_superseded bug creates a mismatch every time a memory is superseded during consolidation. Consolidation runs continuously and generates ~15-30 superseded memories per run (based on consolidation throughput from previous waves). With consolidation running ~6-8×/day, that's ~90-240 superseded events/day → ~3.75-10/hour. This doesn't fully explain 41/hour.

The additional contribution comes from **decay divergence**: every decay cycle applies the importance multiplier to Qdrant but doesn't sync to Neo4j for superseded/stale memories. With double-decay still active, each 6-hour cycle generates fresh divergence for all 1027+ mismatched memories, widening existing gaps rather than creating new ones. However, the reconcile counts distinct memories, not magnitude — so 41 NEW mismatch events per hour means 41 unique memories per hour are crossing the mismatch threshold (>some epsilon) for the first time.
