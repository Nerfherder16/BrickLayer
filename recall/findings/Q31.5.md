## Finding 3: mark_superseded source code

File: `C:/Users/trg16/Dev/Recall/src/storage/neo4j_store.py` lines 384–396:

```python
async def mark_superseded(self, memory_id: str, superseded_by: str):
    """Mark a memory node as superseded in the graph."""
    async with self.driver.session() as session:
        await session.run(
            """
            MATCH (m:Memory {id: $id})
            SET m.superseded_by = $superseded_by,
                m.invalid_at = $invalid_at
            """,
            id=memory_id,
            superseded_by=superseded_by,
            invalid_at=datetime.now(UTC).isoformat(),
        )
```

`mark_superseded` sets `superseded_by` and `invalid_at` on the Neo4j node but does **not** set `importance = 0.0` in Neo4j. The `update_importance` method (lines 424–434) exists and correctly updates Neo4j when called — the mismatch is that `mark_superseded` does not call it. This is the unpatched root cause: every superseded memory creates a new Neo4j vs Qdrant importance mismatch.

**Verdict: WARNING** — The reconcile repair mechanism works correctly and completely when explicitly invoked: `POST /admin/reconcile?repair=true` cleared all 261 mismatches in a single run (0 remaining). This meets the HEALTHY threshold for the repair capability itself. However, the scheduled reconcile runs are scan-only (`repairs_applied=0` in every prior entry) and have never healed a single mismatch. Mismatches accumulate continuously (179 → 264 over the observation window, ~+47%) and are only cleared via manual intervention. The underlying root cause (`mark_superseded` not zeroing Neo4j importance) remains unpatched. The system is not self-healing; mismatches will re-accumulate from zero until the source is fixed or the scheduler is updated to use `repair=true`.

**Actionable recommendations**:
1. (Immediate) Schedule the reconcile cron job with `repair=true` so mismatches are auto-drained on each run.
2. (Root fix) Patch `mark_superseded` to call `update_importance(memory_id, 0.0)` in Neo4j when superseding, eliminating mismatch generation at the source.
