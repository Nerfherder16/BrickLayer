# Frontier Questions — Recall 2.0

Questions for the Recall 2.0 Frontier research session. These are whitepaper-level questions — asking what the ideal architecture looks like, not what's buildable today.

Add freely. Mark PENDING/IN_PROGRESS/DONE. Organized by domain.

**Last updated**: 2026-03-16

---

## Instructions for Adding Questions

- Questions should be **specific and answerable** — not "how should we do retrieval?" but "what retrieval mechanism produces the lowest spurious attractor rate at 20K patterns in a Modern Hopfield Network?"
- Questions should target **first-principles understanding** — what does the ideal system look like without the constraint of existing tools?
- After answering a question, move it to DONE and write the finding to `findings/`.

## Domain 1: Hopfield Network as Memory Substrate

### F-HN-01 [PENDING]
What is the theoretical maximum storage capacity of a Modern Hopfield Network at 1024 dimensions before spurious attractors dominate retrieval? How does this compare to the practical limit at which retrieval precision falls below 95%?

### F-HN-02 [PENDING]
What is the optimal inverse temperature (β) for a Modern Hopfield Network storing 22K+ memory patterns at 1024 dimensions, if the patterns are embeddings from a natural-language encoder (qwen3-embedding:0.6b)? How sensitive is retrieval quality to β miscalibration?

### F-HN-03 [PENDING]
Online Hebbian updates on a Hopfield network cause interference: new patterns partially overwrite old ones. What is the interference rate per update as a function of network size and pattern density? Is there a safe update frequency below which interference is negligible?

### F-HN-04 [PENDING]
Can sparse Hopfield networks (where each memory pattern activates only a subset of neurons) achieve better storage capacity and lower interference than dense networks at the same dimensionality? What is the sparsity level that maximizes capacity without degrading retrieval quality?

### F-HN-05 [PENDING]
In a Modern Hopfield Network, can retrieval reinforcement (Hebbian weight strengthening on retrieved patterns) be implemented in a way that doesn't require access to the full weight matrix? Is there a memory-efficient incremental update that achieves the same effect?

### F-HN-06 [PENDING]
What is the computational cost of one retrieval step (one update of the Modern Hopfield Network update rule) in fp16 precision on an RTX 3090, as a function of stored pattern count N and dimensionality D? At what N does retrieval latency exceed 20ms?

### F-HN-07 [PENDING]
Can the Hopfield energy landscape be used directly for cluster discovery (finding basins of attraction) for consolidation purposes, and if so, what is the computational cost of a basin-of-attraction mapping pass over the full weight matrix?

## Domain 2: Decay and Forgetting

### F-DC-01 [PENDING]
Exponential decay (activation × e^(-λt)) is the simplest model. Are there alternative decay functions (power law, Ebbinghaus forgetting curve) that better model real human memory retention patterns? Would any of these produce measurably better retrieval quality in an AI memory system?

### F-DC-02 [PENDING]
How should per-memory decay rates be set without importance scoring? Specifically: what behavioral features (access frequency, recency, co-retrieval density) are most predictive of long-term retrieval value, and what is the optimal formula for computing λ from those features?

### F-DC-03 [PENDING]
If decay is applied to Hopfield weight matrices (global weight decay), how does this interact with the reinforcement mechanism (Hebbian updates)? At what decay rate does reinforcement fail to overcome decay for frequently accessed memories?

### F-DC-04 [PENDING]
What is the correct model for memory consolidation as it relates to decay? In biological systems, consolidated (semantic) memories decay much more slowly than episodic memories. Should abstract memories generated by consolidation have fundamentally different decay rates? What is the right ratio?

## Domain 3: Retrieval and Reinforcement

### F-RT-01 [PENDING]
In a retrieval system that reinforces retrieved patterns (Hebbian weight update), what prevents a "rich get richer" failure mode where the most frequently retrieved memories dominate the entire energy landscape, making it impossible to retrieve less-accessed memories?

### F-RT-02 [PENDING]
What is the ideal query representation for Hopfield retrieval when the query is a natural-language question from a user? Should the query be the embedding of the question, a hypothetical answer embedding (HyDE), or a combination?

### F-RT-03 [PENDING]
Can co-retrieval patterns be encoded directly into the Hopfield weight matrix (as correlations between frequently co-retrieved patterns), rather than tracked separately in a behavioral graph? What would this look like and what would be lost/gained?

### F-RT-04 [PENDING]
What is the minimum session context required to route retrieval correctly — i.e., to select which tier (LLPC vs Hopfield vs cold) is likely to have the answer before querying each tier sequentially? Is it possible to predict the right tier from the query embedding alone?

### F-RT-05 [PENDING]
Activation-based retrieval returns patterns by activation level, not by cosine similarity. Are there memory retrieval tasks where activation level and cosine similarity diverge, and if so, which produces better results for the Recall 2.0 use case?

## Domain 4: Consolidation

### F-CN-01 [PENDING]
What is the optimal clustering algorithm for discovering consolidatable memory groups in a Hopfield energy landscape? Compare: basin-of-attraction mapping, k-means on embeddings, HDBSCAN, spectral clustering. Which produces the highest-quality abstract memories?

### F-CN-02 [PENDING]
At what level of episodic memory density should consolidation trigger? Specifically: how many semantically similar episodic memories must accumulate before generating a semantic-level abstraction is net positive (adds more retrieval value than the overhead of an additional memory)?

### F-CN-03 [PENDING]
Can consolidation be implemented without a generative LLM call (e.g., using centroid computation + template interpolation), and if so, what quality of abstraction does this produce compared to LLM-generated abstractions?

### F-CN-04 [PENDING]
When consolidation generates an abstract memory from episodic sources, should the source episodic memories be decayed faster (their value is now captured at a higher level), or should they remain at their natural decay rate? What does theory predict, and what does empirical testing show?

## Domain 5: Embedding and Representation

### F-EM-01 [PENDING]
Is qwen3-embedding:0.6b the optimal embedding model for a personal AI memory system, given the constraint of running on an RTX 3090 via Ollama? What alternatives at similar parameter counts should be evaluated (e.g., nomic-embed-text, snowflake-arctic-embed, mxbai-embed-large)?

### F-EM-02 [PENDING]
Should a personal AI memory system use a single embedding model for all memory types, or separate models for different domains (code memories vs. conversational memories vs. factual memories)? What is the retrieval quality gap between unified and domain-specific embeddings?

### F-EM-03 [PENDING]
Is 1024 dimensions the right dimensionality for Hopfield storage given the corpus size and use case? Is there a theoretical or empirical argument for higher (e.g., 3072) or lower (e.g., 512) dimensionality?

### F-EM-04 [PENDING]
What is the correct approach to embedding model drift (when the model is upgraded)? Compare: re-embed everything, maintain multiple embedding spaces, use a learned alignment layer between old and new embedding spaces.

## Domain 6: Multi-Machine Consistency

### F-CR-01 [PENDING]
Are OR-Set, G-Counter, and LWW-Register sufficient CRDT types to model all memory operations in Recall 2.0? Are there memory operations that don't fit cleanly into these types?

### F-CR-02 [PENDING]
For an LWW-Register with Hybrid Logical Clock timestamps, what is the maximum clock skew between two machines before LWW produces causally incorrect results in practice? What NTP synchronization requirement does this imply?

### F-CR-03 [PENDING]
OR-Set tombstones accumulate indefinitely. What is the correct tombstone compaction strategy for a personal memory system where the sync delay between machines is bounded (max N hours offline)?

### F-CR-04 [PENDING]
Can the Hopfield weight matrix itself be treated as a CRDT (e.g., by representing it as a G-Counter of Hebbian updates rather than a weight matrix)? What would this enable for multi-machine Hopfield consistency?

## Domain 7: Context Injection

### F-IN-01 [PENDING]
What is the optimal format for injecting retrieved memories into Claude's context window? Specifically: does structured formatting (XML, Markdown with activation levels) measurably change how well Claude incorporates memories into responses compared to plain text?

### F-IN-02 [PENDING]
Is there a memory injection position effect — do memories injected immediately before the user's message receive more attention weight than memories injected at the top of context? What does the "lost in the middle" literature predict for Claude specifically?

### F-IN-03 [PENDING]
Can Claude be prompted to actively request specific memories from Recall rather than passively receiving injected ones? What prompt structure enables effective active memory retrieval, and what's the latency and quality tradeoff vs. passive injection?

### F-IN-04 [PENDING]
What is the optimal memory injection budget (token count) as a function of query complexity? For simple factual queries (IP address), a small injection budget may be sufficient. For complex architectural queries, more context may be needed.

## Domain 9: Architecture and Implementation

### F-AR-01 [PENDING]
Is there a memory storage substrate other than Modern Hopfield Networks that better satisfies the five requirements (retrieval, reinforcement, decay, consolidation, multi-machine consistency) simultaneously? Compare: Sparse Distributed Memory (Kanerva), Neural Turing Machine memory, Differentiable Neural Computer, Product-Key Memory.

### F-AR-02 [PENDING]
What is the theoretical lower bound on retrieval latency for a 22K-pattern associative memory at 1024 dimensions on an RTX 3090 in fp16? How far is the Modern Hopfield Network implementation from this bound?

### F-AR-03 [PENDING]
Can the three-layer deduplication pipeline (L0 hash, L1 SimHash, L2 semantic) be implemented in under 200ms total including embedding inference? What are the bottlenecks and how can they be eliminated?

### F-AR-04 [PENDING]
What is the right persistence model for the Hopfield weight matrix — periodic full checkpoints, incremental WAL-style updates, or something else? What is the recovery time from checkpoint in the event of a crash?

## Domain 10: Open Frontiers (Not Yet Formulated)

Questions we know we need but haven't formulated yet:

- [ ] Multi-modal memory (code, images) — what architecture supports this without separate models for each modality?
- [ ] Memory provenance and audit trail — what metadata is needed to trace a retrieved memory back to its source?
- [ ] Memory confidence and uncertainty — how should a memory system represent that some memories are more reliable than others?
- [ ] Active forgetting — what is the right user interaction model for intentional memory deletion?
- [ ] Memory isolation by project scope — is global memory better or worse than project-scoped memory?
- [ ] The right open source strategy — does MIT licensing change the architecture decisions?
- [ ] Portability — can this run on Mac/Windows dev machines without an RTX 3090?
