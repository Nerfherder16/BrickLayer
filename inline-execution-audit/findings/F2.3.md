### Change 3: Per-turn receipt reset in masonry-prompt-router.js

Added after the active-campaign `process.exit(0)` block (after line 188 in original):

```js
// Reset Mortar routing receipt per-turn so stale receipts don't carry over
try {
  const receiptPath = "C:/Users/trg16/Dev/Bricklayer2.0/masonry/masonry-state.json";
  if (fs.existsSync(receiptPath)) {
    const rState = JSON.parse(fs.readFileSync(receiptPath, "utf8"));
    rState.mortar_consulted = false;
    fs.writeFileSync(receiptPath, JSON.stringify(rState, null, 2), "utf8");
  }
} catch {}
```

This ensures every new prompt starts with `mortar_consulted: false`. Mortar must stamp the receipt fresh for each turn. Stale receipts from prior sessions or crashed turns cannot persist.

1. User submits prompt
2. `masonry-prompt-router.js` resets `mortar_consulted: false` in masonry-state.json
3. Routing hint (`additionalContext`) injected if prompt has detectable intent
4. Claude processes prompt, sees "[ROUTING] You MUST route through Mortar..."
5. Claude invokes `mortar` subagent (Agent tool)
6. Mortar detects mode, executes Bash receipt write (`mortar_consulted: true`, fresh ISO timestamp)
7. Mortar dispatches specialist agents, returns
8. Claude attempts Write/Edit
9. `masonry-approver.js` fires PreToolUse — `isMortarConsulted()` returns true
10. Gate allows Write/Edit through
11. **No deadlock** — receipt is always written before gate is checked
