## Analysis

### (a) Schema Extension for last_route Persistence

**Current schema** (7 fields, all campaign-tracking):
```json
{
  "last_qid": "string",
  "last_verdict": "string",
  "verdicts": { "VERDICT_NAME": "number" },
  "updated_at": "ISO-8601",
  "active_agent": "string",
  "active_agent_count": "number",
  "mortar_consulted": "boolean",
  "mortar_session_id": "ISO-8601"
}
```

**Proposed addition** -- a `last_route` object at the top level:

```json
{
  "last_route": {
    "agent": "refactorer",
    "effort": "high",
    "prompt_summary": "Refactor authentication service",
    "timestamp": "2026-03-29T14:32:00.000Z",
    "turn_count": 1
  }
}
```

| Field | Type | Purpose |
|-------|------|---------|
| `agent` | `string` | The routing target from the matched INTENT_RULE (e.g., `"refactorer"`, `"developer + test-writer + code-reviewer"`) |
| `effort` | `string` | Effort classification (`"low"` / `"medium"` / `"high"` / `"max"`) |
| `prompt_summary` | `string` (max 80 chars) | First 80 characters of the prompt that established the route, for debuggability |
| `timestamp` | `string` (ISO-8601) | When the route was established |
| `turn_count` | `number` | How many consecutive turns have inherited this route (starts at 1, increments on each inheritance) |

**Expiry logic**: The `last_route` should expire (be cleared to `null`) under any of these conditions:

1. **Fresh intent match**: If `detectIntent(prompt)` returns a non-null result, the new intent replaces `last_route` entirely. The user has provided a new explicit signal -- inheritance is not needed.
2. **Turn count exceeds 5**: After 5 consecutive inherited turns, clear `last_route`. Long inheritance chains increase false-positive risk. The user is likely in a new logical task.
3. **Time-based TTL of 10 minutes**: If `Date.now() - last_route.timestamp > 600000`, clear `last_route`. The user has been idle or switched context. Ten minutes is chosen because typical multi-turn continuation bursts last 2-5 minutes; 10 minutes provides margin without stale inheritance.
4. **Explicit negation prompt**: If the prompt matches a negation pattern (`/^(stop|cancel|never ?mind|forget|start over|new task)/i`), clear `last_route`.

**NOT expired by**: a non-continuation prompt that also has no intent match. In that case, `last_route` remains but is not used (the router exits silently as it does today). This avoids the problem of clearing state prematurely when the user asks a brief question mid-workflow.

### (b) Follow-Up Detection Regex Set

These patterns detect the most common continuation constructs in developer sessions. Organized by category:

```javascript
const FOLLOWUP_PATTERNS = [
  // Explicit continuation directives
  /^(now|next|then|also|additionally)\s/i,
  /\b(now\s+(also|do|update|add|fix|run|build|create|write|test|check))\b/i,
  /\b(also\s+(update|add|fix|run|build|create|write|test|check|do))\b/i,

  // "Same" / repetition references
  /\b(same\s+(for|thing|approach|pattern|way|treatment))\b/i,
  /\b(do\s+the\s+same)\b/i,
  /\b(repeat\s+(that|this|it)\s+(for|on|with))\b/i,
  /\b(do\s+(that|it|this)\s+(again|too|as\s+well))\b/i,

  // Sequential / continuation signals
  /\b(continue|proceed|go\s+ahead|carry\s+on|keep\s+going|next\s+step)\b/i,
  /\b(and\s+then|after\s+that|once\s+that.s\s+done|while\s+you.re\s+at\s+it)\b/i,

  // Scope extension
  /\b(but\s+also|plus\s+(also\s+)?add|and\s+add|and\s+update|and\s+fix)\b/i,
  /\b(don.t\s+forget\s+to|make\s+sure\s+(to\s+)?(also|you))\b/i,

  // Pronoun-heavy referential (weaker signal -- only match with other markers)
  /^(and\s+)?(update|fix|change|modify|refactor|test|check)\s+(it|that|those|them|this)\b/i,
];
```

**Pattern count**: 14 regexes covering 30+ surface forms.

**Coverage estimate against D3.2's modeled prompts**:
- Turn 2 ("Now also update the tests...") -- matched by `/\b(now\s+(also|do|update|...))\b/i` and `/\b(also\s+(update|...))\b/i`
- Turn 3 ("Do the same thing for the UserService module") -- matched by `/\b(same\s+(for|thing|...))\b/i` and `/\b(do\s+the\s+same)\b/i`

### (d) False-Positive Risk Assessment

**High-risk false positive scenarios**:

| Prompt | Prior route | Would inherit? | Correct? |
|--------|------------|---------------|----------|
| "Now what?" (open question after build) | developer | YES (matches `^now\s`) | NO -- user is asking "what next", not continuing the build task |
| "Also, what time is the standup?" (social) | refactorer | YES (matches `^also\s`) | NO -- unrelated question |
| "Continue" (ambiguous) | security agent | YES (matches `\bcontinue\b`) | MAYBE -- depends on whether user means "continue the security audit" or "continue the conversation" |
| "Do the same for the database" (new scope) | uiux-master | YES (matches `do\s+the\s+same`) | NO -- database work is not UI work |

**Mitigation strategies**:

1. **Question-word filter**: If the prompt starts with a question word (`what`, `where`, `how`, `why`, `when`, `who`) AND matches a follow-up pattern, do NOT inherit. Questions seek information, not continuation. This eliminates "Now what?" and "Also, what time..." cases.

2. **Minimum prompt length for inheritance**: Require `prompt.length >= 30` for follow-up inheritance. Very short follow-ups like "Continue" or "Now what?" are ambiguous. Longer prompts like "Now also update the tests in the integration suite" carry enough context to be safely inherited. This is a blunt but effective filter.

3. **Domain-mismatch detection (optional, heavier)**: Before inheriting, run `detectIntent(prompt)` -- if it would match a DIFFERENT rule than last_route.agent, route fresh instead. This catches "Do the same for the database" when last_route was uiux-master. However, this only works when the new prompt has intent keywords, which is exactly the case where inheritance is not needed. Net value: low. Skip this.

**Residual false-positive rate estimate**: With the question-word filter and 30-char minimum, approximately 5-10% of inherited routes will be incorrect. This is acceptable compared to the current 40-60% complete-silence rate. A wrong routing hint is recoverable (Mortar can override); no routing hint at all forces inline execution with no recovery path.

### (e) Campaign-Mode Interaction

**Current behavior** (line 183-188):
```javascript
try {
  const state = JSON.parse(
    fs.readFileSync(path.join(cwd, "masonry-state.json"), "utf8")
  );
  if (state && state.mode) process.exit(0);
} catch {}
```

When `state.mode` is truthy (e.g., `"campaign"`), the router exits at line 187 before reaching `detectIntent()`, `classifyEffort()`, or any proposed follow-up logic. This is a hard early exit.

**Interaction with last_route persistence**:

1. **Writing last_route during campaign mode**: Impossible. The router exits before any routing logic runs. `last_route` will never be written during campaign mode. No interference.

2. **Reading stale last_route after campaign ends**: When campaign mode ends and `state.mode` is cleared, the router resumes processing prompts. If a `last_route` from before the campaign still exists in the state file, the TTL check (10 minutes) will have long expired. No interference.

3. **Campaign hooks writing to the same state file**: `masonry-subagent-tracker.js` writes `active_agent` and `active_agent_count` to `masonry-state.json` (lines 278-285). The proposed `last_route` field is a separate top-level key. Both hooks use `safeWrite` (non-atomic JSON rewrite). There is a **race condition risk**: if both hooks fire simultaneously and read-modify-write the state file, one write can clobber the other's changes. This is a pre-existing risk (already present between subagent-tracker and any other hook touching the file) and is not introduced by this change. Mitigation: use `atomicWrite` (already defined at line 124 of subagent-tracker) instead of `safeWrite` for all state file mutations.

4. **D5.1 campaign blackout is unaffected**: The line 187 early exit is orthogonal to `last_route`. Fixing the campaign blackout (D5.1) requires changing the early-exit condition, not adding persistence. These are independent fixes that do not conflict.

**Verdict on campaign interaction**: No interference. The campaign-mode early exit prevents all routing logic (including last_route reads/writes) from executing during campaigns. The TTL expiry prevents stale inheritance after campaigns end.
