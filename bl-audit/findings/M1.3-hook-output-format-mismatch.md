# M1.3 — masonry-register.js Uses Wrong Output Format for UserPromptSubmit Hook

**Question**: Does masonry-register.js output conform to the Claude Code UserPromptSubmit hook spec for context injection?

**Status**: DONE
**Verdict**: CONFIRMED
**Severity**: Critical
**Confidence**: 0.9

## Summary

`masonry-register.js` writes plain text to stdout. The Claude Code UserPromptSubmit hook spec requires a JSON object with `additionalContext` field for context injection. `recall-retrieve.js` confirms the correct format. Even after fixing the SyntaxError (M1.1), masonry-register.js's plain-text output would be ignored — the Mortar directive still would not reach the model.

## Evidence

### Correct Format (recall-retrieve.js, lines 915-921)

```javascript
const output = JSON.stringify({
  // ... other fields
  additionalContext: context,
});
process.stdout.write(output);
```

This is the Claude Code `hookSpecificOutput` format — a JSON object where `additionalContext` is the string injected into the system context.

### masonry-register.js Format (lines 115-116)

```javascript
const directive = buildRoutingDirective(ctx, project);
process.stdout.write(directive + '\n');
```

This writes plain text (`[MASONRY] Route this prompt through Mortar...`) directly. Without the JSON envelope, the hook's stdout is not recognized as `additionalContext` by Claude Code's hook processor — the text is either silently discarded or treated as an error log.

### Consequence Stack

Both M1.1 (SyntaxError) and M1.3 (wrong output format) are independent failures, either of which alone would prevent Mortar injection:

1. **M1.1** (fixed in `a038099`): Hook crashed before executing — directive never written
2. **M1.3** (unfixed): Hook writes plain text instead of JSON — directive written but not injected

After M1.1 was fixed, the hook runs but its output is still in the wrong format. The Mortar directive still does not reach Claude.

## Fix

Wrap the directive in a JSON envelope:

```javascript
const directive = buildRoutingDirective(ctx, project);
const output = JSON.stringify({ additionalContext: directive });
process.stdout.write(output);
```

Apply the same fix to the guard-warning flush (lines 139-142) and the handoff resume output (lines 195-207 and 214-221).
