# Finding: E1.4 — 3 Same-Name Agents in Both Directories; 2 Confirmed Merge Candidates

**Agent**: economizer
**Date**: 2026-03-22
**Severity**: Medium
**Verdict**: CONFIRMED

## Summary

The combined agent fleet (20 global agents in `~/.claude/agents/`, 35 project agents in `.claude/agents/`) contains 3 agents with identical filenames in both directories (`mortar`, `health-monitor`, `tools-manifest`) and 2 explicit BL 1.x / BL 2.0 version pairs (`synthesizer` / `synthesizer-bl2`, `hypothesis-generator` / `hypothesis-generator-bl2`). Two require consolidation: `tools-manifest` (85% overlap, BL version is a superset — global copy is redundant) and `health-monitor` (70% overlap, diverged purpose but near-identical names cause confusion). One cross-tier overlap was also found: `security` (global, 51 lines) vs `compliance-auditor` (BL, 201 lines) — both perform OWASP audits but via different methods. The versioned BL 1.x / 2.0 pairs are intentional. Total combined fleet: ~12,144 lines across 55 files.

## Evidence

### 3 Name Collisions (Same Filename in Both Directories)

| Agent | Global Lines | BL 2.0 Lines | Overlap | Action |
|-------|-------------|--------------|---------|--------|
| `mortar` | 614 | 150 | ~45% (BRANCHED) | Keep both — different architectures |
| `health-monitor` | 206 | 196 | ~70% | Consolidate or explicitly rename |
| `tools-manifest` | 152 | 149 | ~85% | Delete global (BL is superset) |

**mortar**: Global version (614 lines) is the universal session router for all project types. BL version (150 lines) is a lightweight campaign/conversational router scoped to BL 2.0. Different purposes, same name — acceptable but should be documented.

**health-monitor**: Global description: "VIGIL fleet watchdog — monitors campaign health metrics". BL description: "Activate when the user wants to check live system health — querying live targets and comparing to thresholds." 70% semantic overlap. The BL version is the more authoritative definition (matches the global CLAUDE.md fleet table). The global copy may be an older incarnation that was never cleaned up.

**tools-manifest**: Both versions are reference documents listing available MCP tools. The BL version (149 lines) adds schema validation and is strictly a superset. The global version (152 lines) adds nothing that the BL version doesn't cover. The global copy is dead weight for BL project sessions.

## Proposed Fix

### Priority 1 — Remove redundant global `tools-manifest.md` (High effort/impact ratio: Low/Medium)

`~/.claude/agents/tools-manifest.md` is a 152-line reference document whose content is fully covered by the BL version. For BL 2.0 project sessions, both are loaded unnecessarily.

Action: Remove `~/.claude/agents/tools-manifest.md` or confirm it is legitimately separate.

### Priority 2 — Rename `health-monitor` to clarify versions (Low effort)

Options:
- Rename global copy to `health-monitor-legacy.md` or `campaign-health-monitor.md`
- Update global CLAUDE.md agent table to distinguish the two
- Merge into one agent with a `--mode` parameter (live-targets vs fleet-metrics)

### Priority 3 — Add a "superseded-by" note to BL 1.x agents (Trivial)

Add a frontmatter comment to `synthesizer.md` and `hypothesis-generator.md`:
```yaml
# superseded-by: synthesizer-bl2 for BL 2.0 campaigns
```
This prevents accidental invocation in BL 2.0 contexts.

### Priority 4 — Clarify `security` vs `compliance-auditor` routing

Update the CLAUDE.md agent routing table to specify:
- `security` → open-ended vulnerability assessment, threat modeling, defensive security
- `compliance-auditor` → structured checklist audits (OWASP Top 10 compliance, WCAG)

**Total effort**: Low-Medium across all 4 items
**Impact**: Medium — reduces confusion and eliminates ~300 lines of redundant context per session
