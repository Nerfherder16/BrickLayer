# D4.1 — Agent Registry Path Drift

**Question ID:** D4.1
**Domain:** D4 Config Drift
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** High

## Summary

`masonry/agent_registry.yml` has drifted from the actual agent file layout. Three path prefixes are used inconsistently (`agents/`, `~/.claude/agents/`, `.claude/agents/`), and at least 9 entries under `agents/` point to files that don't exist there. The registry is not a reliable source of truth for agent file locations.

## Evidence

**Three path conventions in the registry:**
- `agents/` — BL1.x root-level agents dir (15 entries)
- `~/.claude/agents/` — global Claude Code agents dir (15 entries)
- `.claude/agents/` — repo-local BL2.0 agents dir (16 entries)

**Stale `agents/` entries (files don't exist at that path):**
`onboard_agent.py`'s `detect_stale_registry_entries()` would flag these as stale. Based on file system checks, at least 9 agents registered under `agents/` exist only at `.claude/agents/`:
- diagnose-analyst, compliance-auditor, research-analyst, fix-implementer, cascade-analyst, benchmark-engineer, competitive-analyst, mcp-advisor, retrospective

**`~/.claude/agents/` prefix errors:**
At least 2 entries registered as global agents (`design-reviewer`, `frontier-analyst`) are actually repo-local at `.claude/agents/`.

**Root cause:** The `agents/` directory was the BL1.x location. BL2.0 moved agents to `.claude/agents/`. When auto-onboarding added new agents, some were added with the old prefix. The dead `masonry-agent-onboard.js` hook (D1.1) would have maintained these entries if it were registered — but without it running, the registry accumulated drift.

## Impact

- High: any tool that reads `file:` from registry to load agent content gets wrong path → FileNotFoundError
- `masonry_drift_check` MCP tool would report 9+ stale entries if called
- Kiln UI showing agent list would show broken file indicators
- DSPy optimization pipeline fails for any agent with a bad `file:` path
- Routing still works (returns agents by name) but injection of agent content into prompts fails

## Files

- `masonry/agent_registry.yml` — drifted, multiple stale paths
- `agents/` — contains only original BL1.x agents
- `.claude/agents/` — contains 34 BL2.0 agents (correct location)

## Recommendation

Run `python masonry/scripts/onboard_agent.py --agents-dir .claude/agents --agents-dir agents --agents-dir ~/.claude/agents` to generate a fresh registry with correct paths. Review output and commit. Then register `masonry-agent-onboard.js` to prevent future drift.
