# M1.1 — masonry-register.js SyntaxError Prevents Mortar Injection

**Question**: Does masonry-register.js successfully inject the Mortar routing directive into every prompt?

**Status**: DONE
**Verdict**: CONFIRMED
**Severity**: Critical
**Confidence**: 1.0

## Summary

`masonry-register.js` crashes at Node.js parse time due to a duplicate `const cwd` declaration inside `async function main()`. The hook has never successfully run. The "Every request → Mortar" policy has never been enforced via this hook mechanism.

## Evidence

### Duplicate Declaration

Line 88 (inside `main()`, before research-project check):
```javascript
const cwd = process.env.CLAUDE_PROJECT_DIR || process.cwd();
```

Line 105 (inside `main()`, after stdin parsing):
```javascript
const cwd = process.env.CLAUDE_PROJECT_DIR || process.cwd();
```

### Verification

```
$ node --check masonry/src/hooks/masonry-register.js
C:\Users\trg16\Dev\Bricklayer2.0\masonry\src\hooks\masonry-register.js:105
  const cwd = process.env.CLAUDE_PROJECT_DIR || process.cwd();
        ^

SyntaxError: Identifier 'cwd' has already been declared
```

Node.js rejects the file at parse time — no code in the file has ever executed.

### Hook Registration

`~/.claude/settings.json` registers `masonry-register.js` as a `UserPromptSubmit` hook. The hook is meant to write the Mortar routing directive to stdout so Claude sees it prepended to every prompt. Because the file fails to parse, the directive `[MASONRY] Route this prompt through Mortar` has never been injected.

## Impact

- Every Claude Code session has operated without the Mortar routing directive
- The CLAUDE.md rule "Every request → Mortar" has never been enforced at the hook level
- Wave 1 data showing agents being invoked directly (not via Mortar) is consistent with this failure — there was no mechanism to remind Claude to use Mortar
- All 43 routing_log.jsonl entries are from subagent-tracker (SubagentStart events), not from Mortar-directed routing

## Fix

Remove the duplicate `const cwd` at line 105. The variable is already in scope from line 88. Line 105 was changed to `const sessionId = input.session_id || 'unknown';` — extracting the actual value it needed.

**Fix status**: APPLIED — commit `a038099` (`fix(hooks): remove duplicate const cwd declaration in masonry-register.js`). Current `node --check` returns clean.
