# M1.6 — mortar.md Missing Routing for Git and Infrastructure Tasks

**Question**: Does mortar.md contain routing trigger patterns that cover all task types, or are coding/git/docs tasks unrepresented?

**Status**: DONE
**Verdict**: CONFIRMED
**Severity**: Medium
**Confidence**: 0.9

## Summary

mortar.md covers campaign, research, and most dev task routing well. However, git operations and infrastructure/homelab tasks are completely absent from the routing table. These task types fall through to inline handling by Claude without specialist dispatch. The "Mortar dispatches ALL work" claim in CLAUDE.md is not fully supported.

### Missing Task Types

**Git operations** — Not in the routing table:
- "commit this", "push to remote", "create a PR", "merge this branch"
- CLAUDE.md says: "After any code changes → git-nerd proactively handles commits"
- mortar.md has no git-nerd routing entry

**Infrastructure / homelab:**
- Docker, Proxmox, CasaOS, VM questions
- CLAUDE.md mentions auto-invoking `/homelab` skill
- Not in mortar.md routing table

**Quick/conversational questions:**
- "What does this code do?", "Explain this error", "What's the best pattern for X?"
- No fallback routing defined (just falls through to inline Claude)

**System status / health checks:**
- "Check docker containers", "Is X running?"
- Covered by CLAUDE.md's auto-invoke skills but not in mortar.md

### CLAUDE.md vs mortar.md Divergence

CLAUDE.md (global rules) has richer routing instructions than mortar.md:
```
After any code changes → git-nerd proactively handles commits and branch hygiene
```
This is defined in CLAUDE.md as a behavioral rule for Claude, not in mortar.md as a Mortar dispatch rule. These two layers have grown out of sync.

## Impact

- Git tasks (very frequent) bypass Mortar → handled inline
- Infrastructure tasks bypass Mortar → handled inline
- The "Mortar dispatches ALL work" claim is aspirational
- Coverage is ~70% for typical day-to-day tasks (campaign + research + core dev covered; git + infra not)

## Recommendation

Add to mortar.md's Dev Task Routing table:
```
| Git: commit, push, PR, branch | git-nerd |
| Infrastructure: Docker, homelab, VMs | devops or health-monitor |
| Quick question / explain code | (inline — no dispatch needed) |
```
Alternatively, acknowledge in CLAUDE.md that Mortar covers research+dev routing and git/infra routing is handled via direct behavioral rules in CLAUDE.md.
