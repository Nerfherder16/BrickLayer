# M1.5 — masonry-register.js Suppresses Mortar Injection in bl-audit Directory

**Question**: Does masonry-register.js suppress Mortar injection in the bl-audit working directory?

**Status**: DONE
**Verdict**: CONFIRMED
**Severity**: Medium
**Confidence**: 1.0

## Summary

`bl-audit/` contains both `program.md` and `questions.md`. `isResearchProject()` returns true for this directory. masonry-register.js exits at line 89 before writing any output, meaning all audit sessions have run without the Mortar routing directive. This is intentional behavior (BL subprocess isolation) but has a consequence: the audit itself was never Mortar-routed.

## Evidence

### Both Files Confirmed Present

```
$ ls bl-audit/program.md bl-audit/questions.md
bl-audit/program.md
bl-audit/questions.md
Both exist
```

### isResearchProject Check (masonry-register.js, lines 81-89)

```javascript
function isResearchProject(dir) {
  return fs.existsSync(path.join(dir, 'program.md')) &&
         fs.existsSync(path.join(dir, 'questions.md'));
}

async function main() {
  const cwd = process.env.CLAUDE_PROJECT_DIR || process.cwd();
  if (isResearchProject(cwd)) return;  // ← exits here for bl-audit
  ...
}
```

When Claude Code runs in `bl-audit/`, `cwd = .../bl-audit`, `isResearchProject()` returns true, and `main()` returns before writing anything to stdout.

## Consequences

1. All 10+ Wave 2 investigation sessions in bl-audit ran without the Mortar routing directive
2. Claude Code handled all audit tasks directly (no Mortar dispatch layer)
3. The sessions used the correct agents, but routing relied on the agent system prompt context (CLAUDE.md) rather than per-prompt directive injection

## Design Intent

This suppression is by design. The CLAUDE.md comment explicitly states:
> "Masonry hooks automatically detect BrickLayer research projects (via `program.md` + `questions.md`) and stay silent inside BL subprocesses. No env var needed."

The intent is to prevent the Mortar routing directive from interfering with the autonomous research loop (which already has its own routing via Trowel). The trade-off: Mortar routing injection is unavailable during the audit itself.

## Action

No fix required — this is intended behavior. The audit finding documents the scope: Mortar injection bypass applies to ALL BL research project directories, not just bl-audit.
