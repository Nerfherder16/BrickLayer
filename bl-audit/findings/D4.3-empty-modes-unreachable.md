# D4.3 — 16 Agents Unreachable Via Mode-Based Routing

**Question ID:** D4.3
**Domain:** D4 Config Drift
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

16 agents in `masonry/agent_registry.yml` have `modes: []` (empty). The deterministic router's `get_agents_for_mode()` returns only agents whose `modes` list contains the requested mode value. Agents with empty modes lists are never returned by this lookup, making them unreachable via the `**Mode**:` field routing path.

## Evidence

From `masonry/src/routing/deterministic.py` line ~106:
```python
m = _MODE_FIELD_RE.search(request_text)
if m:
    mode_value = m.group(1).strip().lower()
    agents = get_agents_for_mode(registry, mode_value)
    if agents:
        return _decision(agents[0].name, f"Mode field: {mode_value}")
```

`get_agents_for_mode(registry, mode)` in `registry_loader.py` filters by `mode in agent.modes`.

**16 agents with `modes: []`** (auto-onboarded from frontmatter that had no `modes:` field, or frontmatter had empty modes):

These agents were onboarded via `masonry-agent-onboard.js` / `onboard_agent.py` but their agent `.md` files lack a `modes:` frontmatter field. The onboarding script correctly reads `modes: []` from frontmatter where the field is absent or empty — but this leaves the agents unreachable via Mode-field routing.

**What modes these agents likely need** (based on their names):
- `diagnose-analyst` → `modes: [diagnose]`
- `compliance-auditor` → `modes: [audit]`
- `research-analyst` → `modes: [research]`
- etc.

**Workaround:** These agents CAN be reached via slash commands (if listed in `_SLASH_COMMANDS`) or via LLM routing (Layer 3), but not via the zero-LLM-cost deterministic Mode field path.

## Impact

- Medium: agents are reachable via other routing layers, just not efficiently
- The primary zero-cost routing path for BL2.0 campaigns (Mode field) bypasses 16 agents
- Campaign waves that set `**Mode**: diagnose` to target `diagnose-analyst` will fail to route via deterministic layer — fall through to LLM routing (1 Haiku call per question)
- Undermines the CLAUDE.md claim: "Deterministic handles 60%+ of routing with zero LLM calls"

## Files

- `masonry/agent_registry.yml` — 16 entries with `modes: []`
- `.claude/agents/*.md` — these agent files lack `modes:` frontmatter
- `masonry/src/routing/deterministic.py` — Mode field routing requires non-empty modes

## Recommendation

Add `modes:` frontmatter to each agent .md file. Then re-run `onboard_agent.py` to update registry entries. The mode values should match BL2.0 question mode labels: `audit`, `research`, `diagnose`, `simulate`, `agent`.
