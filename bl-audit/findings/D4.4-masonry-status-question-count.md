# D4.4 — masonry_status Tool Returns Wrong Question Count for BL2.0

**Question ID:** D4.4
**Domain:** D4 Config Drift
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** High

## Summary

The `masonry_status` MCP tool in `masonry/mcp_server/server.py` counts questions by scanning for lines starting with `### Q` — the BL1.x header format. BL2.0 uses `## D1.1` format. This means `masonry_status` ALWAYS returns `q_total: 0` for any BL2.0 project.

## Evidence

`server.py` masonry_status handler:
```python
q_total = sum(1 for ln in lines if ln.startswith("### Q"))
q_done  = sum(1 for ln in lines if ln.startswith("### Q") and "DONE" in ln)
```

BL2.0 `questions.md` format (this campaign):
```markdown
## D1.1 [audit] Are masonry-agent-onboard.js and masonry-recall-check.js dead code?

**Status**: PENDING
```

BL1.x format that the parser expects:
```markdown
### Q1.1 — Some question title [DONE]
```

The `## D1.1` header does NOT start with `### Q`, so every BL2.0 question is invisible to `masonry_status`. The tool will report `q_total: 0, q_done: 0` for all BL2.0 campaigns.

**Consequence for Kiln:** The Kiln dashboard reads `masonry_status` to display campaign progress (wave %, questions answered). All BL2.0 campaigns show 0% progress regardless of how many questions are answered.

## Impact

- High: core observability tool broken for all BL2.0 campaigns
- Kiln progress display permanently stuck at 0/0
- Any agent that calls `masonry_status` to check campaign health gets wrong data
- The bug has existed since the migration from BL1.x question format to BL2.0 format

## Files

- `masonry/mcp_server/server.py` — masonry_status handler with BL1.x question pattern
- `bl-audit/questions.md` — BL2.0 format (`## D1.1`)
- Any BL2.0 project `questions.md` — all affected

## Recommendation

Update the question count logic to handle both formats:
```python
import re
Q_PATTERN_BL2 = re.compile(r'^## [A-Z]\d+\.\d+')
Q_PATTERN_BL1 = re.compile(r'^### Q')

q_total = sum(1 for ln in lines if Q_PATTERN_BL2.match(ln) or Q_PATTERN_BL1.match(ln))
q_done  = sum(
    1 for i, ln in enumerate(lines)
    if (Q_PATTERN_BL2.match(ln) or Q_PATTERN_BL1.match(ln))
    and any("DONE" in lines[j] for j in range(i+1, min(i+5, len(lines))))
)
```
