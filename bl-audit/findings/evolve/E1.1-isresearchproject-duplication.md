# Finding: E1.1 — `isResearchProject()` Copy-Pasted Into 8 Hook Files

**Agent**: economizer
**Date**: 2026-03-22
**Severity**: Medium
**Verdict**: CONFIRMED

## Summary

`isResearchProject()` is defined independently in 8 hook files. Every copy checks for the same two sentinel files (`program.md` + `questions.md`), but two copies have drifted behaviorally: `masonry-approver.js` adds a null-input guard that the others lack, and `masonry-build-guard.js` re-requires `fs` and `path` inside the function body rather than using module-level imports. The 8 copies total ~24 lines of duplicated logic. A single `masonry-utils.js` shared module would eliminate all copies and provide one source of truth for a security-critical gate that silences all hooks inside BL research sessions.

### Behavioral Divergences

**Divergence 1 — Null safety** (`masonry-approver.js:114-117`):
```js
function isResearchProject(dir) {
  if (!dir) return false;   // ← only this copy has this guard
  return existsSync(join(dir, "program.md")) && existsSync(join(dir, "questions.md"));
}
```
The other 7 copies will throw `TypeError: The "path" argument must be of type string` if `dir` is `undefined` or `null`. The approver guards against this because it is called with file-path–derived directories that may be absent.

**Divergence 2 — Inline requires** (`masonry-build-guard.js:32-37`):
```js
function isResearchProject(dir) {
  const { existsSync } = require('fs');   // ← re-required inside function
  const path = require('path');           // ← re-required inside function
  return existsSync(path.join(dir, 'program.md')) &&
         existsSync(path.join(dir, 'questions.md'));
}
```
Other copies use module-level imports. The inline pattern works (Node.js caches `require()`) but is inconsistent and unnecessary.

**Bonus variant**: `masonry-approver.js` also defines `isResearchProjectFresh()` (lines 215–218) which calls `isResearchProject()` as a building block — this is the only file with this extension.

### Duplication Metrics

- 8 copies × 3 lines body = **24 lines** of duplicated logic
- 8 copies × 1 comment line ("Auto-detect BrickLayer...") = 8 additional near-duplicate comment lines at each call site
- Total potential reduction: ~24 lines of definition + consolidation of call-site boilerplate

## Proposed Fix

Create `masonry/src/hooks/masonry-utils.js` with the canonical definition that incorporates the null guard (the safest variant):

```js
// masonry/src/hooks/masonry-utils.js
const fs = require('fs');
const path = require('path');

/**
 * Returns true if `dir` is the root of a BrickLayer research project.
 * Hooks call this to exit silently when running inside a BL campaign subprocess.
 */
function isResearchProject(dir) {
  if (!dir) return false;
  return fs.existsSync(path.join(dir, 'program.md')) &&
         fs.existsSync(path.join(dir, 'questions.md'));
}

module.exports = { isResearchProject };
```

Replace the 8 inline definitions with:
```js
const { isResearchProject } = require('./masonry-utils');
```

Move `isResearchProjectFresh()` from `masonry-approver.js` into `masonry-utils.js` as well, since it is the natural extension of `isResearchProject()`.

**Effort**: Low (8 files, mechanical substitution)
**Impact**: High — single source of truth for the hook-suppression gate; any future change (e.g., adding `findings/` as a third sentinel) applies once instead of 8 times
