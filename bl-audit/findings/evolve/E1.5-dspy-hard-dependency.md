# Finding: E1.5 — DSPy Listed as Hard Dependency But MCP Server Already Treats It as Optional

**Agent**: economizer
**Date**: 2026-03-22
**Severity**: Low
**Verdict**: CONFIRMED

## Summary

`masonry/requirements.txt` lists `dspy>=2.5` as a hard requirement, but the production code path already treats DSPy as optional: `mcp_server/server.py` imports all DSPy-related modules lazily (inside function bodies, never at module load time). The MCP server starts successfully without DSPy installed. DSPy is only needed when the `masonry_optimize_agent` or `masonry_onboard` endpoints are called — both of which require an active optimization run that has never succeeded (confirmed: `masonry/optimized_prompts/` is empty, contains only `.gitkeep`). The install footprint concern is lessened because DSPy 2.5+ no longer requires PyTorch by default. `httpx` is a legitimate production dependency used in the semantic routing layer (`semantic.py:18`). The fix is to move `dspy>=2.5` to an optional extras group in a `pyproject.toml`.

## Evidence

### DSPy Import Locations

| File | Import Type | Production Path |
|------|-------------|-----------------|
| `src/dspy_pipeline/signatures.py` | Hard (`import dspy` at module top) | Not imported at server startup |
| `src/dspy_pipeline/optimizer.py` | Hard (`import dspy` at module top) | Lazy-imported in server.py |
| `src/dspy_pipeline/generated/economizer.py` | Hard (`import dspy` at module top) | Generated stub, never auto-imported |
| `mcp_server/server.py` | **Lazy** (inside function body, `# noqa: PLC0415`) | Only when endpoint called |
| `scripts/onboard_agent.py` | **Not imported** — `import dspy` appears only in an f-string template (stub generator) | n/a |
| `scripts/run_r23_1.py` | Hard | Ad-hoc script, not production |

**Key insight**: `mcp_server/server.py` uses lazy imports exclusively for all dspy-related code:
```python
# Inside masonry_optimize_agent handler:
from masonry.src.dspy_pipeline.training_extractor import build_dataset  # noqa: PLC0415
from masonry.src.dspy_pipeline.optimizer import configure_dspy, optimize_agent  # noqa: PLC0415
from masonry.src.dspy_pipeline.signatures import ResearchAgentSig  # noqa: PLC0415
```
The `# noqa: PLC0415` suppression confirms this is intentional late-binding.

### Optimization Pipeline Status

```bash
ls masonry/optimized_prompts/
# .gitkeep (only)
```
No optimization has ever completed. The full pipeline (training data extraction → MIPROv2 → prompt export) exists but has never produced output.

### httpx Legitimacy

`httpx` is imported at module level in `masonry/src/routing/semantic.py:18` and used for the Ollama API call in the semantic routing layer. This is a production dependency, not optional. It is appropriate to keep `httpx>=0.27` as a hard requirement.

### DSPy 2.5+ Install Footprint

DSPy 2.5+ (released late 2024) restructured to remove mandatory PyTorch. The core install includes:
- `openai`, `anthropic` (LLM backend clients)
- `pydantic`, `typing-extensions`, `backoff`, `ujson`
- Notably: **no PyTorch by default**

PyTorch is only needed if using embedding-based optimizers (not applicable here — the masonry optimizer uses a keyword-heuristic metric with no LLM judge). The install footprint is significantly lighter than DSPy 1.x.

### pyyaml and pydantic

Both are used throughout the production codebase (`yaml.safe_load` in multiple scripts, `pydantic` models in `schemas/`). These are legitimate hard dependencies.

## Proposed Fix

Convert `masonry/requirements.txt` to a `pyproject.toml` with extras:

```toml
[project]
name = "masonry"
dependencies = [
    "pydantic>=2.10",
    "pyyaml>=6.0",
    "httpx>=0.27",
]

[project.optional-dependencies]
optimize = [
    "dspy>=2.5",
]
dev = [
    "dspy>=2.5",
    "pytest",
]
```

Install with optimization support: `pip install masonry[optimize]`
Install for production routing only: `pip install masonry`

This explicitly documents the optional nature of DSPy and prevents developers from needing the full ML dependency for routing-only use.

**Effort**: Low (rename requirements.txt to pyproject.toml, add extras section)
**Impact**: Low-Medium — cleaner dependency graph; anyone installing just for routing or schemas doesn't get dspy
