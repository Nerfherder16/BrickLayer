## Summary

Three hook files exceed the project's 300-line hard limit (`quality-standards.md`): `masonry-observe.js` (317), `masonry-approver.js` (315), and `masonry-tdd-enforcer.js` (311). Two more approach the limit: `masonry-session-summary.js` (288) and `masonry-statusline.js` (273). Each oversized file contains 2–3 distinct logical concerns that map to clean module boundaries. The most impactful extraction is `masonry-tdd-enforcer.js`'s test-file discovery block (~130 lines) and `masonry-observe.js`'s code-fact extraction block (~136 lines) — both are self-contained and share no callers other than the host file's `main()`. Additionally, fixing E1.1 (shared `masonry-utils.js`) would remove 3 lines from each of these files, though not enough to bring them below the limit.

## Proposed Fix

### Priority 1 — Extract `masonry-code-facts.js` (from observe.js)

Move lines 35–170 to `masonry/src/hooks/masonry-code-facts.js`:
```js
// masonry-code-facts.js — Code-fact extraction for the observe hook
module.exports = { extractCodeFacts };
```
Replace in `masonry-observe.js`:
```js
const { extractCodeFacts } = require('./masonry-code-facts');
```
**Result**: `masonry-observe.js` drops from 317 → ~181 lines. `masonry-code-facts.js` = ~136 lines.

### Priority 2 — Extract `masonry-tdd-helpers.js` (from tdd-enforcer.js)

Move `findTestFile()`, `checkTestImports()`, and classification helpers to `masonry/src/hooks/masonry-tdd-helpers.js`:
```js
module.exports = { findTestFile, checkTestImports, isExempt, isTestFile, isImplementationFile };
```
**Result**: `masonry-tdd-enforcer.js` drops from 311 → ~145 lines.

### Priority 3 — Expand `masonry-utils.js` (E1.1 fix)

Include `readStdin()` in the shared utility module. Remove the 6 independent copies.
**Result**: Each file loses ~8 lines (readStdin), pushing all three below 300.
