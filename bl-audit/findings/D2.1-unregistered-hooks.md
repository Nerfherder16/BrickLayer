# D2.1 — Unregistered Hook Files

**Question ID:** D2.1
**Domain:** D2 Unwired Items
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

Two hook files exist on disk but are not registered in `~/.claude/settings.json`: `masonry-agent-onboard.js` (intentionally or accidentally unregistered) and `masonry-recall-check.js` (correctly NOT registered — it's a subprocess utility, not a direct hook). The two-way matrix shows all other hooks are wired correctly.

|---|---|---|
| masonry-session-start.js | SessionStart ✓ | WIRED |

**Classification:**
- `masonry-agent-onboard.js`: UNWIRED / likely regression. CLAUDE.md explicitly documents it as active under PostToolUse. Its absence means agent auto-onboarding never runs.
- `masonry-recall-check.js`: BY DESIGN. Header comment: "Spawned as a detached background process by masonry-statusline.js". Not expected to be in settings.json.

## Impact

- Agent auto-onboarding silently broken — new agent .md files are never added to registry
- CLAUDE.md advertises "Zero Manual Steps" for onboarding but the mechanism doesn't run
- All other hooks functional — no other wiring gaps

## Files

- `~/.claude/settings.json` — missing masonry-agent-onboard.js PostToolUse entry
- `masonry/src/hooks/masonry-agent-onboard.js` — unwired
- `masonry/src/hooks/masonry-recall-check.js` — correctly unwired (subprocess)

## Recommendation

Add `masonry-agent-onboard.js` to settings.json under PostToolUse for Write and Edit events targeting `agents/*.md` files. Alternatively, remove it and update CLAUDE.md.
