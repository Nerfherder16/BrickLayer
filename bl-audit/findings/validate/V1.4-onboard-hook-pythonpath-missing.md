# V1.4 — masonry-agent-onboard.js Fails Silently Due to Missing PYTHONPATH

**Question**: After D4.4 registration, does masonry-agent-onboard.js successfully fire and update the registry?

**Status**: DONE
**Verdict**: CONFIRMED
**Severity**: High
**Confidence**: 1.0

## Summary

`masonry-agent-onboard.js` correctly detects agent .md files and fires, but spawns `onboard_agent.py` without setting PYTHONPATH. The Python script fails with `ModuleNotFoundError: No module named 'masonry'` silently (`stdio: "ignore"`). The onboard pipeline registered in settings.json does not function end-to-end.

## Evidence

### Hook Spawn Call (masonry-agent-onboard.js, lines 59-67)

```javascript
const cwd = event.cwd || process.cwd();
const scriptPath = path.join(cwd, "masonry", "scripts", "onboard_agent.py");

const child = spawn("python", [scriptPath], {
  detached: true,
  stdio: "ignore",   ← errors silently discarded
  cwd,               ← correct cwd
  // env: NOT SET    ← no PYTHONPATH
});
```

No `env` option is passed to `spawn()`. The child process inherits `process.env`, which does NOT contain `PYTHONPATH=C:/Users/trg16/Dev/Bricklayer2.0` unless it was already set system-wide.

### The Error (confirmed in Phase 2 manual testing)

```
$ python masonry/scripts/onboard_agent.py
Traceback (most recent call last):
  ...
ModuleNotFoundError: No module named 'masonry'
```

When running without PYTHONPATH, `onboard_agent.py` cannot import its own package. With `stdio: "ignore"`, this error is never seen.

### Phase 2 Fix Context

During Phase 2 remediation, the path fix for agent registry entries was applied by running:
```
PYTHONPATH=C:/Users/trg16/Dev/Bricklayer2.0 python masonry/scripts/onboard_agent.py ...
```

The PYTHONPATH requirement is known. The hook doesn't set it.

### Hook Detection Logic (works correctly)

The hook correctly:
- Filters for Write/Edit on `*/agents/*.md` paths (line 36)
- Guards against non-agent .md files by checking YAML frontmatter (lines 44-53)
- Detection is not the problem — spawn env is the problem

## Fix

Add `PYTHONPATH` to the spawn environment:

```javascript
const child = spawn("python", [scriptPath], {
  detached: true,
  stdio: "ignore",
  cwd,
  env: {
    ...process.env,
    PYTHONPATH: cwd,  // cwd = project root = C:/Users/trg16/Dev/Bricklayer2.0
  },
});
```
