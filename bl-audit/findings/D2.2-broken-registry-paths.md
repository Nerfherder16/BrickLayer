# D2.2 — Broken agents/ Prefix Paths in agent_registry.yml

**Question ID:** D2.2
**Domain:** D2 Unwired Items
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** High

## Summary

`masonry/agent_registry.yml` contains 48+ entries using three different path conventions. 9 of the 15 entries with `agents/` prefix point to files that do not exist at that path — the actual files are in `.claude/agents/` (relative to repo root). These 9 agents are effectively unreachable because any path-based file lookup will fail.

## Evidence

**Three path conventions in use:**
1. `agents/` — relative to repo root (15 entries) — BL1.x location
2. `~/.claude/agents/` — global agents dir (15 entries) — global location
3. `.claude/agents/` — repo-local agents dir (16 entries) — BL2.0 location

**Broken `agents/` entries (files NOT at agents/ but exist at .claude/agents/):**
The `agents/` directory at repo root contains only BL1.x agents. Many BL2.0 agents registered under `agents/` prefix were actually written to `.claude/agents/`:

- `agents/diagnose-analyst.md` — MISSING (file is at `.claude/agents/diagnose-analyst.md`)
- `agents/compliance-auditor.md` — MISSING
- `agents/research-analyst.md` — MISSING
- `agents/fix-implementer.md` — MISSING
- `agents/cascade-analyst.md` — MISSING
- `agents/benchmark-engineer.md` — MISSING
- `agents/competitive-analyst.md` — MISSING
- `agents/mcp-advisor.md` — MISSING
- `agents/retrospective.md` — MISSING

**Additionally:** 2 entries in the `~/.claude/agents/` group also use wrong paths:
- `~/.claude/agents/design-reviewer.md` and `~/.claude/agents/frontier-analyst.md` — registered under global path but files are at `.claude/agents/`

**Verified correct entries:**
- `agents/` prefix: `quantitative-analyst.md`, `peer-reviewer.md`, `hypothesis-generator.md` (BL1.x agents) — files actually exist at `agents/`
- `.claude/agents/` prefix: all 16 entries appear to match actual file locations

## Impact

- `detect_stale_registry_entries()` in `onboard_agent.py` would report 9+ stale entries
- `masonry_drift_check` MCP tool would show these as missing
- The deterministic router's `get_agents_for_mode()` can still return these agents by name — but any tool that tries to read the agent file to inject its prompt would fail
- Kiln UI may show broken agent entries

## Files

- `masonry/agent_registry.yml` — 9+ entries with broken `agents/` paths
- `agents/` directory — contains only BL1.x agents
- `.claude/agents/` directory — contains 34 BL2.0 agents

## Recommendation

Audit all `agents/` prefix entries and update paths to `.claude/agents/` for BL2.0 agents. Consider running `python masonry/scripts/onboard_agent.py --agents-dir .claude/agents` to regenerate correct entries.
