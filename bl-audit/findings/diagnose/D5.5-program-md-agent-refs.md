# D5.5 — program.md Agent References

**Question ID:** D5.5
**Domain:** D5 Stale References
**Priority:** MEDIUM
**Verdict:** FALSE_POSITIVE
**Severity:** Info

## Summary

Active `program.md` files (`template/program.md`, `recall/program.md`) do not reference agents by name. They contain loop instructions and context setup but delegate agent selection to Trowel or the human operator. No stale agent name references were found in program.md files.

## Evidence

**`template/program.md`:** Generic BL1.x loop instructions. References "specialist agents" in general language but does not name specific agents. Agents are selected by Trowel or the question's `**Agent**:` field, not hard-coded in program.md.

**`recall/program.md`:** Contains DISABLE_OMC=1 instruction (see D5.3) but does not reference any specific agent names.

**`bl-audit/program.md` (this campaign):** Uses BL2.0 format. Agent selection is controlled by the `**Agent**:` field per question in questions.md, not in program.md.

**CLAUDE.md QUICKSTART section (repo root):** References agent names in invoke instructions (quantitative-analyst, regulatory-researcher, etc.) but this is `CLAUDE.md`, not `program.md`. These agents exist in `.claude/agents/`.

**Potential concern noted:** `CLAUDE.md` does reference `solana-specialist` and `uiux-master` in the agent routing table — both are missing files (covered in D2.6). But that's CLAUDE.md, not program.md.

## Impact

No bug in program.md files. Agent name references in program.md would be a maintenance problem, but none were found.
