# D2.5 — CLAUDE.md Hook Documentation Errors

**Question ID:** D2.5
**Domain:** D2 Unwired Items
**Priority:** MEDIUM
**Verdict:** CONFIRMED
**Severity:** Low

## Summary

`~/.claude/CLAUDE.md` documents the Masonry hook table with two errors: `masonry-agent-onboard.js` is listed as active but is not registered in settings.json (it never runs), and `masonry-context-monitor.js` is listed under the "Stop" event but is actually registered under Stop as async:true — which is correct, but CLAUDE.md's event label matches `settings.json`.

## Evidence

**Error 1 — masonry-agent-onboard.js listed as active:**
```
masonry-agent-onboard.js | PostToolUse (Write/Edit) | Auto-onboard new agents to registry
```
`settings.json`: No PostToolUse entry for `masonry-agent-onboard.js`.
This hook never fires. The documentation creates a false expectation that agent onboarding is automatic.

**Error 2 — masonry-context-monitor.js event label:**
CLAUDE.md: `masonry-context-monitor.js | Stop | Warn when context exceeds 150K tokens (async)`
`settings.json`: `Stop` event, `async: true` for masonry-context-monitor.js.
This is actually correct — the event is Stop. No error here.

**Error 3 — Missing hooks from CLAUDE.md table:**
The following hooks ARE in settings.json but NOT in CLAUDE.md's hook table:
- `masonry-tdd-enforcer.js` (PostToolUse Write|Edit, async:true)

**CLAUDE.md lists 14 hooks in the table; settings.json has 15 registered hooks (excluding masonry-agent-onboard.js which is listed in CLAUDE.md but absent from settings.json, and masonry-recall-check.js which is absent from both).**

## Impact

- Documentation creates false confidence: developers reading CLAUDE.md believe 15 hooks are active when 14 actually fire
- The missing `masonry-tdd-enforcer.js` entry means its (broken — see D3.4) async behavior is undocumented
- onboarding stories in onboarding materials are inaccurate

## Files

- `~/.claude/CLAUDE.md` — hook table with errors
- `~/.claude/settings.json` — authoritative hook registration

## Recommendation

Sync CLAUDE.md hook table with settings.json. Remove `masonry-agent-onboard.js` from the table (or add it to settings.json). Add `masonry-tdd-enforcer.js` to the table with the async:true note.
