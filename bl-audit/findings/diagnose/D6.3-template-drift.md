# D6.3 — Template/ Directory vs Live Masonry Implementation

**Question ID:** D6.3
**Domain:** D6 Structural Problems
**Priority:** MEDIUM
**Verdict:** CONFIRMED
**Severity:** Low

## Summary

`template/.claude/agents/` is substantially in sync with the live `.claude/agents/` — it contains the same 34 BL2.0 agents. However, `template/program.md` is the BL1.x format (simulate.py loop) and does not reflect the BL2.0 program format. New projects started from `template/` would use the old BL1.x research loop rather than the BL2.0 campaign structure.

## Evidence

**`template/.claude/agents/` contents (35 files):**
Contains all current BL2.0 agents: trowel, synthesizer-bl2, hypothesis-generator-bl2, question-designer-bl2, mortar, diagnose-analyst, compliance-auditor, research-analyst, etc. Agent set is current ✓

**`template/program.md` — BL1.x format:**
```markdown
# Autoresearch Program — Generic Template
...
## What You CAN Do
- Modify `simulate.py` — this is the ONLY file you edit.
```
This is the BL1.x simulate.py loop format. BL2.0 uses a questions.md campaign loop without mandatory simulate.py.

**What's missing from template/ for BL2.0:**
- No BL2.0-format `program.md` (with Trowel loop instructions, wave management, finding format)
- `template/program.md` refers to `simulate.py` as "the ONLY file you edit" — incorrect for static-analysis BL2.0 campaigns like this one
- `template/questions.md` (if present) uses BL1.x format — would need BL2.0 `## D1.1` format with Mode/Agent/Priority fields

**What IS current in template/:**
- `.claude/agents/` — fully current with BL2.0 agent fleet ✓
- `constants.py`, `simulate.py` — appropriate for quantitative BL campaigns ✓
- `.gitignore`, `masonry.json` — current ✓

## Impact

- Low: most new BL2.0 projects are research campaigns that do need simulate.py (ADBP, recall, etc.)
- Static-code audit campaigns (like this one) use a different `program.md` that has been written manually
- `CLAUDE.md` `bl-init` skill likely handles the divergence by prompting for campaign type
- Risk: someone copies template/ for a static analysis campaign and follows the simulate.py loop incorrectly

## Files

- `template/program.md` — BL1.x loop format, doesn't cover BL2.0 static campaign mode
- `template/.claude/agents/` — current ✓
- `.claude/agents/` — live reference (34 agents, current)

## Recommendation

Add a BL2.0-format `program.md` template (or a note in template/ README) explaining the two modes: simulate.py campaigns vs. static analysis campaigns. The BL2.0 static format uses a different program.md structure (no simulate.py requirement, Trowel-driven loop).
