# D2.6 — Agent Files Referenced in CLAUDE.md But Not On Disk

**Question ID:** D2.6
**Domain:** D2 Unwired Items
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

`~/.claude/CLAUDE.md` references two specialized agents by name — `solana-specialist` and `uiux-master` — that appear in the Masonry Specialist Agents table, but no corresponding `.md` files exist in `agents/`, `~/.claude/agents/`, or `.claude/agents/`. These agents cannot be invoked.

## Evidence

**CLAUDE.md Masonry Specialist Agents table includes:**
```
uiux-master    | UI/UX design, component build, Figma token extraction, dark dashboard | sonnet
solana-specialist | Solana programs, Anchor, DeFi integration, Token-2022, security audits | opus
```

**File search results:**
- `agents/uiux-master.md` — NOT FOUND
- `~/.claude/agents/uiux-master.md` — NOT FOUND
- `.claude/agents/uiux-master.md` — NOT FOUND
- `agents/solana-specialist.md` — NOT FOUND
- `~/.claude/agents/solana-specialist.md` — NOT FOUND
- `.claude/agents/solana-specialist.md` — NOT FOUND

**`agent_registry.yml` entries:** Neither `uiux-master` nor `solana-specialist` appears in the registry.

**Skills that reference these agents:**
- `masonry/skills/ui-init.md`, `masonry/skills/ui-compose.md`, `masonry/skills/ui-review.md`, `masonry/skills/ui-fix.md` — all invoke `uiux-master` but the agent doesn't exist
- Mortar routing table in CLAUDE.md routes `UI/design` → `uiux-master`

## Impact

- Any `/ui-init`, `/ui-compose`, `/ui-review`, or `/ui-fix` skill invocation that tries to spawn `uiux-master` will fail or fall back to a generic agent
- Mortar's routing table lists `uiux-master` — routing calls for UI work will produce confusing behavior
- ADBP project (Solana) has no specialized agent for smart contract or DeFi work
- Both gaps are silently covered by general-purpose fallback — no hard error, just missing specialist capability

## Files

- `~/.claude/CLAUDE.md` — references both missing agents
- `masonry/skills/ui-init.md` et al — reference uiux-master
- `masonry/agent_registry.yml` — neither agent registered

## Recommendation

Either create the missing agent `.md` files (`uiux-master.md`, `solana-specialist.md`) with proper frontmatter and system prompts, or remove references to them from CLAUDE.md and the UI skills until they're written.
