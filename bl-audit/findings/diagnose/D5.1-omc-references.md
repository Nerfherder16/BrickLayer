# D5.1 — Remaining oh-my-claudecode References

**Question ID:** D5.1
**Domain:** D5 Stale References
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

"oh-my-claudecode" references remain in two functional files: `masonry/skills/masonry-build.md` (uses `subagent_type="oh-my-claudecode:executor"` — a non-functional subagent type) and `mcp-advisor.md` copies in multiple project directories (documents OMC as an installed plugin). Additionally, `~/.claude/CLAUDE.md` mentions OMC in its feedback memory section.

## Evidence

**Functional OMC references (must be removed):**

1. `masonry/skills/masonry-build.md` lines 40, 44, 45:
   ```
   - Use `subagent_type="oh-my-claudecode:executor"` for implementation tasks
   [Agent call 1]: subagent_type="oh-my-claudecode:executor"
   [Agent call 2]: subagent_type="oh-my-claudecode:executor"
   ```
   This skill will fail when invoked — OMC is uninstalled; the executor subagent type doesn't exist.

2. `.claude/agents/mcp-advisor.md` line ~69 and copies in project dirs:
   ```
   oh-my-claudecode plugin as if it's still installed
   ```
   Documents OMC as a recommended plugin — agents reading this for guidance get incorrect information.

**Documentation OMC references (acceptable historical context):**
- `CHANGELOG.md` — historical git commit messages referencing OMC (ok)
- `docs/masonry/MASONRY-SPEC.md` — spec documents why OMC was replaced (ok — historical)

- BL1.x project findings archives — historical research outputs (ok)

## Impact

- `/build` skill (`masonry-build.md`) is broken — invoking it spawns agents with a non-existent `subagent_type`, which either fails silently or falls back to general-purpose
- `mcp-advisor.md` contains incorrect installation guidance that will mislead agents
- The memory note in CLAUDE.md says "oh-my-claudecode is deleted; use Masonry fleet agents instead" — correctly acknowledges removal but the functional files haven't been updated

## Files

- `masonry/skills/masonry-build.md` — must replace OMC executor with standard subagent type
- `.claude/agents/mcp-advisor.md` — must remove OMC plugin reference
- Copies of mcp-advisor.md in project dirs: `projects/adbp2/.claude/agents/`, `projects/ADBP3/.claude/agents/`, etc.

## Recommendation

Update `masonry-build.md` to use `subagent_type="developer"` (or another appropriate agent type) instead of `oh-my-claudecode:executor`. Update `mcp-advisor.md` to remove the OMC plugin reference and replace with current Masonry fleet agent documentation.
