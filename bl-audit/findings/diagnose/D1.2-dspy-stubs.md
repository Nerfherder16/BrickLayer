# D1.2 — Unreferenced DSPy Stubs

**Question ID:** D1.2
**Domain:** D1 Dead Code
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

`masonry/src/dspy_pipeline/generated/` contains 46 auto-generated DSPy signature stub files that are never imported or referenced by any code in the pipeline. Additionally, `masonry/optimized_prompts/` is empty — no agents have ever been successfully optimized.

## Evidence

```
masonry/src/dspy_pipeline/generated/   # 46 .py stub files
masonry/optimized_prompts/             # Empty directory
```

- Grep for any `from generated` or `import generated` in the dspy_pipeline directory: 0 matches
- `optimizer.py` writes to `masonry/optimized_prompts/{agent}.json` but never imports from `generated/`
- `training_extractor.py` produces training data fed to the optimizer, with no reference to stubs
- The stubs appear to have been auto-generated by `masonry-agent-onboard.js` (itself dead per D1.1) but the optimizer uses a different code path

## Impact

- 46 files taking up space and creating cognitive noise
- The entire generated/ directory is effectively scaffolding that was never wired up
- Because `masonry/optimized_prompts/` is empty, ALL agents run with un-optimized prompts — the DSPy optimization feature has never produced output
- The onboarding → stub generation → optimization pipeline is broken at multiple points (dead hook + orphaned stubs)

## Files

- `masonry/src/dspy_pipeline/generated/` — 46 unreferenced stub files
- `masonry/src/dspy_pipeline/optimizer.py` — writes to optimized_prompts/, never reads generated/
- `masonry/optimized_prompts/` — empty, no optimization has run
- `masonry-agent-onboard.js` — the hook that generates stubs (dead per D1.1)

## Recommendation

Either: (a) delete `generated/` and update the onboarding flow to not generate stubs until they're actually used, or (b) wire the stubs into the optimizer. The empty `optimized_prompts/` directory suggests option (a) is the right first step — get one end-to-end optimization working before scaling to 46 agents.
