# D3.3 — Semantic Router Reads Wrong Env Var

**Question ID:** D3.3
**Domain:** D3 Bugs & Logic Errors
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** High

## Summary

`masonry/src/routing/semantic.py` reads `OLLAMA_URL` from the environment but `~/.claude/settings.json` sets `OLLAMA_HOST`. The env var name mismatch means semantic routing ALWAYS uses the hardcoded default URL (`http://192.168.50.62:11434`) regardless of what's configured in settings.json. Additionally, the documented threshold (0.75) does not match the code default (0.60).

## Evidence

**semantic.py line 26:**
```python
_OLLAMA_URL = os.environ.get("OLLAMA_URL", "http://192.168.50.62:11434")
```

**settings.json env section:**
```json
"env": {
  "OLLAMA_HOST": "http://192.168.50.62:11434",
  ...
}
```

The code reads `OLLAMA_URL` but settings injects `OLLAMA_HOST`. On any machine where the Ollama server is at a different address, semantic routing silently falls back to the hardcoded IP — and since the hardcoded IP matches Tim's homelab, this bug has never surfaced on the primary dev machine.

**Threshold mismatch:**
- `semantic.py` line 29: `_DEFAULT_THRESHOLD = 0.60`
- CLAUDE.md: "Ollama cosine similarity at http://192.168.50.62:11434 (threshold 0.75)"

The documented 0.75 threshold would be more discriminating; the actual 0.60 threshold means weaker matches get promoted to semantic routing decisions.

## Impact

- On primary dev machine: no observable impact (hardcoded URL happens to be correct)
- On any other machine (new dev, CI, backup): semantic routing always fails because OLLAMA_URL is never set, meaning requests fall through to Layer 3 (LLM routing) or Layer 4 (fallback)
- Lower threshold (0.60 vs 0.75) causes more false-positive semantic matches, potentially routing requests to wrong agents
- This breaks the "60%+ handled by deterministic + semantic with zero LLM calls" guarantee from CLAUDE.md

## Files

- `masonry/src/routing/semantic.py` — reads wrong env var, uses undocumented threshold
- `~/.claude/settings.json` — sets OLLAMA_HOST, not OLLAMA_URL
- `~/.claude/CLAUDE.md` — documents threshold as 0.75

## Recommendation

Fix `semantic.py` line 26:
```python
_OLLAMA_URL = os.environ.get("OLLAMA_URL") or os.environ.get("OLLAMA_HOST", "http://192.168.50.62:11434")
```
Or update `settings.json` to also set `OLLAMA_URL`. Also update `_DEFAULT_THRESHOLD` to 0.75 to match documented behavior (or update CLAUDE.md to reflect 0.60).
