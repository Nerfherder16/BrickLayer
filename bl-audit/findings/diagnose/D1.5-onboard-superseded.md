# D1.5 — onboard.py Superseded by masonry/scripts/onboard_agent.py

**Question ID:** D1.5
**Domain:** D1 Dead Code
**Priority:** LOW
**Verdict:** CONFIRMED
**Severity:** Low

## Summary

`onboard.py` at the repo root is a BrickLayer 1.x project-scaffolding tool that creates projects under `autosearch/projects/{name}/`. It is completely different from — and superseded by — `masonry/scripts/onboard_agent.py`, which handles agent registration into `agent_registry.yml`. The root `onboard.py` is no longer called by any active code path and constitutes dead code.

## Evidence

- `onboard.py` creates new BL1.x research projects (`AUTOSEARCH_ROOT / "projects"`), reads GitHub URLs, writes `project.json`, `questions.md`, `prepare.md`
- `masonry/scripts/onboard_agent.py` is the BL2.0 replacement for agent-specific registration
- Grep for `onboard.py` references: matches appear only in BL1.x project findings archives (`projects/bricklayer/findings/`), `tests/test_core.py` (may reference legacy path), and self-referential mentions in `questions.md` and simulation files — not in any active hook or script
- No Makefile, no hook, no `__main__` entrypoint is wired to call `onboard.py`
- BL2.0 project bootstrapping is now done via the `/masonry-init` skill and the `bl-init` agent

## Impact

- No runtime impact: file is never called so it cannot break anything
- Adds confusion for new contributors who might try to use it
- `autosearch/projects/` directory layout it targets appears to be a BL1.x path that may no longer be maintained

## Files

- `onboard.py` — BL1.x project scaffolding tool (dead code)
- `masonry/scripts/onboard_agent.py` — BL2.0 agent registration tool (active)

## Recommendation

Archive or remove `onboard.py`. Document in CHANGELOG that project onboarding has moved to `/masonry-init` skill.
