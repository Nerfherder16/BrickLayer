# D2.3 — Global Agent Path Mismatch in Registry

**Question ID:** D2.3
**Domain:** D2 Unwired Items
**Priority:** MEDIUM
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

Several agents registered in `agent_registry.yml` under `~/.claude/agents/` prefix point to files that actually reside in `.claude/agents/` (repo-local). The mismatch means file lookups using the registry path will fail for these agents.

## Evidence

**Confirmed mismatched entries:**
- Registry: `~/.claude/agents/test-writer.md` → Actual location: `C:/Users/trg16/.claude/agents/test-writer.md` ✓ (this one is correct)
- Registry: `~/.claude/agents/health-monitor.md` → Actual location: `C:/Users/trg16/.claude/agents/health-monitor.md` ✓ (this one is correct)
- Registry: `~/.claude/agents/design-reviewer.md` → Actual location: `.claude/agents/design-reviewer.md` (repo-local, NOT global)
- Registry: `~/.claude/agents/frontier-analyst.md` → Actual location: `.claude/agents/frontier-analyst.md` (repo-local, NOT global)

**Context from D2.2:** The `~/.claude/agents/` group in the registry has 15 entries. Some agents that were created as repo-local BL2.0 agents (in `.claude/agents/`) were incorrectly registered with the global path prefix. This happens when `onboard_agent.py` is run from the wrong working directory, causing `Path.relative_to(Path.cwd())` to resolve the path incorrectly.

**Root cause:** `onboard_agent.py` computes relative file paths using `Path.relative_to(Path.cwd())`. If run from the repo root with `--agents-dir ~/.claude/agents`, files in `.claude/agents/` that share a name with files in `~/.claude/agents/` get ambiguous resolution.

## Impact

- Agents like `design-reviewer` and `frontier-analyst` have wrong path in registry
- Any registry consumer that reads the agent file from the `file:` field will look in the wrong directory
- The routing engine will still return these agents by name (uses in-memory registry), but downstream consumers that need the file content will fail

## Files

- `masonry/agent_registry.yml` — mismatched `~/.claude/agents/` entries
- `.claude/agents/design-reviewer.md` — correct location
- `.claude/agents/frontier-analyst.md` — correct location
- `~/.claude/agents/` — does NOT contain these files

## Recommendation

Correct the `file:` field for `design-reviewer` and `frontier-analyst` in `agent_registry.yml` from `~/.claude/agents/{name}.md` to `.claude/agents/{name}.md`. Review all 15 `~/.claude/agents/` entries for correctness.
