# D1.4 — Orphaned mcp_gateway.py

**Question ID:** D1.4
**Domain:** D1 Dead Code
**Priority:** MEDIUM
**Verdict:** CONFIRMED
**Severity:** Low

## Summary

`mcp_gateway.py` at the repo root is a FastMCP 3.1 proxy prototype that is never imported by any other Python file and has no entry in `settings.json`, `~/.claude.json`, or any MCP server config. It is dead code — an experimental approach that was not adopted.

## Evidence

- Grep for `mcp_gateway` across all `.py` files in the repo: only `mcp_gateway.py` itself matches
- No settings.json entry, no `.claude.json` reference, no Makefile or launch script calls it
- The file's docstring shows it was intended as a standalone server (`python mcp_gateway.py --port 8351`) and a `~/.claude.json` snippet for connecting to it — neither has been set up
- The actual MCP server is at `masonry/mcp_server/server.py`, launched from `settings.json` via `node` + `@modelcontextprotocol/sdk`
- Contains a hardcoded API key: `EXA_API_KEY = os.environ.get("EXA_API_KEY", "d1383966-eb3f-4c92-9291-b53f4d0c15d9")` — a security concern independent of the dead-code question

## Impact

- Medium security concern: hardcoded Exa API key is committed in plaintext
- Low operational risk: file is never loaded so it cannot cause runtime failures
- Cognitive overhead: the file suggests a different MCP architecture than what's actually used

## Files

- `mcp_gateway.py` — dead FastMCP proxy prototype

## Recommendation

Remove `mcp_gateway.py`. Before deleting, rotate the hardcoded Exa API key (`d1383966-eb3f-4c92-9291-b53f4d0c15d9`) as it has been committed to git history. If the gateway approach is desired in the future, it can be recreated from scratch.
