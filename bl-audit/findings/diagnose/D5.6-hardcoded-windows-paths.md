# D5.6 — Hardcoded Windows Paths in Hook Files

**Question ID:** D5.6
**Domain:** D5 Stale References
**Priority:** MEDIUM
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

`masonry/src/hooks/masonry-session-start.js` contains a hardcoded Windows absolute path (`C:/Users/trg16/Dev/Bricklayer2.0`) at line 123 that is used to detect the BrickLayer root. This breaks the hook on any machine where the repo is cloned to a different path.

## Evidence

`masonry-session-start.js` line 123:
```javascript
const blRoot = "C:/Users/trg16/Dev/Bricklayer2.0";
```

This path is used to:
- Detect if the current session is within the BrickLayer project
- Build paths to project subdirectories
- Determine whether to show the BL-specific session start UI

**Other hooks use portable path construction:**
- `masonry-stop-guard.js`: uses `cwd` from hook input, `os.tmpdir()`, and `path.join()` — no hardcoded paths
- `masonry-lint-check.js`: uses `cwd` from hook input — no hardcoded paths
- `masonry-approver.js`: uses `cwd` from hook input — no hardcoded paths
- `masonry-observe.js`: uses `cwd` from hook input and reads `masonry.json` for project name — portable

**Impact on session-start specifically:** The hardcoded path means the session-start hook's campaign restoration and context injection only works when running from `C:/Users/trg16/Dev/Bricklayer2.0`. On "casaclaude" or "proxyclaude" (Tim's secondary machines) or any future machine, the hook silently fails to detect the BL project and skips context restoration.

## Files

- `masonry/src/hooks/masonry-session-start.js` line 123 — hardcoded blRoot path

## Recommendation

Replace the hardcoded path with environment-based detection:
```javascript
// Use cwd from hook input, or check for sentinel files (program.md + questions.md)
// to detect BL project rather than matching against a hardcoded path
const blRoot = input.cwd || process.cwd();
const isBlProject = fs.existsSync(path.join(blRoot, 'masonry.json')) ||
                    (fs.existsSync(path.join(blRoot, 'program.md')) &&
                     fs.existsSync(path.join(blRoot, 'questions.md')));
```
