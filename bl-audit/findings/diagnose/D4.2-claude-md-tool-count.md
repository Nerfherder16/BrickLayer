# D4.2 — CLAUDE.md Documents Only 5 of 14 MCP Tools

**Question ID:** D4.2
**Domain:** D4 Config Drift
**Priority:** MEDIUM
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

`~/.claude/CLAUDE.md`'s "Masonry MCP Tools" table lists only 5 tools (`masonry_route`, `masonry_optimization_status`, `masonry_onboard`, `masonry_drift_check`, `masonry_optimize_agent`) but `masonry/mcp_server/server.py` exposes 14 tools. Nine tools are undocumented in CLAUDE.md.

## Evidence

**CLAUDE.md Masonry MCP Tools table (5 entries):**
```
masonry_route               | Route a request through the four-layer pipeline
masonry_optimization_status | Check DSPy optimization status for all agents
masonry_onboard             | Trigger agent auto-onboarding for new .md files
masonry_drift_check         | Run drift detection across all registry agents
masonry_optimize_agent      | Trigger MIPROv2 optimization for a specific agent
```

**server.py tools (14 total) — 9 not in CLAUDE.md:**
- `masonry_status` — current campaign state
- `masonry_questions` — question bank query
- `masonry_nl_generate` — NL to BL questions
- `masonry_weights` — priority weights
- `masonry_git_hypothesis` — questions from git diff
- `masonry_run_question` — run a question
- `masonry_fleet` — agent fleet status
- `masonry_recall_search` — Recall memory search
- `masonry_registry_list` — list registry entries

These 9 tools are documented in `tools-manifest.md` (partially, with some naming differences per D2.4) but absent from CLAUDE.md.

**Impact of omission:** CLAUDE.md is the primary context document injected into every session. Agents learning the system from CLAUDE.md will not know about `masonry_status`, `masonry_questions`, or `masonry_fleet` — the most commonly needed campaign-management tools.

## Files

- `~/.claude/CLAUDE.md` — incomplete MCP tool table
- `masonry/mcp_server/server.py` — authoritative list of 14 tools

## Recommendation

Update CLAUDE.md MCP tools table to list all 14 tools with brief descriptions. Group them: campaign-management tools (status, questions, findings, fleet, weights) vs. optimization/registry tools (route, optimize, drift_check, onboard, registry_list).
