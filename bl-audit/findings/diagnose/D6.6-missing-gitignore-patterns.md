# D6.6 — Runtime Artifacts Missing from .gitignore

**Question ID:** D6.6
**Domain:** D6 Structural Problems
**Priority:** LOW
**Verdict:** CONFIRMED
**Severity:** Low

## Summary

Three runtime artifact patterns are missing from the root `.gitignore`: `masonry-activity-*.ndjson` (session write logs), `agent_db.json` (agent performance database), and `masonry/optimized_prompts/` (DSPy optimization outputs). If generated, these would be committed to git as runtime state.

## Evidence

**Root `.gitignore` confirmed to include:**
- `masonry-state.json` ✓
- `.autopilot/` ✓
- `.ui/` ✓
- `history.db` ✓

**Missing from `.gitignore`:**

1. `masonry-activity-*.ndjson` — Session write activity logs. Written to `os.tmpdir()` at runtime, but the pattern should be in `.gitignore` in case someone runs hooks from a path that resolves tmpdir to the repo. Low likelihood but good hygiene.

2. `agent_db.json` — Agent performance database written by `masonry-tool-failure.js` and read by multiple agents for score tracking. Should not be committed — it contains session-transient scores.

3. `masonry/optimized_prompts/` — DSPy optimization output directory. Currently empty (see D1.2), but when optimization runs, the outputs should not be committed (they're derived from training data and can be regenerated).

**Note:** `masonry-activity-*.ndjson` files are written to `os.tmpdir()` by the hooks, not to the repo. The risk of accidental commit is lower for this pattern. `agent_db.json` and `masonry/optimized_prompts/` are higher priority.

## Impact

- Low current risk: `masonry/optimized_prompts/` is empty; `agent_db.json` may or may not currently exist
- If any of these files get committed, history accumulates runtime noise and potential scoring data from other machines
- `agent_db.json` contains agent performance scores that vary per machine/campaign — committing it would cause merge conflicts on multi-machine workflows

## Files

- `.gitignore` — missing three patterns
- `masonry-activity-*.ndjson` (in tmpdir) — pattern to add
- `agent_db.json` (in repo root) — pattern to add
- `masonry/optimized_prompts/` — pattern to add

## Recommendation

Add to root `.gitignore`:
```
masonry-activity-*.ndjson
agent_db.json
masonry/optimized_prompts/
```
