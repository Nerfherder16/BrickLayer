# D1.1 — Dead Hook Files

**Question ID:** D1.1
**Domain:** D1 Dead Code
**Priority:** HIGH
**Verdict:** CONFIRMED
**Severity:** Medium

## Summary

`masonry-agent-onboard.js` is dead code — it was designed as a PostToolUse hook but is not registered in `~/.claude/settings.json`. `masonry-recall-check.js` is NOT dead — it is intentionally spawned as a detached background process by `masonry-statusline.js`.

## Evidence

**masonry-agent-onboard.js — DEAD:**
- File header: `// PostToolUse hook — triggered when Write/Edit creates a new agent file`
- Searches settings.json: no entry for `masonry-agent-onboard.js`
- CLAUDE.md documents it as an active hook: `masonry-agent-onboard.js | PostToolUse (Write/Edit) | Auto-onboard new agents to registry`
- The hook will never execute because the settings.json registration is absent

**masonry-recall-check.js — NOT DEAD:**
- File header: `// Spawned as a detached background process by masonry-statusline.js`
- `masonry-statusline.js` line ~50 spawns it via `spawn('node', [recallCheckPath], { detached: true, stdio: 'ignore' })`
- It is not a settings.json hook — it is a subprocess utility called by the statusline script
- No registration needed or expected

## Impact

- Agent auto-onboarding silently fails: new `.md` files written to `agents/` directories are never added to `agent_registry.yml`
- CLAUDE.md implies onboarding is automatic ("Zero Manual Steps"), but the hook that performs the onboarding is never triggered
- Agents written during a session stay at `tier: "draft"` only if they were manually onboarded or were pre-existing
- Zero runtime errors from the dead hook (it simply never runs)

## Files

- `masonry/src/hooks/masonry-agent-onboard.js` — dead PostToolUse hook
- `~/.claude/settings.json` — missing registration entry
- `masonry/src/hooks/masonry-statusline.js` — correctly spawns masonry-recall-check.js
- `masonry/src/hooks/masonry-recall-check.js` — not dead, used as subprocess

## Recommendation

Register `masonry-agent-onboard.js` in `settings.json` under PostToolUse for Write and Edit events, or remove the hook and update CLAUDE.md to remove the "Zero Manual Steps" claim. If re-registering, test that it correctly parses frontmatter and appends to `agent_registry.yml`.
