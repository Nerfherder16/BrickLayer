**Agent**: economizer
**Date**: 2026-03-22
**Severity**: Low
**Verdict**: FALSE_POSITIVE (threshold not breached — but conditional; see below)

## Summary

The 50K token threshold is **not breached** for auto-loaded context. Every session (regardless of project type) loads ~22,587 tokens of instructions before any user message. Agent definition files are NOT auto-loaded — they are only injected into spawned sub-agent contexts when the Agent tool is invoked. The true overhead concern is that **4 Figma/UI rules files (9,390 tokens)** are loaded in every session including non-UI code audit sessions, representing irrelevant context that cannot be suppressed per-project.

## Analysis

### What the 22.6K Tokens Buys

- **3,363 tokens** (global CLAUDE.md): Architecture routing, Tim's dev context, project list, MCP servers, workflow reference — high utility per token
- **2,848 tokens** (BL CLAUDE.md): Full BL 2.0 loop instructions, directory layout, orchestration architecture — necessary for all BL sessions
- **3,948 tokens** (6 universal rules): Security, quality standards, spec workflow, TDD, verification — relevant in every dev session
- **12,423 tokens** (6 UI/Figma rules): Design philosophy, Figma MCP workflow, designer guide, React/Tailwind, UI workflow, Figma-to-code — **relevant only when doing UI work**

### Actionable Redundancy: UI Context in Non-UI Sessions

The 6 Figma/UI rules files add 12,423 tokens to every session, including this code audit campaign, which will never touch a Figma file. Claude Code does not currently support per-project rules file suppression — all `~/.claude/rules/*.md` files load universally.

**Workaround options:**
1. Move UI-specific rules to `~/.claude/rules/ui/` and only include them when working in UI-adjacent projects (manual — no current mechanism to auto-exclude)
2. Consolidate the 6 UI files into 2 (a design-tokens reference + a workflow reference) — currently 12,423 tokens → could be reduced to ~5,000–6,000 tokens with de-duplication and summary compression
3. Accept as-is — 12,423 tokens is 6.2% of a 200K context window

## Verdict Qualification

**FALSE_POSITIVE** on the 50K threshold (auto-loaded context is 22.6K, not 50K+). However:
- A **conditional concern exists**: if per-session rules continue to grow, the threshold could be approached. Current trajectory: rules grew from ~1,000 lines (Wave 1 baseline) to 1,701 lines now — a 70% increase.
- The **primary inefficiency is not volume but relevance**: 55% of auto-loaded rules tokens (12,423 of 22,587) are UI-specific content irrelevant to non-UI sessions.

## Proposed Fix

1. **No immediate action required** — 50K threshold not breached
2. **Optional consolidation**: Merge 6 UI rules files into 2 compact references, reducing UI context overhead from 12,423 → ~5,500 tokens. Effort: Medium. Impact: Low.
3. **Growth monitoring**: If rules/ grows beyond 2,500 lines, revisit — that would put auto-loaded context at ~27K tokens (54% of threshold) and compression would become worthwhile.
