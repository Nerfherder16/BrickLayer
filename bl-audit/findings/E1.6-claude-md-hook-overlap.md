# Finding: E1.6 — CLAUDE.md Rules Files Are Complementary to Hooks, Not Redundant; True Overlap Is Minimal

**Agent**: economizer
**Date**: 2026-03-22
**Severity**: Low
**Verdict**: FALSE_POSITIVE

## Summary

The hypothesis that CLAUDE.md and rules files contain redundant content already enforced by hooks does not hold at a significant level. Hooks enforce **outcomes** (a test file must exist, lint must pass, Stop is blocked with pending tasks). Rules files describe **process** (how to run the RED-GREEN-REFACTOR cycle, how to structure specs, what the file schemas look like). These are complementary, not duplicated. The CLAUDE.md "Masonry Hooks" table (14 entries, ~25 lines) provides routing signal — Claude needs to know what runs automatically to avoid duplicating it. Total confirmed redundancy across all 12 rules files + CLAUDE.md: ~15–20 lines, concentrated in two places: a small overlap between `spec-workflow.md` and `ui-design-workflow.md` state machines, and the `tdd-enforcement.md` "Enforcement" section which partly restates content in `spec-workflow.md`. The main context concern is total size (1,953 lines), not internal redundancy.

## Evidence

### Hook vs. Rules Mapping

| Hook | What it enforces | Corresponding rules file | Overlap type |
|------|-----------------|--------------------------|--------------|
| `masonry-tdd-enforcer.js` | Test file existence (blocks in /build mode) | `tdd-enforcement.md` (63 lines) | Process guidance; hook enforces outcome, rule describes method. **LOW overlap.** |
| `masonry-lint-check.js` | ruff + prettier + eslint after every write | `verification-checklist.md` (59 lines) | Lint section (~15 lines) partly describes automated behavior. **LOW overlap.** |
| `masonry-build-guard.js` | Blocks Stop with pending `.autopilot/` tasks | `spec-workflow.md` (73 lines) | Rule defines schemas the hook reads; not descriptively redundant. |
| `masonry-context-safety.js` | Blocks ExitPlanMode during active build | `spec-workflow.md` (73 lines) | Rule defines mode values the hook checks. **Non-redundant — provides schema context.** |
| `masonry-stop-guard.js` | Blocks Stop with uncommitted changes | None | No rules file describes this hook. |

### True Overlap Found

**1. tdd-enforcement.md "Enforcement" section (lines 58–64, 7 lines)**

```markdown
## Enforcement
TDD discipline is enforced by the /build workflow:
- Worker agents are instructed to write tests first
- Orchestrator validates all tests pass before marking DONE
- `progress.json` is proof of compliance
```

This restates content from `spec-workflow.md`'s "Agent Mode Architecture" section. Not harmful, but ~7 lines could be removed from `tdd-enforcement.md` with a cross-reference.

**2. spec-workflow.md and ui-design-workflow.md parallel structure**

Both files open with a state machine diagram, then define mode values, directory schema, and progress.json schema. The `progress.json` schema is defined identically in both files (same fields, same status values). That's ~15 lines of true duplication.

**3. CLAUDE.md "Masonry Hooks" table (lines ~100–130, ~25 lines)**

The hooks table describes each hook's event and purpose — but this IS needed routing signal. Without it, Claude would not know `masonry-tdd-enforcer.js` runs on every write or that `masonry-build-guard.js` blocks Stop during builds. This is not redundant — it's required context to prevent Claude from duplicating hook behavior manually.

## Proposed Fix

### Trim confirmed overlaps (~22 lines)

1. **`tdd-enforcement.md` "Enforcement" section**: Replace 7 lines with: `See spec-workflow.md — enforcement is built into the /build orchestrator.`

2. **`ui-design-workflow.md` and `spec-workflow.md` progress.json schemas**: Move the shared schema definition to one file and cross-reference from the other. Saves ~15 lines.

### Do NOT remove

- CLAUDE.md hooks table — provides required routing signal
- Any of the 8 purely-additive files — all serve distinct purposes
- `tdd-enforcement.md` exemptions list — no hook covers this, agents need it

**Effort**: Trivial (edit 2 files)
**Impact**: Low — true redundancy is ~22 lines, a negligible context savings
