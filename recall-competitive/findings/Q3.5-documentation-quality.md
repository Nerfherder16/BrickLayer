## Evidence

### mem0 — Mintlify-hosted, comprehensive

**URL**: https://docs.mem0.ai

**Platform**: Mintlify (professional docs hosting, OpenAI uses this too)

**Structure** (observed from sidebar scrape):
- Getting Started
  - Overview
  - Python SDK Quickstart (5-step, working code in every step)
  - Node SDK Quickstart
- Self-Hosting Features
  - Graph Memory
  - Enhanced Metadata Filtering
  - Reranker-Enhanced Search
  - Async Memory
  - Multimodal Support
  - Custom Fact Extraction Prompt
  - REST API Server
  - OpenAI Compatibility
- Configuration (LLMs, Vector Databases, Embedding Models, Rerankers)
- API Reference (full REST API with request/response schemas)
- Release Notes (changelog)
- Cookbooks (integration examples)
- Integrations (LangChain, LlamaIndex, etc.)

**Quickstart quality**: The Python quickstart is 4 numbered steps with code blocks, expected output shown, and a "What's Next?" section linking to memory operations, configuration, and advanced features. Time to first working memory: under 5 minutes.

**API Reference**: Full OpenAPI-derived reference at https://docs.mem0.ai/api-reference. Every endpoint has request body schema, required/optional fields, authentication header example, and response schema with example JSON.

**Search**: Built-in docs search with AI assistant ("Ask AI" button).

**LLMs.txt**: `https://docs.mem0.ai/llms.txt` — a complete documentation index optimized for AI coding tools. Signals that mem0 actively thinks about LLM-driven discovery of their API.

Source: https://docs.mem0.ai/open-source/python-quickstart (scraped live)

### Zep — Fern-hosted, professional

**URL**: https://docs.getzep.com (current v3), https://help.getzep.com (older, some CE content)

**Platform**: Fern (professional docs platform, similar to Mintlify)

**Structure** (from live scrape):
- Key Concepts
- Quickstart (Python, TypeScript, Go)
- Cookbooks
- SDK Reference (Python, TypeScript, Go — per-method documentation)
- Mem0 Migration Guide (actively poaching mem0 users)
- Graphiti docs (separate section for their open-source temporal KG)

**Notable feature**: **Mem0 Migration guide** at https://docs.getzep.com/mem0-to-zep — a dedicated page helping mem0 users migrate to Zep. This is aggressive positioning that Recall has no equivalent of.

**SDK Reference**: Full method-level documentation with example code in 3 languages. Every SDK method has a working Python snippet and a "Try it" button in the API reference.

**Quick Start quality**: 3-line initialization. Sub-200ms retrieval latency called out explicitly in the quickstart text.

**MCP server**: Zep has an MCP server for AI coding tools. Also provides `llms.txt` at the docs root.

Source: https://docs.getzep.com/ (scraped live), https://help.getzep.com/quick-start-guide

### Letta — Starlight/Astro, most polished UX

**URL**: https://docs.letta.com

**Platform**: Starlight (Astro-based) — the most visually polished of the three

**Structure** (from live scrapes):
- Tutorials (First steps: hello-world tutorial, step-by-step with expected output for every code block)
- Guides (Docker, Build with Letta)
- API Reference (full REST, every endpoint)
- Models (OpenAI, Anthropic, Ollama — each has its own page)

**Notable features**:
- "Open in Claude" / "Open in ChatGPT" / "Open in Cursor" buttons on every page — deeplinks that open the page content directly in AI coding tools
- "Ask Ezra" — a built-in AI assistant (named character) available on every page
- "Copy Markdown" button on every page — makes doc content immediately usable in LLM context windows
- OG images generated per page (`https://docs.letta.com/og/tutorials/hello-world.png`)

**Hello-world tutorial quality**: 5 steps, every step has code in both Python and TypeScript tabs, expected output shown after every code block, "Complete Example" section at the bottom with full working code in one place.

**API Reference**: OpenAPI-derived, all endpoints documented. Authentication section explains bearer tokens. SDK auto-handles auth.

Source: https://docs.letta.com/tutorials/hello-world/ (scraped live), https://docs.letta.com/api-reference/overview/

**What exists**:

**What is missing**:

- Changelog / release notes
- LLMs.txt (how does an AI coding tool discover Recall's API surface?)

## Mitigation Recommendation

**Immediate (half a day)**:
1. **GitHub Pages site**: A minimal `index.html` or Mintlify/Docusaurus deploy from the existing markdown is a half-day project. Even a single well-written page at `recall.projectname.io` signals that the project is alive and developer-accessible.

2. **README quickstart block**: Add a 10-line "Get started" code block to the top of the README that shows `pip install recall-sdk` (once SDK exists), `store()`, and `search()` with expected output. This is the first thing a developer sees.

**Short term (1-2 days)**:
3. **API reference generation**: The FastAPI server already generates an OpenAPI schema at `/docs` (SwaggerUI) and `/redoc`. The self-hosted instance at `http://192.168.50.19:8200/docs` is the API reference — it just needs to be published. Deploy a static export of the OpenAPI spec to GitHub Pages.

4. **LLMs.txt**: Create `recall.projectname.io/llms.txt` listing all endpoints, memory types, and integration methods. This is a 50-line file and signals maturity to AI-native developers.

**Medium term (1 week)**:
5. **Mintlify or Docusaurus deploy**: Free tier on both. Move the existing markdown docs there. Add a quickstart guide. This puts Recall in the same visual tier as mem0.

6. **"Ezra" equivalent**: Even a single Discord channel or a GitHub Discussions section gives developers a place to ask questions — currently there is no visible community surface.

**The most critical single action**: A hosted docs site at a real URL. The GitHub README is fine for contributors but invisible to developers evaluating tools via web search. Mintlify free tier deploys from GitHub in under 30 minutes.
