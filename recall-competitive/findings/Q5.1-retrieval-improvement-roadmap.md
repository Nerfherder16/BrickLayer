## Evidence

### Current Retrieval Baseline (from Q2.5, Q2.1)

- **Embedding**: qwen3-embedding:0.6b — 64.33 MTEB Multilingual, 61.83 MTEB English v2. Strong.
- **Retrieval**: Pure cosine similarity — no BM25, no hybrid
- **Reranker**: sklearn GBM — fast but no semantic cross-attention; feedback-starved
- **Query understanding**: Zero — no expansion, no BFS graph traversal, no HyDE

### Ranked Improvements by ROI

#### Tier 1: High Impact, Low Effort (do first)

**1. Cross-encoder reranking** — Impact: HIGH, Effort: 2-3 days
- Replace sklearn GBM with BGE MiniLM-L6-v2 (< 150MB, CPU inference)
- Cross-encoders use query+memory pairwise attention — semantically superior to GBM feature scoring
- Latency penalty: +30-80ms CPU — acceptable for homelab
- mem0 Platform added reranker (proprietary cloud model); no self-hosted competitor has cross-encoder
- Score delta: `reranking_quality` 0.42 → 0.70+ (estimated)

**2. Embedding model upgrade** — Impact: HIGH, Effort: 1 day
- qwen3-embedding:0.6b (current) → qwen3-embedding:4b
- MTEB score delta: 61.83 → 68.46 (+6.6 points English v2 Retrieval)
- Fits on RTX 3090 (24GB VRAM) alongside qwen3:14b — concurrent inference feasible
- Cost: ollama pull command + config update. Zero code change.
- Score delta: `semantic_retrieval_quality` 0.73 → 0.82+ (estimated)

#### Tier 2: Medium Impact, Medium Effort

**3. Graph-augmented query expansion** — Impact: MEDIUM-HIGH, Effort: 1 week
- When retrieving, expand query with: (a) related entities from Neo4j 1-hop BFS, (b) SUPPORTS/CAUSED_BY edge traversal
- Zep does this implicitly via entity BFS; Recall's typed edges make this MORE expressive
- Requires: API endpoint that does Qdrant vector search → Neo4j entity lookup → re-rank combined results
- Score delta: `query_understanding` 0.45 → 0.65+ (estimated)

**4. BM42 hybrid retrieval** — Impact: MEDIUM (for Recall's use case), Effort: 2-3 days
- Qdrant natively supports BM42 (sparse-dense hybrid)
- BM42 advantage over BM25: replaces TF component with attention weights (no IDF penalty on short text)
- For Recall's use case (sub-200-char fact strings): measured benefit likely 5-15% recall improvement
- Note from Q1.5: for sub-200-char memories specifically, pure vector is 90%+ competitive with hybrid
- Score delta: `hybrid_retrieval` 0.10 → 0.50+ (estimated, even partial implementation counts)

#### Tier 3: High Effort, Uncertain ROI for Homelab

**5. HyDE (Hypothetical Document Embeddings)** — Impact: MEDIUM, Effort: 2-3 weeks
- Generate a hypothetical "ideal memory" for the query using Ollama, embed it, use that embedding for search
- Works well for factual knowledge retrieval where query phrasing differs from stored phrasing
- Adds 200-500ms per query (Ollama inference); significant for interactive Claude Code sessions
- **Not recommended** for Recall's primary use case — homelab latency budget is tight

**6. LLM-based reranking** — Impact: HIGH, Effort: 1 week
- Prompt Ollama to score each retrieved memory for relevance to query
- Latency: +1-3s per query (serial Ollama inference on 5-10 candidates)
- **Not recommended** for interactive sessions; potential value for batch/async retrieval
- Cross-encoder reranking (Tier 1) achieves 80% of benefit at 10% of the latency cost
