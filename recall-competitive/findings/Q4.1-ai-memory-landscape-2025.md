## Evidence

### The Five Taxonomy Categories

The market has converged on five distinct categories of AI memory systems by early 2025. This taxonomy is now cited in multiple analyst comparisons.

**Category 1: Personalization / User-Preference Systems**
- Build fact stores about individual users (preferences, name, past decisions)
- Primary representatives: mem0, SuperMemory
- Architecture: vector + optional graph
- Positioning: cross-app "memory passport" following the user
- mem0 specifically describes itself as: "a memory passport that follows users across apps and agents"

**Category 2: Temporal Knowledge Graph Systems**
- Model relationships between entities over time; track fact changes
- Primary representatives: Zep, Graphiti (its open-source engine)
- Architecture: temporal KG with hybrid retrieval (semantic + BM25 + graph traversal)
- Graphiti arXiv paper submitted Jan 20, 2025; 23,703 GitHub stars as of March 2026
- Claim: outperforms MemGPT on Deep Memory Retrieval benchmark (94.8% vs 93.4%)
- Source: https://arxiv.org/abs/2501.13956

**Category 3: Agent-Tier / OS-Inspired Memory**
- Treats the LLM context window like an OS managing RAM; agent controls what moves in/out of context
- Primary representatives: Letta (formerly MemGPT)
- Architecture: tiered — in-context (working), external (archival), recursive paging
- 21K GitHub stars; renamed from MemGPT in 2024
- Key limitation: requires capable model to drive tool-based paging reliably

**Category 4: Coding-Tool / MCP-Native Memory**
- Persistent memory specifically for developer AI tools (Claude Code, Cursor, Windsurf, Cline)
- Delivered via MCP server; stores across sessions; targets developer context not user chat
- Representatives: OpenMemory MCP (mem0), Engram, minime-mcp, Recall (System-Recall)
- OpenMemory MCP launched May 2025 on ProductHunt — 416 upvotes, Product of the Day #2
- Source: https://hunted.space/product/openmemory-mcp-2
- Engram: self-hosted, Apache 2.0, TypeScript, SQLite+vector, graph visualization, MCP server
- GitHub: https://github.com/zanfiel/engram (24 stars, very new March 2026)

**Category 5: OS-Level Passive Capture**
- Records everything the user sees/does on their machine; builds searchable memory from OS activity
- Was: Rewind.ai (shut down December 19, 2025 — acquired by Meta via Limitless)
- Now: Pieces for Developers (150K+ developers, 9-month memory, cross-app capture, local processing)
- Source: https://9to5mac.com/2025/12/05/rewind-limitless-meta-acquisition/

### Rewind.ai Status: Confirmed Dead (December 2025)

Rewind.ai's parent company, Limitless AI (formerly Rewind AI), was acquired by Meta in December 2025. The Rewind Mac app was shut down on December 19, 2025 — 14 days after the acquisition announcement. Users had 14 days notice. $27M raised. The company pivots to Meta's AI wearables (smart glasses) team.

"Rewind.ai got killed by Meta" — r/remotework thread, March 2026.
"The AI pendant era is officially over" — Mindstream newsletter, Dec 9, 2025.
Source: https://www.mindstream.news/p/the-ai-pendant-era-is-officially-over

This removes one significant privacy-first local capture competitor from the market. Pieces is the remaining player in category 5 but it is not as privacy-pure (cloud options exist).

### New Entrants Since Mid-2024 Not Previously on Radar

1. **OpenMemory MCP** (mem0 product, May 2025): Local MCP server for cross-tool memory. Launched Product Hunt with 416 upvotes. Direct competitor to Recall in category 4. Backed by mem0's $24M and brand recognition. URL: https://mem0.ai/blog/introducing-openmemory-mcp

2. **Graphiti** (Zep open-source, August 2024): The temporal KG engine underlying Zep released as standalone open-source. 23K stars. Now has its own MCP server for Claude/Cursor. URL: https://github.com/getzep/graphiti

3. **Hindsight** (vectorize.io, 2025): MIT-licensed, Ollama-native memory. Explicitly documents "Run Hindsight with Ollama: No API Keys Needed." ~4K stars, growing. URL: https://hindsight.vectorize.io

4. **Pieces for Developers LTM-2** (2025 update): Second-generation OS-level memory agent. Captures from all apps, 9-month retention, 150K+ developer users. Cross-IDE (VS Code, JetBrains). URL: https://pieces.app/features/long-term-memory

5. **Engram** (March 2026, very new): Self-hosted MCP memory with graph visualization. TypeScript + SQLite + embeddings. Very early (24 stars), but positioned identically to Recall. URL: https://github.com/zanfiel/engram

6. **ByteDance OpenViking** (March 2026, very new): Open-sourced "memory layer" for agents. File-system-style structure (viking://user/memories/, viking://agent/skills/). L0/L1 tiered compression. No GitHub stars data yet — announced March 13, 2026.

7. **Mnemora** (March 2026): Serverless AWS-native open-source agent memory. 4 memory types, sub-10ms reads. Very early (4 stars). URL: https://github.com/mnemora-db/mnemora

8. **Remembra** (March 2026): "Universal memory layer for AI applications. Self-host in minutes." MIT license, MCP support, knowledge graph. 6 stars. URL: https://github.com/remembra-ai/remembra

### OpenAI Memory: No Self-Hostable Version

OpenAI's built-in memory feature (ChatGPT) remains cloud-only with no self-hostable or API-accessible version for personal memory persistence. The only developer-accessible path is through the Assistants API thread management, which is session-level, not cross-session semantic memory.

### Coding-Tool Native Memory Features (Cursor, Windsurf)

- **Windsurf**: Has a built-in "Memory" feature accessible via Cascade. Users can say "remember X" and it stores to `.windsurfrules` or similar. March 2025 blog: "Mimicking Windsurf's Memory Feature in Cursor." This is rule-based, not semantic vector memory.
- **Cursor**: No native persistent memory. Multiple high-upvote forum threads requesting it (Jan 2025, March 2025, Oct 2025, Feb 2026). Users solve this by: (a) adopting MCP memory servers, (b) using Cline with memory plugins, (c) manually maintaining markdown context files.
- **Claude Code**: No native persistent memory beyond session. Recall is the self-hosted solution specifically targeting this gap.

## Where Recall Sits in the Taxonomy

Recall is squarely in **Category 4: Coding-Tool / MCP-Native Memory**, with a secondary lean toward **Category 5 approaches** (automatic capture via hooks, no explicit user command needed).

Recall's unique combination:
- Ollama-native (extraction + embedding via local models — no cloud LLM required)
- Hooks-based automatic capture (observe-edit.js, session-summary.js — passive like Pieces, not command-based)
- MCP server delivery
- Self-hosted on homelab infrastructure (Qdrant + Neo4j + Redis + PostgreSQL)
- Zero vendor lock-in by design

No other player in Category 4 combines all of these. OpenMemory MCP requires mem0's extraction LLM calls (not Ollama-extraction by default). Engram is close architecturally but brand new and has no hooks-based auto-capture.
