## Evidence

### mem0 — Advanced Retrieval (Platform tier)

Source: https://docs.mem0.ai/platform/features/advanced-retrieval (scraped 2026-03-15)

mem0 Platform offers three retrieval enhancement layers:

1. **Keyword Search** (hybrid/BM25-like expansion): Expands results to include specific entity names and technical terms. Latency: +10ms. Increases recall at slight precision cost.

2. **Reranking**: "Reorders results using deep semantic understanding to put the most relevant memories first." Latency: +150–200ms additional. The docs describe it as significantly improving accuracy and ordering. The mechanism is not named explicitly in public docs — likely a cross-encoder or Cohere Rerank tier behind their managed API. Available via `rerank=True` flag.

3. **Memory Filtering**: LLM-based precision pass that removes off-topic results after retrieval. Latency: +200–300ms additional. Maximizes precision at potential recall cost.

Quoted from docs: "While basic search uses semantic similarity, these advanced options help you find exactly what you need, when you need it."

**Key caveat**: Advanced Retrieval is a Platform (cloud) feature. The OSS mem0 (`pip install mem0ai`) does NOT expose `rerank=True` or `filter_memories=True`. These are managed-API-only capabilities. This is a critical distinction for Recall's self-hosted comparison.

mem0 OSS source (from prior Q1.1 research and arxiv.org/html/2504.19413v1): The OSS pipeline does LLM-based extraction + vector similarity search. No reranking layer is documented in the OSS code path.

### Zep — Retrieval Architecture

Source: https://help.getzep.com/retrieving-memory (scraped 2026-03-15)

Zep's Context Block is described as combining: "semantic search, full text search, and breadth first search to return context that is highly relevant." This is de facto hybrid retrieval (vector + BM25-style full-text + graph BFS).

Zep's Graphiti engine handles retrieval through the knowledge graph structure — not a simple vector cosine search. Graph traversal naturally provides reranking-like re-weighting based on relationship structure and temporal validity. There is no explicit "reranker model" mentioned; relevance is structural.

P95 latency for context block: <200ms (Zep claim). This already includes their multi-modal search fusion.

No HyDE, query expansion, or cross-encoder reranking is mentioned in Zep's public docs. Zep's quality advantage is graph structure, not retrieval technique sophistication.

### Letta — Retrieval Architecture

Source: https://docs.letta.com/guides/core-concepts/memory/archival-memory/ (scraped 2026-03-15)

Letta uses semantic vector search on archival memory via `archival_memory_search`. The docs describe it as understanding "concepts and meaning, not just exact keywords." No reranking, query expansion, or hybrid retrieval documented. Agent-controlled insertion and retrieval — no automatic pipeline. Retrieval is fundamentally on-demand by the agent, not pre-emptive.

However, these benchmarks are for long-document RAG, not short factual memory retrieval (sub-200-char facts). The gains narrow significantly for short-text corpora where vector similarity already performs well.

### sklearn Gradient Boost Reranker — Quality Profile

No published benchmark compares sklearn GBM rerankers to cross-encoders on short-text memory retrieval specifically. However, the architectural profile is clear:

- **sklearn GBM reranker**: Trained on user feedback signals (clicks, accepts, edits). Feature-based: embeds query-memory pair and uses learned features. Fast (<5ms). Quality depends entirely on the training distribution — if Recall's training data is thin, the model underperforms a zero-shot cross-encoder. No semantic understanding of tokens, only representation similarity features.
- **Cross-encoder (BGE, ms-marco MiniLM)**: Token-level attention across query+document pair. Captures fine-grained semantic interactions. Zero-shot capability. Add latency of 50–150ms for self-hosted small models.
- **Cohere/Voyage API reranker**: Best quality, 150–300ms API call, cost per query. Not viable for self-hosted systems.

Key finding: For short factual memories (e.g., "User has a dog named Max"), vector cosine similarity is already quite precise. The query-document mismatch that motivates HyDE and heavy reranking (long documents, vocabulary mismatch) is reduced. Recall's sklearn reranker is providing value mainly for ambiguous or overlapping memories — not a total quality failure, but it is feedback-starved and architecturally limited compared to cross-encoder approaches.

### HyDE and Query Expansion in Memory Systems

HyDE (Hypothetical Document Embeddings) is actively used in RAG pipelines in 2025/2026 (multiple implementations found). For memory systems specifically, no leading competitor has published HyDE integration. The query-to-memory mismatch is lower than query-to-document because memories are already compressed fact statements rather than long prose. HyDE benefit would be most pronounced for memories with unusual phrasing or domain jargon. No competitor has shipped this for memory retrieval specifically.

Query expansion (keyword search in mem0) is present in mem0 Platform. Zep has implicit expansion via graph BFS. Letta has none. Recall has none.

## Mitigation Recommendation

**Short-term (low effort, high ROI)**:
Replace or augment the sklearn reranker with a self-hosted cross-encoder. BGE Reranker v2 M3 or ms-marco-MiniLM-L6-v2 run locally via sentence-transformers. Both are <150MB, run on CPU, and add 30–80ms latency for a 20-candidate reranking pass. This is a weekend-scale change and would provide zero-shot semantic reranking without dependency on training feedback data.

Estimated quality improvement for memory retrieval: 10–25% improvement in top-1 precision for ambiguous queries (lower than document RAG gains because memories are shorter). Worth the latency for batch/background retrieval; optional for real-time hook injection.

**Medium-term (moderate effort)**:
Add keyword expansion to the search path. A simple BM25 pass over the memory text index (Qdrant supports sparse vectors) combined with the existing dense vector would qualify as hybrid retrieval. mem0 Platform charges for this; Recall could offer it self-hosted.

**Low priority**:
HyDE — adds an LLM call per query (100–500ms), benefit for short factual memories is unclear and likely negative due to hallucinated hypothetical answer drifting from fact style. Do not implement until benchmarked on Recall's actual memory corpus.
