### Major Architectural Shift: Zep CE → Graphiti

Zep Community Edition has been **deprecated and moved to a legacy folder**. Zep's open-source strategy pivoted to **Graphiti** — a temporal knowledge graph framework. As of early 2025, "self-hosted Zep" effectively means self-hosted Graphiti + cloud-connected backend.

Zep Cloud is the supported product for the full feature set. CE is no longer recommended.

### Graphiti Architecture

Graphiti (github.com/getzep/graphiti, 20K+ stars) is a temporal context graph framework:
- **Graph backends**: Neo4j 5.26+, FalkorDB, Kuzu, Amazon Neptune
- **LLM requirement**: Requires LLM API access (OpenAI default; Anthropic, Groq, Google, Azure supported) — **no local-only path** without custom integration
- **MCP server**: Available (v1.0 released 2025), bundles FalkorDB in single container
- **Retrieval**: Embedding similarity + temporal validity windows

### Temporal Memory Design

Graphiti's key differentiator is **temporal validity**: facts are not deleted when they change — they're invalidated with temporal metadata (valid_from, valid_until). This allows historical reconstruction.

Three-tier structure:
1. **Episode subgraph**: Raw input (messages, text, JSON) — non-lossy source of truth
2. **Semantic entity subgraph**: Extracted entities and relationships
3. **Community subgraph**: Higher-order clusters across entities

### Graph Schema Comparison

| Dimension | Recall | Graphiti/Zep | Gap |
|-----------|--------|--------------|-----|
| Graph backend | Neo4j (fixed) | Neo4j, FalkorDB, Kuzu (pluggable) | Graphiti more flexible |
| Relationship types | RELATED_TO, CAUSED_BY, SUPPORTS, CONTRADICTS, SUPERSEDES (typed) | Generic LLM-extracted entity-relationship triples + temporal metadata | Recall more semantically typed |
| Temporal tracking | Recency boost in retrieval; no validity windows | Full temporal validity windows (valid_from/valid_until per fact) | Graphiti ahead on temporal precision |
| Auto-linking | Neo4j auto-linker creates edges between related memories | Entities extracted and linked at ingest time | Comparable |
| Local-only support | Full — Ollama for everything | Requires cloud LLM API (no local path tested) | Recall ahead |

### Memory Lifecycle Management

Graphiti: **no automatic decay, pruning, or consolidation pipeline**. Uses temporal invalidation instead — old facts persist but are marked invalid. No GC, no hygiene, no consolidation process.

This is architecturally different from Recall's pipeline but not equivalent. Benefits of Recall's approach (lower storage growth, cleaner retrieval) vs Graphiti's benefits (full historical audit trail).

### Self-Hosting Complexity

Graphiti self-hosted (MCP server): Python + LLM API key + graph DB (FalkorDB bundled in one container). Simpler than Recall in some ways, but **requires cloud LLM API** — no homelab-only path without custom config.

- `auto_consolidation`: 0.82 → 0.84 (no auto-consolidation in Graphiti either)

## Mitigation Recommendation

1. **Add temporal validity tracking** to Recall's Neo4j schema: `valid_from`, `valid_until` fields on memory nodes. When a memory is superseded, mark it invalid rather than deleting. Low cost, high historical audit value.
2. **Position local-first as explicit differentiator vs Graphiti**: Graphiti cannot operate without cloud LLM API. Recall can. This is a meaningful moat for privacy-first users.
