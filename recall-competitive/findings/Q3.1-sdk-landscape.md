## Evidence

### mem0 — the gold standard

**Python SDK** (`pip install mem0ai`): 2,034,758 downloads/month (PyPI, as of March 2026). The complete integration in 8 lines:

```python
from mem0 import Memory

m = Memory()  # zero-config; uses embedded Qdrant + SQLite by default

messages = [
    {"role": "user", "content": "Hi, I'm Alex. I love basketball and gaming."},
    {"role": "assistant", "content": "Hey Alex! I'll remember your interests."}
]
m.add(messages, user_id="alex")
results = m.search("What do you know about me?", filters={"user_id": "alex"})
```

Source: https://docs.mem0.ai/open-source/python-quickstart

**Zero-config path**: `Memory()` with no arguments creates an embedded Qdrant vector store at `/tmp/qdrant`, SQLite history at `~/.mem0/history.db`, and uses OpenAI `gpt-4.1-nano` for extraction. No Docker. No infrastructure. Just `pip install` and go.

**Methods exposed**: `add()`, `search()`, `get()`, `get_all()`, `update()`, `delete()`, `delete_all()`, `history()`, `reset()` — full CRUD plus history.

**TypeScript/JS SDK**: `npm install mem0ai`, imports from `'mem0ai/oss'`. Full parity with Python SDK.

```ts
import { Memory } from "mem0ai/oss";
const memory = new Memory();
await memory.add(messages, { userId: "alice" });
const results = await memory.search("What do you know?", { userId: "alice" });
```

Source: https://docs.mem0.ai/open-source/node-quickstart

**Cloud API**: Separate `MemoryClient` for the hosted tier, same method signatures. API key = `Authorization: Token <key>`.

**Authentication**: API key header for cloud; no auth for embedded/local use.

### Zep — cloud-only SDK, CE is different

**Python SDK** (`pip install zep-cloud`): Cloud SDK. API key required. 3-line integration:

```python
from zep_cloud.client import Zep
zep_client = Zep(api_key=API_KEY)
response = zep_client.thread.add_messages(thread_id, messages=messages)
```

**TypeScript/JS**: `npm install @getzep/zep-cloud`. Also has a Go SDK.

**Community Edition**: Zep CE repo was **archived on April 2, 2025** (read-only). The current Zep stack (v3) is cloud-only or requires self-hosting via Graphiti (their open-source temporal knowledge graph). The old CE Python SDK was `pip install zep-python` (deprecated).

**SDK coverage**: `thread.add_messages`, `memory.get` (context assembly), `graph.search`, `graph.add`. Notable: no standalone `Memory` class — you work with threads and users, and Zep assembles context automatically.

Source: https://help.getzep.com/quick-start-guide, https://help.getzep.com/adding-memory

### Letta — agent-centric SDK

**Python SDK** (`pip install letta-client`): Agent-first design. Not a memory store you call directly — you create a stateful agent and send it messages. Memory is a side effect.

```python
from letta_client import Letta
client = Letta(api_key=os.getenv("LETTA_API_KEY"))
agent = client.agents.create(
    model="openai/gpt-4.1",
    memory_blocks=[{"label": "human", "value": "Name: User"}]
)
response = client.agents.messages.create(agent_id=agent.id, input="Hello!")
```

**TypeScript/JS**: `npm install @letta-ai/letta-client`. Full parity.

**Self-hosted path**: `client = Letta(base_url="http://localhost:8283")` — point at your Docker server, no API key needed locally.

**Methods exposed**: `agents.create`, `agents.messages.create`, `agents.blocks.list`, `agents.blocks.update`. Memory is implicitly managed by the agent; no explicit `add_memory()` call.

Source: https://docs.letta.com/api-reference/overview/, https://docs.letta.com/tutorials/hello-world/

### Recall — current state

**No Python SDK exists.** Integration is via:
1. Claude Code hook configuration (YAML/JSON in `.claude/settings.json`)
2. Raw HTTP calls to the FastAPI server at `http://192.168.50.19:8200`
3. MCP tools (`recall_store`, `recall_search`) exposed via the MCP server

There is no `pip install recall` or `npm install recall`. The GitHub repo (github.com/Nerfherder16/System-Recall) shows no published package on PyPI or npm. Documentation exists as markdown files in the repo, not a hosted site.

A raw HTTP integration currently looks like this (based on the MCP server pattern):

```python
import requests
BASE = "http://192.168.50.19:8200"
# Store
requests.post(f"{BASE}/memories", json={"content": "...", "memory_type": "semantic"})
# Search
requests.post(f"{BASE}/memories/search", json={"query": "...", "limit": 5})
```

This is undocumented, requires knowing the server address, and has no authentication surface exposed to external callers.

## Mitigation Recommendation

**Immediate (1-2 days)**: Publish `recall-sdk` to PyPI as a thin wrapper around the existing REST API. Minimum viable surface:

```python
from recall import RecallClient

client = RecallClient(base_url="http://localhost:8200")
client.store("I prefer Python for backend work", memory_type="semantic")
results = client.search("programming preferences", limit=5)
client.delete(memory_id="abc123")
```

This is a ~200-line wrapper. No new backend endpoints needed — just wrap the existing FastAPI routes.

**Short term (1 week)**:
- Add `npm install @recall/client` TypeScript SDK
- Publish PyPI package with semantic versioning
- Add a 10-line quickstart to the README that demonstrates store + search

**Medium term (2-4 weeks)**:
- Add `RECALL_API_KEY` authentication to the FastAPI server (currently none)
- Publish a hosted docs page (GitHub Pages or ReadTheDocs — minimal effort, high visibility signal)

The SDK gap is Critical because it is the **first thing a developer evaluates** when choosing a memory library. mem0's zero-friction `pip install + 8 lines` is nearly impossible to match without publishing a package. Recall is invisible on PyPI.
